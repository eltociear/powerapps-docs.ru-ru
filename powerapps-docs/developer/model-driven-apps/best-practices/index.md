# <a name="best-practices-and-guidance-for-model-driven-apps"></a>Рекомендации по приложениям на основе модели

Создание приложений на основе модели — это ориентированный на компоненты подход к разработке приложений, более адаптированных для решения конкретных задач. При настройке приложения на основе модели разработчику следует учитывать общепринятые рекомендации. 

В этом разделе вы узнаете о проблемах, которые мы выявили, их возможных последствиях и способах их устранения. Мы поясним, почему нужно выполнять те или иные действия, чтобы избежать проблем в будущем. В результате вы сможете повысить удобство использования и производительность среды, а также обеспечить ее поддержку. В документации используются существующие сведения из руководств для разработчиков и администраторов.

# <a name="targeted-customization-types"></a>Целевые типы настройки
Документация предназначена для следующих типов настройки:

- разработка приложения на основе модели;
- разработка формы сущности;
- Создание скриптов для клиента
- веб-ресурсы.

# <a name="sections"></a>Разделы
Каждая статья состоит из большинства или всех следующих разделов:

- Заголовок — описание рекомендации.
- Категория — одна или несколько областей, в которых могут возникнуть проблемы, если не следовать рекомендации.
- Возможные последствия — уровень риска (высокий, средний или низкий) для среды в случае, если рекомендация не соблюдается.
- Симптомы — возможные признаки того, что рекомендация не соблюдается.
- Рекомендация — рекомендованные действия; могут приводиться примеры.
- Неоптимальная схема — описание или пример несоблюдения рекомендации.
- Дополнительные сведения — вспомогательная информация, расширяющая представление.
- См. также — ссылки на дополнительные материалы по вопросам, упомянутым в статье.

# <a name="categories"></a>Категории
Каждая статья относится к одной или нескольким из следующих категорий:

- Использование — ненадлежащее использование определенного API-интерфейса, шаблона или конфигурации.
- Проектирование — изъяны в проекте настройки.
- Производительность — настройка или шаблон, которые могут негативно сказаться на производительности в таких областях, как использование памяти, загрузка ЦП, сетевой трафик или пользовательский интерфейс.
- Безопасность — потенциальные уязвимости настройки, которые могут стать мишенями для атак в среде выполнения.
- Готовность к обновлению — настройка или шаблон, из-за которых может возрастать риск неудачного обновления версии.
- Миграция по сети — настройка или шаблон, из-за которых может возрастать риск неудачной миграции по сети.
- Удобство поддержки — настройка, из-за которой необоснованно увеличивается объем работы по внесению изменений разработчиком, требуемая частота изменений или вероятность возникновения регрессий.
- Возможность поддержки — настройка или шаблон, которые выходят за пределы официальных условий поддержки, в том числе из-за использования упраздненных API-интерфейсов или реализации запрещенных методик.
