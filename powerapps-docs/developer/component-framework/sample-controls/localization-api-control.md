# <a name="implementing-localization-api-component"></a>Реализация компонента API локализации

Этот образец демонстрирует, как локализация выполняется для пользовательских элементов управления. В этом примере мы используем [компонент инкремента](increment-control.md) для локализации текста, который отображается на кнопке приращение на основе выбранного пользователем языка. 

Платформа компонентов PowerApps использует концепцию реализации веб-ресурсов строки (RESX), которая используется для управления локализованными строками, отображаемыми в любом пользовательском интерфейсе. Дополнительные сведения: [String (RESX)-ресурсы](https://docs.microsoft.com/dynamics365/customer-engagement/developer/resx-web-resources).

## <a name="code"></a>Код

```TypeScript
import { IInputs, IOutputs } from "./generated/ManifestTypes";
export class TSLocalizationAPI
  implements ComponentFramework.StandardControl<IInputs, IOutputs> {
  // Value of the field is stored and used inside the control
  private _value: number;
  // PowerApps component framework framework delegate which will be assigned to this object which would be called whenever an update happens.
  private _notifyOutputChanged: () => void;
  // label element created as part of this control
  private label: HTMLInputElement;
  // button element created as part of this control
  private button: HTMLButtonElement;
  // reference to the control container HTMLDivElement
  // This element contains all elements of our custom control example
  private _container: HTMLDivElement;
  /**
   * Empty constructor.
   */
  constructor() {}
  /**
   * Used to initialize the control instance. Controls can kick off remote server calls and other initialization actions here.
   * Data-set values are not initialized here, use updateView.
   * @param context The entire property bag available to control via Context Object; It contains values as set up by the customizer mapped to property names defined in the manifest, as well as utility functions.
   * @param notifyOutputChanged A callback method to alert the framework that the control has new outputs ready to be retrieved asynchronously.
   * @param state A piece of data that persists in one session for a single user. Can be set at any point in a controls life cycle by calling 'setControlState' in the Mode interface.
   * @param container If control is marked control-type='standard', it receives an empty div element within which it can render its content.
   */
  public init(
    context: ComponentFramework.Context<IInputs>,
    notifyOutputChanged: () => void,
    state: ComponentFramework.Dictionary,
    container: HTMLDivElement
  ) {
    // Creating the label for the control and setting the relevant values.
    this.label = document.createElement("input");
    this.label.setAttribute("type", "label");
    this.label.addEventListener("blur", this.onInputBlur.bind(this));
    //Create a button to increment the value by 1.
    this.button = document.createElement("button");
    // Get the localized string from localized string
    this.button.innerHTML = context.resources.getString(
      "PCF_LocalizationSample_ButtonLabel"
    );
    this.button.classList.add("LocalizationSample_Button_Style");
    this._notifyOutputChanged = notifyOutputChanged;
    //this.button.addEventListener("click", (event) => { this._value = this._value + 1; this._notifyOutputChanged();});
    this.button.addEventListener("click", this.onButtonClick.bind(this));
    // Adding the label and button created to the container DIV.
    this._container = document.createElement("div");
    this._container.appendChild(this.label);
    this._container.appendChild(this.button);
    container.appendChild(this._container);
  }
  /**
   * Button Event handler for the button created as part of this control
   * @param event
   */
  private onButtonClick(event: Event): void {
    this._value = this._value + 1;
    this._notifyOutputChanged();
  }
  /**
   * Input Blur Event handler for the input created as part of this control
   * @param event
   */
  private onInputBlur(event: Event): void {
    let inputNumber = Number(this.label.value);
    this._value = isNaN(inputNumber)
      ? ((this.label.value as any) as number)
      : inputNumber;
    this._notifyOutputChanged();
  }
  /**
   * Called when any value in the property bag has changed. This includes field values, data-sets, global values such as container height and width, offline status, control metadata values such as label, visible, etc.
   * @param context The entire property bag available to control via Context Object; It contains values as set up by the customizer mapped to names defined in the manifest, as well as utility functions
   */
  public updateView(context: ComponentFramework.Context<IInputs>): void {
    // This method would re render the control with the updated values after we call NotifyOutputChanged
    //set the value of the field control to the raw value from the configured field
    this._value = context.parameters.value.raw;
    this.label.value = this._value != null ? this._value.toString() : "";
    if (context.parameters.value.error) {
      this.label.classList.add("LocalizationSample_Input_Error_Style");
    } else {
      this.label.classList.remove("LocalizationSample_Input_Error_Style");
    }
  }
  /**
   * It is called by the framework prior to a control receiving new data.
   * @returns an object based on nomenclature defined in manifest, expecting object[s] for property marked as “bound” or “output”
   */
  public getOutputs(): IOutputs {
    // custom code goes here - remove the line below and return the correct output
    let result: IOutputs = {
      value: this._value
    };
    return result;
  }
  /**
   * Called when the control is to be removed from the DOM tree. Controls should use this call for cleanup.
   * i.e. canceling any pending remote calls, removing listeners, etc.
   */
  public destroy(): void {}
}
```

Чтобы локализовать существующий проект, достаточно создать дополнительные файлы ресурсов (RESX), по одному для конкретного языка, как упоминалось в строковых веб-ресурсах, и включить их как часть файла манифеста элемента управления в узле [ресурсов](../reference/resources.md) .  

Платформа компонентов PowerApps определяет язык пользователя и возвращает строки из файла ресурсов конкретного языка при попытке получить доступ к строке с помощью метода `context.resources.getString`.

В этом примере на двух языках `Spanish` и `Finnish` с кодами языков 3082 и 1035, соответственно определенными. Мы сделали копию примера `Increment component` и переименовали ее в `Localization API`. Все соответствующие файлы, включая файлы во вложенных папках, переименовываются соответствующим образом.

В папке strings в разделе `TS_LocalizationAPI` добавляются два дополнительных RESX файлов с суффиксами, соответствующими испанскому и финским, как 3082 и 1035. Имена файлов в создаваемых файлах должны заканчиваться на `{filename}.3082.resx` и `{filename}.1035.resx`, поскольку платформа использует это соглашение об именовании, чтобы указать, какой файл ресурсов следует выбрать для чтения строк для пользователя.

Убедитесь, что ключи, используемые для строк во всех этих файлах ресурсов, имеют одинаковое имя на всех языках. Теперь, когда компонент отображается в пользовательском интерфейсе, мы видим в коде, что извлекается значение, которое будет отображаться на кнопке с помощью `context.resources.getString("PCF_LocalizationSample_ButtonLabel")`.

При выполнении этой строки кода платформа компонентов PowerApps автоматически определяет язык пользователя и выбирает значение для метки кнопки, используя ключ, указанный в соответствующем файле языка, который мы определили. Ниже приведен текст, отображаемый для каждого из трех языков, поддерживаемых для этого образца компонента.
  
|LanguageCode |Отображаемое значение |
|---|---|
|3082 |инкременто |
|1033 |Производим |
|1035 |лисäис |
