Манифест — это файл метаданных, определяющий компонент. Это файл `XML`, описывающий следующее:

- Пространство имен компонента.
- Тип данных, которые можно настроить — либо поле, либо набор данных.
- Любые свойства, которые могут быть настроены в приложении при добавлении компонента.
- Список файлов ресурсов, необходимых для компонента. 
  - Один из них должен быть веб-ресурсом JavaScript. Этот код JavaScript должен включить функцию, которая создает экземпляр объекта. Этот код реализует интерфейс, предоставляющий методы, которые требуются для работы компонента. Это называется библиотекой реализации компонентов.
- Имя функции JavaScript в библиотеке реализации компонентов, которая будет возвращать объект, который применяет требуемый интерфейс компонента.

Когда пользователь настраивает пользовательский компонент в приложении Canvas или приложении, управляемом моделью, данные в манифесте отфильтровывают доступные компоненты, чтобы для конфигурации были доступны только допустимые компоненты контекста. Свойства, определенные в манифесте для компонента, подготавливаются к просмотру как поля конфигурации, чтобы пользователь, настроив компонент, мог указать значения. Затем эти значения свойств доступны функции Component во время выполнения.
