# <a name="add-code-components-to-a-field-or-entity-in-model-driven-apps"></a>Добавление компонентов кода в поле или сущность в приложениях, управляемых моделью

Компоненты кода позволяют преобразовывать поля, которые традиционно содержат текст, в визуализации. Аналогичным образом можно использовать компоненты кода для преобразования наборов данных, например представления, для отображения в более визуальной отрисовке, а не в виде списка записей. компоненты кода могут отображаться в виде визуализаций на формах, панелях мониторинга, представлениях и сетках домашней страницы. 


   > [!div class="mx-imgBorder"] 
   > (../../maker/model-driven-apps/media/slider-control.PNG "Элемент управления ползунок") ![настраиваемого ползунка]для поля

## <a name="add-a-code-component-to-a-field"></a>Добавление компонента кода в поле

Выполнив действия, описанные в этой процедуре, измените поле Метка по умолчанию и текстовое поле в поле **сумма бюджета** на компонент кода ползунка в сущности возможной сделки. Аналогичные действия можно использовать для замены существующего поля на компонент кода или настройки компонента кода для настраиваемого поля.

1. Откройте обозреватель решений.

2. Разверните узел **сущности**, разверните нужную сущность, например сущность **возможности** , выберите **формы**, а затем откройте форму, например **главную** форму.

3. В редакторе форм дважды щелкните поле, в которое необходимо добавить компонент кода, например поле **сумма бюджета** в главной форме возможности. Кроме того, можно создать настраиваемое поле.

4. На странице **Свойства поля** выберите вкладку **элементы управления** и нажмите кнопку **Добавить элемент управления**.

5. На странице Добавление элемента управления выберите нужный компонент, например компонент **линейного ползунка** , а затем нажмите кнопку **Добавить**.

   > [!div class="mx-imgBorder"] 
   > ![Добавить элемент управления "линейный ползунок]"(../../maker/model-driven-apps/media/add-slider.PNG "Добавить ползунок \"линейный") "

6. Выберите клиент, на котором должен отображаться компонент.

   - **Веб-узел**. Чтобы сделать компонент кода доступным из любого веб-браузера, выберите параметр веб-узел рядом с компонентом. Обратите внимание, что установка веб-параметра включает отображение компонента в веб-браузерах на ПК, компьютерах Mac и мобильных устройствах.

   - **Телефон**. Чтобы сделать компонент кода доступным на телефонах, использующих Dynamics 365 для телефонов, выберите параметр телефон рядом с компонентом.

   - **Планшетный ПК**. Чтобы сделать компонент кода доступным на планшетных устройствах под ОС Dynamics 365 для планшетов, выберите параметр планшет рядом с компонентом.

   > [!div class="mx-imgBorder"] 
   > ![Выберите клиентские приложения для просмотра пользовательского элемента управления](../../maker/model-driven-apps/media/choose-client.png "выберите клиентские приложения для просмотра пользовательского элемента управления") 

7. Щелкните значок карандаша рядом с параметром **min**, **Max**, and **Step**, задайте значение свойства и нажмите кнопку **ОК**.  
   
   > [!div class="mx-imgBorder"] 
   > ![Добавление свойств пользовательского элемента управления](../../maker/model-driven-apps/media/ccf-add-properties.png "Добавление свойств пользовательского элемента управления")

   - **Min**. Задайте минимальное допустимое значение. Вы можете привязать введенное статическое значение или привязать значение к существующему полю. В этом примере выполняется **Привязка к статическому значению** **Currency** , а минимальное значение, которое можно указать, равно *нулю*.  
  
       - **Привязка к статическому значению**. Выбор типа данных, например целого числа (целое число. нет), валюты, плавающей запятой (FP) или десятичного числа. Затем введите число, представляющее минимальное допустимое значение для поля.  
  
       - **Привязка к значениям поля**. Выберите поле из списка, которое будет использоваться как минимальное допустимое значение.  
  
   - **Макс**. Задайте максимальное допустимое значение для поля. Как и в случае с минимальным значением, можно привязать статическое значение, которое вы вводите, или привязать значение к существующему полю, как описано выше. В этом примере **Привязка к статическому значению** — **Currency** , а максимальное значение, которое можно указать, — **1 000 000 000**.  
  
   - **Шаг**. Представляет единицу, которую нужно увеличить или уменьшить при добавлении к текущему значению или вычитании из него. Например, для суммы бюджета можно выбрать 100 долларов США инкрементс\декрементс.  
  
   - **Скрыть элемент управления по умолчанию**. При выборе этого параметра компонент скрывается, поэтому ни компонент, ни данные не отображаются ни на одном из клиентов, которые не поддерживают компонент кода.   
  
8. Нажмите кнопку **ОК**, чтобы закрыть страницу Свойства поля.  
  
9. Чтобы активировать настройку, в форме сущность выберите **сохранить**, а затем щелкните **опубликовать**.  
  
10. Нажмите кнопку **сохранить и закрыть** , чтобы закрыть редактор форм.  
  
## <a name="add-code-component-to-an-entity"></a>Добавление компонента кода в сущность

Чтобы добавить в сетку или представление компонент кода, например компонент набора данных или простой компонент таблицы, выполните следующие действия:

  - Перейдите в раздел **параметры > настройки** и щелкните элемент **Настройка системы**.
  - Щелкните стрелку рядом с вкладкой **сущности** а выберите сущность, для которой нужно добавить компонент кода. 
  - Перейдите на вкладку **элементы управления** и щелкните **Добавить элемент управления**.
  - На странице Добавление элемента управления выберите нужный компонент, например простой компонент таблицы, а затем нажмите кнопку **Добавить**.
  - Выберите клиент, на котором должен отображаться компонент.


## <a name="see-the-code-component-in-action"></a>Просмотр компонента кода в действии  

 Откройте запись, включающую в себя поле с компонентом кода, например форму возможности из предыдущего примера, и посмотрите, как изменилось поле. Теперь поле отображается как компонент Slider вместо текстового поля.
