# <a name="limitations"></a>Ограничений 

С выпуском платформы PowerApps Component Framework теперь можно создавать собственные компоненты кода для улучшения взаимодействия с пользователем в приложениях и приложениях Canvas на основе моделей. Несмотря на то, что вы можете создавать собственные компоненты, существуют некоторые ограничения, которые ограничивают разработчиков, реализующие некоторые функции в компонентах кода. Ниже приведены некоторые ограничения.

1. В экспериментальном предварительном просмотре для приложений Canvas поддерживаются *только типы компонентов, а* не компоненты типа *набора данных* . 
2. Common Data Service зависимые API, включая WebAPI вместе с другими API, недоступны для экспериментальной предварительной версии. Сведения о доступности отдельных API для этого экспериментального выпуска см. в статье [Справочник по API инфраструктуры компонентов PowerApps](reference/index.md).
3. Компоненты кода должны объединять весь код, включая содержимое внешней библиотеки, в основной набор кодов. Пример того, как интерфейс командной строки PowerApps может помочь при объединении содержимого внешней библиотеки в набор компонентов, см. в разделе пример [компонента угловой переворот](sample-controls/angular-flip-control.md) .

   > [!NOTE]
   > Поддержка общих библиотек в компонентах, использующих узлы библиотек в манифесте компонента, пока не поддерживается. Мы рассмотрим эту функцию и добавим эту возможность в будущем выпуске.
4. Определение нескольких компонентов в одном файле манифеста пока не поддерживается.
5. Вызов процессов и действий пока не поддерживается. Вызывать диалоговые окна можно только с помощью метода [навигации](reference/navigation.md) .
6. Вызов одного компонента из другого компонента кода пока не поддерживается.
7. Текущий шрифтовый ресурс (. ТФФ) пока не поддерживается.
