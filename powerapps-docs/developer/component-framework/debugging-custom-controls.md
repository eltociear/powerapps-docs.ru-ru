# <a name="debug-code-components"></a>Компоненты кода отладки

После завершения реализации логики компонента кода можно начать тестирование и отладку компонента кода с помощью команды `npm start`. Это позволяет построить компонент кода и открыть его в локальной тестовой программе.

> [!div class="mx-imgBorder"]
> Окружение тестирования ![1](media/test-harness-1.png "Тестовая программа 1")

Как показано на рисунке выше, откроется окно браузера с четырьмя разделами. Компонент кода отображается на левой панели, а в правой панели — в разделах **входные данные контекста**, **входы**и **выходы** .

- **Входные данные контекста** позволяют указать конструктивный фактор и протестировать компонент кода с каждым форм-фактором (Web, Tablet, Phone). Это полезно, когда компонент кода зависит от конкретной возможности форм-фактора.
- **Входные данные** — это интерактивный пользовательский интерфейс, отображающий все свойства и их [типы](manifest-schema-reference/types.md) или [группы типов](manifest-schema-reference/type-group.md) , определенные в файле [манифеста](manifest-schema-reference/manifest.md) . Он позволяет получить основные сведения о фиктивных данных для каждого свойства. 
- **Выходные данные визуализируются** при каждом вызове метода- [выходов](reference/control/getoutputs.md) компонента.  

     > [!div class="mx-imgBorder"]
     > Окружение тестирования ![2].(media/test-harness-2.png "Тестовая программа 2")

> [!NOTE]
> Если вы хотите изменить файл `ControlManifest.Input.xml` или создать дополнительные свойства, необходимо перезапустить процесс отладки, прежде чем они появятся в разделе входные данные.

## <a name="watch-mode-in-test-harness"></a>Режим просмотра в тестовой программе

Тестовая программа поддерживает режим `watch`, который можно использовать для проектов платформы компонентов PowerApps. Чтобы включить режим `watch`, запустите тестовую программу с помощью команды `npm start watch`. В `watch` режиме изменения, внесенные в любой из следующих компонентов, автоматически отражаются в тестовой программе без необходимости перезапуска.

1.  файл `index.ts`.
2.  файл `ControlManifest.Input.xml`.
3.  Импортированные библиотеки в `index.ts`.
4.  Все ресурсы, перечисленные в файле манифеста.

## <a name="debug-code-components-using-native-browsers"></a>Отладка компонентов кода с помощью собственных браузеров

Для отслеживания поведения компонента можно использовать возможности отладки в браузере. Каждый браузер предоставляет средство отладки, помогающее выполнить отладку кода в браузере. Как правило, отладку можно активировать в браузере, нажав клавишу **F12** , чтобы отобразить собственное средство разработки, используемое для отладки.

Например, в **Microsoft ребро**:

- Нажмите клавишу **F12** , чтобы открыть инспектор.
- Выберите свой компонент.
- На верхней панели перейдите к **отладчику** и найдите имя компонента, описанное в файле манифеста в строке поиска. Например, введите имя компонента, например `Hello World component`.

     > [!div class="mx-imgBorder"]
     > ![](media/debug-control.png "Компонент") отладки компонента отладки

> [!NOTE]
> Всегда рекомендуется устанавливать точки останова в таких методах жизненного цикла компонента, как [init](reference/control/init.md) и [упдатевиев](reference/control/updateview.md).

Вы также можете взаимодействовать с компонентом локально в режиме реального времени и наблюдать за элементами в модели DOM, установив точку останова на вкладке *источники* , как показано ниже:
