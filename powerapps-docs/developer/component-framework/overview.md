# <a name="powerapps-component-framework-overview"></a>Обзор платформы компонентов PowerApps

Платформа компонентов PowerApps позволяет профессиональным разработчикам и руководителям приложений создавать компоненты кода для приложений на основе моделей и приложений Canvas (экспериментальная Предварительная версия) для предоставления пользователям возможности просмотра и работы с данными в формах, представлениях, и панели мониторинга. Пример.

- Замените поле, содержащее числовое текстовое значение, `dial` или `slider` компонентом кода.
- Преобразование списка в совершенно другую визуальную среду, привязанную к набору данных, например `Calendar` или `Map`.

> [!IMPORTANT]
> - Платформа компонентов PowerApps находится в экспериментальном режиме для приложений Canvas и в общедоступной версии для приложений на основе модели. Это означает, что все API-интерфейсы, которые поддерживаются для приложений на основе моделей, могут еще не поддерживаться в приложениях Canvas.
> - По умолчанию платформа компонентов PowerApps включена для приложений, управляемых моделью. Сведения о включении этой функции для приложений Canvas см. в разделе [доступность для приложений Canvas](component-framework-for-canvas-apps.md).
> - [!INCLUDE[cc_preview_features_definition](../../includes/cc-preview-features-definition.md)]
> - Приложения Canvas поддерживают только тип *полей* компонентов кода, а не тип *набора данных* .


Используйте платформу компонентов PowerApps для создания компонентов кода, которые можно использовать в рамках всех возможностей PowerApps. В отличие от веб-ресурсов HTML, компоненты кода подготавливаются к просмотру в одном и том же контексте, загружаются в то же время, что и любые другие компоненты, что обеспечивает удобство работы пользователей. Разработчики могут объединить все файлы HTML, CSS и TypeScript или JavaScript в один файл пакета решения. Компоненты кода могут многократно использоваться многократно в разных сущностях и формах.

Компоненты кода имеют доступ к обширному набору API-интерфейсов платформы, которые предоставляют такие возможности, как управление жизненным циклом компонентов, контекстные данные и доступ к метаданным, простой доступ к серверу через веб-API, методы форматирования программного обеспечения и данных, функции устройств, например камеру расположение и микрофон, а также простые в вызове элементы UX, такие как диалоговые окна, уточняющие запросы и полная отрисовка страниц.  

Разработчики и руководители приложений могут использовать современные веб-методики, а также работать с внешними библиотеками для создания сложных пользовательских взаимодействий. Платформа автоматически обрабатывает жизненный цикл компонента, удерживает бизнес-логику приложения и оптимизирует производительность (без асинхронных IFrame). Определение компонента, зависимости и конфигурации могут быть упакованы в [решение](https://docs.microsoft.com/dynamics365/customer-engagement/customize/solutions-overview) и перемещены между средами и могут поставляться через [AppSource](https://appsource.microsoft.com/en-us/marketplace/apps?page=1&product=dynamics-365).
