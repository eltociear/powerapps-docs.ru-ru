# <a name="licensing-overview"></a>Общие сведения о лицензировании

## <a name="about-powerapps-licenses"></a>Лицензии PowerApps

- PowerApps лицензируется отдельно для каждого пользователя. 
- Пользовательские лицензии назначаются по имени пользователя. Для запуска приложений каждому пользователю требуется отдельная лицензия. 
- Лицензии PowerApps не ограничивают создание приложений.
- Для PowerApps доступны два автономных плана — план 1 и 2. 
- План 1 для PowerApps обеспечивает доступ к Common Data Service для приложений, позволяя хранения и администрировать данные. Пользователи могут запустить приложения холста, встроенные в Common Data Service для приложений, использовать соединители уровня "Премиум", получать доступ к локальным данным или данным в пользовательских приложениях. 
- План 2 для PowerApps позволяет пользователям запускать приложения на основе моделей с использованием подключаемых модулей кода и рабочих процессов в реальном времени. Дополнительные сведения см. на [странице цен PowerApps](https://powerapps.microsoft.com/pricing/).  
- Кроме автономных планов, возможности PowerApps также добавлены в определенные планы Office 365 и Dynamics 365. Благодаря этому пользователи могут включать соответствующие функции PowerApps и Microsoft Flow в решения Office 365 и Dynamics 365 и настраивать их. См. дополнительные сведения о [приложениях и планах Dynamics 365 с поддержкой полной лицензии PowerApps (план 2)](#powerapps-for-dynamics-365).

Ниже описаны различия между функциями, поддерживаемыми этими группами лицензий.

## <a name="pricing"></a>Цены
Сведения об актуальной стоимости каждой лицензии PowerApps (план 1 и 2) см. на [странице цен на PowerApps](https://powerapps.microsoft.com/pricing/). Сведения об актуальной стоимости каждой лицензии Microsoft Flow см. на [странице цен на Microsoft Flow](https://preview.flow.microsoft.com/pricing/).

### <a name="powerapps-standalone-plan-1-and-plan-2"></a>Автономные планы для PowerApps 1 и 2
Автономные планы для PowerApps позволяют пользователям создавать и запускать приложения в источниках данных, отличающихся от Office 365, включая Salesforce, а также локальные и пользовательские источники данных. Эти планы также включают доступ к Common Data Service для приложений для хранения и администрирования данных. См. дополнительные сведения о [Common Data Service для приложений](../maker/common-data-service/data-platform-intro.md).  

- Подписки Microsoft PowerApps (план 1) предназначены для пользователей, которым требуется запускать приложения холста и получать доступ к локальным данным, данным в пользовательских приложениях и облачных службах с помощью соединителей уровня "Премиум".
- Подписки Microsoft PowerApps (план 2) предназначены для пользователей и администраторов, которым требуется больше возможностей. Такие пользователи могут запускать приложения на основе моделей, которые могут включать настраиваемые подключаемые модули кода и рабочие процессы в реальном времени. Такие пользователи имеют доступ к важным функциям управления, таким как просмотр использования и настройка политик. 

### <a name="powerapps-plan-2-free-trial"></a>Бесплатная пробная версия PowerApps (план 2)
Пользователи могут использовать PowerApps (план 2) бесплатно в течение 30 дней. Во время пробного периода пользователи имеют доступ ко всем возможностям в PowerApps (план 2). См. дополнительные сведения о [самостоятельной регистрации для использования PowerApps](../maker/signup-for-powerapps.md). 

По истечении пробного периода пользователям доступны следующие возможности:

- Пользователи с доступом к PowerApps или Microsoft Flow через приложения и планы Dynamics 365 или Office 365 могут по-прежнему получать доступ к PowerApps или Microsoft Flow. Но такие пользователи потеряют доступ к функциям, предназначенным исключительно для плана 2, как описано на странице с ценами на PowerApps. Некоторые приложения и планы Dynamics 365 поддерживают PowerApps (план 2).
- Пользователи без доступа через выбранные приложения и планы Office 365 и Dynamics 365 могут запросить продление пробной версии или приобрести автономный план. См. дополнительные сведения о [приобретении PowerApps для организации](signup-for-powerapps-admin.md).

> [!NOTE]
>  Чтобы приобрести PowerApps для организации, нужны права глобального администратора Office 365 или администратора выставления счетов на клиенте. Либо необходимо создать клиент.

## <a name="resource-capacity-is-included-with-each-license"></a>В каждой лицензии включены дополнительные ресурсы
Индивидуальные лицензии предоставляются вместе с дополнительными ресурсами, используемыми при выполнении приложения или потока. Эти ресурсы обеспечивают управление хранилищем данных и потоковое выполнение. Ресурсы, включенные в индивидуальные лицензии, объединены на уровне клиента, и при их исчерпании пользователи могут приобрести дополнительную емкость с помощью дополнительной лицензии. Дополнительные сведения см. на [странице с ценами PowerApps](https://powerapps.microsoft.com/pricing).

> [!NOTE]
> В Common Data Service для каждого экземпляра (база данных) максимальная емкость в 10 ГБ больше не поддерживается. Вы можете приобрести свободное место объемом до 30 ТБ, которое будет использоваться экземплярами в клиенте.

## <a name="powerapps-licensing-examples"></a>Примеры лицензирования PowerApps
Рассмотрим пример. В компании ABC Inc. работает 1000 сотрудников, 700 из которых имеют лицензию Office 365 корпоративный 3. На раннем этапе опытный пользователь создает бизнес-приложение, которое упрощает способ отслеживания заказов. Позже отдел кадров работает с ИТ-отделом над внедрением приложения для отчетности об отпусках и отсутствии на рабочем месте, которое основано на Common Data Service.

### <a name="order-tracking-app"></a>Приложение для отслеживания заказов
ABC Inc. начинает разработку приложения для лицензированных пользователей Office 365. Приложение объединяет данные о клиентах и данные конфигурации товаров, хранящиеся в списках Office 365 SharePoint, с документами заказов клиентов, которые хранятся в Box.com. Так как это приложение получает доступ только к данным, хранящимся в Office 365, а общая облачная служба доступна через стандартный соединитель, уже имеющиеся лицензии Office 365 покрывают создание и использование этого приложения.

**Требуемые лицензии:**  имеющихся 700 лицензий Office 365 корпоративный 3 будет достаточно.

### <a name="time-and-absence-app"></a>Приложение для учета рабочего времени и отсутствия на рабочем месте
Проанализировав скорость и сложность запуска приложения для отслеживания заказов, команда отдела кадров компании ABC рассчитывает на помощь ИТ-отдела в создании приложения для учета рабочего времени и отсутствия на рабочем месте, которое будет внедрено во всех отделах компании. Все сотрудники должны будут использовать это приложение для учета рабочего времени, дней отпуска и дней отсутствия из-за болезни.

Для реализации этого приложения ИТ-отдел выбирает Common Data Service в качестве системы для хранения данных о рабочих часах и отсутствии на рабочем месте. Common Data Service предоставляет функции безопасности и настройки политики данных, которые необходимы ИТ-отделу для получения сведений, относящихся к сотрудникам. Они назначают двух сотрудников ИТ-отдела для работы над проектом. Они должны создать базу данных и смоделировать данные приложения для учета рабочего времени и отсутствия на рабочем месте в Common Data Service. Эти сотрудники также несут ответственность за отслеживание использования приложения и установку политики, применяемой к этим данным.

**Требуемые лицензии:**

- **PowerApps (план 2) – 10 лицензий**. Для каждого из 10 ИТ-администраторов, которые будут настраивать среды для тестирования и развертывания приложения, а также устанавливать политики безопасности данных, потребуется PowerApps (план 2).
- **PowerApps (план 1) – 990 лицензий**. 700 пользователям Office 365 должны иметь лицензию на PowerApps (план 1), так как это приложение использует данные, хранимых за пределами Office 365 (т. е. в Common Data Service). Другим 290 пользователям без лицензии Office 365 или PowerApps (план 2) эта лицензия потребуется, чтобы получить права на запуск приложения.
