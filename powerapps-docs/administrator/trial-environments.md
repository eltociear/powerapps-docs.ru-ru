# <a name="about-trial-environments"></a>Общие сведения о пробных средах

В настоящее время можно создавать два типа сред Common Data Service (CDS) для приложений: пробные и рабочие. Пробная среда удобна для ознакомления с приложениями Dynamics 365 для взаимодействия с клиентами и предоставляется бесплатно. Срок действия пробной среды истекает через 30 дней.

Откройте страницу **Среды**, чтобы увидеть имеющиеся у вас типы сред и дату истечения срока их действия:

> [!div class="mx-imgBorder"] 
> ![Среды PowerApps](media/powerapps-environments75b.png "PowerApps environments")

## <a name="convert-a-trial-environment-to-production"></a>Преобразование пробной среды в рабочую

При использовании пробной среды, если вы создали ресурсы, которые хотите хранить больше чем 30 дней, преобразуйте пробную версию среды в рабочую.

Для преобразования в рабочую среду должны быть выполнены следующие условия.

**Наличие подходящего плана PowerApps**. План, который позволяет создавать рабочие среды, например PowerApps (план 2). Сведения о планах PowerApps, которые включают рабочие среды, см. в разделе [Выберите подходящий план для своей команды](https://powerapps.microsoft.com/pricing/). Сведения о том, как определить свои планы PowerApps, см. в [соответствующем разделе](#how-do-i-identify-my-plans).
**Наличие доступной рабочей квоты**. Существует ограниченное число рабочих сред, которые можно создать с помощью плана. Например, в PowerApps (план 2) можно создать две рабочие среды. Если вы уже создали две рабочие среды, вы не сможете создать третью среду, пока не удалите одну из существующих. См. дополнительные сведения о [создании среды](environments-overview.md#creating-an-environment).

Выполните следующие действия, чтобы преобразовать пробную среду в рабочую.

1. Перейдите к [https://admin.powerapps.com/environments](https://admin.powerapps.com/environments) и войдите в систему с правами администратора.
 
2. Откройте страницу **Среды** и выберите пробную среду, которую хотите преобразовать в рабочую:

    > [!div class="mx-imgBorder"] 
    > ![Выберите пробную среду](media/powerapps-environments75b-select-trial.png "Select trial environment")

3. На вкладке **Сведения** выберите **Преобразовать**:

    > [!div class="mx-imgBorder"] 
    > ![Выберите "Преобразовать"](media/powerapps-trial-select-convert.png "Select Convert")

4. Выберите **Подтвердить**:

    > [!div class="mx-imgBorder"] 
    > ![Выберите "Подтвердить"](media/powerapps-trial-select-confirm.png "Select Confirm")

При наличии в среде базы данных ее преобразование в рабочую среду может занять несколько часов. Вы можете отслеживать ход выполнения с помощью уведомлений на вкладке **Сведения**:

  > [!div class="mx-imgBorder"] 
  > ![Преобразование началось](media/powerapps-trial-conversion-started.png "Conversion started")

## <a name="frequently-asked-questions"></a>Часто задаваемые вопросы

### <a name="who-can-convert-a-trial-environment-to-a-production-environment"></a>Кто может преобразовать пробную среду в рабочую?

Для преобразования пробной среды в рабочую необходимо выполнить определенные условия.

**Наличие подходящего плана PowerApps**. Вам нужен план, который позволяет создавать рабочую среду, например PowerApps (план 2). Сведения о планах PowerApps, которые включают рабочие среды, см. в разделе [Выберите подходящий план для своей команды](https://powerapps.microsoft.com/pricing/). Сведения о том, как определить свои планы PowerApps, см. в [соответствующем разделе](#how-do-i-identify-my-plans).
**Наличие доступной рабочей квоты**. Существует ограниченное число рабочих сред, которые можно создать с помощью плана. Например, в PowerApps (план 2) можно создать две рабочие среды. Если вы уже создали две среды, вы не сможете создать третью среду, пока не удалите одну из существующих.

### <a name="what-if-i-dont-have-available-quota-for-production-environments"></a>Что делать, если нет доступной квоты для рабочих сред?

Обратитесь к глобальному администратору Office 365 или администратору клиента Azure Active Directory (Azure AD), чтобы:
- он назначил вам PowerApps (план 2); 
- найти другого пользователя, у которого есть доступная квота рабочей среды.

Кроме того, вы можете приобрести план PowerApps.

### <a name="can-every-office-365-global-admin-or-azure-ad-tenant-admin-convert-a-trial-environment-to-a-production-environment"></a>Может ли любой глобальный администратор Office 365 или администратор клиента Azure AD преобразовать пробную среду в рабочую?

Нет. Глобальные администраторы и администраторы клиентов Azure AD должны располагать доступной квотой для рабочих сред, чтобы иметь возможность преобразовать пробную среду в рабочую.

### <a name="is-there-a-way-to-recover-a-deleted-trial-environment"></a>Можно ли восстановить удаленную пробную среду?

Мы не можем гарантировать восстановление удаленной пробной среды, однако приложим все усилия, чтобы восстановить ее в течение 7 дней после удаления. Мы не сможем восстановить базу данных среды, однако приложения и потоки (созданные с помощью PowerApps) восстановить можно.

### <a name="how-can-i-retain-my-data-and-resources-if-i-dont-have-a-way-to-convert-the-trial-environment-to-a-production-environment"></a>Как сохранить данные и ресурсы, если у меня нет возможности преобразовать пробную среду в рабочую?

Ресурсы и данные можно экспортировать в другую среду. Если вы хотите сохранить их на более длительный срок, рекомендуется создать рабочую среду или отдельную среду (используя план сообщества PowerApps) и экспортировать ресурсы в эту среду. 

Ниже приведены некоторые рекомендации по экспорту ресурсов.

|Тип ресурса в среде  |Как экспортировать ресурсы?  |
|---------|---------|
|Приложения (на основе холста и модели) и потоки     |Для экспорта приложений и потоков из одной среды можно использовать [пакеты](environment-and-tenant-migration.md).         |
|Данные в базе данных (среда Common Data Service (CDS) для приложений)     |У вас есть несколько вариантов:<br/><ul><li>[экспорт в Excel](../user/export-data-excel.md) и сохранение данных; [импорт данных](../user/import-data.md) в другую среду;</li><br/><li>использование [служб интеграторов данных](data-integrator.md) и API для экспорта данных в другую среду.</li></ul> |

Мы удаляем пробные среды, для которых отсутствие действий с базой данных среды наблюдается в течение 30 дней.

### <a name="how-can-i-create-a-production-or-an-individual-environment"></a>Как создать рабочую или личную среду?

У вас должен быть план PowerApps с поддержкой создания рабочих сред. См. дополнительные сведения о [создании среды](environments-overview.md#creating-an-environment).

Личные среды можно создавать, зарегистрировавшись в [плане сообщества PowerApps](https://powerapps.microsoft.com/communityplan/). Обратите внимание, что существуют ограничения на совместное использование приложений в личных средах: эти среды предназначены только для личного использования.

### <a name="how-do-i-identify-my-plans"></a>Как узнать, какие планы у меня есть?

Чтобы узнать, какие планы у вас есть, щелкните значок **шестеренки** в правом верхнем углу сайта PowerApps и выберите пункт **Планы**.

> [!div class="mx-imgBorder"] 
> ![Выберите планы](media/powerapps-plans.png "Select Plans")
