# <a name="what-is-common-data-service"></a>Что такое Common Data Service?
Служба Common Data Service позволяет безопасно хранить данные, которые используются в бизнес-приложениях, и управлять ими. Данные в службе Common Data Service хранятся в наборе сущностей. *Сущность* — это набор записей, используемых для хранения данных, похожий на таблицу для хранения данных в базе данных. Служба Common Data Service содержит базовый набор стандартных сущностей, которые применяются в типичных сценариях, однако можно также создавать настраиваемые сущности специально для вашей организации и заполнять их данными с помощью Power Query. Затем авторы приложений могут использовать Power Apps для создания полнофункциональных приложений на основе этих данных.

![Снимок экрана с обзором платформы бизнес-приложений.](./media/data-platform-cds-intro/platform.png "Обзор платформы")

Сведения о приобретении плана для использования службы Common Data Service см. в [информации о ценах](../../administrator/pricing-billing-skus.md).

## <a name="why-use-common-data-service"></a>Для чего используют службу Common Data Service?
Стандартные и настраиваемые сущности в службе Common Data Service служат для безопасного хранения данных в облаке. Сущности позволяют создавать определения бизнес-данных в организации, которые можно использовать в приложениях. Если вы не уверены, что сущности действительно окажутся вам полезны, обратите внимание на перечисленные ниже преимущества.

* **Простота управления**. &ndash; Метаданные и данные хранятся в облаке. Не нужно беспокоиться о том, каким образом они хранятся.
* **Простота обеспечения безопасности**. &ndash; Данные хранятся в безопасности, и доступ к ним могут получить только пользователи с соответствующими правами. Безопасность на основе ролей позволяет управлять доступом к сущностям для разных пользователей в организации.
* **Доступ к данным Dynamics 365**. &ndash; Данные из приложений Dynamics 365 также хранятся в службе Common Data Service, что позволяет быстро создавать приложения, в которых используются данные Dynamics 365, и расширять приложения с помощью Power Apps.
* **Широкие возможности метаданных**. &ndash; Типы данных и отношения между данными используются непосредственно в Power Apps.
* **Логика и проверка**. &ndash; Определите вычисляемые поля, бизнес-правила, рабочие процессы и потоки бизнес-процессов, чтобы обеспечить качество данных и реализовать бизнес-процессы.
* **Средства повышения производительности**. &ndash; Сущности, доступные в надстройках для Microsoft Excel, повышают производительность и обеспечивают доступ к данным.

## <a name="dynamics-365-and-common-data-service"></a>Использование Dynamics 365 и Common Data Service

Такие приложения Dynamics 365, как Dynamics 365 Sales, Dynamics 365 Customer Service или Dynamics 365 Talent, также используют службу Common Data Service для хранения и защиты используемых ими данных. С помощью Power Apps и службы Common Data Service можно создавать приложения непосредственно на основе базовых бизнес-данных, которые уже используются в Dynamics 365. При этом не нужно выполнять интеграцию.

* **Создание приложений на основе данных Dynamics 365**. &ndash; Оперативно создавайте приложения на основе бизнес-данных в Power Apps или с помощью пакета SDK для профессиональных разработчиков.
* **Управление бизнес-логикой и бизнес-правилами с возможностью повторного использования**. &ndash; К Power Apps применяются бизнес-правила и бизнес-логика, уже определенные в ваших сущностях Dynamics 365. Это обеспечивает согласованность данных независимо от того, каким образом или с помощью какого приложения ваши пользователи осуществляют к ним доступ.
* **Повторное использование навыков в Dynamics 365 и Power Apps**. &ndash; Пользователи, у которых уже есть навыки в области работы с Power Apps или Dynamics 365, теперь могут использовать их на новой платформе службы Common Data Service. Сущности, формы, диаграммы, и т. д теперь можно создавать во всех приложениях.

    > [!NOTE]
    > Для приложений Finance и Operations сейчас требуется конфигурация [интегратора данных](/power-platform/admin/data-integrator), чтобы бизнес-данные из Finance и Operations были доступны в службе Common Data Service.

## <a name="integrating-data-into-the-common-data-service"></a>Интеграция данных в Common Data Service

При создании приложения обычно используются данные из нескольких источников. Иногда это можно реализовать на уровне приложения. Но в некоторых случаях, интегрировав эти данные в общее хранилище и используя единый набор средств логики, можно упростить создание приложения и обслуживание данных, а также управление ими. Служба Common Data Service позволяет интегрировать данные из нескольких источников в единое хранилище, которое затем можно использовать в Power Apps, Flow и Power BI вместе с данными, уже доступными в приложениях Dynamics 365.

* **Запланированная интеграция с другими системами**. &ndash; Данные, хранящиеся в другом приложении, можно регулярно синхронизировать со службой Common Data Service, чтобы использовать данные других приложений в Power Apps.
* **Преобразование и импорт данных с помощью PowerQuery**. &ndash; Преобразовать данные при импорте из нескольких подключенных источников в Common Data Service можно с помощью PowerQuery — широко используемого в Excel и Power BI средства.
* **Однократный импорт данных**. &ndash; Простую функцию импорта и экспорта файлов Excel и CSV можно использовать для однократного нечастого импорта данных в службу Common Data Service.

Дополнительные сведения об интеграции данных в Common Data Service см. в разделе [Добавление данных в сущность в Common Data Service с помощью Power Query](data-platform-cds-newentity-pq.md).

## <a name="interacting-with-entities"></a>Взаимодействие с сущностями
При разработке приложения можно использовать как стандартные, так и настраиваемые сущности (в том числе в сочетании). В службе Common Data Service доступен набор стандартных сущностей. Они разработаны в соответствии с рекомендациями и охватывают стандартные понятия и сценарии в организации.

![Снимок экрана со списком сущностей.](./media/data-platform-cds-intro/entitylist.png "Список сущностей")

Полный список сущностей см. в [справочнике по сущностям](https://docs.microsoft.com/powerapps/developer/common-data-service/reference/about-entity-reference).

Функциональные возможности стандартных сущностей можно расширить, создав одну или несколько пользовательских сущностей для хранения уникальных корпоративных данных. Дополнительные сведения см. в статье о [создании настраиваемой сущности](create-custom-entity.md).

## <a name="logic-and-validation"></a>Логика и проверка
Сущности в службе Common Data Service могут использовать логику на стороне сервера и проверки для обеспечения качества данных и сокращения объема повторяющегося кода в каждом приложении, создающем и использующем данные в сущностях.

* **Бизнес-правила** проверяют данные в нескольких полях и сущностях и выводят предупреждения и сообщения об ошибках вне зависимости от приложения, в котором создаются данные. Дополнительные сведения см. в статье о [создании бизнес-правила](./data-platform-create-business-rule.md).
* **Потоки бизнес-процессов** предоставляют пользователям указания по согласованному вводу данных и выполнению повторяющихся процедур. Потоки бизнес-процессов в настоящее время поддерживаются только для приложений на основе модели. Дополнительные сведения см. в статье [Обзор потоков бизнес-процессов](/dynamics365/customer-engagement/customize/business-process-flows-overview).
* **Рабочие процессы** позволяют автоматизировать выполнение бизнес-процессов без участия пользователя. Дополнительные сведения см. в [обзоре рабочих процессов](/dynamics365/customer-engagement/customize/workflow-processes).
* **Бизнес-логика с кодом** поддерживает сложные сценарии разработки и позволяет расширять возможности приложений посредством кода. Дополнительные сведения см. в статье [Применение бизнес-логики с помощью кода](../../developer/common-data-service/apply-business-logic-with-code.md).

## <a name="security"></a>Безопасность
В службе Common Data Service реализована комплексная модель безопасности, которая обеспечивает целостность данных и конфиденциальность пользователей, способствуя повышению эффективности совместной работы и доступа к данным. Сочетая возможности управления безопасностью на основе подразделений, ролей, записей и полей, вы можете определять общие параметры доступа к информации, которую пользователи размещают в среде Common Data Service. Дополнительные сведения: [Безопасность в Common Data Service](/power-platform/admin/wp-security)
