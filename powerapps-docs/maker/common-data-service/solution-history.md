# <a name="view-the-history-of-a-solution"></a>Просмотр журнала решения
Можно просмотреть подробные сведения об операциях решения в области **Решения** управляемого моделью приложения. Операция может быть импортом, экспортом или удалением решения. Журнал решения содержит такие сведения, как версия решения, издатель решения, тип операции, времени начала и завершения операции, а также состояние операции.

> [!div class="mx-imgBorder"] 
> ![](media/solutions-history-custom-view.png "Настраиваемое представление журнала решений")

## <a name="view-solution-history"></a>Просмотр журнала решений
1. Выберите **Настройки**, затем выберите **Журнал решений**.

     > [!div class="mx-imgBorder"] 
     > ![](media/solution-history-sitemap.png "Область журнала решений")

     > [!NOTE]
     > Чтобы перейти в область **Параметры** из управляемого моделью приложений единого интерфейса PowerApps, выберите **Параметры** ![Параметры](../model-driven-apps/media/powerapps-gear.png) на панели инструментов приложения, затем выберите **Дополнительные параметры**. 

2. По умолчанию отображается представление **Настраиваемый журнал решений**. Следующие представления доступны из области **Журнал решений**. 
- **Журнал всех решений**. Отображается журнал решений для внутренних системных решений и настраиваемых решений. 
- **Настраиваемый журнал решений**. Отображается журнал решений только для настраиваемых решений. 
- **Журнал внутренних решений**. Отображается журнал решений только для внутренних системных решений. 

Каждая запись журнала решений доступна только для чтения и включает следующие свойства: 
- **Время начала**. Время начала операции. 
- **Время окончания**. Время завершения операции. 
- **Версия решения**. Версия решения. 
- **Имя издателя**. Имя издателя, который связан с операцией. Дополнительные сведения: [Изменение префикса издателя решения](change-solution-publisher-prefix.md)  
- **Операция**. Операция, например импорт, экспорт или удаление. Дополнительные сведения: [Импорт, обновление и экспорт решений](import-update-export-solutions.md)
- **Вложенная операция**. Обозначает тип операции, например импорт нового решения или обновление существующего решения. 
- **Статус**. Текущий статус операции, такой как **Завершена** или **Не завершена**. 
- **Результат**. Результат выполнения операции, например **Успех** или **Сбой**. 
- **Код ошибки**. Код ошибки, возвращенный из операции. Код ошибки 0 означает, что операция успешно завершена. 

### <a name="view-solution-operation-error-details"></a>Просмотр сведений об ошибке операции решения 
Когда операция решения включает сбой, можно выбрать его для отображения страницы с дополнительными сведениями об ошибке. 

> [!div class="mx-imgBorder"] 
> ![](media/solution-history-with-failure.png "Журнал решения с ошибкой операции")

На странице сведений содержатся сведения, включая **Сообщение об исключении**, которые могут помочь диагностировать первопричину сбоя операции. Некоторые ошибки, включая ошибки зависимостей решений, могут содержать ссылки на **слои решения**, чтобы упростить диагностику проблемы. **Идентификаторы действия** может быть полезен в случаях, где необходимо обратиться в службу технической поддержки клиентов Майкрософт. 

> [!div class="mx-imgBorder"] 
> ![](media/solution-history-error-details.png "Сведения об ошибке операции решения")
