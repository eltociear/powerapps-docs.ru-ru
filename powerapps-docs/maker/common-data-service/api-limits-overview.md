# <a name="common-data-service-api-limits-overview"></a>Обзор границ API Common Data Service

Границы API Common Data Service помогают обеспечить уровни обслуживания, доступность и качество. Границы API Common Data Service являются частью границ запросов Power Platform и распределений. В этой теме рассмотрены границы, относящиеся к Common Data Service для приложений на основе модели в Dynamics 365 (например, Dynamics 365 Sales и Dynamics 365 Customer Service), Power Apps и Power Automate с подключением к Common Data Service/приложениям на основе модели в Dynamics 365. 

Для получения информации о границах для всех областей в пределах Power Platform см. [Границы запросов Power Platform и распределений](/power-platform/admin/api-request-limits-allocations).

Существуют две категории границ, которые применяются для Common Data Service: границы *Объем обслуживания* и *Защита служб*.

## <a name="entitlement-limits"></a>Границы объема обслуживания

Эти границы представляют количество запросов, которые пользователи имеют право делать каждый день. Выделенное ограничение зависит от типа лицензии, назначенной каждому пользователю.

Если какой-либо пользователь превысит свой объем обслуживания для запросов, администратор получит уведомление и сможет назначить объем запросов Power Apps и Power Automate для этого пользователя. Пользователям не будет заблокировано использование приложений для случайных и разумных перегрузок в этот момент времени.

Для Common Data Service запросы API включают все операции с данными, которые взаимодействуют с записями сущностей, в которых записи создаются, извлекаются, обновляются или удаляются (CRUD). Специальные операции, такие как *общий доступ* и *назначение* включены, потому что они считаются обновлениями. Эти запросы могут поступать от любого клиента или приложения и с использованием любой конечной точки. К ним относятся, помимо прочего, операции, выполняемые подключаемыми модулями, асинхронные рабочие процессы, пользовательские элементы управления и операции $batch (ExecuteMultiple). Существует небольшой набор внутренних системных операций, которые исключены, такие как операции входа в систему, выхода из системы и системных операций с метаданными.

> [!IMPORTANT]
> Распределения запросов API Power Platform включают использование Power Automate, AI Builder и API Connector. Все запросы через соединитель, которые приводят к запросу Common Data Service будут представлять 1 запрос Power Platform.

Для получения подробной информации об этих границах объема обслуживания см. [Распределения запросов Microsoft Power Platform на основе лицензий](/power-platform/admin/api-request-limits-allocations#microsoft-power-platform-requests-allocations-based-on-licenses).

Для получения информации о просмотре и распределении подключаемых модулей для производительности см. [Подключаемые модули для производительности](/power-platform/admin/capacity-add-on).

Для получения информации о приобретении отдельных подключаемых модулей для производительности см. [Руководство по лицензированию Power Apps и Power Automate](https://go.microsoft.com/fwlink/?linkid=2085130).

## <a name="service-protection-limits"></a>Границы защиты служб

Чтобы обеспечить постоянную доступность и производительность для всех, мы применяем некоторые ограничения на использование API с Common Data Service. Ограничения API защиты служб помогают гарантировать, что пользователи, выполняющие приложения, не могут мешать друг другу на основании ограничений ресурсов. Ограничения не повлияют на обычных пользователей платформы. Это затронет только приложения, в которых может выполняться большое число запросов API. Эти ограничения помогут обеспечить определенный уровень защиты от случайных и неожиданных скачков числа запроса, которые могут повлиять на доступность и характеристики производительности платформы Common Data Service.

Мы ограничиваем количество одновременных подключений на одну учетную запись пользователя, количество запросов API на подключение и количество времени выполнения, которое можно использовать для каждого подключения. Они оцениваются в течение пятиминутного скользящего окна. При превышении одного из этих ограничений платформа выдаст исключение.

> [!NOTE]
> Границы защиты служб применяются ко всем запросам, а не только к операциям CRUD с сущностями, которые учитываются в рамках границ объема обслуживания.
> 
> Поскольку подключаемые модули и настраиваемые действия рабочих процессов выполняются на сервере независимо от вошедшего в систему пользователя, границы API для защиты служб не применяются к вызовам API, выполняемым из кода подключаемого модуля.

Поскольку границы защиты служб обычно встречаются только в приложениях, выполняющих большой объем операций с данными, мы рекомендуем разработчикам, создающим эти приложения, применять шаблоны для повторных попыток выполнения операций через период времени, когда возвращаются эти исключения. Это позволит приложению реагировать на исключения, отправляемые службой, и сокращать общее количество запросов, и достигать максимально возможной пропускной способности.

Для получения информации о конкретных ошибках, которые могут быть получены, и о том, как разработчики могут применять шаблоны для реагирования на эти ошибки, см. [Границы API для защиты служб](../../developer/common-data-service/api-limits.md).
