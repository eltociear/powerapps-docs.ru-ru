# <a name="import-solutions"></a>Импорт решений 

 Решения можно импортировать вручную, используя описанные ниже действия. Импортируйте только решения,полученные из надежного источника. Настройки могут включать код, передающий данные во внешние источники.   
  
1.  Войдите в [Power Apps](https://make.powerapps.com/?utm_source=padocs&utm_medium=linkinadoc&utm_campaign=referralsfromdoc) и выберите **Решения** в левой области навигации.  
  
2.  На панели команд выберите **Импорт**.  

    > [!div class="mx-imgBorder"]  
    > ![Импорт решения](media/solution-import.png "Импорт решения") 
  
3.  На странице **Выбор пакета решения** выберите **Обзор**, чтобы найти сжатый (ZIP или CAB) файл, содержащий решение, которое следует импортировать. 
  
4.  Выберите **Далее**.  
  
5.  Отображаются сведения о решении. Выберите **Импорт**.  
  
6. Может потребоваться подождать несколько секунд, пока не будет завершен импорт. Просмотрите результаты и выберите **Закрыть**.  
  
 Если вы импортировали какие-либо изменения, которые требуется опубликовать, настройки необходимо опубликовать, прежде чем они станут доступны. 
  
 Если импорт не завершился успешно, появится отчет со всеми зафиксированными ошибками или предупреждениями. Выберите **Загрузить файл журнала** для получения сведений о том, что вызвало ошибку импорта. Чаще всего импорт заканчивается ошибкой из-за того, что решение не содержит некоторых требуемых компонентов.  
  
 Загрузив файл журнала, вы получите XML-файл, который можно открыть с помощью Office Excel для просмотра содержимого.  
  
> [!NOTE]
>  Невозможно изменить активный набор правил маршрутизации. Поэтому при импорте решения, которое включает активный набор правил маршрутизации, в среду, в которой правило с таким идентификатором уже существует, импорт завершится ошибкой. Дополнительные сведения: [Создание правил для автоматического перенаправления обращений](https://docs.microsoft.com/dynamics365/customer-engagement/customer-service/create-rules-automatically-route-cases)
