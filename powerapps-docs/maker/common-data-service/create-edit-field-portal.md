# <a name="create-and-edit-fields-for-common-data-service-using-power-apps-portal"></a>Создание и изменение полей для Common Data Service с помощью портала Power Apps

[Портал Power Apps](https://make.powerapps.com/?utm_source=padocs&utm_medium=linkinadoc&utm_campaign=referralsfromdoc) предоставляет простой способ создания и редактирования полей сущностей с помощью Common Data Service.

Портал позволяет настроить наиболее распространенные параметры, но определенные параметры можно задать только с помощью обозревателя решений. <br />Дополнительные сведения. 
- [Создание и изменение полей для Common Data Service](create-edit-fields.md)
- [Создание и изменение полей для Common Data Service с помощью обозревателя решений Power Apps](create-edit-field-solution-explorer.md)

## <a name="view-fields"></a>Просмотр полей

1. На [портале Power Apps](https://make.powerapps.com/?utm_source=padocs&utm_medium=linkinadoc&utm_campaign=referralsfromdoc)выберите **данные** > **сущности** и выберите сущность с полями, которые необходимо просмотреть.
2. Выбрав вкладку **поля** , можно выбрать следующие представления: 

 |Вид|Описание|
 |--|--|
 |**Каждого**| Показывает все поля для сущности|
 |**Управляемых**| Показывает только управляемые и стандартные поля для сущности|
 |**Настройки**|Показывает только настраиваемые поля для сущности|
 |**Параметры**|Показывает только стандартные поля для сущности|

## <a name="create-a-field"></a>Создание поля

При просмотре полей на панели команд нажмите кнопку **Добавить поле** , в котором будет отображаться панель **свойств поля** .

![Свойства поля](media/field-properties.png)

Изначально доступны только три свойства поля:

 |Свойство|Описание|
 |--|--|
 |**Отображаемое имя**|Текст, отображаемый для поля в пользовательском интерфейсе.|
 |**Name** (Наименование)|Уникальное имя в среде. Имя будет создано на основе введенного отображаемого имени, но его можно изменить перед сохранением. После создания поля имя нельзя изменить, так как на него могут ссылаться ваши приложения или код. В начале имени будет добавлен префикс настройки для **издателя Common Data Service по умолчанию** .|
 |**Тип данных**|Управляет сохранением значений, а также их форматированием в некоторых приложениях. После сохранения поля Тип данных нельзя изменить с помощью исключения преобразования текстовых полей в поля счетчика.|
 |**Обязательно**| Запись не может быть сохранена без данных в этом поле. |
 |**Возможностью поиска**| Это поле отображается в расширенном поиске и доступно при настройке представлений. |
 |**Вычисляемое или свертка**| Используется для автоматизации ручных вычислений. Используйте значения, даты или текст.|
 |**Дополнительные параметры**| Добавьте описание и укажите максимальную длину и режим IME для поля.

Можно задать дополнительные параметры в зависимости от выбранного **типа данных**.

## <a name="field-data-types"></a>Типы данных полей

Существует много различных типов полей, но только некоторые из них можно создать. Дополнительные сведения обо всех типах полей см. в разделе [типы полей и типов данных](types-of-fields.md)полей.

При создании поля **тип данных** предоставляет следующие возможности.

### <a name="text"></a>Text 

В стандартных текстовых полях может храниться до 4 000 символов. По умолчанию для параметра [максимальной длины](#max-length) задано меньшее значение, которое можно изменить.

|Тип данных|Описание|
|--|--|
|**Текст**|Текстовое значение, предназначенное для отображения в однострочном текстовом поле.|
|**Текстовая область**|Текстовое значение, предназначенное для отображения в многострочном текстовом поле. Если требуется более 4 000 символов, используйте [многострочный текстовый](#multi-line-field) тип данных.|
|**Email**|Текстовое значение, проверенное как адрес электронной почты и отображаемое в виде ссылки mailto в поле. |
|**URL-адрес**|Текстовое значение, проверенное как URL-адрес и отображаемое в виде ссылки для открытия URL-адреса.|
|**Символ деления**|Текстовое значение для символа-разделителя, которое будет отображать ссылку, открывающуюся для отображения цитаты для символа биржевого тикера. |
|**Факс**|Текстовое значение, проверяемое как номер телефона, отображаемое как ссылка для инициации телефонного звонка с помощью Skype. |
|**AutoNumber**|Настраиваемое сочетание цифр и букв, автоматически создаваемое сервером при каждом создании записи. Дополнительные сведения: [поля автонумерации](autonumber-fields.md) |

#### <a name="max-length"></a>Максимальная длина

Поля, в которых хранится текст, имеют абсолютный максимальный размер в зависимости от типа. Параметр **Max length** задает меньшее значение, чем максимальное, относящееся к вашей среде. Вы можете увеличить эту максимальную длину, но не следует понижать ее, если в системе есть данные, превышающие меньшее значение.

### <a name="whole-number"></a>Целое число

Эти поля хранят данные в виде числа, но включают различные параметры представления и проверки.

|Тип данных|Описание|
|--|--|
|**Целое число**|Числовое значение, представленное в текстовом поле.|
|**Длитель**|Числовое значение, представленное в виде раскрывающегося списка, содержащего интервалы времени. Пользователь может выбрать значение в списке или ввести целое число, обозначающее количество минут. Длительность должна быть задана в формате: "x минут", "x hours" или "x Days". Часы и дни могут также быть указаны с помощью десятичных знаков, например "x. x hours" или "x. x дн.". Указанные значения должны быть выражены в минутах, значения в подминуте будут округляться до ближайшей минуты.|
|**Стандарт**|Числовое значение, представленное в виде раскрывающегося списка, содержащего список часовых поясов.|
|**Язык**|Числовое значение, представленное в виде раскрывающегося списка, содержащего список языков, включенных для среды. Если никакие другие языки не включены, отображается только базовый язык. Сохраненное значение — это значение кода локали (LCID) для языка.|


### <a name="date-time"></a>Дата и время

Используйте эти поля для хранения значений времени. Значения можно хранить как раньше 1/1/1753 12:00 AM.

|Тип данных|Описание|
|--|--|
|**Дата и время**|Значение даты и времени.|
|**Только дата**|Значение даты и времени, которое отображает только дату. Значение времени сохраняется в системе как 12:00 AM (00:00:00).|

Можно также задать конкретное **поведение** для полей даты и времени в **дополнительных параметрах**.

- **Локальная пользователь** : отображает значения, преобразованные в в местном часовом поясе текущего пользователя. Это значение по умолчанию для новых полей.
- **Только дата**: это поведение доступно только для типа **даты** . Отображает значения без преобразования часового пояса. Используйте этот параметр для данных, таких как дни рождения и годовщины.
- **Независимо от часового пояса**: отображает значения без преобразования часового пояса.

Дополнительные сведения см. в разделе [Behavior and format of the Date and Time field](behavior-format-date-time-field.md) (Поведение и формат поля "Дата и время").

### <a name="other-data-types"></a>Другие типы данных

|Тип данных|Описание|
|--|--|
|**Валюта**|Значение Money для всех валют, настроенных для среды. Можно задать уровень точности или выбрать точность на основе определенной валюты или одной стандартной точности, используемой Организацией. Дополнительные сведения: [Использование полей валюты](types-of-fields.md#using-currency-fields)|
|**Десятичное число**| Десятичное значение с точностью до 10 пунктов. Дополнительные сведения: [использование правильного типа числа](types-of-fields.md#using-the-right-type-of-number)|
|**Файл**| Для хранения двоичных данных.|
|**Число с плавающей запятой**|Число с плавающей запятой, имеющее до 5 точек точности. Дополнительные сведения: [использование правильного типа числа](types-of-fields.md#using-the-right-type-of-number)|
|**Изображение**|Отображает один образ для каждой записи в приложении. Каждая сущность может иметь одно поле изображения. **Имя** , введенное при создании поля изображения, будет пропущено. Поля изображения всегда имеют имя "Ентитимаже".|
|**Поиск**| Создает ссылку на одну запись для одного типа целевой записи.|
|**Набор параметров множественного выбора**|Отображает список параметров, в которых можно выбрать несколько вариантов.|
|<a name="multi-line-field"></a>**Многострочный текст**|Текстовое значение, предназначенное для отображения в многострочном текстовом поле. Ограничение не должна превышать 1 048 576 символов. Кроме того, можно задать меньшую [максимальную длину](#max-length). |

|**Два параметра**|Отображает два параметра, где можно выбрать только один. Вы можете выбрать, какие метки будут отображаться для каждого параметра. Значения по умолчанию: **Да** и **нет**.|

## <a name="save-new-field"></a>Сохранить новое поле

После задания свойств **Отображаемое имя**, **имя** и **тип данных** можно нажать кнопку **Готово** , чтобы закрыть панель **свойств поля** . 

Можно продолжить редактирование сущности и добавить дополнительные поля или вернуться и продолжить редактирование этого поля. Поля не будут созданы до нажатия кнопки **Сохранить сущность** для сохранения всех изменений в сущности.

![Кнопка "Сохранить сущность"](media/save-entity-button.png)

Вы также можете нажать кнопку **отменить** , чтобы отменить внесенные изменения.
 
## <a name="edit-a-field"></a>Изменение поля

При просмотре полей выберите поле, которое нужно изменить. Можно изменить **Отображаемое имя** , но нельзя изменить **имя** и **тип данных** , если вы сохранили изменения в сущности, чтобы добавить поле.

### <a name="general-properties"></a>Общие свойства
Каждое поле имеет следующие свойства, которые можно изменить.

|Свойство|Описание|
|--|--|
|**Обязательно**|Если выбран этот элемент, запись не может быть сохранена без данных в этом поле.|
|**Возможностью поиска**|Отмените выбор этого поля для полей неиспользуемой сущности.  Если поле доступно для поиска, оно появляется в **расширенном поиске** и доступно при настройке представлений. Отмена выбора этого параметра уменьшит число параметров, отображаемых для пользователей с помощью функции расширенного поиска.|
|**Описание**|Найдено в **дополнительных параметрах**. Введите инструкции для пользователя о том, для чего предназначено поле. Эти описания отображаются в виде всплывающих подсказок для пользователя в приложениях, управляемых моделью, при наведении указателя мыши на метку поля.|

> [!NOTE]
> **Создание обязательных полей**: Будьте внимательны при создании обязательных полей. Пользователи будут испытывать трудности с использованием приложения, если они не могут сохранять записи из-за отсутствия нужных данных для ввода в обязательное поле. Пользователи могут ввести неверные данные, просто сохранив запись и приступить к работе.
>
>**Динамическое задание требования**. в приложениях на основе модели можно использовать бизнес-правила или скрипты форм для изменения уровня требований, так как данные в записи изменяются по мере того, как пользователи работают над ней. Дополнительные сведения: [Создание бизнес-правил и рекомендаций по применению логики в форме](../model-driven-apps/create-business-rules-recommendations-apply-logic-form.md)
>
>**Расширенный поиск доступности**. в настоящее время Расширенный поиск доступен только для приложений, управляемых моделью, с помощью веб-клиента. Расширенный поиск в настоящее время недоступен в клиентах единой системы.

## <a name="calculated-or-rollup"></a>Вычисляемое или свертка

Можно задать настраиваемое поле как **вычисляемое** или поле **свертки** . Поля, которые не являются вычисляемыми или полями свертки, иногда называют *простыми* полями.

### <a name="calculated"></a>Вычисляемое

С помощью вычисляемого поля можно ввести формулу для присвоения значения полю. Для этих типов данных можно задать вычисляемые поля: **Валюта**, **Дата и время**, **только дата**, **десятичное число**, **Длительность**, **Электронная почта**, **язык**, **набор параметров множественного выбора**, **набор параметров**, **текст**, **Текстовая область**, **символ**разделителя, **Часовой пояс**, **два параметра**, **URL-адрес**и **целое число**.

Дополнительные сведения: [определение вычисляемых полей для автоматизации ручных вычислений](define-calculated-fields.md)

### <a name="rollup"></a>исправлений

С помощью поля ROLLUP можно задать агрегатные функции, которые будут периодически выполняться для задания числового значения для поля. Для этих типов данных можно задать вычисляемые поля: **Валюта**, **Дата и время**, **только дата**, **десятичное число**, **Длительность**, **язык**, **Часовой пояс**и **целое число**.

Дополнительные сведения: [Определение полей свертки, которые агрегируют значения](define-rollup-fields.md)

## <a name="number-field-options"></a>Параметры числового поля

Каждый тип числового поля имеет абсолютное минимальное и максимальное значения. В этих абсолютных значениях можно задать соответствующее **минимальное** и **Максимальное** значения. Это необходимо для того, чтобы Common Data Service проверить значения данных, которые необходимо сохранить в поле.

Для типов данных **числа с плавающей** запятой и **десятичного числа** можно указать число **десятичных разрядов**.


## <a name="option-set-field-options"></a>Параметры набора параметров

Поля, предоставляющие набор параметров, могут включать собственный набор *локальных* параметров или ссылаться на общий набор *глобальных* параметров, которые могут использоваться в нескольких полях.

Использование глобального набора параметров полезно при создании одного и того же набора параметров для нескольких полей. Если задан глобальный набор параметров, то в одном месте необходимо поддерживать только набор вариантов. 

При выборе **параметра множественный выбор** или **Установка** типа данных конструктор выводит набор доступных глобальных наборов параметров для выбора из и предоставляет возможность создать **новый набор параметров**.

![Выберите тип набора параметров](media/option-set-options.png)

При выборе **варианта «создать»** поведение по умолчанию — создать новый глобальный набор параметров.

> [!NOTE]
> При редактировании параметров для нового глобального набора параметров **Отображаемое имя** и **имена** предназначены для глобального набора параметров, а не для поля. Значения по умолчанию соответствуют значениям полей, но их можно изменить во время изменения глобального параметра, чтобы оно отличалось от создаваемого в данный момент поля.

Если необходимо создать локальный набор параметров, необходимо нажать кнопку **Просмотреть другие** и выбрать **локальный набор параметров**.

![Локальный набор параметров](media/local-option-set.png)

> [!NOTE]
> Если вы определяете каждый набор параметров как глобальный параметр, установка списка глобальных наборов параметров будет расти и может быть трудно управлять. Если известно, что набор параметров будет использоваться только в одном месте, используйте локальный набор параметров.

## <a name="delete-a-field"></a>Удаление поля

С помощью роли безопасности Системный администратор можно удалить все настраиваемые поля, которые не являются частью управляемого решения. При удалении поля все данные, хранящиеся в нем, будут утеряны. Единственный способ восстановить данные из удаленного поля — восстановление базы данных из точки восстановления, созданной перед удалением поля.

> [!NOTE]
> Прежде чем можно будет удалить настраиваемое поле, необходимо удалить все зависимости, которые могут существовать в других компонентах решения. 

При [просмотре полей](#view-fields)при выборе настраиваемого поля, которое можно удалить из списка, появляется команда **удалить поле** и включена.

![Удаление поля с помощью портала](media/delete-field-portal.png)

Чтобы удалить поле, используйте команду **удалить поле** . После удаления поля необходимо сохранить изменения в сущности.

![Сохранить сущность после удаления поля](media/delete-field-portal-save-entity.png)

> [!NOTE]
> Если возникает ошибка, связанная с зависимостями, то для обнаружения зависимостей необходимо использовать обозреватель решений. Дополнительные сведения: [Проверка зависимостей полей](create-edit-field-solution-explorer.md#check-field-dependencies)

## <a name="ime-mode"></a>Режим IME

Любое поле, предоставляющее прямой ввод текста, имеет режим IME. Редактор методов ввода (IME) используется для восточно-азиатских языков, например японского. Редакторы IME позволяют пользователю вводить тысячи различных символов, используемых в восточноазиатских языках, с помощью стандартной клавиатуры с 101-клавишей.
