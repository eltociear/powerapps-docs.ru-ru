# <a name="use-solutions-in-powerapps"></a>Использование решений в PowerApps

 Внутри PowerApps можно просматривать список решений, выбрав **Решения** в левой части навигационной панели. После этого можно выбрать решение для просмотра всех его компонентов. 
 
> [!div class="mx-imgBorder"]  
> ![Демонстрационное решение со всеми компонентами](media/solution-all-items-list.PNG "Демонстрационное решение со всеми компонентами")  
 
> [!NOTE]
>  Опыт работы с решениями доступен только онлайн и для среды версии 9.1.0.267 и более поздних версий. Чтобы проверить свою версию, последовательно перейдите ...[Центр администрирования PowerApps](https://admin.powerapps.com/)> **Среды** > Выберите вашу среду > вкладка **Детали**. Для сред более ранней версии при выборе решения оно открывается в классическом интерфейсе.  
 
 Вы можете просматривать все компоненты решения, прокручивая элементы. Если в списке более 100 элементов, вы можете выбрать **Загрузить следующие 100 элементов**, чтобы увидеть больше. 
 
> [!div class="mx-imgBorder"]  
> ![Загрузить еще компоненты](media/load-more.PNG "Загрузить еще компоненты")  

 ## <a name="search-and-filter-in-a-solution"></a>Поиск и фильтрация в рамках решения
 
 Можно также выполнить поиск определенного компонента по имени. 
 
> [!div class="mx-imgBorder"]  
> ![Поиск компонента](media/solution-search-box.png "Поиск компонента")  
 
 Или отфильтруйте все элементы в списке по типу компонента.
  
> [!div class="mx-imgBorder"]  
> ![Фильтровать компоненты по типу](media/solution-filter.PNG "Фильтровать компоненты по типу")  
 
 ## <a name="contextual-commands"></a>Контекстные команды
 
 При выборе каждого компонента действия, доступные в командной строке, будут меняться в зависимости от типа компонента, который вы выбрали, а также от того, является ли решение решением по умолчанию или управляемым решением. 
 
> [!div class="mx-imgBorder"]  
> ![Определенные команды компонентов](media/component-commands.png "Определенные команды компонентов")  
 
 Когда вы не выбираете какой-либо компонент, на панели команд будут отображаться действия, применяемые к самому решению. 
 
> [!div class="mx-imgBorder"]  
> ![Определенные команды решений](media/solution-commands.PNG "Определенные команды решений")  
 
 ## <a name="create-components-in-a-solution"></a>Создание компонентов в решении
 В случае с неуправляемыми решениями или решениями по умолчанию вы можете использовать команду **Создать** для создания различных типов компонентов. Это приведет вас к другому созданию в зависимости от выбранного вами типа компонента. После того, как вы закончите создание компонента, он будет добавлен в решение. 
 
> [!div class="mx-imgBorder"]  
> ![Создание нового компонента в решении](media/solution-new-component.PNG "Создание нового компонента в решении")  
 
 ## <a name="add-an-existing-component-to-a-solution"></a>Добавление существующего компонента в решение
 
 При работе с решениями, которые являются неуправляемыми и не являются решениями по умолчанию, можно использовать команду **Добавить существующее**, чтобы добавить в них компоненты, которые еще не присутствуют в решении.  
 
> [!div class="mx-imgBorder"]  
> ![Добавить существующий компонент в решение](media/solution-add-existing-component.PNG "Добавить существующий компонент в решение")  
  
 В случае управляемых решений только определенные команд будут доступны, и появится сообщение, как описано ниже. Вам понадобится найти компонент в решении под названием **Решение по умолчанию** и попробовать изменить его там или добавить его в другое неуправляемое решение, которое вы создали. Компонент может не настраиваться. Дополнительные сведения: [Управляемые свойства](solutions-overview.md#managed-properties)

> [!div class="mx-imgBorder"]  
> ![Управляемое решение](media/managed-solution.PNG "Управляемое решение")  

 Многие из настроек, которые могут вам понадобиться, предполагают использование сущностей. Можно использовать фильтр **Сущности**, чтобы отобразить список всех сущностей в данном решении, которые можно настроить тем или иным образом. При подробном ознакомлении с сущностью можно также развернуть каждую сущность, чтобы увидеть компоненты решения, которые входят в состав сущности, как показано для сущности организации на следующем снимке экрана. 
   
> [!div class="mx-imgBorder"]  
> ![Демонстрационное решение, показывающее развернутую сущность организации](media/solution-entity-account.png "Демонстрационное решение, показывающее развернутую сущность организации")  

## <a name="classic-solution-explorer"></a>Классический обозреватель решений

В PowerApps можно просматривать классический обозреватель решений выбрав **Решения** в левой части навигационной панели, а затем **Перейти в классический режим** в командной строке. Классический обозреватель решений — это тот, который ранее был доступен в области **Параметры > Расширенные настройки** в PowerApps. Если вы являетесь пользователем Dynamics 365 для Customer Engagement, вы используете классический обозреватель решений для работы с решениями.  

## <a name="known-limitations"></a>Известные ограничения

- Пользовательские соединители недоступны в решении.
- Приложения на основе холста необходимо открывать после завершения импорта решения для обновления подключений.
- Если приложение на основе холста упаковано в управляемое решение, его можно по-прежнему изменить в целевой среде.
- Зависимости не доступны для приложений на основе холста.
- Удаление управляемого решения нельзя будет откатить до другой версии приложения на основе холста. 
-   Доступ в приложение на основе холста (CRUD и безопасность) регулируется полностью через PowerApps, а не в базе данных Common Data Service (Common Data Service).
-   API-интерфейсы Common Data Service для вызова приложений на основе холста блокируются и ничего не возвращают. 
-   Приложения на основе холста и потоки, созданные в решении, не могут совместно использоваться в качестве совладельца группы безопасности AAD.
-   Приложения на основе холста не отображаются в классическом обозревателе решений.
- Потоки, запускаемые кнопкой, недоступны в решениях.
- Приложения на основе холста, запускаемые кнопкой, недоступны в решениях.
- Потоки, запускаемые из приложений от Microsoft 365, такие как SharePoint и в Excel, недоступны в решениях.
