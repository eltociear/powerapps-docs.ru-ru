# <a name="use-solutions-in-power-apps"></a>Использование решений в Power Apps

 Внутри Power Apps можно просматривать список решений, выбрав **Решения** в левой части навигационной панели. После этого можно выбрать решение для просмотра всех его компонентов. 
 
> [!div class="mx-imgBorder"]  
> ![Демонстрационное решение со всеми компонентами](media/solution-all-items-list.PNG "Демонстрационное решение со всеми компонентами")  
 
> [!NOTE]
>  Опыт работы с решениями доступен только онлайн и для среды версии 9.1.0.267 и более поздних версий. Чтобы проверить свою версию, перейдите на вкладку ...[Центр администрирования Power Apps](https://admin.powerapps.com/)> **Среды** > Выберите вашу среду > **Подробности**. Для сред более ранней версии при выборе решения оно открывается в классическом интерфейсе.  
 
 Вы можете просматривать все компоненты решения, прокручивая элементы. Если в списке более 100 элементов, вы можете выбрать **Загрузить следующие 100 элементов**, чтобы увидеть больше. 
 
> [!div class="mx-imgBorder"]  
> ![Загрузить дополнительные компоненты](media/load-more.PNG "Загрузить дополнительные компоненты")  

 ## <a name="search-and-filter-in-a-solution"></a>Поиск и фильтрация в рамках решения
 
 Можно также выполнить поиск определенного компонента по имени. 
 
> [!div class="mx-imgBorder"]  
> ![Поиск компонента](media/solution-search-box.png "Поиск компонента")  
 
 Или отфильтруйте все элементы в списке по типу компонента.
  
> [!div class="mx-imgBorder"]  
> ![Фильтровать компоненты по типу](media/solution-filter.PNG "Фильтровать компоненты по типу")  
 
 ## <a name="contextual-commands"></a>Контекстные команды
 
 При выборе каждого компонента действия, доступные в командной строке, будут меняться в зависимости от типа компонента, который вы выбрали, а также от того, является ли решение решением по умолчанию или управляемым решением. 
 
> [!div class="mx-imgBorder"]  
> ![Команды, специфичные для компонентов](media/component-commands.png "Команды, специфичные для компонентов")  
 
 Когда вы не выбираете какой-либо компонент, на панели команд будут отображаться действия, применяемые к самому решению. 
 
> [!div class="mx-imgBorder"]  
> ![Команды, специфические для решений](media/solution-commands.PNG "Команды, специфические для решений")  
 
 ## <a name="create-components-in-a-solution"></a>Создание компонентов в решении
 В случае с неуправляемыми решениями или решениями по умолчанию вы можете использовать команду **Создать** для создания различных типов компонентов. Это приведет вас к другому созданию в зависимости от выбранного вами типа компонента. После того, как вы закончите создание компонента, он будет добавлен в решение. 
 
> [!div class="mx-imgBorder"]  
> ![Создание нового компонента в решении](media/solution-new-component.PNG "Создание нового компонента в решении")  
 
 ## <a name="add-an-existing-component-to-a-solution"></a>Добавление существующего компонента в решение
 
 При работе с решениями, которые являются неуправляемыми и не являются решениями по умолчанию, можно использовать команду **Добавить существующее**, чтобы добавить в них компоненты, которые еще не присутствуют в решении.  
 
> [!div class="mx-imgBorder"]  
> ![Добавление существующего компонента в решение](media/solution-add-existing-component.PNG "Добавление существующего компонента в решение")  
  
 В случае управляемых решений только определенные команд будут доступны, и появится сообщение, как описано ниже. Следует добавить его в другое неуправляемое решение, которое вы создали для настройки компонента. Компонент может не настраиваться. Дополнительные сведения: [Управляемые свойства](solutions-overview.md#managed-properties)

> [!div class="mx-imgBorder"]  
> ![Управляемое решение](media/managed-solution.PNG "Управляемое решение")  

 Многие из настроек, которые могут вам понадобиться, предполагают использование сущностей. Можно использовать фильтр **Сущности**, чтобы отобразить список всех сущностей в данном решении, которые можно настроить тем или иным образом. При подробном ознакомлении с сущностью можно также развернуть каждую сущность, чтобы увидеть компоненты решения, которые входят в состав сущности, как показано для сущности организации на следующем снимке экрана. 
   
> [!div class="mx-imgBorder"]  
> ![Демонстрационное решение, показывающее расширенный объект организации](media/solution-entity-account.png "Демонстрационное решение, показывающее расширенный объект организации")  

## <a name="classic-solution-explorer"></a>Классический обозреватель решений

В Power Apps можно просматривать классический обозреватель решений выбрав **Решения** в левой части навигационной панели, а затем **Перейти в классический режим** в командной строке. Классический обозреватель решений — это тот, который ранее был доступен в области **Параметры > Расширенные настройки** в Power Apps. 

## <a name="known-limitations"></a>Известные ограничения

Применяются следующие ограничения для использования приложений холста, потоков и настраиваемых соединителей в решениях. 

- Потоки приложения на основе холста, запускаемые кнопкой, в настоящее время не поддерживаются в решениях. Создайте приложение и поток вне решения, затем экспортируйте файл .msapp для переноса приложений холста со встроенным потоком, запускаемым кнопкой. 
- Если приложение на основе холста упаковано в управляемое решение, его невозможно изменить и заново опубликовать в целевой среде. Используйте неуправляемые решения, если приложения требуют редактирования в целевой среде. 
- Подключения требуют проверки подлинности и разрешения, что требует интерактивной пользовательской сессии и поэтому не может быть транспортировано через решения. После импорта решения воспроизведите решение для проверки подлинности подключений. Вы также можете создать соединения в целевой среде перед импортом решения. 
-   Приложения на основе холста в качестве совладельца группы безопасности Azure Active Directory (AAD), не могут быть добавлены в решения. Отмените общий доступ к приложению перед добавлением его в решение.
-   Приложения на основе холста не отображаются в классическом обозревателе решений. Используйте современное взаимодействие.
-   Доступ в приложение на основе холста (CRUD и безопасность) регулируется полностью через Power Apps, а не в базе данных Common Data Service.
- Операции базы данных, такие как создание резервной копии, восстановление и копирование, не поддерживаются для приложений холста и потоков. Эти операции могут повредить приложения холста и потоки.
- Удаление управляемого решения нельзя откатить до другой версии приложения на основе холста. Напротив, все версии приложений удаляются.
- Импорт решения, содержащего поток, автоматически не приведет к созданию или связыванию необходимых соединений. Поток необходимо изменить для исправления подключений.
  - Если используются управляемые решения, при этом создается активная настройка в неуправляемом слое. Следовательно, последующие обновления решения в потоке не будут отражены. 
- Потоки, созданные из решений, не будут отображаться в списке "Team Flows". Необходимо получить доступ к ним через решение. 
- Потоки, запускаемые кнопкой, недоступны в решениях.
- Потоки, запускаемые из приложений от Microsoft 365, такие как Excel, недоступны в решениях.
- Потоки, подключаемые к SharePoint, недоступны в решениях.
- Потоки в решениях не поддерживают делегированную проверку подлинности. Например, доступ к потоку не будет автоматически предоставляться на основе наличия доступа к списку SharePoint, из которого был создан поток.
- Пользовательские соединители, созданные вне решений, нельзя добавлять в решения в это время.
