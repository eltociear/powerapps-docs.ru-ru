# <a name="create-guided-help-learning-path-for-your-app"></a>Создание управляемой справки (схемы обучения) для приложения

Используйте схему обучения, чтобы предоставить пользователям управляемую, встроенную в приложение справку, которую можно настроить согласно требованиям среды, определенному использованию и бизнес-процессу организации. 

> [!IMPORTANT]
> Схема обучения доступна только со старыми приложениями веб-клиента. Используйте настраиваемые страницы справки для приложений единого интерфейса. Дополнительные сведения: [Создание управляемой справки для приложения единого интерфейса](../common-data-service/create-custom-help-pages.md)

Схема обучения облегчает освоение и адаптацию приложений и организационные процессы, гарантирует, что данные вводятся и интерпретируются согласованно, и снижает количество ошибок и звонков в службу поддержки, создаваемых пользователями. [Просмотрите короткое видео (1:50) о схеме обучения](https://community.dynamics.com/crm/b/crmvideos/archive/2016/05/09/introducing-learning-path-for-dynamics-crm).

## <a name="how-is-learning-path-different-from-customizable-help"></a>Как схема обучения отличается от настраиваемой справки?  
 Настраиваемая справка позволяет переопределить справку приложений [!INCLUDE[pn_crm_shortest](../../includes/pn-crm-shortest.md)] по умолчанию и перенаправить пользователей в организации на другой URL-адрес для получения справки. Либо можно переопределить справку для сущности с широкими возможностями настройки, чтобы лучше описать бизнес-процесс. 

 Схема обучения позволяет добавить настраиваемую справку, которую пользователи смогут просмотреть в приложении при открытии страницы, выполнении действия или выборе кнопки справки (?).   

 Дополнительные сведения о настраиваемой справке см. в разделе [Настройка справки](https://technet.microsoft.com/library/dn832079.aspx).

## <a name="prerequisites"></a>Необходимые компоненты  

 Для создания содержимого схемы обучения необходимо:  

- Используйте PowerApps или [!INCLUDE[pn_crm_online_shortest](../../includes/pn-crm-online-shortest.md)].  

- Согласиться на использование схемы обучения. Этот параметр включен по умолчанию, но мог быть отключен.  

   Чтобы убедиться, что схема обучения включена: на панели навигации выберите **Параметры** ![Значок параметров](media/optionsbutton.png "Значок «Параметры»") > **Согласиться на схему обучения**.  

   Дополнительные сведения: [Переключатель плана обучения (управляемой справки)](/dynamics365/customer-engagement/admin/on-off-switch-for-learning-path-guided-help).  

- Иметь роль настройщика системы или системного администратора либо другую роль, с привилегиями на создание схемы обучения.  

- Включить разработку схемы обучения. При этом создается группа безопасности авторов схемы обучения [!INCLUDE[pn_Office_365](../../includes/pn-office-365.md)].  

- Быть участником группы безопасности авторов схемы обучения [!INCLUDE[pn_Office_365](../../includes/pn-office-365.md)].  

  Можно создавать содержимое схемы обучения для модулей веб-приложений и модулей приложений единого интерфейса. Это включает [!include[](../../includes/pn-dyn-365-tablets.md)].  

<a name="TurnOnLP"></a>   
## <a name="turn-on-learning-path-for-your-organization"></a>Включение схемы обучения для организации  
 Схема обучения — дополнительная функция, которую можно включить или отключить для организации. Можно отобразить содержимое схемы обучения, включенное в [!INCLUDE[pn_crm_shortest](../../includes/pn-crm-shortest.md)], создать собственное содержимое схемы обучения для пользователей или и то, и другое.  

1. Войдите в [PowerApps](https://make.powerapps.com/?utm_source=padocs&utm_medium=linkinadoc&utm_campaign=referralsfromdoc) или [!INCLUDE[pn_crm_shortest](../../includes/pn-crm-shortest.md)], используя учетную запись администратора.  

2. Откройте **Параметры** и выберите **Администрирование** в разделе **Система**. Дополнительные сведения: [Параметры](/powerapps/maker/model-driven-apps/advanced-navigation#settings)

3. На странице **Администрирование** выберите **Системные параметры**.  

4. На вкладке **Общие сведения** в разделе **Задать URL-адрес настраиваемой справки** выберите **Да** для параметров **Включить схему обучения** и **Включить разработку схемы обучения**.  

    Можно включить схему обучения или настраиваемую справку, но не обе справки одновременно. Подтвердите, чтоб для параметров **Использовать настраиваемую справку для настраиваемых сущностей** и **Присоединить параметры к URL-адресу** задано значение **Нет**.  

     ![Диалог системных параметров с параметрами для выбора включения разработки схемы обучения](media/lp-system-settings.png "Диалоговое окно системных параметров с параметрами для выбора включения разработки схемы обучения")  

5. Нажмите **ОК**.

## <a name="add-a-user-to-the-office-365-learning-path-authors-security-group"></a>Добавление пользователя в группу безопасности авторов схемы обучения Office 365  
 Если вы не являетесь участником группы безопасности авторов схемы обучения [!INCLUDE[pn_Office_365](../../includes/pn-office-365.md)], появится следующее сообщение об ошибке при открытии библиотеки содержимого схемы обучения.  

 ![Сообщение об ошибке, указывающее, что вы не входите в группу безопасности схемы обучения](media/lp-o365-security-group.png "Сообщение об ошибке, указывающее, что вы не входите в группу безопасности схемы обучения")  

#### <a name="add-a-user"></a>Добавление пользователя  

1. Перейдите на портал администрирования клиента [!INCLUDE[pn_Office_365](../../includes/pn-office-365.md)], выбрав кнопку **Переход к другим приложениям** в левом верхнем углу страницы при входе в [!INCLUDE[pn_crm_shortest](../../includes/pn-crm-shortest.md)], а затем выберите **Администрирование**.  

    Может отобразиться запрос на повторный ввод пароля.  

2. На странице **Центр администрирования** выберите **Группы**.  

3. На странице **Группы** выберите группу безопасности **Авторы схемы обучения**.  

4. В строке **Участники** выберите **Изменить** для добавления пользователей в группу.  

5. Выберите **+ Добавить участников**, а затем укажите или найдите пользователей, которых необходимо добавить в группу.  

6. Выберите **Сохранить** по завершении добавления пользователей.  

> [!NOTE]
>  Другой способ назначить группу учетной записи пользователя — выберите **Пользователи** > **Активные пользователи**, выбрать пользователя, которого нужно добавить, а затем выбрав **Изменить** рядом с разделом **Членства в группе** для выбора групп, в которые требуется добавить пользователя.  

<a name="MultipleOrgs"></a>   
## <a name="how-does-learning-path-work-with-multiple-organizations"></a>Как схема обучения работает с несколькими организациями?  
 При публикации содержимого схемы обучения можно использовать среды публикации для управления тем, в какие организации, связанные с клиентом, будет публиковаться содержимое. Чтобы опубликовать различное содержимое в различные организации, создайте несколько сред публикации и добавьте каждую организацию в одну или несколько из них.  

<a name="SecurityRoles"></a>   
## <a name="learning-path-and-common-data-service-security-roles"></a>Схема обучения и роли безопасности Common Data Service 
 
 В Common Data Service роли безопасности используются для определения того, какое содержимое схемы обучения отображается, когда пользователь выбирает кнопку справки, переходит на страницу или выполняет определенное действие в Common Data Service.  

 Роли, используемые в схеме обучения, — это те же роли, которые используются в организации Common Data Service для безопасности и доступа к данным, но можно создать содержимое схемы обучения для любой или всех ролей безопасности Common Data Service.  Как правило, требуется, чтобы роли безопасности в конструкторе Схемы обучения совпадали со средой Common Data Service. Однако можно упростить пользовательский интерфейс в конструкторе, скрыв некоторые роли безопасности Common Data Service из конструктора. Если позже вы решите, что вам требуется использовать роль безопасности, которая была удалена из схемы обучения, вы можете синхронизировать роли между схемой обучения и Common Data Service.  

 Если в организации несколько подразделений, роли безопасности могут иметь отношения "родительская-дочерняя". Синхронизируются только роли безопасности в корневом подразделении.  

 Дополнительные сведения о ролях безопасности: [Роли безопасности и привилегии](/dynamics365/customer-engagement/admin/security-roles-privileges)  

<a name="Precedence"></a>   
### <a name="learning-path-role-precedence"></a>Приоритет ролей схемы обучения  
 Если пользователю в организации назначено несколько ролей безопасности, используется приоритет для определения того, какая назначенная роль безопасности используется для отображения содержимого схемы обучения. Если содержимое схемы обучения связывается с ролью безопасности, любой пользователь, назначенный этой роли, сможет просмотреть содержимое схемы обучения, даже если ему назначена роль безопасности с более высоким приоритетом, которая не связана с содержимым схемы обучения.  

 Каждая роль в схеме обучения имеет числовое значение. Первая роль в списке имеет самый высокий приоритет, и последующие роли имеют более низкий приоритет. Когда пользователю назначается роль, которая вызывает отображение содержимого схемы обучения, он видит это содержимое, даже если ему была назначена роль с более низким приоритетом, которая не связана с содержимым. Например, если пользователю назначается роль с приоритетом 1 и роль с приоритетом 20, пользователь сможет просматривать содержимое схемы обучения, определенное только для роли с приоритетом 1.  

 Если создать различное содержимое для различных ролей на той же странице приложения на основе модели или экране, пользователи смогут просматривать содержимое, связанное с ролью с более высоким приоритетом.  

<a name="ManageRoles"></a>   
### <a name="manage-security-roles-and-precedence"></a>Управление ролями безопасности и приоритетом  
 Можно управлять тем, какие роли безопасности будут доступны в конструкторе схемы обучения, и задавать порядок ролей для определения приоритета при выполнении схемы обучения. Если пользователь имеет две роли и разное содержимое опубликовано для данного контекста каждой из этих ролей, пользователь сможет просматривать содержимое для роли, которая находится выше по порядку в списке.  

<a name="ConfigureRoles"></a>   
#### <a name="configure-security-roles"></a>Настройка ролей безопасности  

1. Войдите в PowerApps с помощью учетной записи, имеющей соответствующие разрешения схемы обучения.  

2. Откройте раздел **Библиотека содержимого**.  

3. Выберите **Конфигурация** в верхней части экрана.  

4. Для синхронизации ролей безопасности с вашими ролями безопасности Common Data Service выберите **Синхронизировать роли**.  

5. Чтобы настроить порядок приоритета для ролей, используемых в схеме обучения, используйте стрелку вверх или вниз для перемещения роли на более высокую или низкую позицию в списке.  

    Приоритет определяется порядком ролей, перечисленных на этой странице.  

6. Для удаления роли из схемы обучения выберите **Удалить** рядом с ролью.  

   > [!NOTE]
   >  Эта роль не будет удалена из [!INCLUDE[pn_crm_shortest](../../includes/pn-crm-shortest.md)]. Она удаляется в конструкторе схемы обучения для определения того, как содержимое отображается для пользователей. Вы всегда можете восстановить скрытую роль, выбрав **Синхронизировать роли**.  

7. После внесения изменений выберите **Сохранить**.  

8. Выберите **Назад** для возврата в библиотеку содержимого.  

<a name="MobileApp"></a>   
## <a name="create-learning-path-controls-for-includepn_dyn_365_tabletsincludespn-dyn-365-tabletsmd"></a>Создание элементов управления "Схема обучения" для [!INCLUDE[pn_dyn_365_tablets](../../includes/pn-dyn-365-tablets.md)]  
 Можно создать элементы управления схемы обучения для [!INCLUDE[pn_dyn_365_tablets](../../includes/pn-dyn-365-tablets.md)] так же, как создаются элементы управления для веб-клиента. Для этого следует использовать имитатор мобильного приложения в веб-браузере, чтобы у вас был доступ к мобильному пользовательскому интерфейсу для закрепления элементов управления "Схема обучения". Этот имитатор должен использоваться только для этой цели.  


### <a name="display-the-mobile-app-interface-simulator-in-a-web-browser"></a>Отображение имитатора интерфейса мобильного приложения в веб-браузере  

1. Выполните вход в [!INCLUDE[pn_crm_shortest](../../includes/pn-crm-shortest.md)].  

2. Скопируйте имя сервера для организации Common Data Service из URL-адреса, который отображается в браузере, например *<https://contososales.crm.dynamics.com/>*.  

    Н забудьте вставить символ косой черты (/) после ".com".  

3. Определите уникальное имя организации (также называется экземпляром), для которой требуется создать элементы управления схемы обучения. Чтобы получить уникальное имя, на карте сайта выберите **Параметры** > **Настройки**, а затем на странице **Настройка** выберите **Ресурсы для разработчиков**. Скопируйте значение для поля **Уникальное имя**, которое отображается в разделе **Ссылка на экземпляр**.  

   ![Имя организации Dynamics, отображаемое на панели сведений о пользователе](media/lp-org-name.png "Имя организации Dynamics, отображаемое на панели сведений о пользователе")  

4. Добавьте следующее к первой части URL-адреса организации, заменив \<имя_организации> на уникальное имя организации, определенное на предыдущем шаге:  

    nga/main.htm?org=*\<имя_организации>*  

    Ваш URL-адрес должен выглядеть аналогично следующему: https://contososales.crm.dynamics.com/nga/main.htm?org=orgb557e46a  

5. Добавьте следующее к URL-адресу на предыдущем шаге, заменив \<имя_сервера> на имя сервера на первом шаге в этой процедуре:  

    &server=*\<имя_сервера>*  

    Ваш URL-адрес должен выглядеть аналогично следующему: https://contososales.crm.dynamics.com/nga/main.htm?org=orgb557e46a&server=https://contososales.crm.dynamics.com/.  

6. Откройте новую вкладку или окно браузера и скопируйте полный созданный URL-адрес, вставьте его в новую вкладку или окно браузера, а затем выберите ВВОД.  

    При первом подключении к интерфейсу мобильного приложения отобразится экран приветствия, пока система обрабатывает метаданные и настройки загрузок. После завершения отобразится рабочее пространство.  

   > [!NOTE]
   >  При подключении к версии мобильного приложения интерфейса для проверки подлинности пользователя используются учетные данные для входа в веб-интерфейс. Вам потребуется оставить веб-интерфейс открытым, чтобы избежать ошибок отказа в доступе при открытии интерфейса мобильного приложения.  

7. Закройте рабочее пространство для отображения домашней страницы интерфейса мобильного приложения в браузере. Можно открыть библиотеку содержимого для создания или изменения элементов управления "Схема обучения". Дополнительные сведения: [Библиотека содержимого](#ContentLibrary)  


<a name="ContentLibrary"></a>   
## <a name="content-library"></a>Библиотека контента  
 Библиотека содержимого отображает содержимое, созданное и доступное для вашей организации, в дополнение к командам для создания, управления и взаимодействия с элементами управления. Для создания или изменения элементов управления схемы обучения сначала подключитесь к интерфейсу клиента, для которого требуется создать элементы управления, а затем откройте библиотеку содержимого.  

**Чтобы открыть библиотеку содержимого из интерфейса веб-клиента по умолчанию, выполните одно из следующих действий:**  

-   На боковой панели выберите кнопку **Библиотека содержимого**.  

     ![Значок библиотеки содержимого, отображаемый на боковой панели схемы обучения](media/lp-sidebar-cl-icon.png "Значок библиотеки содержимого, отображаемый на боковой панели схемы обучения")  

-   Выберите плитку **обучение** на карте сайта, затем выберите **Библиотека содержимого**.  

     ![Значок библиотеки содержимого на карте сайта приложения на основе модели](media/lp-sitemap-content-library.png "Значок библиотеки содержимого на карте сайта приложения на основе модели")  

**Чтобы открыть библиотеку содержимого из имитатора интерфейса мобильного приложения:**  

1.  Выберите кнопку многоточия в кружке (...) в правом нижнем углу экрана.  

    ![Кнопка многоточия для отображения значков схемы обучения](media/lp-cl-ellipses.png "Кнопка многоточия для отображения значков схемы обучения")  

2.  Выберите **Библиотека содержимого схемы обучения**.

<a name="ContentTypes"></a>   
## <a name="learning-path-content-types-guided-tasks-and-sidebars"></a>Типы контента схемы обучения. Управляемые задачи и боковые панели  
 Можно создать два типа содержимого в схеме обучения: управляемые задачи и боковые панели.  

<a name="GT"></a>   
### <a name="guided-tasks"></a>Управляемые задачи  
 Управляемая задача обычно является последовательностью шагов, хотя она также может быть одним шагом. Пользователь может запустить управляемую задачу, выбрав ссылку на боковой панели, перейдя на страницу либо выбрав ссылку на странице, для которой создается содержимое. На каждом шаге пользователь выбирает кнопку **Далее** или выполняет определенное действие для перехода к следующему шагу либо для завершения управляемой задачи.  

 Управляемые задачи полезны для ведения пользователей через общие или новые задачи. Они также могут использоваться для обеспечения того, что задачи выполняются согласованно в организации, или того, что данные вводятся определенным образом для поддержки процессов или бизнес-правил организации. Можно включить ссылки, видеоматериалы, а также другие сведения в управляемые задачи, чтобы помочь пользователям изучать и получать дополнительную информацию о части пользовательского интерфейса, на который ссылается шаг.  

<a name="Sidebar"></a>   
### <a name="sidebars"></a>Боковые панели  
 Боковая панель отображается на экране, когда пользователь выбирает кнопку справки, переходит на страницу или выбирает ссылку либо кнопку на странице, для которой создается содержимое. Можно также создать боковые панели "Домашняя страница", которые отображаются, когда пользователь открывает страницу или экран либо выбирает значок "Домашняя страница" на боковой панели.  

 ![Значок "Домашняя страница" на боковой панели](media/sidebar-home.png "Значок "Домашняя страница" на боковой панели")  

 Можно также определить боковые панели "Ошибка", которые отображаются, когда возникает проблема с отображением нужной боковой панели. Можно включить ссылки, видеоматериалы, а также другие сведения в боковые панели, чтобы помочь пользователям ознакомиться и изучить отображаемую страницу или форму либо действия, которые они могут выполнить на странице или в форме.

1.  Определите способ запуска задачи и назначьте роли, к которым применяется содержимое.  

2.  С помощью редактора процессов добавьте шаги, которые будут отображаться для пользователей по мере выполнения шагов управляемой задачи.

### <a name="add-steps-with-the-flow-editor"></a>Добавление шагов в редактор процессов  

1. Добавьте заголовок в место, где отображается *Заголовок управляемой задачи*. Это заголовок, который увидят пользователи.  

2. Выберите, следует ли отображать только элементы управления с фиксированными кодами. Зарегистрированные элементы управления выделяются зеленым цветом, а незарегистрированные элементы управления выделяются синим цветом при перемещении плитки для закрепления в пользовательском интерфейсе. При закреплении шага на элементе управления, у которого нет фиксированного кода, на него может повлиять будущее обновление до [!INCLUDE[pn_crm_shortest](../../includes/pn-crm-shortest.md)]. Обновления не повлияют на элементы управления с фиксированными кодами.  

3. Выберите **Добавить новый шаг** и выберите тип шага, который нужно использовать для первого шага управляемой задачи.  

   |Тип кнопки|Описание|  
   |-----------------|-----------------|  
   |**Шаг с кнопкой "Далее"**|Этот шаг имеет кнопку "Далее", которую можно нажать для перехода к следующему шагу в процессе. Если это последний шаг в процессе, кнопка "Далее" не будет отображаться. Перетащите плитку для закрепления шага в том месте, в котором он должен отображаться в приложении.|  
   |**Шаг с действием пользователя**|Этот шаг не имеет кнопку "Далее". Пользователь получает запрос на выбор элемента пользовательского интерфейса, на котором закреплен шаг. Если в результате этого выбора происходит перенаправление на другую страницу или какое-либо изменение состояния пользовательского интерфейса, следите за тем, чтобы следующий шаг был прикреплен к измененному состоянию интерфейса. Перетащите плитку для закрепления шага в том месте, в котором он должен отображаться в приложении.<br /><br /> Закрепите этот тип шага на активном элементе управления в пользовательском интерфейсе, например кнопке или ссылке.|  
   |**Действие пользователя с кнопкой "Далее"**|Этот шаг имеет кнопку "Далее". Выбор кнопки "Далее" имеет тот же эффект, что и выбор элемента пользовательского интерфейса, на котором закреплен шаг. Если в результате этого выбора происходит перенаправление на другую страницу или какое-либо изменение состояния пользовательского интерфейса, следите за тем, чтобы следующий шаг был прикреплен к измененному состоянию интерфейса. Перетащите плитку для закрепления шага в том месте, в котором он должен отображаться в приложении.|  
   |**Шаг обучения**|У этого шага имеется настраиваемая кнопка, выступающая в качестве конечного действия. Этот шаг может находиться только в конце потока управляемой задачи. Его можно использовать для ссылки на боковую панель схемы обучения. Перетащите плитку для закрепления шага в том месте, в котором он должен отображаться в приложении.|  

   > [!NOTE]
   >  Если вы уже запланировали управляемую задачу и знаете, какие шаги требуется добавить, вы можете добавить их все сейчас. Каждый добавляемый шаг будет отображаться в редакторе процессов в порядке добавления. Можно изменить порядок шагов после их добавления, перетащив шаги вверх или вниз в списке.  

4. Выберите тип шага, который требуется добавить, затем перетащите шаг в пользовательский интерфейс, чтобы закрепить его на элементе управления. Вам может потребоваться несколько попыток, чтобы наловчиться размещать шаг там, где надо.  

   > [!NOTE]
   >  Вы можете удерживать плитку до 15 секунд. Если вы на закрепите ее в течение 15 секунд, плитка останется незакрепленной и указатель мыши изменится на нормальный.  

   ![Редактор потока управляемых задач](media/lp-gt-flow-editor.png "Редактор потока управляемых задач")  

5. После расположения шага в требуемом месте отпустите кнопку мыши, чтобы закрепить его на элементе управления. Шаг отобразится в выбранном расположении. Для перемещения шага используйте кнопку **Перетащите меня** на панели рядом с шагом.  

   ![Значок "Перетащите меня" в пузырьке управляемой задачи](media/lp-gt-bubble-drag-me.png "Значок "Перетащите меня" в пузырьке управляемой задачи")  

6. Добавьте содержимое в шаг с помощью элементов управления, отображаемых рядом с ним. Доступны следующие параметры:  

   - **Тип контента**: добавление текста или видео в шаг. Выберите **Изменить**, чтобы просмотреть дополнительные параметры. Можно изменить размер шрифта, цвет и стиль текста, а также добавить эскиз для видеозаписи.  

   - **Размещение**: выбор расположения шага в элементе управления, на котором он закреплен. Доступны следующие варианты: "Автоматический выбор позиции" (выбрано по умолчанию), "Вверху слева", "Вверху справа", "Внизу слева", "Внизу справа", "Слева сверху", "Слева снизу", "Справа сверху" и "Справа снизу".  

   - **Копировать**: создание копии шага с идентичным содержимым, закрепленной в том же месте. Она будет вставлена в поток управляемой задачи сразу же после исходного шага.  

7. Выберите **Сохранить** по завершении расположения и добавления содержимого в шаг а затем либо закройте шаг с помощью кнопки "Закрыть" в правом верхнем углу шага, либо выберите стрелку в левом верхнем углу экрана для возврата в редактор процессов.  

   > [!NOTE]
   >  Вы всегда можете изменить шаг позже, поэтому не тревожитесь, если вы случайно закрыли его, не закончив работу с ним.  

8. Чтобы добавить или изменить следующий шаг в управляемой задаче, выберите стрелку вправо ![Значок шеврона для возврата в редактор процессов](media/lp-chevron.png "Значок шеврона для возврата в редактор процессов") в левом верхнем углу экрана, чтобы отобразить редактор процессов.  

9. Добавьте дополнительные шаги, которые требуется включить в управляемую задачу, сохраняя каждый шаг после добавления содержимого.  

    > [!NOTE]
    >  При изменении расположения или копировании шага с помощью кнопки **Копировать** на панели инструментов все несохраненные изменения в этом шаге будут потеряны. Не забудьте часто сохранять изменения.  

10. Закончив добавлять шаги в управляемую задачу, выберите **Сохранить**.  

11. Выберите **Предварительный просмотр**, чтобы протестировать управляемую задачу и просмотреть, как она будет отображаться для пользователей.  

    > [!NOTE]
    >  Чтобы опубликовать управляемую задачу, сначала ее нужно предварительно просмотреть. Когда вы закрываете шаг или используйте стрелку в левом верхнем углу экрана в режиме предварительного просмотра, вы видите кнопки **Зарегистрировать** и **Опубликовать** в конструкторе схемы обучения.  

12. Если вы довольны изменениями, зарегистрируйте их и опубликуйте управляемую задачу либо опубликуйте ее позднее из библиотеки содержимого.

<a name="Videos"></a>   
## <a name="use-videos-in-your-learning-path-controls"></a>Использование видео в элементах управления "Схема обучения"  
 Только видео, размещенные в YouTube, поддерживаются в элементах управления схемы обучения. Если вы планируете использовать видео в элементах управления схемы обучения, необходимо иметь учетную запись и канал YouTube для отправки видео. Невозможно настроить ссылки на видео в канале, для которого задано состояние "Частный", но можно настроить ссылки на видео, если для канала задано состояние "Общедоступный" или "Не в списке". Можно также настроить канал так, чтобы несколько пользователей могли управлять содержимым видео для организации.  

 При добавлении видео в элементы управления можно внедрить видео в элемент управления. Если требуется, чтобы пользователи просматривали видео на новой вкладке или в новом окне браузера, вы можете добавить раздел текста на боковую панель, а затем добавить ссылку на видео в раздел текста.  

 При внедрении видео, которые отображаются на боковой панели или в управляемой задаче, вы будете использовать ссылку, получаемую от YouTube. Схема обучения будет автоматически обновлять ссылку для внедрения видео и изменять размер видео в соответствии с размером боковой панели или плитки управляемой задачи. Пользователь может выбрать параметр **На весь экран** для просмотра видео в полноэкранном режиме. Если пользователь приостанавливает воспроизведение или воспроизведение завершено, YouTube может автоматически отобразить ссылки на другие видео, которые могут заинтересовать пользователя. Это можно запретить, изменив ссылку в элементе управления, чтобы она включала **?rel=0** в конце.  

 Например, после создания и отправки видео в канал скопируйте URL-адрес видео, предоставленный YouTube, — **https://youtu.be/4TrYMB4pjyw**. Чтобы внедрить это видео в элемент управления, введите этот URL-адрес в поле **Введите URL-адрес видео** для элемента управления.  

 После сохранения элемента управления схема обучения изменяет URL-адрес на: **https://www.youtube.com/embed/4TrYMB4pjyw**. Чтобы отключить отображение ссылок на другие видео при приостановке видео или по окончании воспроизведения, измените URL-адрес так, чтобы он включал **?rel=0** в конце, чтобы URL-адрес выглядел аналогично следующему: **https://www.youtube.com/embed/4TrYMB4pjyw?rel=0**.  

Дополнительные сведения об использовании YouTube: [Центр справки YouTube](https://go.microsoft.com/fwlink/?linkid=847120)  

<a name="PublishLP"></a>   
## <a name="publish-learning-path-content"></a>Публикация содержимого схемы обучения  
 Пользователи смогут просматривать содержимое схемы обучения только после его публикации. Можно опубликовать только содержимое, которое зарегистрировано.  

1.  Откройте библиотеку содержимого.  

2.  Установите флажок рядом с каждой управляемой задачей и боковой панелью, которые требуется опубликовать. Убедитесь, что элемент управления, который требуется опубликовать, зарегистрирован.  

3.  Выберите **Опубликовать** в верхней части страницы, а затем выберите **Опубликовать**.  

4.  На странице **Публикация элементов управления** выберите среды публикации, в которых требуется опубликовать содержимое, и выберите **Опубликовать**.  

<a name="PublishingGroup"></a>   
### <a name="about-publishing-groups"></a>О группах публикации  
 Содержимое схемы обучения публикуется в группу публикации. Если вы включаете схему обучения для организации, группа публикации создается с тем же именем, что и название организации. Можно создавать дополнительные группы публикации по мере необходимости. Можно добавить несколько организаций в группу публикации, и организация может быть участником нескольких групп публикации, так что вы можете настроить содержимое и легко опубликовать его в различные организации.  

<a name="CreatePG"></a>   
### <a name="create-a-publishing-group"></a>Создание группы публикации  

1.  Откройте библиотеку содержимого.  

2.  Выберите **Конфигурация** в верхней части экрана.  

3.  Выберите **Конфигурация публикации**.  

4.  На странице **Конфигурация публикации** выберите **Новая среда публикации**.  

5.  Введите имя и необязательное описание.  

6.  Выберите организации, которые требуется включить в группу публикации. Будут отображены только организации, для которых у вас есть разрешения.  

7.  Выберите **Сохранить**, затем выберите **ОК**.  

<a name="ExportandImport"></a>   
## <a name="export-and-import-learning-path-content"></a>Экспорт и импорт содержимого схемы обучения  
 Можно экспортировать созданное содержимое, например для совместного использования с автором другой организации или создания резервных копий. Функция экспорта создает сжатый ZIP-файл, содержащий JSON-файлы, используемые для содержимого в схеме обучения. Будет создана одна папка в ZIP-файле для всех выбранных боковых панелей или управляемых задач схемы обучения.  

<a name="ExportProc"></a>   
### <a name="export-your-learning-path-content"></a>Экспорт содержимого схемы обучения  

1.  В библиотеке содержимого установите флажок рядом с содержимым для экспорта.  

     Можно экспортировать содержимое, не регистрируя его.  

2.  Выберите **Управление** в верхней части страницы, а затем выберите **Экспорт**.  

3.  Выберите вариант, который требуется использовать для сохранения созданного ZIP-файла, а затем выберите имя файла и папку.  

    > [!NOTE]
    >  Имя файла по умолчанию для ZIP-файла то же при каждом экспорте, поэтому обязательно используйте уникальное имя, чтобы избежать перезаписи ранее экспортированных файлов.  

### <a name="import-your-learning-path-content"></a>Импорт содержимого схемы обучения  

1.  В библиотеке содержимого выберите **Управление**, а затем выберите **Импорт**.  

2.  Выберите **Обзор**, чтобы выбрать ранее экспортированный файл, который планируется импортировать, или перетащите файл в поле **Перетащите сюда элементы управления** в диалоговом окне.  

    > [!CAUTION]
    >  Если импортируется элемент управления, он перезапишет и заменит любую версия того же элемента управления, который уже существует в библиотеке, даже если существующий элемент управления новее.  

3.  Подтвердите, что отображаемое имя файла — это файл, который требуется импортировать, а затем выберите **Импорт**.  

4.  В диалоговом окне подтверждения выберите **ОК**..  

<a name="Localize"></a>   
## <a name="localize-learning-path-controls"></a>Локализация элементов управления "Схема обучения"  
 Можно локализовать содержимое в элементах управления, создаваемых в схеме обучения, чтобы они отображались для пользователей на языке, который они выбрали для [!INCLUDE[pn_crm_shortest](../../includes/pn-crm-shortest.md)]. Чтобы локализовать собственные элементы управления, можно просто экспортировать их, локализовать строки, отображаемые для пользователей, а затем импортировать элемент управления, который включает локализованное содержимое. Можно импортировать элемент управления в ту же организацию либо в любую другую организацию по желанию. Можно локализовать тот же элемент управления на несколько языков, а затем импортировать только конкретные языки в определенные организации, поддерживающие пользователей с выбранным языком. Поддержка локализации в схеме обучения соответствует стандарту OASIS XML Localisation Interchange File Format (XLIFF) 2.0. Доступны бесплатные средства и учебники для работы с этим распространенным форматом. Дополнительные сведения: [XLIFF версии 2.0](https://docs.oasis-open.org/xliff/xliff-core/v2.0/os/xliff-core-v2.0-os.html)  

 Дополнительные сведения о языковых настройках пользователя в [!INCLUDE[pn_crm_shortest](../../includes/pn-crm-shortest.md)]: [Установка личных параметров](/dynamics365/customer-engagement/basics/set-personal-options)  

1.  Выберите элемент управления, который необходимо локализовать, в библиотеке содержимого.  

2.  Выберите **Локализовать**, затем выберите **Экспорт**.  

    ![Кнопка "Экспорт" в меню локализации схемы обучения](media/lp-localize-export.png "Кнопка "Экспорт" в меню локализации схемы обучения")  

3.  Выберите вариант, который требуется использовать для сохранения созданного ZIP-файла, а затем выберите имя файла и папку.  

    > [!NOTE]
    >  Имя файла по умолчанию для ZIP-файла то же при каждом экспорте, поэтому обязательно используйте уникальное имя, чтобы избежать перезаписи ранее экспортированных файлов.  

4.  После локализации содержимого в библиотеке содержимого выберите **Локализовать**, а затем выберите **Импорт**.  

5.  Выберите **Обзор**, чтобы выбрать ранее экспортированный файл, который планируется импортировать, или перетащите файл в поле **Перетащите сюда элементы управления** в диалоговом окне.  

    > [!CAUTION]
    >  Если импортируется элемент управления, он перезапишет и заменит любую версия того же элемента управления, который уже существует в библиотеке, даже если существующий элемент управления новее.  

6.  Подтвердите, что отображаемое имя файла — это файл, который требуется импортировать, а затем выберите **Импорт**.  

7.  В диалоговом окне подтверждения выберите **ОК**..  

8.  Опубликуйте локализованный элемент управления в требуемые среды публикации, чтобы сделать локализованный элемент управления доступным для пользователей. Локализованное содержимое автоматически будет отображаться для пользователей, которые выбрали тот же язык для своего пользовательского интерфейса.
