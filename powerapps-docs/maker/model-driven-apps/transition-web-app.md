# <a name="quick-start-for-transitioning-your-legacy-web-client-application-to-unified-interface"></a>Краткое руководство по переводу старого приложения веб-клиента на единый интерфейс

В платформе единого интерфейса также используются принципы гибкого веб-дизайна для оптимального просмотра и взаимодействия на экранах любых размеров, на любых устройствах и в любой ориентации. В этом кратком руководстве объясняется, как перевести старое приложение веб-клиента на единый интерфейс с помощью новой непроизводственной среды. 

> [!VIDEO https://www.microsoft.com/videoplayer/embed/RE3JwWU]

Для использования существующей непроизводственной среды для перевода приложения веб-клиента см. [Краткое руководство по использованию существующей среды для проверки старого приложения веб-клиента с единым интерфейсом](transition-web-app-existing.md). 
## <a name="prerequisites"></a>Необходимые компоненты
- Старое приложение веб-клиента. 
- Хотя это не обязательно, рекомендуется использовать непроизводственную среду для проверки приложения, чтобы это не повлияло на текущее развертывание или этапы разработки. Дополнительные сведения: [Управление экземплярами в песочнице](/dynamics365/admin/manage-sandbox-instances)

## <a name="prepare-the-environment"></a>Подготовка среды
Во-первых, выберите непроизводственную среду и включите режим **Использовать только единый интерфейс**, в результате чего единый интерфейс будет использоваться для всех управляемых моделью приложений в среде. Это также включает любое модули приложений Dynamics 365, первоначально настроенные для старого веб-клиента.

1. Выполните вход в [PowerApps](https://make.powerapps.com/?utm_source=padocs&utm_medium=linkinadoc&utm_campaign=referralsfromdoc), выберите **Среда**, затем выберите среду песочницы. 

2. Выберите **Параметры** > **Поведение**, затем включите **Использовать только единый интерфейс**.

   > [!div class="mx-imgBorder"] 
   > ![Настройка использования только единого интерфейса](media/use-unified-interface-only-pac.png)

Вы также можете установить это в области настроек. Перейдите в раздел **Параметры** > **Администрирование** > **Системные параметры**, затем на вкладке **Общие** задайте для параметра **Включить только единый интерфейс** значение **Да**.

> [!div class="mx-imgBorder"] 
> ![Использовать только единый интерфейс](media/use-unified-interface-only.png "Использовать только единый интерфейс")


> [!NOTE]
> Если требуется переключить среду обратно в ее предыдущее состояние, можно переключить параметр единого интерфейса для возврата к первоначальному интерфейсу. Дополнительные сведения: [Включить только единый интерфейс](/dynamics365/customer-engagement/admin/enable-unified-interface-only)

## <a name="run-and-validate-your-application-in-the-unified-interface"></a>Запуск и проверка приложения в едином интерфейсе
Запустите приложения, которые ранее были приложениями веб-клиента. Обратите внимание, что после включения параметра **Включить только единый интерфейс** все доступные приложения в среде используют единый интерфейс, даже если приложение было изначально настроено для веб-клиента.

Для запуска приложения войдите в [PowerApps](https://make.powerapps.com/?utm_source=padocs&utm_medium=linkinadoc&utm_campaign=referralsfromdoc), выберите **Приложения**, затем выберите приложение, которое требуется запустить. Кроме того, вы можете перейти непосредственно на страницу **Мои приложения**, например, *https://contoso.crm.dynamics.com/apps/*.

### <a name="validate-your-app-processes-and-customizations"></a>Проверка приложения, процессов и настроек 
Рекомендуется проверить все варианты использования. Можно начать с самых важных вариантов использования или группировать их в логические схемы дизайна. Поскольку единый интерфейс базируется на отзывчивом дизайне, рекомендуется выполнять тесты на различных устройствах с различным разрешением экрана. По мере тестирования приложения вы сможете убедиться, что настройки с совместимы с единым интерфейсом, а также определить, имеются ли какие-либо функции, которые требуют переработки или у которых отсутствует функциональность. Создайте план для проверки этих элементов и размещайте вопросы и отзывы на нашем форуме сообщества. 

> [!IMPORTANT]
> Текущая версия Common Data Service и приложения на основе модели в Dynamics 365 все еще содержат ряд устаревших функций. Следует проверить приложение на любые устаревшие функции и при необходимости заменить их новыми возможностями. Дополнительные сведения: [Предстоящие важные изменения (устаревшие функции)](/dynamics365/get-started/whats-new/customer-engagement/important-changes-coming)

### <a name="dynamics-365-apps"></a>Приложения Dynamics 365
Если используются приложения Dynamics 365 Field Service или Dynamics 365 Project Service Automation и нужно протестировать единый интерфейс, потребуется создать новую среду песочницы и сделать копию производственной среды для обновления Field Service и Project Service Automation до последней версии перед проверкой этих приложений в едином интерфейсе. Для этого выполните следующие шаги.

1. Создайте новую среду песочницы из [Центра администрирования Power Platform](https://admin.powerplatform.microsoft.com/environments) или [Центра администрирования Dynamics 365](https://port.crm.dynamics.com/). Дополнительные сведения: [Добавление экземпляра в подписку](/dynamics365/customer-engagement/admin/add-instance-subscription)

2. Скопируйте производственную среду, в которой находятся приложения Dynamics 365 Field Service или Dynamics 365 Project Service Automation, в новую среду песочницы. Для этого в центре администрирования Power Platform откройте производственную среду, затем выберите **Копировать**.

    > [!div class="mx-imgBorder"] 
    > ![Копировать среду](media/ppac-copy-environment.png "Копировать среду")

3. На странице **Копировать среду** выберите **Все содержимое**, выберите свою новую среду песочницы в списке **Выберите среду для перезаписи**, затем выберите **Копировать**. 

    > [!div class="mx-imgBorder"] 
    > ![Перезаписать среду](media/ppac-copy-overwrite.png "Перезаписать среду")

4. Появится диалоговое окно **Перезаписать среду**. Убедитесь, что выбрана правильная среда и выбраны правильные параметры, затем выберите **Подтвердить**. 

5. Если копирование выполнено успешно, отображается подтверждающее уведомление. 

6. В строке меню выберите **Управление решениями** для открытия области **Решений**. 

    > [!div class="mx-imgBorder"] 
    > ![Управление решениями](media/ppac-manage-solutions.png "Управление решениями")

    > [!IMPORTANT]
    > Если **режим администрирования** включен, необходимо отключить его, чтобы была видна область **Решения**. Дополнительные сведения: [Режим администрирования](/power-platform/admin/sandbox-environments#administration-mode)

7. Найдите решение Field Service или Project Service Automation и выберите его. Параметр **Обновить** должен быть доступен. Выберите его, чтобы обновить решение. 

    > [!div class="mx-imgBorder"] 
    > ![Обновление решения](media/ppac-upgrade-solution.png "Обновление решения")
    
> [!NOTE]
> Последние версии Field Service и Project Service Automation доступны на едином интерфейсе по умолчанию для новых созданных экземпляров. Чтобы произвести обновление существующей среды с установленными более ранними версиями, необходимо запросить обновление, обратившись в [Службу поддержки пользователей Майкрософт](https://go.microsoft.com/fwlink/?LinkId=853505). 

Дополнительные сведения: [Последние версии Dynamics 365 for Field Service](/dynamics365/customer-engagement/field-service/version-history#latest-versions) и [Домашняя страница обновления Dynamics 365 for Project Service Automation](/dynamics365/customer-engagement/project-service/upgrade-psa-home-page)

## <a name="next-steps"></a>Дальнейшие шаги
В зависимости от ваших заключений, ваша группа реализации или партнер могут оценить объем работы, которую необходимо выполнить для перевода приложений на единый интерфейс, а также определить возможные улучшения удобства использования. В связи с наличием многочисленных новых функций и возможностей, доступных в едином интерфейсе, имеется возможность увеличить ценность для пользователей приложений. 

Переход на единый интерфейс — это замечательная возможность создать современный интерфейса пользователя и перепроверить существующие процессы, чтобы убедиться, что они все еще правильны или нуждаются в улучшении. Это также удобное время для проверки, отражает ли приложение требования бизнеса и может ли существующее приложение быть распространен на несколько приложений для различных рабочих групп и ролей.
Дополнительные сведения: [Разработка управляемых моделью приложений с помощью конструктора приложений](design-custom-business-apps-using-app-designer.md)
