# <a name="enable-and-use-customizable-help"></a>Включение и использование настраиваемой справки
Настраиваемая справка позволяет предоставить вашу собственную контекстную информацию для пользователей управляемых моделью приложений, заполняющих формы. 

> [!NOTE]
> Вместо создания и сопровождения вашей собственной системы справки, пользовательские панели справки и управляемые задачи доступны, которые можно использовать для создания справки, которая предоставляет вашему приложению единого интерфейса настраиваемое взаимодействие справки в продукте, которое настроено для вашей организации. Дополнительные сведения: [Создание управляемой справки для приложения единого интерфейса](../common-data-service/create-custom-help-pages.md)

В управляемых моделью приложениях можно заменить справку по умолчанию на настраиваемую справку на глобальном уровне (среды) или уровне сущности. В настраиваемой справке содержимое отображается с помощью ссылок справки и больше подходит для ваших пользовательских или настраиваемых сущностей. С помощью одного глобального URL-адреса можно переопределить ссылки готовой справки для всех настраиваемых сущностей. URL-адреса на уровне сущности переопределяют ссылки готовой справки в сетках и формах определенной настраиваемой сущности. Можно включить дополнительные параметры в URL-адрес, например код языка и имя сущности. Эти параметры позволяют разработчику добавить возможность перенаправления пользователя на страницу с нужным языком или контекстом сущности в приложении. Параметры настраиваемой справки на уровне сущности поддерживают решения, поэтому их можно упаковать как часть решения и перенести между средами либо распределить в решениях. 

## <a name="set-up-customizable-help"></a>Настройка настраиваемой справки
Настраиваемая справка может быть задана на глобальном уровне или уровне сущностей. 

### <a name="set-customizable-help-at-the-global-level"></a>Задание настраиваемой справки на глобальном уровне
Пользователи с ролью безопасности администратора могут использовать параметры, чтобы переопределить справку по умолчанию на глобальном уровне. 
1. Откройте управляемое моделью приложение, затем на панели команд выберите **Параметры** ![Параметры](../model-driven-apps/media/powerapps-gear.png) > **Дополнительные параметры**.
2. Перейдите в **Параметры** > **Администрирование**.
3. Выберите **Системные параметры**, затем выберите вкладку **Общие сведения**. 
4. В поле **Задать URL-адрес настраиваемой справки** выберите и определите следующие глобальные параметры настраиваемой справки: 
     - **Использовать настраиваемую справку для настраиваемых сущностей**. Выберите, чтобы включить.  
     - **URL-адрес глобальной настраиваемой справки**. Введите URL-адрес вашей настраиваемой справки. 
     - **Присоединить параметры к URL-адресу**. Выберите значение **Да**, чтобы разрешить добавление таких параметров, как код языка или имя сущности, к **URL-адресу справки**, указанный в определении сущности. Дополнительные сведения: [Добавление параметров к URL-адресу](#append-parameters-to-url)  

    > [!div class="mx-imgBorder"] 
    > ![Глобальный параметр настраиваемой справки](media/customizable-help-global-setting.png)

5. Нажмите **ОК**.

### <a name="set-customizable-help-for-a-specific-entity"></a>Задание настраиваемой справки для конкретной сущности
После включения настраиваемой справки на глобальном уровне настройщики системы могут переопределить глобальный URL-адрес справки для сущности в определении сущности. 

1. Откройте обозреватель решений.
2. Разверните **Сущности**, затем выберите требуемую сущность. 
3. На вкладке **Общие** в разделе **Справка** определения сущности в поле **URL-адрес справки** введите URL-адрес страницы настраиваемой справки. 

    > [!div class="mx-imgBorder"] 
    > ![Настройка сущности настраиваемой справки](media/customizable-help-entity-setting.png)

#### <a name="append-parameters-to-url"></a>Присоединить параметры к URL-адресу
Как описано ранее, чтобы разрешить добавление параметров в **URL-адрес справки** для определения сущности, задайте для параметра **Присоединить параметры к URL-адресу** на вкладке **Системные параметры** > **Общие** значение **Да**. 

Примеры настроек, которые можно присоединять к URL-адресу:

- Код языка пользователя: userlcid
- Имя сущности: entity
- Точка входа: диаграмма иерархии или форма
- Идентификатор формы: formid
