# <a name="faqs-transition-to-unified-interface"></a>Вопросы и ответы: переход на единый интерфейс

Этот раздел предоставляет ответы на наиболее распространенные вопросы о единый интерфейс и о переходе от устаревшего веб-клиента к единый интерфейс.

## <a name="faqs-unified-interface-and-legacy-web-client"></a>Вопросы и ответы: Единый интерфейс и устаревший веб-клиент

### <a name="why-should-i-move-to-unified-interface"></a>Почему я должен перейти на единый интерфейс?

Видение Microsoft для продуктивного рабочего места требует современный интерфейс, который включает в себя функции и улучшения, содержащиеся в единый интерфейс, который обеспечивает более единообразное взаимодействие с пользователем, повышение производительности и эффективности работы пользователя. Для получения дополнительных сведений см. [сборник схем единого интерфейса](https://docs.microsoft.com/powerapps/maker/model-driven-apps/unified-interface-playbook)

### <a name="are-my-isvs-compatible"></a>Совместимы ли мои независимые поставщики ПО?

Независимым поставщикам ПО было предложено принять меры по переходу на единый интерфейс. Обратитесь непосредственно к своему независимому поставщику ПО относительно его временная шкала для совместимости с единый интерфейс.

### <a name="what-is-the-retraining-effort"></a>Что такое меры по переподготовке?

С технологической точки зрения переподготовка должна быть минимальной, поскольку ваша бизнес-логика и схема остаются неизменными. Переподготовка должна быть направлена на новых возможностях в навигации и общем внешнем виде. Дальнейшее обучение может потребоваться, если вы приняли бизнес-решение изменить разработку, чтобы оптимизировать ее для новых возможностей и функций.

### <a name="will-i-need-to-re-create-my-solution-customizations"></a>Нужно ли мне заново создавать настройки своего решения?

Нет, вам не нужно повторно создавать какие-либо настройки, если вы следуете [Руководство разработчика Common Data Service](/powerapps/developer/common-data-service/overview) и в курсе любых изменений. Тем не менее, могут появиться новые возможности для выполнения наших последних инвестиций, которые могут привести к изменениям настройки.

### <a name="how-long-will-it-take-to-transitionwhat-is-the-work-effort"></a>Сколько времени займет переход/каковы усилия работы?

Усилия работы здесь будут варьироваться в зависимости от сложности вашей среды. Мы рекомендуем начать с пилотного/POC-приложения на основе модели, чтобы начать тестирование. После этого вы можете работать с бизнесом и планировать полное развертывание единого интерфейса. См. [сборник схем единого интерфейса](https://docs.microsoft.com/powerapps/maker/model-driven-apps/unified-interface-playbook) и [контрольный список планирования](https://aka.ms/UIChecklist) для получения дополнительных рекомендаций по началу работы и нашим [техническим документам](https://download.microsoft.com/download/A/F/3/AF3D45A7-4F38-41BE-8956-1DF7A4A5AFDB/approaching-unified-interface-transition.pdf).

### <a name="where-can-i-find-public-information-about-the-unified-interface-direction"></a>Где я могу найти публичную информацию о направлении единого интерфейса?

Стратегическое направление единого интерфейса уже находится в открытом домене благодаря таким событиям, как:
- [MBAS: BRK2073 Microsoft Power Apps : выполните один пользовательский интерфейс — будущее приложений на основе холста, на основе модели и единый интерфейс в Power Apps](https://powerusers.microsoft.com/t5/MBAS-Gallery/Microsoft-PowerApps-Run-one-UI-the-future-of-canvas-model-driven/td-p/301580)
- [BRK3031 Лучшие практики внедрения для Dynamics 365: переход на современный единый интерфейс](https://powerusers.microsoft.com/t5/Microsoft-Business-Applications/Implementation-Best-Practices-for-Dynamics-365-Making-the-move/m-p/299928)


### <a name="as-an-on-premises-customer-looking-to-move-to-the-cloud-what-is-the-best-approach-to-adopt-unified-interface"></a>Как локальный клиент, желающий перейти в облако, каков наилучший подход к использованию единый интерфейс?

Настоятельно рекомендуется планировать переход на единый интерфейс одновременно с переходом на облако. Начните тестирование своего устаревшего веб-клиента в пределах единого интерфейса и после переноса подтвердите совместимость единого интерфейса, прежде чем выпускать его для сообщества пользователей. Это уменьшит усилия по обучению и позволит пользователям освоить некоторые из новых функций и возможностей, которые может предложить единый интерфейс. Для получения дополнительной информации, свяжитесь с вашим представителем Microsoft.

### <a name="when-will-isvs-be-notified-of-the-need-to-move-to-unified-interface"></a>Когда независимые поставщики ПО будут уведомлены о необходимости перехода на единый интерфейс?

Независимым поставщикам ПО было предложено принять меры по переходу на единый интерфейс. Обратитесь непосредственно к своему независимому поставщику ПО относительно его временная шкала для совместимости с единый интерфейс.

### <a name="how-are-language-translations-handled"></a>Как обрабатываются языковые переводы?

Процесс языкового перевода в основном не изменился, за исключением карты сайта. Области, группы и дочерние области карты сайта локализуются в редакторе карты сайта.

### <a name="i-dont-have-budget-or-resources-to-do-a-project-is-the-deadline-a-fixed-date"></a>У меня нет бюджета или ресурсов, чтобы сделать проект. Является ли крайний срок фиксированной датой?

Мы заблокировали срок перехода с устаревшего веб-клиента к 1 декабря 2020 года. Мы рекомендуем провести некоторое тестирование как можно скорее, чтобы понять сложность проекта для вашей организации, чтобы получить совместимость. В большинстве случаев для быстрого продвижения вперед и обнаружения преимуществ для пользователя не требуется обширных изменений. Свяжитесь с Microsoft, чтобы предоставить любые отзывы, которые могут остановить ваш переход, создав обращения к службе поддержки.

### <a name="were-working-on-transitioning-to-unified-interface-but-have-hit-some-issues-how-can-we-report-to-microsoft"></a>Мы работаем над переходом на единый интерфейс, но столкнулись с некоторыми проблемами; как мы можем отправить отчет в Microsoft?

Рассмотрите наши [ресурсы](https://community.dynamics.com/365/unified-interface/), чтобы помочь с руководством и зарегистрировать обращение к службе поддержки, следуя стандартной процедуре. Вы также можете уведомить своего партнера, представителя Microsoft или рабочую группу FastTrack (если применимо).

### <a name="when-should-i-start-moving-to-unified-interface"></a>Когда я должен начать переходить на единый интерфейс?

Вы можете переходить сразу же; у нас уже есть много клиентов, переведенных в работу на единый интерфейс. Начните тестирование как можно скорее. Получите доступ к сайту Сообщества: <https://community.dynamics.com/365/unified-interface/> для полезного содержимого. Используйте процедуру поддержки Standard для регистрации любых проблем.

### <a name="what-are-the-risks-associated-with-moving-to-unified-interface"></a>Какие риски связаны с переходом на единый интерфейс?

Мы видим риск в том, чтобы остаться на устаревшем веб-клиенте, потому что наше инвестиционное и стратегическое направление переходит на единый интерфейс. Если вы планируете начать переход, рассмотрите наш [сайт сообщества](https://community.dynamics.com/365/unified-interface/) и [краткие руководства по началу работы](https://docs.microsoft.com/powerapps/maker/model-driven-apps/transition-web-app-existing). Это поможет вам выявить пробелы (если таковые имеются) и составить план по их устранению до полного развертывания.

### <a name="i-have-a-new-feature-idea-for-unified-interface-where-do-i-log-it"></a>У меня есть новая идея для единый интерфейс; где я могу зарегистрировать ее?

Посетите наш портал идей (<https://powerusers.microsoft.com/t5/PowerApps-Ideas/idb-p/PowerAppsIdeas>), выберите **Предложить идею** и пометьте элемент категорией «единый интерфейс».

### <a name="is-there-any-supporting-content-to-help-me-plan-and-execute-my-transition-to-unified-interface"></a>Есть ли вспомогательные содержимое, которое поможет мне спланировать и выполнить мой переход на единый интерфейс?

Да, у нас есть новая группа сообщества (<https://community.dynamics.com/365/unified-interface/>), которая предоставляет полезное содержимое, интерактивный форум и блог, чтобы увидеть все последние обновления и сведения о единый интерфейс.

### <a name="how-does-this-move-impact-on-premises-customer-and-what-will-happen-when-the-web-client-is-deprecated-do-web-client-forms-become-unsupported"></a>Как этот переход влияет на локального клиента и что произойдет, если веб-клиент будет нерекомендуемым (формы веб-клиента станут неподдерживаемыми)?

В настоящее время это объявление действует только для онлайн-клиентов. Мы продолжим поддерживать устаревший веб-клиент для локальных развертываний.

### <a name="is-the-timeline-applicable-for-customers-using-government-community-cloud-gcc-and-other-data-centers-such-as-china"></a>Применима ли временная шкала для клиентов, использующих Government Community Cloud (GCC) и другие центры обработки данных, такие как Китай?

Мы планируем выпускать обновления единый интерфейс для этих центров обработки данных в рамках стандартной программы выпуска.

### <a name="what-action-should-i-take-as-a-unified-service-desk-customer-who-is-still-on-the-web-client"></a>Какие действия я должен предпринять как клиент Unified Service Desk, который все еще находится в веб-клиенте?

Клиенты Unified Service Desk, которые все еще находятся в веб-клиенте, должны перенести свои конфигурации из веб-клиента в единый интерфейс. Это потребует обновления до последних версий [Unified Service Desk](https://docs.microsoft.com/dynamics365/unified-service-desk/download-unified-service-desk?view=dynamics-usd-4.1) (как клиентские, так и серверные компоненты). У нас есть [помощник по переносу](https://docs.microsoft.com/dynamics365/unified-service-desk/admin/migration-steps-web-client-unified-interface-configuration?view=dynamics-usd-4.1), чтобы помочь с этим.

### <a name="are-there-any-other-deprecations-that-i-should-plan-for"></a>Есть ли какие-либо другие уведомления об устаревании, которые я должен планировать?

Рассмотрите нашу [страницу уведомлений об устаревании](https://docs.microsoft.com/power-platform/important-changes-coming), чтобы определить, есть ли другие области, которые также требуют планирования.

### <a name="where-can-i-find-further-information-on-the-transition-service"></a>Где я могу найти дополнительную информацию о службе перехода?

Чтобы ускорить процесс перехода, мы указываем предполагаемую целевую дату для переключения сред с устаревшего веб-клиента на единый интерфейс через нашу службу перехода. Клиенты должны будут утвердить дату до какого-либо действия, а затем получат серию сообщений с напоминанием о связи по мере приближения даты. Сроки также могут быть перенесены, если они не подходят для бизнеса, до крайнего срока 1 декабря 2020 года. Для получения дополнительных сведений см. следующий раздел.


## <a name="faqs-transitioning-to-unified-interface"></a>Вопросы и ответы: переход на единый интерфейс

### <a name="where-can-i-go-to-see-the-transition-dates-that-have-been-suggested-for-my-environments"></a>Где я могу посмотреть даты перехода, которые были предложены для моей среды? 

Используйте портал перехода для управления датой перехода вашей среды: <https://runone.powerappsportals.com>.

### <a name="how-do-i-gain-access-to-the-portal"></a>Как получить доступ к порталу?

Выполните следующие действия:
1. Посетите сайт <https://runone.powerappsportals.com>.
2. Выполните вход с учетными данными администратора клиента, которым требуется управлять.
3. Выберите **Моя среда** и просмотрите все среды, для которых применяется рекомендуемая дата.

### <a name="i-see-my-environment-has-a-date-suggested-for-transition-can-i-change-this-date"></a>Я вижу, что у моей среды есть дата, предложенная для перехода. Можно ли изменить эту дату?

Да, это возможно, если у вас есть роль **глобальный администратор**, **администратор службы Dynamics 365** или **администратор Power Platform** для клиента. 

Даты, связанные с вашей средой, это предложение, требующее одобрения. Утвердите, если дата подходит для вашей организации.  

Для изменения даты выберите раскрывающийся значок рядом со средой и просмотрите запись. Затем роли администратора клиента смогут перенести дату перехода на более раннюю или более позднюю дату.

Чтобы перенести на более раннюю дату, обновите существующую дату до предпочитаемого вами варианта в списке. Также можно [переключиться вручную](transition-web-app.md) , если наши даты не подходят. 
 
Чтобы запланировать на более позднюю дату, нажмите кнопку переноса даты перехода. Предлагаемые даты будут доступны в раскрывающемся списке После утверждения дата будет обновлена на портале соответственно. Примите обновленную дату, если вы хотите, чтобы ваша среда была переведена. 
 
Изменения даты будут рассмотрены и предоставлены, если дата наступит до 1 декабря 2020 года. Дата будет обновлена в рамках портала после подтверждения. 

> [!NOTE]
> Если у вас есть одобренный переход и до запланированной даты осталось менее 48 часов, вы не сможете изменить дату. Аналогично, вы не можете запросить дату после 1 декабря 2020 года, так как устаревший веб-клиент больше не будет доступен.

### <a name="what-will-happen-if-i-dont-opt-in-and-approve-a-suggested-transition-date-for-my-environment"></a>Что произойдет, если я не соглашаюсь и не подтверждаю предлагаемую дату перехода для своей среды?

В вашей среде не произойдет никаких изменений, если вы не утвердите предложенную дату на портале. Когда предложенная дата истечет, мы постараемся предоставить вам на рассмотрение другую дату в будущем.  
 
> [!NOTE]
> После 1 декабря 2020 года все среды будут обновлены до единый интерфейс.

### <a name="my-transition-date-is-within-48-hours-and-i-cant-change-the-date-within-the-portal-how-can-i-stop-the-transition-from-taking-place"></a>Моя дата перехода наступает в течение 48 часов, и я не могу изменить дату на портале. Как я могу остановить переход?

Возможность изменить дату перехода для среды доступна только ранее 48 часов до перехода. Для остановки процесса после этого периода обратитесь в службу поддержки. 

> [!NOTE]
> Мы не можем гарантировать, что переход можно остановить, если запрос сделан после фиксации даты на портале (48 часов или менее).

### <a name="i-have-environments-without-a-scheduled-date-can-i-update-these-to-include-a-date"></a>У меня есть среда без запланированной даты. Могу я обновить это для включения дат?

Да, если у вас есть роль **глобальный администратор**, **администратор службы Dynamics 365** или **администратор Power Platform** для клиента, выберите среду и выберите дату, нажав кнопку переноса даты перехода.

Мы обновим портал с датой для подтверждения. Сообщения электронной почты с уведомлениями также будут отправлены глобальным администраторам клиента при приближении даты перехода. Это будет следовать стандартной процедуре напоминания, описанной в этом документе.

### <a name="is-there-a-recommended-checklist-i-should-run-through-before-transition"></a>Существует ли рекомендованный контрольный список, который я должен выполнить перед переходом?

Проверьте вспомогательное содержимое, доступное на [сайте сообщества](https://community.dynamics.com/365/unified-interface/). Мы также имеет [контрольный список перехода](https://aka.ms/UIChecklist), помогающий эффективно планировать. Просмотрите его внимательно для обеспечения комфортного перехода на единый интерфейс.

### <a name="my-environment-has-been-transitioned-but-i-am-finding-blocking-issues-for-my-users-and-want-to-move-back-to-the-legacy-web-client-is-this-possible"></a>Моя среда была перенесена, но я обнаружил проблемы с блокировкой для своих пользователей и хочу вернуться к устаревшему веб-клиенту. Возможно ли это?

Да, вы сможете вернуться к старому веб-клиенту в течение не более 10 дней после перехода. Вы можете сделать [переключение вручную](https://docs.microsoft.com/power-platform/admin/enable-unified-interface-only) в течение первых 10 дней или подать [стандартный запрос поддержки](https://admin.powerplatform.microsoft.com/support?referer=mbssupport) и установить тип проблемы "Переход от устаревшего веб-клиента на единый интерфейс", поскольку ручной переключатель будет отключен. 

> [!NOTE]
> Эти 10 дней должны быть до 1 декабря 2020 года, так как устаревший веб-клиент больше не будет доступен после этой даты.

### <a name="i-want-to-transition-after-december-1-2020-is-that-possible"></a>Я хочу перейти после 1 декабря 2020 года. Это возможно?

С 1 декабря 2020 года устаревший веб-клиент больше не будет доступен. У нас нет возможности отложить дату на дату после этого периода.

Если возникли какие-либо блокирующие проблемы, зарегистрируйте их с помощью стандартного процесса поддержки как можно быстрее.

### <a name="what-is-the-standard-reminder-procedure-throughout-this-process"></a>Что представляет собой стандартную процедуру напоминания в течение этого процесса?

Связь будет происходить волнообразно, по крайней мере, за 30 дней до предполагаемого интервал времени, но вы можете войти на портал (<https://runone.powerappsportals.com>) в любое время для просмотра статуса. Майкрософт отправит следующее сообщение:

-    Начальное сообщение для каждой среды, в которой есть предложенная дата перехода.
-    Если вы утвердили дату, вы получите напоминание за два дня до того, как даты будут заблокированы на портале. 
-    Окончательное напоминание будет отправлено за два дня до перехода. В нем будет указано, что дата перехода заблокирована и будет выполнен переход.
-    После перехода появится заключительное сообщение, подтверждающее успех (или наличие проблемы)

Сообщения можно просмотреть с помощью следующих каналов:
-    Центр сообщений в клиенте Microsoft 365. Они обычно видны ролям, таким как глобальный администратор, администратор сервиса, читатель сообщений сервиса.
-    Рассылка по электронной почте.  Электронные письма отправляются только роли системного администратора для конкретной среды или на адреса электронной почты, которые были добавлены на вкладку "Уведомления" на портале администрирования среды.

> [!NOTE]
> Вы получите сообщение электронной почты для каждой среды, в которой ваша учетная запись пользователя имеет роль системного администратора.

### <a name="i-have-requested-a-date-to-postpone-but-still-receiving-e-mails-and-message-center-posts-that-my-environment-is-set-to-transition-should-i-be-concerned"></a>Я попросил отложить дату, но все равно получаю сообщения электронной почты и записи центра сообщений, что для моей среды задан переход. Следует ли беспокоиться?

Наша первая рекомендация — проверить портал перехода (<https://runone.powerappsportals.com/>), поскольку это будет единственным источником окончательной информации для всех ваших сред. Если дата обновлена, то скорее всего наша система коммуникаций отправила сообщение, прежде чем мы обновили список сообщений. 

Если дата на портале не обновлена на новую дату, отправьте запрос в техническую поддержку в соответствии с вашей стандартной процедурой.

Переход будет выполнен только для дат, утвержденных администратором. 

### <a name="if-i-already-have-an-environment-transitioned-to-unified-interface-will-i-still-be-able-to-switch-back-to-the-legacy-web-client-manually"></a>Если я уже имеют среду, переведенную на единый интерфейс, смогу ли я переключиться обратно на старый веб-клиент вручную?

Если с момента перевода среды прошло не менее 10 дней, мы отключим ручное переключение на старый веб-клиент. 

Если вы обнаружите, что это было отключено и у вас есть требование переключиться назад, создайте [стандартный запрос поддержки](https://admin.powerplatform.microsoft.com/support?referer=mbssupport) и установите тип проблемы "Переход от устаревшего веб-клиента на единый интерфейс".

### <a name="is-there-a-specific-day-and-time-when-approved-transitions-will-take-place"></a>Есть ли конкретный день и время, когда будут выполнены утвержденные переходы?

Мы не предвидим никаких простоев при выполнении этого перехода. Однако мы будет осуществлять переход только в пятницу, следуя тем же срокам обслуживания, которые указаны в наших стандартных политиках и сообщениях. Дополнительные сведения: [График обслуживания](https://docs.microsoft.com/power-platform/admin/policies-communications#maintenance-timeline)

### <a name="are-environments-from-all-data-centers-included-within-this-transition-service"></a>Включены ли среды из всех центров обработки данных в эту службу перехода?

В настоящее время среды из определенных центров обработки данных, таких как Government Community Cloud (GCC), отсутствуют на портале. Мы предоставим рекомендуемые даты перехода для этих сред к июню 2020 года. Клиенты, которые хотят перейти на единый интерфейс, всегда могут [переключиться вручную](/power-platform/admin/enable-unified-interface-only#how-to-enable-unified-interface-only-mode) в любое время до 1 декабря 2020 года.
