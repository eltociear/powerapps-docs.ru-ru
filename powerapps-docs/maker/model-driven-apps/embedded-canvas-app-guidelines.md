# <a name="guidelines-on-working-with-embedded-canvas-apps"></a>Рекомендации по работе с внедренными приложениями на основе холста
[!INCLUDE [cc-beta-prerelease-disclaimer](../../includes/cc-beta-prerelease-disclaimer.md)]

В этом разделе представлены рекомендации по работе с внедренными приложениями на основе холста, а также полезные советы по устранению возможных неполадок.

-   Внедренные приложения на основе холста поддерживаются только с управляемыми моделью приложениями единого интерфейса.
-   Можно включить только одно внедренное приложение на основе холста в одной форме. 
     - В форму можно добавить несколько внедренных приложений на основе холста, но одновременно может быть включено только одно из них.
     - При попытке включить несколько внедренных приложений на основе холста в управляемой моделью форме, отображается сообщение "В форме можно включить только одно приложение на основе холста".
     - Чтобы включить или отключить внедренное приложение на основе холста, см. раздел [Включение внедренного приложения на основе холста](#enable-an-embedded-canvas-app) и [Отключение внедренного приложения на основе холста](#disable-an-embedded-canvas-app).
-   Внедренные приложения на основе холста можно создавать, изменять и воспроизводить только через управляемую моделью форму, в которой они размещены.
     - Нельзя создать внедренные приложения на основе холста непосредственно вне контекста управляемой моделью формы.
     - Аналогично, не поддерживается изменение или воспроизведение внедренного приложения на основе холста вне контекста управляемой моделью формы.

     > [!NOTE]
     > Хотя иногда можно открыть внедренное приложение на основе холста вне управляемого моделью приложения, это не поддерживается.

-   Обратите внимание на следующее при использовании элемента управления вложенной сетки для добавления внедренного приложения на основе холста в управляемую моделью форму.
     - Данные (поля и значения), отправленные во внедренное приложение на основе холста во время выполнения, определяются представлением, которое указано как **Представление по умолчанию** в разделе **Источник данных** свойств элемента управления вложенной сетки. Используйте только поля во внедренном приложении на основе холста, которые включены в представление, или добавьте их к представлению при необходимости. Все поля, которые не включены в представление, отображаются с пустыми значениями во время выполнения. 
     - Условия фильтра для представления не используются во время разработки. Следовательно, данные, которые отображаются при создании внедренных приложений на основе холста, не отфильтрованы, это просто список нескольких первых записей, к которым имеется доступ. Во время выполнения условия фильтра для представления применяются надлежащим образом и отображаются только соответствующие данные.
-   При использовании элемента управления полей для добавления внедренного приложения на основе холста в управляемую моделью форму всегда используйте поле, которое гарантировано имеет значение. Если поле не имеет значения, внедренное приложение на основе холста не обновляется в ответ на любое изменение данных в управляемой моделью форме, в которой размещено приложение.
-   Публикация управляемой моделью формы не означает публикацию также и внедренного приложения на основе холста.
     - Внедренные приложения на основе холста должны быть опубликованы независимо от управляемой моделью формы, в которой они размещены. Дополнительные сведения: [Публикация приложения](../canvas-apps/save-publish-app.md#publish-an-app).
-   Если открытие PowerApps Studio для создания или изменения внедренного приложения на основе холста с помощью кнопки **Настройка** в свойствах элемента управления приложения на основе холста заблокировано из-за блокировки всплывающих окон браузера, нужно включить сайт web.powerapps.com или временно отключить средство блокирования всплывающих окон, затем снова выбрать **Настройка**.
-   Внедренные приложения на основе холста не отображаются при создании новой записи, так как требуется, чтобы им был передан контекст записи.
-   Объект ModelDrivenFormIntegration.Data доступен только для чтения. 
     - Чтобы записать данные обратно, необходимо использовать соединитель Common Data Service. Дополнительные сведения: [Common Data Service](/connectors/commondataservice/)
-   Объект ModelDrivenFormIntegration.Data является списком записей. 
     - Даже текущая запись передается во внедренное приложение на основе холста через объекта ModelDrivenFormIntegration.Data в виде списка, который содержит единственную запись.
     - Чтобы напрямую указать запись, можно использовать [функцию First](../canvas-apps/functions/function-first-last.md). Пример: First(ModelDrivenFormIntegration.Data).Name
-   Необходимо в максимально возможной степени исключить ручное изменение идентификатора приложения в свойствам элемента управления приложения на основе холста.
     - Идентификатор приложения на основе холста создается автоматически и заполняется для вас. 
     - Если по какой-либо причине необходимо вручную изменить его, необходимо убедиться, что любой используемый идентификатор приложения соответствует *внедренному* приложению на основе холста, а не просто автономному приложению на основе холста. 
     - Внедренное приложение на основе холста также должно быть создано с тем же контекстом данных, который будет отправляться управляемой моделью формой.
     - После обновления идентификатора приложения выберите **Настройка**, чтобы открыть его в PowerApps Studio и установить соединение с новым приложением.
     - Внесите небольшое изменение в приложение, чтобы перевести его в несохраненное состояние, затем сохраните и опубликуйте приложение.
- Если при просмотре управляемой моделью формы с внедренным приложением на основе холста отображается сообщение об ошибке "Это приложение не найдено", убедитесь, что внедренное приложение на основе холста находится в том же решении, что и управляемая моделью форма.
- Если при просмотре управляемой моделью формы с внедренным приложением на основе холста отображается сообщение об ошибке "Похоже, у вас нет доступа к этому приложению. Попросите ответственного предоставить вам доступ", убедитесь, что автор предоставил вам доступ к внедренному приложению на основе холста. Дополнительные сведения: [Общий доступ к внедренному приложению на основе холста](share-embedded-canvas-app.md).

## <a name="enable-an-embedded-canvas-app"></a>Включение внедренного приложения на основе холста
1. Выберите поле или элемент управления вложенной сетки, который был настроен для отображения как внедренное приложение на основе холста.
2. В диалоговом окне **Свойства поля** (или **Задать свойства** для вложенной сетки) выберите вкладку **Элементы управления**.
3. В списке элементов управления выберите **Приложение на основе холста**, затем выберите параметр **Веб**.
4. Нажмите **ОК**.

## <a name="disable-an-embedded-canvas-app"></a>Отключение внедренного приложения на основе холста
1. Выберите поле или элемент управления вложенной сетки, который был настроен для отображения как внедренное приложение на основе холста.
2. В диалоговом окне **Свойства поля** (или **Задать свойства** для вложенной сетки) выберите вкладку **Элементы управления**.
3. В списке элементов управления выберите элемент управления по умолчанию, затем выберите параметр **Веб**.
4. Нажмите **ОК**.

## <a name="known-issues-and-limitations-with-embedded-canvas-apps"></a>Известные проблемы и ограничения, связанные с внедренными приложениями на основе холста
- Пользовательский элемент управления приложения на основе холста поддерживается только для использования с клиентом типа **Веб**. В настоящее время клиенты типов **Телефон** и **Планшет** не поддерживаются. Дополнительные сведения: [Использование настраиваемых элементов управления для визуализации данных управляемых моделью приложений](use-custom-controls-data-visualizations.md)
- При создании новой записи внедренное приложение на основе холста в форме не отображается даже после сохранения записи. 
-    Объект ModelDrivenFormIntegration.Data в настоящее время не работает с элементами управления "Форма отображения" и "Форма редактирования".
- Нельзя использовать привилегию **Приложение на основе холста** в роли безопасности для предоставления пользователю доступа к внедренному или автономному приложению на основе холста. Дополнительные сведения об общем доступе к внедренному приложению на основе холста см. в разделе: [Общий доступ к внедренному приложению на основе холста](share-embedded-canvas-app.md).
- При обратной записи тех же данных, которые показываются в управляемой моделью форме, в которой размещено приложение, в форме будут и далее отображаться старые данные, пока она не будет обновлена. Простой способ сделать это — использовать метод [RefreshForm](embedded-canvas-app-actions.md).
- Если вы не видите IntelliSense для [методов для выполнения предопределенных действий](embedded-canvas-app-actions.md) во внедренных приложениях холста, которые были созданы до того, как стала доступна данная функция, то сохраните, закройте и снова откройте приложение.
