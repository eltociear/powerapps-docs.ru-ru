# <a name="guidelines-on-working-with-embedded-canvas-apps"></a>Рекомендации по работе с внедренными приложениями на основе холста
В этом разделе представлены рекомендации по работе с внедренными приложениями на основе холста, а также полезные советы по устранению возможных неполадок.

-   Внедренные приложения на основе холста поддерживаются только с управляемыми моделью приложениями единого интерфейса.
-   Можно включить только одно внедренное приложение на основе холста в одной форме. 
     - В форму можно добавить несколько внедренных приложений на основе холста, но одновременно может быть включено только одно из них.
     - При попытке включить несколько внедренных приложений на основе холста в управляемой моделью форме, отображается сообщение "В форме можно включить только одно приложение на основе холста".
     - Чтобы включить или отключить внедренное приложение на основе холста, см. раздел [Включение внедренного приложения на основе холста](#enable-an-embedded-canvas-app) и [Отключение внедренного приложения на основе холста](#disable-an-embedded-canvas-app).
-   При добавлении внедренного приложения на основе холста в управляемую моделью форму всегда используйте поле, которое гарантировано имеет значение. Если поле не имеет значения, внедренное приложение на основе холста не обновляется в ответ на любое изменение данных в управляемой моделью форме, в которой размещено приложение.
-   Публикация управляемой моделью формы не означает публикацию также и внедренного приложения на основе холста.
     - Внедренные приложения на основе холста должны быть опубликованы независимо от управляемой моделью формы, в которой они размещены. Дополнительные сведения: [Публикация приложения](../canvas-apps/save-publish-app.md#publish-an-app).
-   Если открытие Power Apps Studio для создания или изменения внедренного приложения на основе холста с помощью кнопки **Настройка** в свойствах элемента управления приложения на основе холста заблокировано из-за блокировки всплывающих окон браузера, нужно включить сайт make.powerapps.com или временно отключить средство блокирования всплывающих окон, затем снова выбрать **Настройка**.
-   Внедренные приложения на основе холста не отображаются при создании новой записи, так как требуется, чтобы им был передан контекст записи.
-   Объект ModelDrivenFormIntegration.Item доступен только для чтения. 
     - Чтобы записать данные обратно, необходимо использовать соединитель Common Data Service. Дополнительные сведения см. в разделе [Common Data Service](/connectors/commondataservice/)
-   Внедренные приложения на основе холста можно создавать только через управляемую моделью форму, в которой они размещены. 
    - Добавление существующих приложений на основе холста в качестве внедренный в управляемые моделью формы в настоящее время не поддерживается.
    - Поддержка внедрения существующего приложения на основе холста в управляемую моделью форму с использование идентификатора приложения будет обеспечена в будущем обновлении.
- Если при просмотре управляемой моделью формы с внедренным приложением на основе холста отображается сообщение об ошибке "Это приложение не найдено", убедитесь, что внедренное приложение на основе холста находится в том же решении, что и управляемая моделью форма.
- Если при просмотре управляемой моделью формы с внедренным приложением на основе холста отображается сообщение об ошибке "Похоже, у вас нет доступа к этому приложению. Попросите ответственного предоставить вам доступ", убедитесь, что автор предоставил вам доступ к внедренному приложению на основе холста. Дополнительные сведения: [Общий доступ к внедренному приложению на основе холста](share-embedded-canvas-app.md).
- Добавление приложения на основе холста в элемент управления вложенной сетки больше не доступно.
    - В предварительной версии, создатели могут добавить приложение на основе холста в элемент управления вложенной сетки. Теперь, когда внедрение приложения на основе холста в формы на основе модели находится в общем доступе, добавление встроенного приложения на основе холста в форму на основе модели упрощается. 
    - Это облегчает работу разработчиков, поскольку им не нужно заранее решать, передавать ли текущую запись (основная форма) в качестве контекста данных или список записей, связанных с текущей записью (основная форма). 
    - Разработчики всегда начинают с поля и могут получить доступ как к текущей записи (основной формы), так и к списку записей, связанных с текущей записью (основной формы).
    - Для получения доступа к списку связанных записей в приложении на основе холста разработчики могут использовать соединитель Common Data Service и функцию [фильтр](../canvas-apps/functions/function-filter-lookup.md) с включенной функцией [Улучшение источников данных и представлений Common Data Service](https://powerapps.microsoft.com/blog/improved-data-source-selection-and-common-data-service-views/) в приложении на основе холста.  
    Например, чтобы получить доступ к представлению *Активные контакты* сущности *Контакты*, разработчики могут использовать: *Фильтр(Контакты, "Контакты (представления)". "Активные контакты")*.
    - Существующие приложения на основе холста использующие элемент управления вложенной сетки будут продолжать работать. Однако рекомендуем осуществлять перенос этих приложений, чтобы использовать поле вместо этого. Дополнительные сведения: [Перенос внедренных приложений на основе холста на управляемых моделью формах, которые используют список записей, связанных с текущей записью (основной формы)](embedded-canvas-app-migrate-from-preview.md#migrating-embedded-canvas-apps-on-model-driven-forms-that-use-a-list-of-records-related-to-the-current-main-form-record).

## <a name="enable-an-embedded-canvas-app"></a>Включение внедренного приложения на основе холста
1. Выберите поле, которое было настроено для отображения как внедренное приложение на основе холста.
2. В диалоговом окне **Свойства поля** выберите вкладку **Элементы управления**.
3. В списке элементов управления выберите **Приложение на основе холста**, затем выберите параметр **Веб**.
4. Нажмите **ОК**.

## <a name="disable-an-embedded-canvas-app"></a>Отключение внедренного приложения на основе холста
1. Выберите поле, которое было настроено для отображения как внедренное приложение на основе холста.
2. В диалоговом окне **Свойства поля** выберите вкладку **Элементы управления**.
3. В списке элементов управления выберите элемент управления по умолчанию, затем выберите параметр **Веб**.
4. Нажмите **ОК**.

## <a name="known-issues-and-limitations-with-embedded-canvas-apps"></a>Известные проблемы и ограничения, связанные с внедренными приложениями на основе холста
- Пользовательский элемент управления приложения на основе холста поддерживается только для использования с клиентом типа **Веб**. В настоящее время клиенты типов **Телефон** и **Планшет** не поддерживаются.
- Элемент управления ModelDrivenFormIntegration не содержит значения полей связанной сущности. 
  - Например, когда элемент управления ModelDrivenFormIntegration подключен к сущности учетной записи, использование *ModelDrivenFormIntegration.Item."Основной контакт". "Полное имя"* не вернет значение. 
  - Для доступа к полям связанных сущностей разработчики могут использовать любое из приведенных здесь выражений:
    - *Поиск(учетные записи, учетная запись = GUID(First(ModelDrivenFormIntegration.Data).ItemId))."Основной контакт". "Полное имя"*  
      - *ItemId* пуст во время разработки, но будет иметь значение во время выполнения.
    - *Поиск(учетные записи, учетная запись = ModelDrivenFormIntegration.Item.Account)."Основной контакт". "Полное имя"* (Это выражение легче читать, но предыдущее выражение будет работать немного лучше.)
- Нельзя использовать привилегию **Приложение на основе холста** в роли безопасности для предоставления пользователю доступа к внедренному или автономному приложению на основе холста. Дополнительные сведения об общем доступе к внедренному приложению на основе холста см. в разделе: [Общий доступ к внедренному приложению на основе холста](share-embedded-canvas-app.md).
- При обратной записи тех же данных, которые показываются в управляемой моделью форме, в которой размещено приложение, в форме будут и далее отображаться старые данные, пока она не будет обновлена. Простой способ сделать это — использовать метод [RefreshForm](embedded-canvas-app-actions.md#refreshformshowprompt).
