# <a name="faqs-for-timeline-control"></a>Вопросы и ответы для элемента управления "Временная шкала"

## <a name="why-do-i-receive-the-message-records-could-not-be-loaded-because-of-unexpected-error"></a>Почему я получают сообщение "Не удалось загрузить записи из-за непредвиденной ошибки"?

Раздел **Временная шкала** извлекает данные сведений и отображает в форме карточек. По умолчанию временная шкала извлекает данные для 10 стандартных сущностей действий, а именно:

-   Адрес электронной почты
-   Задача
-   Разрешение инцидентов
-   Факс
-   Закрытие возможной сделки
-   Письмо
-   Встреча
-   Звонок

Когда вы выполните следующие процедуры как администратор, пользователи увидят ошибку во время выполнения:

**Процедура**
-   Создайте любые дополнительные пользовательские действия
-   Включение пользовательских действий для мобильных устройств
-   Выберите **Форма карточки** для всех пользовательских действий 

**Ошибка.** Не удалось загрузить записи из-за непредвиденной ошибки.

   > [!div class=mx-imgBorder] 
   > ![Не удалось загрузить записи из-за непредвиденной ошибки.](media/timeline-error1.png "Не удалось загрузить записи из-за непредвиденной ошибки.")

Эта ошибка вызвана тем, что число сущностей действия для извлечения данных превысило максимальный предел 10.

   > [!div class=mx-imgBorder] 
   > ![Количество сущностей ссылки в запросе превысило максимальный предел](media/timeline-error2.png "[Количество сущностей ссылки в запросе превысило максимальный предел")

### <a name="workaround"></a>Решение

Чтобы обойти эту проблему, вы должны уменьшить количество объектов до 10 или меньше. Для этого выполните описанные ниже шаги.

1.  Войдите в среду `https://<YourOrgURL>.dynamics.com/apps`.

2.  Откройте управляемое моделью приложение, затем на панели команд выберите **Параметры** ![Параметры](../model-driven-apps/media/powerapps-gear.png) > **Дополнительные параметры**.

3.  Перейдите в раздел **Параметры** > **Настройка** > **Настроить систему**. Откроется страница обозревателя решений в новом окне браузера.

4.  Разверните **Сущности** в разделе **Компоненты** на панели решения по умолчанию.

5.  Выберите сущность и выберите **Формы**. Например, выберите сущность **Организация**.

6.  Выберите запись **Организация для интерактивного взаимодействия**, которая является типом формы **Основная**. Форма **Организация для интерактивного взаимодействия** откроется в новом окне браузера.

   > [!div class=mx-imgBorder] 
   > ![Выберите форму сущности с интерактивным взаимодействием в названии](media/account-interactive-experience.png "Выберите форму сущности с интерактивным взаимодействием в названии")

   Для единого интерфейса нужно использовать имя формы, которое имеет `<Entity> for Interactive experience`.

7.  Дважды щелкните поле **Вкладки разговора** в разделе **Временная шкала**. Отображается диалог **Свойства вкладки "Действия"**.

    > [!div class=mx-imgBorder] 
    > ![Дважды щелкните поле в социальной области](media/timeline-conversation-tabs-field.png "Дважды щелкните поле в социальной области")  

8.  Выберите параметр **Показать выбранные** для поля **Показывать эти действия** в контейнере **Фильтровать по**.

9.  Выберите действия, которых требуется отображать пользователям.

10. Выберите **ОК**, затем выберите **Сохранить**.

11. Выберите **Опубликовать**, чтобы опубликовать настройку.


## <a name="why-i-cant-assign-or-delete-an-activity-from-the-timeline"></a>Почему я не могу назначить или удалить действие с временной шкалы?

Если вы используете правило **HideCustomActions**, чтобы скрыть кнопки, такие как **Назначить** и **Удалить** в определении панели команд ленты, то те кнопки, которые присутствуют в элементе управления "Временная шкала", не будут работать. Кнопки на панели команд совпадают с кнопками в элементе управления "Временная шкала"; поэтому, когда пользователь выбирает кнопку **Назначить** или **Удалить** на временной шкале, отображается сообщение об ошибке.

**У вас нет разрешения для выполнения этого действия. Обратитесь к системному администратору.**

Чтобы устранить эту проблему, отмените скрытие кнопок в определениях панели команд.


## <a name="why-my-users-see-different-activities-and-records-in-their-my-activities-stream-in-the-dashboard"></a>Почему мои пользователи видят различные действия и записи в своем потоке "Мои действия" на панели мониторинга?

Поток **Мои действия** на панели мониторинга показывает записи и действия, которые принадлежат конкретному пользователю. Например, пользователь **A** видит записи и действия, которые принадлежат пользователю **A**, а пользователь **В** видит записи и действия, которые принадлежат пользователю **В**.
