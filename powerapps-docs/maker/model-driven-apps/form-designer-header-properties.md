# <a name="configure-header-properties-in-the-form-designer"></a>Настройка свойств верхнего колонтитула в конструкторе форм

Создатели могут контролировать плотность верхних колонтитулов форм управляемых моделью приложений, чтобы они соответствовали потребностям любого, кто использует форму.

> [!NOTE]
> Эта функция доступна только в тех средах, в которых используется предварительная версия функций волны 2 выпуска 2019 года. Дополнительные сведения: [Раннее получение волны 2 выпуска 2019 года](https://docs.microsoft.com/power-platform/admin/preview-october-2019-updates). 

## <a name="high-density-header"></a>Заголовок высокой плотности

Заголовок формы высокой плотности гарантирует, что ключевая информация постоянно отображается для пользователей. С помощью заголовка высокой плотности заголовок записи никогда не усекается. Даже длинные заголовки записей отображаются с помощью нескольких строк. Аналогично, заголовок высокой плотности также обеспечивает, что до четырех значений полей напрямую видны в заголовке и никогда не усекаются и не скрываются.  

Для обеспечения того, что ключевая информация всегда отображается, инфраструктура отображает доступные только для чтения значения полей, и пользователи не могут напрямую изменять значения полей в заголовке. Элементы визуализации, такие как настраиваемые компоненты или веб-ресурсы, также не допускаются.

Если в форме не указана плотность заголовков или при создании новой формы инфраструктура по умолчанию задает заголовок высокой плотности.

> [!div class="mx-imgBorder"] 
> ![Заголовок формы высокой плотности](media/form-header-high-density.png "Заголовок формы высокой плотности")
    
## <a name="low-density-header"></a>Заголовок низкой плотности
Заголовок формы низкой плотности позволяет пользователям сразу изменять значения полей в заголовке. Он также допускается элементы визуализации, такие как настраиваемые компоненты и веб-ресурсы.  
  
Однако часто в результате ключевая информация усекается или не сразу видна. Заголовок низкой плотности усекает заголовок записи, а также значения полей, отображаемых в заголовке. Часто только одно или два поля непосредственно видны в заголовке, остальные уходят в переполнение и отображаются во всплывающем окне, требуя дополнительного щелчка.

> [!div class="mx-imgBorder"] 
> ![Заголовок формы низкой плотности](media/form-header-low-density.png "Заголовок формы низкой плотности")

### <a name="configuring-header-density"></a>Настройка плотности заголовка

Чтобы настроить плотность заголовка управляемой моделью формы, выполните следующие действия:
1.  Откройте конструктор форм для [создания или редактирования формы](create-and-edit-forms.md).
2.  Выберите заголовок формы, выбрав заголовок в представлении предварительного просмотра формы или с помощью [представления дерева](using-tree-view-on-form.md).
3.  В области свойств установите флажок **Высокая плотность** для использования заголовка высокой плотности или снимите его, чтобы использовать заголовок формы низкой плотности.
4.  На панели команд выберите **Сохранить** для сохранения формы или выберите **Опубликовать**, чтобы сохранить изменения и сделать их видимыми пользователям.

> [!NOTE]
> Используйте новый конструктор форм. Классический конструктор форм не предусматривает возможность настройки плотности заголовка.

## <a name="header-flyout"></a>Всплывающее окно верхнего колонтитула
Всплывающее окно заголовка отображается, когда пользователь выбирает шеврон в заголовке формы. Оно позволяет пользователям изменять значения полей и также отображать элементы визуализации, такие как настраиваемые компоненты и веб-ресурсы, являющиеся частью заголовка формы.

Поведение всплывающего окна заголовка зависит от настройки плотности заголовка.

### <a name="high-density-header-flyout"></a>Всплывающее окно заголовка высокой плотности
В случае заголовка формы высокой плотности во всплывающем окне заголовка отображаются все поля заголовка, включая четыре поля, которые отображаются непосредственно в заголовке. Инфраструктура по умолчанию показывает всплывающее окно, когда используется заголовок высокой плотности. Создатели могут управляться видимостью всплывающего окна заголовка с заголовком высокой плотности.

> [!div class="mx-imgBorder"] 
> ![Всплывающее окно заголовка с заголовком высокой плотности](media/form-header-flyout-high-density.png "Всплывающее окно заголовка с заголовком высокой плотности")

### <a name="low-density-header-flyout"></a>Всплывающее окно заголовка низкой плотности
С заголовком формы низкой плотности во всплывающем окне заголовка отображаются только непоместившиеся поля, например поля, которые форма не может отображать непосредственно в заголовке на основе ширины формы. Всплывающее меню заголовка также автоматически отображается или скрывается на основании количества полей в заголовке и ширины формы. Создатели не могут управлять видимостью всплывающего окна заголовка при использовании заголовка низкой плотности.

> [!div class="mx-imgBorder"] 
> ![Всплывающее окно заголовка с заголовком низкой плотности](media/form-header-flyout-low-density.png "Всплывающее окно заголовка с заголовком низкой плотности")

### <a name="show-or-hide-the-header-flyout"></a>Отображение или скрытие всплывающего окна заголовка
Чтобы показать или скрыть всплывающее окно заголовка для управляемой моделью формы, выполните следующие действия:

1.  Откройте конструктор форм для [создания или редактирования формы](create-and-edit-forms.md).
2.  Выберите заголовок формы в предварительном просмотре формы или используйте [представление дерева](using-tree-view-on-form.md), чтобы выбрать его.
3.  В области свойств выберите **Высокая плотность** для использования заголовка формы высокой плотности. 
4.  В области свойств установите флажок **Показать всплывающее меню верхнего колонтитула**, чтобы всплывающее меню верхнего колонтитула было видно, или снимите его, чтобы скрыть всплывающее меню верхнего колонтитула.
5.  На панели команд выберите **Сохранить** для сохранения формы или выберите **Опубликовать**, чтобы сохранить изменения и сделать их видимыми пользователям.

> [!NOTE]
> - Используйте новый конструктор форм. Классический конструктор форм не предусматривает возможность отображения или скрытия всплывающее меню заголовка.   
> - Видимостью всплывающего окна заголовка можно управлять только при использовании заголовка формы высокой плотности. При использовании заголовка низкой плотности всплывающее меню заголовка автоматически отображается или скрывается на основании количества полей в заголовке и ширины формы.
> - Изображение для сущности будет отображаться в заголовке, только если для сущности определен атрибут **Основное изображение** и свойство формы **Показать изображение в форме**. Дополнительные сведения: [Поля изображения](../common-data-service/types-of-fields.md#image-fields). <br />
    Разработчики могут определять изображение для сущности с помощью атрибута [EntityMetadata.PrimaryImageAttribute](https://docs.microsoft.com/dotnet/api/microsoft.xrm.sdk.metadata.entitymetadata.primaryimageattribute?view=dynamics-general-ce-9). 


## <a name="form-designer-messages-related-to-form-headers"></a>Сообщения конструктора форм, связанные с заголовками формы
При изменении формы с помощью классического или нового конструктора форм могут отображаться сообщения, связанные с заголовками формы. Ниже можно найти сведения о каждом сообщении и почему оно отображается.

### <a name="weve-upgraded-your-form-to-show-a-high-density-header-that-displays-more-data-you-can-edit-this-setting-in-the-properties-of-the-header"></a>Вы обновили свою форму для отображения заголовка высокой плотности, в котором отображается больше данных. Можно изменить этот параметр в свойствах заголовка. 
Это сообщение отображается в конструкторе форм, когда создатель создает новую главную форму (включая действие "Сохранить как") или изменяет главную форму, которая ранее не была настроена для плотности заголовка.  
  
По умолчанию инфраструктура использует верхний колонтитул высокой плотности, и это сообщение помогает создателям быть осведомленными об этом поведении. Создатели могут переопределить значение инфраструктуры по умолчанию в любой момент, вручную настроив плотность заголовка формы, как было указано выше.

### <a name="this-form-isnt-using-high-density-header-access-the-setting-in-the-header-properties-high-density-header-helps-display-more-data"></a>В этой форме не используется заголовок с высокой плотностью. Для отображения дополнительных данных откройте этот параметр в новом конструкторе форм. Заголовок высокой плотности помогает отображать больше данных. 
Это сообщение отображается в конструкторе форм, когда создатель открывает основную форму для редактирования, для которой настроено использование заголовка низкой плотности. 

Это сообщение помогает повысить осведомленность о заголовке высокой плотности и его преимуществах.

### <a name="field-moved-to-header-flyout-the-header-supports-displaying-up-to-four-read-only-field-values-the-field-field-display-name-will-now-only-be-available-from-the-flyout"></a>Поле перемещено во всплывающее меню заголовка: заголовок поддерживает отображение до четырех доступных только для чтения значений полей. Поле *[отображаемое имя пользователя]* теперь будет доступно только во всплывающем меню.
Это сообщение отображается в конструкторе форм, если форма использует заголовок высокой плотности с видимым всплывающим меню заголовка.  
  
В заголовке высокой плотности отображаются доступные только для чтения значения первых четырех полей в заголовке. Когда создатели добавляют поле в заголовок в четыре первых положения, это приводит к тому, что существующее поле, которая ранее отображалось непосредственно в заголовке, становится расширенным и видимым только во всплывающем меню заголовка.      

Это сообщение информирует создателя об изменении и запрашивает подтверждение, требуется ли продолжить выполнять действие.

### <a name="header-field-limit-exceeded-the-header-supports-displaying-up-to-four-read-only-field-values-remove-unused-fields-to-add-more"></a>Превышен предел числа полей заголовка: заголовок поддерживает отображение до четырех доступных только для чтения значений полей. Удалите неиспользуемые поля для добавления дополнительных. 
Это сообщение отображается в конструкторе форм, если форма использует заголовок высокой плотности со скрытым всплывающим меню заголовка.  
  
В заголовке высокой плотности отображаются доступные только для чтения значения до четырех полей в заголовке. Поскольку всплывающее меню заголовка скрыто, пользователи не смогут просмотреть дополнительные поля.  

Это сообщение информирует создателя, что уже имеются четыре поля в заголовке, и не позволяет добавить дополнительные поля в заголовке, которые пользователи не смогут увидеть.

### <a name="header-does-not-display-custom-components-the-header-supports-displaying-up-to-four-read-only-field-values-remove-the-custom-component-from-the-field-before-adding-it-to-the-header"></a>Заголовок не отображает настраиваемые компоненты: заголовок поддерживает отображение до четырех доступных только для чтения значений полей. Удалите настраиваемый компонент из поля перед добавлением в верхний колонтитул.  
Это сообщение отображается в конструкторе форм, если форма использует заголовок высокой плотности со скрытым всплывающим меню заголовка.  
  
В заголовке высокой плотности отображаются доступные только для чтения значения полей в заголовке. Поскольку всплывающее меню заголовка скрыто, пользователи не смогут просматривать никакие настраиваемые компоненты, связанные с полями в заголовке.  

Это сообщение информирует создателя, что он пытается добавить поле со связанным настраиваемым компонентом в верхний колонтитул, и они должны удалить настраиваемый компонент перед добавлением поля в верхний колонтитул. Это связано с тем, что пользователи не смогут увидеть настраиваемый компонент в заголовке.

### <a name="header-displays-read-only-field-values-the-header-supports-displaying-up-to-four-read-only-field-values-to-provide-editability-to-the-user-add-the-field-to-a-section-in-the-form"></a>В заголовке отображаются значения полей только для чтения: заголовок поддерживает отображение до четырех доступных только для чтения значений полей. Для предоставления возможности редактирования пользователю добавьте поле в раздел в форме.  
Это сообщение отображается в конструкторе форм, если форма использует заголовок высокой плотности со скрытым всплывающим меню заголовка.  
  
В заголовке высокой плотности отображаются доступные только для чтения значения полей в заголовке. Поскольку всплывающее меню заголовка скрыто, пользователи не смогут редактировать значения полей.  
  
Это сообщение информирует создателя, что любые добавляемые в верхний колонтитул поля будут доступны только для чтения, и все поля, которые должны быть доступны пользователям для редактирования, также должны быть добавлены в раздел в форме.

### <a name="header-field-values-are-not-editable-moving-a-field-from-the-body-to-the-header-will-display-as-a-read-only-value-to-maintain-editability-copy-the-field-to-the-header"></a>Значения полей заголовка не являются изменяемыми: при перемещении поля из текста в заголовок оно будет отображать доступное только для чтения значение. Для сохранения возможности редактирования скопируйте поле в верхний колонтитул.  
Это сообщение отображается в конструкторе форм, только если форма использует заголовок высокой плотности со скрытым всплывающим меню заголовка.  
  
В заголовке высокой плотности отображаются доступные только для чтения значения полей в заголовке. Поскольку всплывающее меню заголовка скрыто, пользователи не смогут редактировать значения полей.  

Это сообщение информирует создателя, что он пытается переместить поле из тела формы в верхний колонтитул формы. В результате оно будет доступно только для чтения. Это позволяет создателю выбрать перемещение поля в верхний колонтитул или добавление копии поля в верхний колонтитул. При перемещении поля в верхний колонтитул оно станет недоступно в первоначальном расположении в теле формы для пользователей для редактирования. Добавление копии поля в заголовок оставит поле в первоначальном расположении без изменений, позволяя пользователям продолжать его изменять в теле формы.

### <a name="form-headers-now-default-to-high-density-to-display-more-data-use-the-new-form-designer-to-edit-header-density"></a>Заголовки форм теперь по умолчанию имеют высокую плотность для отображения дополнительных данных. Используйте новый конструктор форм для редактирования плотности заголовка.  
Это сообщение отображается в классическом конструкторе форм, когда создатель открывает основную форму для редактирования, и для нее настроено использование заголовка низкой плотности. 

Это сообщение помогает повысить осведомленность о заголовке высокой плотности и его преимуществах, и что создатели должны теперь использовать новый конструктор форм для установки плотности заголовка.  

### <a name="this-form-is-using-high-density-header-for-the-best-authoring-experience-with-this-form-use-the-new-form-designer"></a>В этой форме используется заголовок с высокой плотностью. Чтобы максимально эффективно использовать эту форму для разработки, используйте новый конструктор форм. 
Это сообщение отображается в классическом конструкторе форм, когда создатель открывает основную форму для редактирования, и для нее настроено использование заголовка высокой плотности.  
  
Классический конструктор форм не обеспечивает возможности разработки в режиме WYSIWYG. Он также не обнаруживает и не предотвращает или не предупреждает создателей о последствиях изменения, которые они вносят в заголовок формы. Например, при редактировании формы, в которой используется заголовок высокой плотности и всплывающее меню заголовка скрыто, классический конструктор форм не запрещает создателям добавлять более четырех полей в верхний колонтитул, даже если это поле не будет доступно пользователям.  
  
Это сообщение информирует создателя, что при редактировании формы, в которой используется заголовок высокой плотности, необходимо использовать новый конструктор форм. Это помогает обеспечить, что создатель отдает себе отчет о влиянии изменений на верхний колонтитул формы.
