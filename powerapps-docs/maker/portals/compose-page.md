# <a name="compose-a-page"></a>Создание страницы

[!include[cc-beta-prerelease-disclaimer](../../includes/cc-beta-prerelease-disclaimer.md)]

После добавления необходимых веб-страниц и управления иерархией их в на карте сайта можно добавить в различные компоненты. Редактор WYSIWYG позволяет добавлять и изменять требуемые компоненты на холсте легко. Можно добавлять и изменять на холсте на следующие компоненты:

- Разделы
    - Раздел из одного столбца
    - Раздел из двух столбцов
    - Раздел из трех столбцов
- Компоненты портала
    - Text
    - Изображение
    - IFrame
    - Форма
    - Список
    - Навигационная цепочка

> [!NOTE]
> Если вы настроите портал с помощью конструктора портала, пользователи веб-сайта заметят снижение производительности. Мы порекомендовали. чтобы внести изменения в непиковые часы на эксплуатационном портале. 

## <a name="use-the-wysiwyg-editor"></a>Использование редактора WYSIWYG

1.  [Изменение портал](manage-existing-portals.md#edit) чтобы открыть его в конструкторе портала.  

2.  Выберите страницу, для которого требуется добавить компонент.

3.  Выберите редактируемый элемент на холсте.

    > [!NOTE]
    > Изменяемые элементы разграничены границей.

4.  Выберите **Компоненты** ![значок "Компоненты"](media/components-icon.png "значок \"Компоненты\"") из панели инструментов в левой части экрана.  

5.  Выберите компонент для добавления.

    > [!div class=mx-imgBorder]
    > ![Панель "Компоненты"](media/components-pane.png "Панель \"Компоненты\"")  

    Выбранный компоненты добавляются на холст внутри редактируемого элемента.

6.  Для удаления компонента выберите его на холсте затем выберите **Удалить** на панели команд вверху страницы.

    > [!div class=mx-imgBorder]
    > ![удалить компонент](media/delete-component.png "Удалить компонент")  

## <a name="add-sections"></a>Добавить разделы

Разделы позволяют указать структуру страницы и расположить портала компоненты соответственно. После добавления разделов на страницу вы можете добавлять компоненты портала в разделы в соответствии с требованиями.

1.  [Изменение портал](manage-existing-portals.md#edit) чтобы открыть его в конструкторе портала.

2.  Выберите страницу, на которую необходимо добавить раздел.

3.  Выберите редактируемый элемент на холсте.

4.  Выберите **Компоненты** ![значок "Компоненты"](media/components-icon.png "значок \"Компоненты\"") из панели инструментов в левой части экрана.

5.  В разделе **Макет раздела**выберите тип раздела для вставки.

6.  В области свойств в правой части экрана введите или выберите следующие сведения:

    - **Минимальная высота**: введите минимальную высоту раздела. Если вы добавляете компонент, который занимает больше места, чем указанная высота, раздел расширяется для размещения компонента. По умолчанию минимальная высота составляет 100 пикселей. Можно также ввести высоту в точках (pt) и процентах (%).

        > [!div class=mx-imgBorder]
        > ![Выравнивание в разделе](media/section-props-height.png "Выравнивание в разделе")  

    - **Выравнивание**: выберите, должен ли компонент в разделе быть выровнен по центру, по левому или правому краю.

        > [!div class=mx-imgBorder]
        > ![Выравнивание в разделе](media/section-props-align.png "Выравнивание в разделе")  

    - **Фон**: Выберите, хотите ли вы иметь цвет или изображение в качестве фона раздела.

        - **Заливка**: Выберите цвет фона.

            > [!div class=mx-imgBorder]
            > ![Цвет заливки раздела](media/section-props-fill.png "Цвет заливки раздела")  

        - **Изображение**: выберите Изображение из списка. Если вы хотите загрузить новое изображение, выберите **Загрузить изображение**.

            > [!div class=mx-imgBorder]
            > ![Добавить изображение в раздел](media/section-props-image.png "Добавить изображение в раздел")  

7.  Добавьте необходимый раздел портала в раздел.

### <a name="add-text-box"></a>Добавить текстовое поле

1.  [Изменение портал](manage-existing-portals.md#edit) чтобы открыть его в конструкторе портала.  

2.  Выберите страницу, для которого требуется добавить компонент.

3.  Выберите редактируемый элемент на холсте.

4.  Выберите **Компоненты** ![значок "Компоненты"](media/components-icon.png "значок \"Компоненты\"") из панели инструментов в левой части экрана.  

5.  В разделе **Компоненты портала** выберите **Текст**.

6.  Введите необходимый текст в текстовое поле.

7.  Чтобы форматировать текст, выбрать текст, чтобы показать параметры формата. Изменение размера и стиль шрифта по мере необходимости.

    > [!div class=mx-imgBorder]
    > ![компонент текста](media/text-component.png "Компонент текста")  

8. В области свойств в правой части экрана выберите следующие сведения:

    - **Выравнивание**: выберите, должен ли текст быть выровнен по центру, по левому или правому краю.

    - **Цвет шрифта**: Выберите цвет для текста.

        > [!div class=mx-imgBorder]
        > ![Выберите выравнивание и цвет текста](media/text-props.png "Выберите выравнивание и цвет текста")  
 

### <a name="add-image"></a>Добавить изображение

1.  [Изменение портал](manage-existing-portals.md#edit) чтобы открыть его в конструкторе портала.  

2.  Выберите страницу, для которого требуется добавить компонент.

3.  Выберите редактируемый элемент на холсте.

4.  Выберите **Компоненты** ![значок "Компоненты"](media/components-icon.png "значок \"Компоненты\"") из панели инструментов в левой части экрана.  

5.  В разделе **Компоненты портала** выберите **Изображение**. Местозаполнитель изображения добавляется на холст.

6.  В области свойств в правой части экрана введите следующие сведения:

    - **Изображение**. Выберите этот параметр если нужно выбрать существующее изображение или отправить новое. Если требуется выбрать ранее загруженное изображение, выберите изображение из списка **Выбор изображения**. Чтобы загрузить новое изображение, выберите **Загрузить изображение**. Все загруженные изображения включены в библиотеку изображений, которую можно выбрать снова с помощью списка **Выбор изображения**.

        > [!div class=mx-imgBorder]
        > ![свойства изображения](media/image-props.png "Свойства изображения")  

        > [!NOTE]
        > - Можно загружать только изображения типа png, svg, jpg, и jpeg с максимальным размером 5 МБ.
        > - Нельзя передать изображение с тем же именем. Необходимо изменить имя изображения, чтобы отправить его снова.

    - **Внешний URL-адрес**: выберите этот параметр если вы хотите загрузить изображение с внешнего URL-адреса. Введите URL-адрес в поле **URL-адрес**. Допускаются только защищенные ссылки, т. е. ссылка должна обязательно начинаться с https://. Если у вас есть изображения, хранящиеся в вашей сети доставки содержимого, вы можете указать ссылку в этом поле.

        > [!div class=mx-imgBorder]
        > ![внешний URL-адрес изображения](media/image-ext-url.png "внешний URL-адрес изображения")  

    -   **Параметры форматирования**

        - **Ширина**: Введите ширину изображения.

        - **Высота**: Введите высоту изображения.

    > [!NOTE]
    > Также можно выбрать изображение на холсте и перетаскивать ручки, чтобы изменить его размеры.

### <a name="add-iframe"></a>Добавить IFrame

1.  [Изменение портал](manage-existing-portals.md#edit) чтобы открыть его в конструкторе портала.  

2.  Выберите страницу, для которого требуется добавить компонент.

3.  Выберите редактируемый элемент на холсте.

4.  Выберите **Компоненты** ![значок "Компоненты"](media/components-icon.png "значок \"Компоненты\"") из панели инструментов в левой части экрана.  

5.  В разделе **Компоненты портала** выберите **IFrame**. На холст добавляется местозаполнитель IFrame.

6.  В области свойств в правой части экрана введите следующие сведения:

    - **Ширина**: введите ширину IFrame.

    - **Высота**: Введите высоту IFrame.

    - **Ссылка**. Введите URL-адрес веб-сайта, который будет отображаться в IFrame. Допускаются только защищенные ссылки, т. е. ссылка должна обязательно начинаться с https://. По умолчанию в качестве значения доступно <https://www.bing.com>.

        > [!div class=mx-imgBorder]
        > ![свойства IFrame](media/iframe-props.png "Свойства IFrame")  

    > [!NOTE]
    > Также можно выбрать IFrame на холсте и перетаскивать ручки, чтобы изменить его размеры.

### <a name="add-form"></a>Добавление формы

Форма — это управляемая данными конфигурация, которую вы используете для добавления формы для сбора данных на портале без необходимости для разработчика загружать форму на портал. [Формы создаются в Common Data Service](https://docs.microsoft.com/en-us/powerapps/maker/model-driven-apps/form-designer-overview), и вы можете использовать их в веб-страницах на портале или совместно со списками для создания полных веб-приложений.  

1.  [Изменение портал](manage-existing-portals.md#edit) чтобы открыть его в конструкторе портала.  

2.  Выберите страницу, для которого требуется добавить компонент.

3.  Выберите редактируемый элемент на холсте.

4.  Выберите **Компоненты** ![значок "Компоненты"](media/components-icon.png "значок \"Компоненты\"") из панели инструментов в левой части экрана.  

5.  В разделе **Компоненты портала** выберите **Форма**.

6.  В области свойств в правой части экрана выберите один из следующих параметров:

    - **Создать**: Создать новую форму.
    - **Использовать существующий**: Используйте существующую форму.

7. Введите сведения или сделайте выбор для следующего:

    - **Имя**: имя формы.

    - **Сущность**: Имя сущности, из которой будет загружена форма.

    - **Макет формы**: Имя формы в целевой сущности, в Common Data Service которое должно быть отображено.

    - **Режим**: выберите один из следующих параметров:

        - **Вставка**: указывает, что форма должна вставить новую запись при отправке.

        - **Редактировать**: указывает, что форма должна изменить существующую запись.

        - **Только для чтения**: указывает, что форма должна отображать недоступную для редактирования форму существующей записи.

        > [!NOTE]
        > Параметр по умолчанию для режимов **Редактировать** и **Только для чтения** установлен как имя параметра строки запроса передаваемое как идентификатор в URL-адрес. Для изменения этих значений возникает необходимость открыть приложение управления порталом и обновить свойства формы.

    - **При успешном завершении**: выберите один из следующих параметров:

        - **Показывать сообщение об успехе**: Требуется, чтобы сообщение отображалось пользователю при успешной отправке формы. Можно также выбрать **Скрыть форму при успешном завершении**, чтобы скрыть форму при успешной отправке.

        - **Перенаправить на веб-страницу**: Перенаправляет пользователя на выбранную веб-страницу на портале. Необходимо выбрать веб-страницу из списка **Перенаправить на веб-страницу**.

        - **Перенаправить на URL-адрес**: Перенаправляет пользователя на указанный URL-адрес. Необходимо ввести URL-адрес в поле **Перенаправить на URL-адрес**.

    - **Показывать captcha для анонимных пользователей**: Отображает captcha анонимным пользователям.

    - **Показывать captcha для пользователей, прошедших проверку подлинности**: Отображает captcha прошедшим проверку подлинности пользователям.

    - **Включить разрешения сущности**: Разрешения сущности. которые следует рассматривать для формы. По умолчанию не выбрано. Если выбрано, для доступа любого пользователя к форме требуются явные разрешения. Дополнительные сведения: [Разрешение сущности](https://docs.microsoft.com/en-us/dynamics365/customer-engagement/portals/assign-entity-permissions)

        > [!div class=mx-imgBorder]
        > ![свойства формы](media/form-props.png "Свойства формы")

### <a name="add-list"></a>Добавление списка

Список — это управляемая данными конфигурация, которая используется для добавления веб-страницы, которая будет отображать список записей, при этом разработчику не требуется отображать таблицу на портале. Списки используют [представления Common Data Service](https://docs.microsoft.com/en-us/powerapps/maker/model-driven-apps/create-and-edit-views) для отображения записей на портале.  

1.  [Изменение портал](manage-existing-portals.md#edit) чтобы открыть его в конструкторе портала.  

2.  Выберите страницу, для которого требуется добавить компонент.

3.  Выберите редактируемый элемент на холсте.

4.  Выберите **Компоненты** ![значок "Компоненты"](media/components-icon.png "значок \"Компоненты\"") из панели инструментов в левой части экрана.  

5.  В разделе **Компоненты портала** выберите **Список**.

6.  В области свойств в правой части экрана выберите один из следующих параметров:

    - **Создать**: Создать новый список.
    - **Использовать существующий**: Используйте существующий список.

7.  Введите сведения или сделайте выбор для следующего:

    - **Имя**: имя списка.

    - **Сущность**: Имя сущности, из которой будет загружены представления.

    - **Представления**: список представлений целевой сущности, которые требуется обработать. Можно выбрать несколько представлений для отображения записей в списке. Первое выбранное представление является представлением по умолчанию.

    - **Создать новую запись**: Позволяет пользователю создавать запись. Выберите новую веб-страницу содержащую форму для создания новой записи.

    - **Просмотр сведений**: Позволяет пользователю просматривать подробных сведений. Выберите веб-страницу содержащую форму для отображения сведений.

    - **Изменить запись**: Позволяет пользователю изменять запись. Выберите новую веб-страницу содержащую форму для изменения записи.

    - **Удалить запись**: Позволяет пользователю удалить запись.

    - **Пустое сообщение списка**: Сообщение, которое будет отображаться, когда нет записей для отображения.

    - **Количество записей на странице**: Целочисленное значение для указания количества записей, отображаемых на странице.

    - **Включить поиск в списке сущностей**: Позволяет пользователю искать записи в списке.

    - **Включить разрешения сущности**: Разрешения сущности. которые следует рассматривать для списка. По умолчанию не выбрано. Если выбрано, для доступа любого пользователя к форме требуются явные разрешения. Дополнительные сведения: [Разрешение сущности](https://docs.microsoft.com/en-us/dynamics365/customer-engagement/portals/assign-entity-permissions)  

    > [!div class=mx-imgBorder]
    > ![свойства списка](media/list-props.png "Свойства списка")

## <a name="add-a-custom-menu"></a>Добавление настраиваемого меню

По умолчанию меню на веб-сайте создается автоматически на основе иерархии веб-страниц. Это называется меню **по умолчанию**. Для создания настраиваемого меню необходимо создать набор веб-ссылок в приложении управления порталом. Дополнительные сведения см. в разделе [Управление веб-ссылками](https://docs.microsoft.com/en-us/dynamics365/customer-engagement/portals/manage-web-links)

После создания набора веб-ссылок:

1.  [Изменение портал](manage-existing-portals.md#edit) чтобы открыть его в конструкторе портала.

2.  Выберите компонент заголовка. 

3.  В свойствах в правой части экрана, выберите название набора веб-ссылок из списка **Меню навигации**.

    > [!div class=mx-imgBorder]
    > ![Меню навигации](media/navigation-menu.png "Меню навигации")

## <a name="use-code-editor"></a>Использование редактора кода

Чтобы просмотреть источник компонента на холсте, выберите его, а затем выберите значок редактора исходного кода **&lt;/&gt;** в нижнем колонтитуле.

> [!div class=mx-imgBorder]
> ![значок редактора кода](media/code-editor-icon.png "Значок редактора кода")  

Исходный код отображается в панели **Редактор кода** в нижней части экрана. Ранее внесенные вами изменения отражаются в исходном коде. Чтобы внести изменения, обновите исходный код и выберите **сохранить**. Изменения отражены на холсте.

> [!div class=mx-imgBorder]
> ![редактор кода](media/code-editor.png "Редактор кода") 

> [!NOTE]
> Можно также добавлять теги Liquid в редакторе исходного кода для расширенной конфигурации. Дополнительные сведения см. в разделе [Работа с шаблонами Liquid](https://docs.microsoft.com/en-us/dynamics365/customer-engagement/portals/custom-templates-dynamic-content)
