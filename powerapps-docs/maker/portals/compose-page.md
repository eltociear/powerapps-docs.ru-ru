# <a name="compose-a-page"></a>Создание страницы

После добавления необходимых веб-страниц и управления их иерархией в Sitemap можно добавить различные компоненты. Редактор WYSIWYG позволяет легко добавлять и изменять необходимые компоненты на холсте. На холсте можно добавлять и изменять следующие компоненты:

- Разделы
    - Раздел из одного столбца
    - Раздел из двух столбцов
    - Раздел из трех столбцов
- Компоненты портала
    - Text
    - Изображение
    - Элементы
    - Форма
    - Числ
    - Пути

> [!NOTE]
> При настройке портала с помощью PowerApps Portals Studio пользователи веб-сайта могут заметить влияние на производительность. Мы рекомендуем выполнить изменения в нерабочее время на активном портале. 

## <a name="use-the-wysiwyg-editor"></a>Использование редактора WYSIWYG

1.  [Измените портал](manage-existing-portals.md#edit) , чтобы открыть его в PowerApps Portals Studio.  

2.  Выберите страницу, на которую нужно добавить компонент.

3.  Выберите редактируемый элемент на холсте.

    > [!NOTE]
    > Редактируемые элементы демаркатед границей.

4.  В левой части **экрана выберите** ![значок](media/components-icon.png "Значок "компоненты"") компоненты компонентов из воле.  

5.  Выберите добавляемый компонент.

    > [!div class=mx-imgBorder]
    > ![область компонентов](media/components-pane.png "Область компонентов")  

    Выбранный компонент добавляется на холст внутри редактируемого элемента.

6.  Чтобы удалить компонент, выберите его на холсте, а затем выберите **Удалить** на панели команд в верхней части страницы.

    > [!div class=mx-imgBorder]
    > ![удалить компонент](media/delete-component.png "Удалить компонент")  

## <a name="add-sections"></a>Добавить разделы

Разделы позволяют определить структуру страницы и соответствующим образом расположить компоненты портала. После добавления разделов на страницу можно добавить компоненты портала в разделы в соответствии с требованиями.

1.  [Измените портал](manage-existing-portals.md#edit) , чтобы открыть его в PowerApps Portals Studio.

2.  Выберите страницу, на которую нужно добавить раздел.

3.  Выберите редактируемый элемент на холсте.

4.  В левой части **экрана выберите** ![значок](media/components-icon.png "Значок "компоненты"") компоненты компонентов из воле.

5.  В разделе **макет раздела**выберите тип раздела для вставки.

6.  На панели свойств в правой части экрана введите или выберите следующие сведения:

    - **Минимальная высота**: введите минимальную высоту раздела. При добавлении компонента, который занимает больше пространства, чем указанная высота, раздел разворачивается в соответствии с компонентом. По умолчанию минимальная высота — 100 пикс. Можно также ввести высоту в точках (Пт) и процентах (%).

        > [!div class=mx-imgBorder]
        > ![Выравнивание в разделе](media/section-props-height.png "Выравнивание в разделе")  

    - **Выравнивание**: укажите, должен ли компонент в разделе быть выровнен по левому краю, по центру или по правому краю.

        > [!div class=mx-imgBorder]
        > ![Выравнивание в разделе](media/section-props-align.png "Выравнивание в разделе")  

    - **Фон**: выберите, если требуется использовать цвет или изображение в качестве фона раздела.

        - **Заливка**: выберите цвет фона.

            > [!div class=mx-imgBorder]
            > ![Цвет заливки в разделе](media/section-props-fill.png "Цвет заливки в разделе")  

        - **Изображение**: выберите изображение из списка. Если вы хотите отправить новый образ, выберите **Отправить изображение**.

            > [!div class=mx-imgBorder]
            > ![Добавить изображение в раздел](media/section-props-image.png "Добавить изображение в раздел")  

7.  Добавьте необходимый компонент портала в раздел.

### <a name="add-text-box"></a>Добавление текстового поля

1.  [Измените портал](manage-existing-portals.md#edit) , чтобы открыть его в PowerApps Portals Studio.  

2.  Выберите страницу, на которую нужно добавить компонент.

3.  Выберите редактируемый элемент на холсте.

4.  В левой части **экрана выберите** ![значок](media/components-icon.png "Значок "компоненты"") компоненты компонентов из воле.  

5.  В разделе **компоненты портала**выберите **текст**.

6.  Введите требуемый текст в текстовое поле.

7.  Чтобы отформатировать текст, выделите текст, чтобы отобразить параметры формата. При необходимости измените размер и стиль шрифта.

    > [!div class=mx-imgBorder]
    > ![компонент текста](media/text-component.png "Компонент текста")  

8. На панели Свойства в правой части экрана выберите следующие сведения.

    - **Выравнивание**: укажите, должен ли текст быть выровнен по левому краю, по центру или по правому краю.

    - **Цвет шрифта**: выберите цвет для текста.

        > [!div class=mx-imgBorder]
        > ![Выбор выравнивания и цвета текста](media/text-props.png "Выбор выравнивания и цвета текста")  
 

### <a name="add-image"></a>Добавить изображение

1.  [Измените портал](manage-existing-portals.md#edit) , чтобы открыть его в PowerApps Portals Studio.  

2.  Выберите страницу, на которую нужно добавить компонент.

3.  Выберите редактируемый элемент на холсте.

4.  В левой части **экрана выберите** ![значок](media/components-icon.png "Значок "компоненты"") компоненты компонентов из воле.  

5.  В разделе **компоненты портала**выберите **изображение**. Заполнитель изображения добавляется на холст.

6.  На панели свойств в правой части экрана введите следующие сведения:

    - **Изображение**. Выберите этот параметр, если хотите выбрать существующий образ или передать новый. Если вы хотите выбрать ранее загруженное изображение, выберите его в списке **выбрать изображение** . Чтобы отправить новое изображение, выберите **Отправить изображение**. Все отправленные изображения включаются в библиотеку изображений, которую можно выбрать снова с помощью списка **выбрать изображение** .

        > [!div class=mx-imgBorder]
        > ![Свойства изображения](media/image-props.png "Свойства изображения")  

        > [!NOTE]
        > - Можно отправлять только изображения типа PNG, SVG, JPG и JPEG с максимальным размером 5 МБ.
        > - Невозможно отправить образ с тем же именем. Необходимо изменить имя образа, чтобы передать его снова.

    - **Внешний URL-адрес**: Выберите этот параметр, если вы хотите передать изображение из внешнего URL-адреса. Введите URL-адрес в поле **внешний URL-адрес** . Принимаются только защищенные ссылки, т. е. https://является обязательным. Если у вас есть образы, хранящиеся в сети доставки содержимого, можно указать ссылку в этом поле.

        > [!div class=mx-imgBorder]
        > ![внешний URL-адрес изображения](media/image-ext-url.png "Внешний URL-адрес изображения")  

    -   **Параметры форматирования**

        - **Ширина**: введите ширину изображения.

        - **Высота**: введите высоту изображения.

    > [!NOTE]
    > Можно также выбрать изображение на холсте и перетащить маркеры, чтобы изменить его размер.

### <a name="add-form"></a>Добавить форму

Форма — это управляемая данными конфигурация, которая используется для добавления формы для получения данных на портале без необходимости для разработчика отображать форму на портале. [Формы создаются в Common Data Service](https://docs.microsoft.com/powerapps/maker/model-driven-apps/form-designer-overview) и их можно использовать на веб-страницах на портале или в сочетании с списками для создания полных веб-приложений.  

1.  [Измените портал](manage-existing-portals.md#edit) , чтобы открыть его в PowerApps Portals Studio.  

2.  Выберите страницу, на которую нужно добавить компонент.

3.  Выберите редактируемый элемент на холсте.

4.  В левой части **экрана выберите** ![значок](media/components-icon.png "Значок "компоненты"") компоненты компонентов из воле.  

5.  В разделе **компоненты портала**выберите **форма**.

6.  На панели Свойства в правой части экрана выберите один из следующих вариантов.

    - **Создать**. Создание новой формы.
    - **Использовать существующий**: используйте существующую форму.

7. Введите сведения или сделайте выбор для следующих элементов:

    - **Name**: имя формы.

    - **Сущность**— имя сущности, из которой будет загружена форма.

    - **Макет формы**: имя формы в целевой сущности в Common Data Service, подготове к просмотру.

    - **Режим**: выберите один из следующих вариантов:

        - **INSERT**: указывает, что форма должна вставлять новую запись после отправки.

        - **Изменить**: указывает, что форма должна изменять существующую запись.

        - **Только для чтения**: указывает, что в форме должна отображаться нередактируемая форма существующей записи.

        > [!NOTE]
        > Параметр по умолчанию для режимов **Edit** и **ReadOnly** задается как имя параметра строки запроса, передаваемое как идентификатор в URL-адресе. Чтобы изменить эти значения, необходимо открыть приложение управления портала и обновить свойства формы.

    - **При успешном выполнении**: выберите один из следующих вариантов:

        - **Показать сообщение об успешном выполнении**: необходимо, чтобы пользователь отображал сообщение для успешной отправки формы. Можно также выбрать параметр **Скрыть форму при успешном завершении** , чтобы скрыть форму после успешной отправки.

        - **Перенаправление на веб-страницу**: перенаправляет пользователя на выбранную веб-страницу на портале. Необходимо выбрать веб-страницу из списка **перенаправление в веб-страницу** .

        - **Перенаправлять на URL-адрес**: перенаправляет пользователя по указанному URL-адресу. Необходимо ввести URL-адрес в поле " **перенаправить в URL-адрес** ".

    - **Показать CAPTCHA для анонимных пользователей**: отображает CAPTCHA для анонимных пользователей.

    - **Показать CAPTCHA для прошедших проверку пользователей**: отображает CAPTCHA для пользователей, прошедших проверку подлинности.

    - **Включить разрешения для сущности**: разрешения сущности, которые следует учитывать в форме. По умолчанию он не выбран. Если этот флажок установлен, для доступа к форме пользователям требуются явные разрешения. Дополнительные сведения: [разрешение сущности](configure/assign-entity-permissions.md)

        > [!div class=mx-imgBorder]
        > ![Свойства формы](media/form-props.png "Свойства формы")

### <a name="add-list"></a>Добавить список

Список — это управляемая данными конфигурация, используемая для добавления веб-страницы, которая будет отображать список записей без необходимости для разработчика отобразить сетку на портале. Списки используют [Common Data Service представления](https://docs.microsoft.com/powerapps/maker/model-driven-apps/create-and-edit-views) для отображения записей на портале.  

1.  [Измените портал](manage-existing-portals.md#edit) , чтобы открыть его в PowerApps Portals Studio.  

2.  Выберите страницу, на которую нужно добавить компонент.

3.  Выберите редактируемый элемент на холсте.

4.  В левой части **экрана выберите** ![значок](media/components-icon.png "Значок "компоненты"") компоненты компонентов из воле.  

5.  В разделе **компоненты портала**выберите **Список**.

6.  На панели Свойства в правой части экрана выберите один из следующих вариантов.

    - **Создать**. Создайте новый список.
    - **Использовать существующую**: Используйте существующий список.

7.  Введите сведения или сделайте выбор для следующих элементов:

    - **Имя**: имя списка.

    - **Сущность**: имя сущности, из которой будут загружены представления.

    - **Представления**: список представлений целевой сущности, которые должны быть отображены. Можно выбрать несколько представлений для отображения записей в списке. Представление, выбранное первым, будет представлением по умолчанию.

    - **Создание новой записи**. позволяет пользователю создать запись. Выберите веб-страницу, содержащую форму для создания новой записи.

    - **Просмотреть сведения**: позволяет пользователю просматривать подробные сведения. Выберите веб-страницу, содержащую форму, чтобы отобразить подробные сведения.

    - **Изменить запись**: разрешает пользователю изменять запись. Выберите веб-страницу, содержащую форму для изменения записи.

    - **Удалить запись**. позволяет пользователю удалить запись.

    - **Пустое сообщение списка**: сообщение, отображаемое при отсутствии записей для отображения.

    - **Число записей на страницу**: целое значение, определяющее количество записей, отображаемых на странице.

    - **Включить поиск в списке сущностей**: позволяет пользователю искать записи в списке.

    - **Включить разрешения для сущности**: разрешения сущности, которые следует учитывать для списка. По умолчанию он не выбран. Если этот флажок установлен, для доступа к форме пользователям требуются явные разрешения. Дополнительные сведения: [разрешение сущности](configure/assign-entity-permissions.md)  

    > [!div class=mx-imgBorder]
    > ![Свойства списка](media/list-props.png "Свойства списка")

## <a name="add-a-custom-menu"></a>Добавление пользовательского меню

По умолчанию меню на веб-сайте создается автоматически на основе иерархии веб-страниц. Он называется меню **по умолчанию** . Чтобы создать настраиваемое меню, необходимо создать набор веб-ссылок в приложении управления портала. Дополнительные сведения: [Управление веб-ссылками](configure/manage-web-links.md)

После создания набора веб-ссылок:

1.  [Измените портал](manage-existing-portals.md#edit) , чтобы открыть его в PowerApps Portals Studio.

2.  Выберите компонент заголовка. 

3.  В свойствах в правой части экрана выберите имя набора веб-ссылок из списка **меню навигации** .

    > [!div class=mx-imgBorder]
    > ![Меню навигации](media/navigation-menu.png "Меню навигации")

## <a name="use-code-editor"></a>Использовать редактор кода

Чтобы просмотреть источник компонента на холсте, выберите компонент, а затем щелкните значок редактора исходного кода **&lt;/&gt;** в нижнем колонтитуле.

> [!div class=mx-imgBorder]
> ![значок редактора кода](media/code-editor-icon.png "Значок редактора кода")  

Исходный код отображается в области **редактора кода** в нижней части экрана. Внесенные ранее изменения обновляются в исходном коде. Чтобы внести изменения, обновите исходный код и нажмите кнопку **сохранить**. Изменения отражены на холсте.

> [!div class=mx-imgBorder]
> ![редактор кода](media/code-editor.png "Редактор кода") 

> [!NOTE]
> Для расширенной конфигурации можно также добавить теги жидкостей в редакторе исходного кода. Дополнительные сведения: [Работа с шаблонами жидкостей](liquid/liquid-overview.md)
