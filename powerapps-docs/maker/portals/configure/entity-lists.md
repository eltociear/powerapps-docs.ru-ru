# <a name="about-entity-lists"></a>О списках сущностей

Список сущностей — это управляемая данными конфигурация, которая используется для добавления веб-страницы, которая будет отображать список записей, при этом разработчику не требуется отображать таблицу на портале. Используя списки сущностей, можно открыть записи для отображения на порталах.

Таблица поддерживает сортировку и будет разбиваться на страницы, если число записей больше указанного размера страницы. Если указана **Веб-страница для представления сведений**, каждая запись будет содержать ссылку на страницу и идентификатор записи будет добавлен к строке запроса вместе с идентификатором имени параметра строки запроса. Список сущностей также поддерживает несколько представлений. Если указано несколько представлений, раскрывающийся список обрабатывается таким образом, чтобы пользователь мог переключаться между разными представлениями.

Данные могут также фильтроваться по текущему пользователю портала, организации родительского клиента текущего пользователя портала и текущему веб-сайту портала. Если существует значения для обоих условий фильтрации **Атрибут пользователя портала** и **Атрибут организации**, портал представляет раскрывающийся список, чтобы пользователь мог просмотреть свои собственные данные (Мои) или данные организации родительского клиента.

## <a name="add-an-entity-list-to-your-portal"></a>Добавление списка сущностей в портал

Список сущностей содержит отношения с веб-страницами и различными свойствами для управления инициализацией списка записей на портале. Отношения с веб-страницами обеспечивают динамическое извлечение определения списка для указанного узла страницы на веб-сайте. Для просмотра существующих представлений сущностей или для создания новых представлений сущностей перейдите к пункту **Порталы** > **Списки сущностей**.

> [!Note]
> - Список сущностей должен быть связан с веб-страницей для определенного веб-сайта, чтобы этот список был виден на этом сайте.
> - Набор параметров с выбором нескольких вариантов не поддерживается в списках сущностей.

Веб-страницы, связанные со списком сущности, можно просмотреть, выбрав ссылку **Веб-страницы**, имеющуюся в ссылках навигации **В отношении** в самом левом меню. При создании списка сущностей на первом шаге требуется выбрать сущность, для которой требуется отобразить список на портале. После этого требуется выбрать одно или несколько представлений приложений на основе модели для отображения.

При создании или изменении веб-страницы можно указать список сущностей в поле поиска, предусмотренном на форме "Веб-страница". Шаблон страницы обычно будет шаблоном "Страница", однако может быть одним из нескольких других шаблонов, предназначенных для содержимого, так как главные шаблоны содержат необходимую логику для определения, требуется ли отображать список сущностей.

## <a name="add-custom-javascript"></a>Добавление пользовательского кода JavaScript

Вкладка "Параметры" в форме содержит текстовую область, в которую можно ввести пользовательский код [!INCLUDE[pn-javascript](../../../includes/pn-javascript.md)]; если ваша страница включает библиотеку jQuery, то ее также можно использовать здесь. Настраиваемый блок скрипта будет добавлен в нижнюю часть веб-страницы непосредственно перед закрывающим тегом формы на странице.

![Пример пользовательского кода JavaScript](../media/custom-javascript-example.png "Пример пользовательского кода JavaScript")  

Список получает данные асинхронно, и когда он будет готов, он запускает событие `loaded`, которое может ожидать ваш настраиваемый код [!INCLUDE[pn-javascript](../../../includes/pn-javascript.md)] и выполнять какие-то действия с элементами в сетке. Ниже приведен тривиальный пример кода:
```
$(document).ready(function (){
$(".entitylist.entity-grid").on("loaded", function () {
$(this).children(".view-grid").find("tr").each(function (){
// do something with each row
$(this).css("background-color", "yellow");
});
});
}); 
```

Поиск определенного поля атрибута и получение его значения, чтобы, возможно, изменить отображение значения. Ниже приведен код, который получает каждую ячейку таблицы, содержащую значение атрибута `accountnumber`. Замените `accountnumber` на атрибут, соответствующий вашей сущности, и посмотрите.
```
$(document).ready(function (){
   $(".entitylist.entity-grid").on("loaded", function () {
      $(this).children(".view-grid").find("td[data-attribute='accountnumber']").each(function (i, e){
         var value = $(this).data(value);
         // now that you have the value you can do something to the value
      });
   });
});
```
## <a name="entity-list-configuration"></a>Конфигурация списка сущностей

Можно легко включить и настроить действия (создание, изменение, удаление, и т. д.) для записей в списке сущностей. Также можно переопределить подписи по умолчанию, их размер и другие атрибуты, чтобы список сущностей отображался именно так, как вам нужно.

Эти параметры находятся в разделе "Конфигурация" формы списка сущностей. По умолчанию отображаются только **Основные параметры**. Выберите **Дополнительные параметры** для просмотра дополнительных параметров.

**Общие параметры действий**

Обычно действия сущности имеют параметры, которые можно настроить. Во всех случаях это сделано для того, чтобы предоставить дополнительные параметры для настройки, и эти поля не являются обязательными. Простое добавление действия позволяет выполнить это действие на портале, если были предоставлена соответствующая привилегия в разрешениях сущности.

## <a name="securing-entity-lists"></a>Защита списков сущностей

Чтобы защитить список сущностей, необходимо настроить разрешения сущностей для сущности, для которой отображаются записи, а также задать для логического значения **Включить разрешения сущности** для записи списка сущностей в значение true.

Действие защиты списка сущностей обеспечит, что для любого пользователя, который получает доступ к странице, отображаются только записи, для которых ему представлено разрешение. Этого можно добиться путем добавления дополнительного фильтра в представлениях приложений на основе модели, которые открываются с помощью списка. Этот фильтр будет фильтровать только записи, которые доступны для пользователя с разрешением **Чтение**.

Кроме того, все действия, которые определены для списка, будут учитывать соответствующие разрешения для этой операции для каждой записи. То есть, если для записи есть разрешение "Редактировать", действие редактирования будет включено для этой записи. То же самое справедливо для удаления, создания и т. п. Обратите внимание, что если нет доступных записей, то при загрузке списка будет отображаться сообщение об этом.

Однако при разработке веб-сайта рекомендуется определить, что если у пользователя нет роли, у которой имеются какие-либо разрешения для сущности (то есть, никогда не будет ситуации, в которой пользователь не видит никаких записей), этот пользователь вообще не получит доступа к этой странице. В идеальном случае страница должна быть защищена с помощью разрешений доступа к веб-странице.

## <a name="adding-a-view-details-page"></a>Добавление страницы просмотра сведений

Установив веб-страницу для подстановки представления сведений на веб-страницу, сведения записи, указанной в сетке, можно просматривать как доступные только для чтения или редактируемые, в зависимости от конфигурации связанной формы или страницы.

Эта страница может быть полностью настроенным шаблоном страницы, возможно, созданным с помощью Liquid. В, вероятно, самом распространенном сценарии страницей сведений будет веб-страница, содержащая форму сущности или веб-форму.

Важно помнить, что каждая запись, перечисленная в сетке, будет иметь гиперссылку на страницу сведений, и ссылка будет содержать именованный параметр строки запроса с идентификатором этой записи. Имя параметра строки запроса зависит от имени параметра строки запроса идентификатора, указанного в списке сущностей. Наконец, следует отметить, что целевая веб-страница сведений должна также знать этот параметр строки запроса для получения идентификатора записи, который требуется для запроса и загрузки ее данных.

**Использование формы сущности для отображения сведений**

Режим может быть редактированием или только для чтения, в зависимости от ваших потребностей.

**Использование страницы сведений для функции создания**

Можно использовать настраиваемую страницу, форму сущности или веб-форму одинаковым образом для функции создания. Это является альтернативой определению действия создания в форме. Невозможно определить как действие создания, *так и* пользовательскую страницу для действия "Создание": приоритет имеет пользовательское действие.

Если назначить веб-страницу поиску создания в списке сущностей, и не указать действие создания с помощью конфигурации, в списке будет отображаться кнопка создания; эта кнопка будет направлять пользователя на настраиваемую страницу, назначенную для создания.

## <a name="entity-list-filter-configuration"></a>Конфигурация фильтра списка сущностей

Добавить возможность фильтровать записи в списке сущностей легко; просто включите параметр фильтрации и выберите один или несколько типов фильтров для отображения пользователям. Можно фильтровать по атрибутам, которые соответствуют тексту, предоставленному пользователем, или выбирать из ряда параметров. Можно даже создавать практически любые типы фильтров, которые можно представить, с помощью расширенного поиска.

**Включение фильтра списка сущностей**

В разделе фильтра метаданных установите флажок "Включено". Будет добавлена область фильтра в список сущностей, когда он отображается. Пока вы не определите хотя бы один тип фильтра, поле будет пустым.

Можно определить, как отображается область фильтра в списке сущностей, с помощью параметра ориентации. По умолчанию область фильтра отображается горизонтально над списком сущностей. При вертикальной ориентации область фильтра отображается как поле слева от списка сущностей.

![Параметры фильтра метаданных](../media/metadata-filter-settings.png "Параметры фильтра метаданных")  

**Типы фильтров**

|**Тип фильтра**      |**Описание**                                                                                                                                                                                                                               |
|----------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Текстовый фильтр          | Фильтрация списка сущностей с помощью текстового поля для поиска соответствующего текста в выбранном атрибуте заданной сущности.                                                                                                                               |
| Набор фильтров атрибутов | Фильтрация списка сущностей с помощью ряда флажков, каждый из которых попытается сопоставить свое состояние с определенным атрибутом указанной сущности.                                                                                           |
| Поле поиска           | Фильтрация списка сущностей с помощью ряда флажков, каждый из которых представляет отношение между записью для указанной сущности и записью для связанной сущности.                                                                         |
| Набор фильтров по интервалу     | Подобно набору фильтров атрибутов, за исключением того, что каждый флажок может представлять два условия, а не одно (например, больше или равно 0 И меньше 100).                                                                    |
| Динамическое поле выбора | Аналогично выбору значения из списка выбора в наборе фильтров атрибута. Динамический набор списка выбора не требует указания параметров списка выбора, по которым требуется выполнять фильтрацию; вместо этого создается полный список параметров при загрузке списка сущностей. |
| Динамическое поле поиска   | Аналогично набору поиска. Динамический набор подстановки не требует указания параметров подстановки, по которым требуется выполнять фильтрацию; вместо этого создается полный список параметров при загрузке списка сущностей.                                           |
| Фильтр FetchXML      | Фильтрация списка сущностей с помощью условие фильтрации на языке FetchXML.                                                                                                                                                                                     |

**Текстовый фильтр**

Текстовый фильтр добавляет текстовое поле в область фильтрации списка сущностей, которая связана с атрибутом типа сущности списка сущностей. Если пользователь применяет фильтр, в списке сущностей отображаются только те записи, выбранный атрибут которых содержит это значение.

Чтобы добавить текстовый фильтр, нажмите кнопку **+ Текстовый фильтр**.

![Добавление текстового фильтра](../media/add-text-filter.png "Добавление текстового фильтра")  

В текстовом фильтре используются следующие атрибуты:

|**Название**     |**Описание**                                                                                                                                        |
|--------------|--------------------------------------------------------------------------------------------------------------------------------------------------------|
| Атрибут    | Имя атрибута с выбранным типом сущности списка сущностей, по которому следует выполнить фильтрацию. *Только атрибуты с типом "Строка" допустимы для текстового фильтра.*                                                                                   |
| Отображаемое имя | Переопределяет подпись для фильтра, когда отображается список сущностей. По умолчанию здесь автоматически задается имя выбранного атрибута. |

**Набор фильтров атрибутов**

Набор фильтров атрибутов добавляет ряд параметров для фильтрации списка сущностей, связанных с одним атрибутом выбранного типа сущности списка сущностей. Если пользователь применяет фильтр, в списке сущностей отображаются только те записи, которые точно соответствует по крайней мере одному из выбранных параметров.

![Параметры фильтра атрибутов](../media/set-attribute-filter.png "Параметры фильтра атрибутов")

В наборе фильтров атрибутов используются следующие атрибуты:

|**Название**     |**Описание**                                                                                                                                        |
|--------------|--------------------------------------------------------------------------------------------------------------------------------------------------------|
| Атрибут    | Имя атрибута с выбранным типом сущности списка сущностей, по которому следует выполнить фильтрацию. Только атрибуты со следующими типами допустимы для текстового фильтра: String, BigInt, Decimal, Double, Integer, Money, Picklist, DateTime и Boolean.
|Отображаемое имя | Переопределяет подпись для фильтра, когда отображается список сущностей. По умолчанию здесь автоматически задается имя выбранного атрибута.
| Параметры |      Список возможных значений, по которым требуется выполнить фильтрацию. Дополнительные сведения см. ниже.                                                                              |

**Параметры набора фильтров атрибутов**

Набор фильтров атрибутов обычно имеет произвольное число параметров, за исключением атрибутов Picklist и Boolean. Набор фильтров атрибутов Boolean может иметь только один или два параметра &mdash; один параметр "истина", и один параметр "ложь". Набор фильтров атрибутов Picklist может иметь не более одного параметра для каждого возможного значения в списке выбора.

Параметры имеют следующие атрибуты:

|**Название**     |**Описание**                                                                                                                                                                                  |
|--------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Оператор     | Оператор сравнения используется для фильтрации результатов, например, "равно", "меньше чем" и т. п. Список операторов для параметра будет зависеть от типа атрибута, выбранного для фильтра. Например, числовые типы (Decimal) будут иметь такие операторы, как "меньше чем" или "больше чем", в то время как строковые атрибуты (String) будут использовать такие операторы, как "начинается на"или "содержит". Операторы для списка выбора и логического значения всегда "равно".                                                                                                                                               |
| Значение        | Фактическое значение, используемое для данного условия фильтра.                                                                                                                                                 |
| Отображаемое имя | Переопределяет отображаемое имя для этого параметра в поле фильтра. По умолчанию здесь задается такое же значение, что и в атрибуте значения.                                                             |

**Поле поиска**

Набор подстановки добавляет ряд параметров для фильтрации списка сущностей, связанных с соответствующим объектом с выбранным типом сущности списка сущностей. Если пользователь применяет фильтр, в списке сущностей отображаются только те записи, которые точно соответствует по крайней мере одной из связанных записей.

![Набор подстановки](../media/lookup-set.png "Поле поиска")  

В наборе подстановки используются следующие атрибуты:

|**Название**     |**Описание**                                                                                                                                           |
|--------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------|

| Отображаемое имя | Переопределяет подпись для фильтра, когда отображается список сущностей. По умолчанию здесь автоматически задается имя выбранного отношения. |
| Параметры      | Список возможных значений, по которым требуется выполнить фильтрацию. Дополнительные сведения см. ниже.                                                                                 |

**Параметры набора подстановки**

Набор подстановки обычно может иметь любое число параметров, которое ограничено только числом связанных записей выбранного связанного типа.

Параметры имеют следующие атрибуты:

|**Название**     |**Описание**                                                                                                                      |
|--------------|--------------------------------------------------------------------------------------------------------------------------------------|

| Отображаемое имя | Переопределяет отображаемое имя для этого параметра в поле фильтра. По умолчанию здесь задается такое же значение, что и в атрибуте значения. |

**Набор фильтров по интервалу**

Набор фильтров по интервалу добавляет ряд параметров, каждый с одним или несколькими условиями, в область фильтра. Если пользователь применяет фильтр, в списке сущностей отображаются только те записи, которые точно соответствует всем условиям по крайней мере одного из выбранных параметров.

![Параметры фильтра диапазона](../media/set-range-filter.png "Параметры фильтра диапазона")  

В наборе фильтров по интервалу используются следующие атрибуты:

|**Название**     |**Описание**                                                                                                                                        |
|--------------|--------------------------------------------------------------------------------------------------------------------------------------------------------|
| Атрибут    | Имя атрибута с выбранным типом сущности списка сущностей, по которому следует выполнить фильтрацию. Только атрибуты со следующими типами допустимы для текстового фильтра: String, BigInt, Decimal, Double, Integer, Money, DateTime.                       |
| Отображаемое имя | Переопределяет подпись для фильтра, когда отображается список сущностей. По умолчанию здесь автоматически задается имя выбранного атрибута. |
| Параметры      | Список возможных значений, по которым требуется выполнить фильтрацию. Дополнительные сведения см. ниже.                                                                              |

**Параметры набора фильтров по интервалу**

Набор фильтров по интервалу может иметь любое количество параметров. Каждый параметр создает условие фильтрации с одним или двумя дополнительными условиями, оба из которых должны быть выполнены, чтобы условие стало истинным.

Параметры имеют следующие атрибуты:

|**Название**              |**Описание**                                                                                                                                                                                |
|---------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Оператор 1            | Первый оператор сравнения используется для фильтрации результатов, например, "равно" или "меньше чем". Список операторов для параметра будет зависеть от типа атрибута, выбранного для фильтра. Например, числовые типы (Decimal) будут иметь такие операторы, как "меньше чем" или "больше чем", в то время как строковые атрибуты (String) будут использовать такие операторы, как "начинается на"или "содержит". Операторы для списка выбора и логического значения всегда "равно".                                                                                                                                             |
| Значение 1               | Первое значение, используемое для данного условия фильтра.                                                                                                                                                |
| Оператор 2 (необязательно) | Второй оператор сравнения используется для фильтрации результатов, например, "равно" или "меньше чем". Список операторов для параметра будет зависеть от типа атрибута, выбранного для фильтра. Например, числовые типы (Decimal) будут иметь такие операторы, как "меньше чем" или "больше чем", в то время как строковые атрибуты (String) будут использовать такие операторы, как "начинается на"или "содержит". Операторы для списка выбора и логического значения всегда "равно".                                                                                                                                             |
| Значение 2 (необязательно)    | Второе значение, используемое для данного условия фильтра.                                                                                                                                               |
| Отображаемое имя          | Переопределяет отображаемое имя для этого параметра в поле фильтра. По умолчанию оно будет задаваться динамически на основе выбранных операторов и значений.                                         |

**Динамическое поле выбора**

Динамическое поле выбора добавляет ряд параметров для выполнения фильтрации, которые представляют все значения указанного поля списка выбора. Это отличается от выбора списка выбора в наборе фильтров по атрибутам. В наборе фильтров по атрибутам необходимо указать набора параметров, которые будут доступны пользователю для фильтрации по ним; в динамическом поле выбора нужно только указать поле списка выбора, и полный набор параметров будет представлен автоматически. Если нужна большая свобода управления, рекомендуется использовать набор фильтров по атрибутам.

![Параметры динамического списка выбора](../media/set-dynamic-picklist.png "Параметры динамического списка выбора")  

В динамическом поле выбора используются следующие параметры:

|**Название**     |**Описание**                                                                                                                                        |
|--------------|--------------------------------------------------------------------------------------------------------------------------------------------------------|
| Атрибут    | Имя атрибута списка выбора с выбранным типом сущности списка сущностей, по которому следует выполнить фильтрацию.                                                             |
| Отображаемое имя | Переопределяет подпись для фильтра, когда отображается список сущностей. По умолчанию здесь автоматически задается имя выбранного атрибута. |

**Динамическое поле поиска**

Динамическое поле поиска добавляет динамический ряд параметров для фильтрации списка сущностей, связанных с соответствующим объектом с выбранным типом сущности списка сущностей. Если пользователь применяет фильтр, в списке сущностей отображаются только те записи, которые точно соответствует по крайней мере одной из связанных записей.

Это отличается от набора подстановки. В после поиска необходимо вручную задать связанные сущности, по которым требуется выполнить фильтрацию. В динамическом поле поиска необходимо только указать отношение, по которому выполнять фильтрацию, и список параметров создается на основе указанного представления связанных сущностей.

![Параметры динамической подстановки](../media/set-dynamic-lookup.png "Параметры динамической подстановки")  

В динамическом поле поиска используются следующие параметры:

|**Название**                      |**Описание**                                                                                                                                                                      |

Только сущности с отношением "один ко многим" или "многие ко многим" с выбранным типом сущности списка сущностей отображаются как параметры для этого типа фильтра.                                     |

| Столбец меток                  | Поле из представления, предоставляющее значение "Имя" каждой сущности.                                                                                                                    |
| Поиск фильтра по отношению | Указывает связь между сущностью, определенной полем отношения и пользователем, выполнившим вход. Если сущность, указанное в поле отношения, также имеет отношение с контактом, можно ограничить список параметров фильтра до тех, которые связаны с пользователем, выполнившим вход.  |
| Отображаемое имя                  | Переопределяет подпись для фильтра, когда отображается список сущностей. По умолчанию здесь автоматически задается имя выбранного отношения.                            |

**Фильтр FetchXML**

Фильтр диапазона может создать или простой фильтр текстового поля, аналогичный текстовому фильтру, или набор параметров, аналогичный фильтрам других типов. Он позволяет создать вручную практически любой тип фильтра для списка сущностей с помощью языка FetchXML.

## <a name="entity-list-map-view"></a>Представление карты списка сущностей

Со списками сущностей можно включать и настраивать представление карты для данных на основе карт [!INCLUDE[pn-bing](../../../includes/pn-bing.md)] с возможностью поиска мест рядом с адресом. Заполняя записи значениями широты и долготы и указывая необходимые параметры конфигурации, перечисленные в этом разделе, можно отображать записи в виде вешек на карте. Любая запись. у которой нет значений широты или долготы, будет исключена из поиска. При начальной загрузке страницы отображаются все записи в пределах исходного значения поля "Значения расстояния" (в милях или км в зависимости от указанных единиц измерения расстояния) от центральной широты и долготы по умолчанию. Указанное представление игнорируется, если используется представление "Карта", и запрос расстояния применяется к набору данных для возвращения результатов, которые можно показать на карте.
>[!Note] 
>Этот параметр не поддерживается в среде государственного облака в Германии. Раздел "Представление карты" не будет отображаться в этой среде.


## <a name="entity-list-calendar-view"></a>Представление календаря списка сущностей

Используйте представление календаря списка сущностей для отображения списка сущностей в виде календаря, при этом каждая отдельная запись настраивается как отдельное событие.

Для отображения записей с использованием календаря эти записи должны включать по крайней мере поле даты. Чтобы события имели точное время начала и завершения, должны присутствовать соответствующие поля, и т. д. Если эти поля настроены, на портале появится представление календаря списка сущностей.

## <a name="enhanced-view-filter-for-entity-lists"></a>Улучшенный фильтр представления для списков сущностей

Если включен, сущность можно опубликовать в канале OData. Протокол OData представляет собой протокол прикладного уровня для взаимодействуя с данными посредством веб-служб RESTful. Данные из этого канала можно просматривать в веб-браузере, потреблять клиентским веб-приложением или импортировать их в [!INCLUDE[pn-excel-short](../../../includes/pn-excel-short.md)].

## <a name="entity-list-odata-feeds"></a>Каналы OData списка сущностей

Можно использовать разрешения сущности, если требуется защита записей, но если необходимо просто предоставить фильтр в качестве набора параметров фильтра, относящихся к текущему пользователю портала, можно воспользоваться функцией списка сущностей. Эта функция поддерживает фильтрацию текущего пользователя, головной организации пользователя или веб-сайта с любой глубиной. Просто создайте фильтр представления в соответствии с любой отдельной записью контакта, и код заменит его значение на фактическое значение во время выполнения &mdash; не требуется назначать значения полей в разделе "Условия фильтра".

> [!Note]
> Опубликованный канал OData является анонимным и не имеет никаких проверок подлинности; следовательно, важно не включать каналы oData для данных, которые не подходят для анонимного доступа к порталу.
