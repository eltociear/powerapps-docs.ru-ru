# <a name="invite-contacts-to-your-portals"></a>Приглашение контактов на ваши порталы

Используйте функцию приглашения порталов, чтобы пригласить контакты на свой портал с помощью автоматизированных сообщений электронной почты, созданных в системе Common Data Service. Приглашенные вами люди получат сообщение электронной почты, которое вы можете полностью настроить, со ссылкой на ваш портал и кодом приглашения. Данный код можно использовать, чтобы получить специальный доступ, настроенный вами. С помощью этой функции вы можете:

- Отправлять одиночные или групповые приглашения
-   Указать срок действия, если требуется
-   Указать пользователя или контакт портала в качестве приглашающего лица, если требуется
-   Автоматически назначать приглашенные контакты для организации после использования приглашения
-   Автоматически выполнить бизнес-процесс после использования приглашения
-   Автоматически назначать приглашенные контакты веб-ролям после использования приглашения

Использовать приглашение можно с помощью любого из многих вариантов проверки подлинности. Документацию по проверке подлинности на портале см. в разделе [Задание удостоверения аутентификации для портала](set-authentication-identity.md) и выберите модель, применимую к вашим версии и конфигурации портала. При использовании приглашения пользователь принимает все параметры, предоставленные администратором. Действие по использованию приглашения будет создано для приглашения и контакта.

Приглашения отправляются с помощью бизнес-процесса **Отправить приглашение**. По умолчанию в бизнес-процессе создается универсальное сообщение электронной почты, которое отправляется на основной адрес электронной почты приглашенного контакта. Адреса электронной почты в полях CC и BCC игнорируются для обеспечения безопасного подключения. Бизнес-процесс **Отправить приглашение** содержит шаблон электронной почты, который следует отредактировать, чтобы он содержал конкретное сообщение для вашего портала и правильную гиперссылку на **страницу использования приглашения** на портал.

Для редактирования шаблона электронной почты бизнес-процесса **Отправка приглашения** найдите и деактивируйте его. После его деактивации можно изменить шаблон электронной почты, чтобы отправлялось требуемое сообщение, и указать прямую ссылку на **страницу использования приглашения** на вашем портале.

> [!NOTE]
> Приглашение отправляется только по основному адресу электронной почты (emailaddress1) контакта. Приглашение не отправляется на дополнительный (emailaddress2) или альтернативный адрес электронной почты (emailaddress3) в записи контакта.

## <a name="create-invitations-from-portal-management-app"></a>Создание приглашений из приложения управления порталами

1.  Откройте [Приложение управления порталом](configure-portal.md).

2.  Перейдите в раздел **Порталы** > **Контакты**.
    Можно также открывать страницу **Контакты** из панели [Общий доступ](../manage-existing-portals.md#share). 

3.  Выберите контакт или откройте запись контакта, которого требуется пригласить.

4.  На панели команд выберите **Создать приглашение**.

5.  На странице **Приглашение** введите соответствующие значения в поля. Дополнительные сведения: [Атрибуты приглашения](#invitation-attributes)

6.  Нажмите кнопку **Сохранить**.

7.  На панели команд выберите **Поток** > **Отправить приглашение**.

    > [!div class="mx-imgBorder"]
    > ![Бизнес-процесс отправки приглашения](../media/send-invitation-portal-app.png "Бизнес-процесс отправки приглашения")

8.  В окне подтверждения выберите **OK**. Приглашение высылается выбранному контакту.

    > [!div class="mx-imgBorder"]
    > ![Подтверждение для отправки приглашения](../media/confirm-invitation-portal-app.png "Подтверждение для отправки приглашения")

### <a name="send-multiple-invitations"></a>Отправка нескольких приглашений

Можно создавать приглашения для ваших контактов и затем отправить все приглашения сразу.

1.  Создайте приглашения для требуемых контактов, затем перейдите к пункту **Порталы** > **Приглашения**.

2.  Выберите созданные приглашения.

3.  На панели команд выберите **Поток** > **Отправить приглашение**.

    > [!div class="mx-imgBorder"]
    > ![Бизнес-процесс отправки приглашения](../media/send-invitation-portal-app.png "Бизнес-процесс отправки приглашения")

4.  В окне подтверждения выберите **OK**. Приглашения будут отправлены выбранным контактам.

    > [!div class="mx-imgBorder"]
    > ![Подтверждение для отправки нескольких приглашений](../media/confirm-multiple-invites-portal-app.png "Подтверждение для отправки нескольких приглашений")

## <a name="invitation-attributes"></a>Атрибуты приглашения

|  Имя    |    Описание    |
|-------|------------|
|                 Имя                  |                                                                                                      Описательное имя, помогающая распознать приглашение.                                                                                                      |
|                 Тип                  |                                             **Один** или **Группа**. Одиночное приглашение позволяет пригласить только один контакт и использовать только одно приглашение. Группа позволяет пригласить несколько контактов и использовать несколько приглашений.                                              |
|             Владелец/отправитель              | Пользователь, который будет отправителем сообщения электронной почты при отправке приглашения. Это можно переопределить в бизнес-процессе **Отправить приглашение**, если созданное сообщение уже содержит кто-либо в поле "От кого". |
|            Код приглашения            |                                                                 Уникальный код для приглашения, который будет знать только приглашенный. Он создается автоматически при создании нового приглашения.                                                                  |
|              Дата окончания срока действия              |                                                                                     Дата, указывающая, когда приглашение станет недействительным для использования. Необязательно.                                                                                     |
|                Приглашающий                |                                                                                               Можно использовать, когда контакт является отправителем приглашения. Необязательно.                                                                                                |
|          Приглашенный контакт (контакты)           |                                                                                                             Контакты, которых требуется пригласить на портал.                                                                                                              |
|           Назначить организации           |                                                                        Запись организации, которую следует связать в качестве родительского клиента контакта, использующего приглашение, при использовании приглашения. Необязательно.                                                                        |
| Выполнить бизнес-процесс для активирующего контакта |                                                         Бизнес-процесс, который должен быть выполнен при использовании приглашения. Бизнес-процесс будет передан активирующему контакту в качестве основной сущности. Необязательно.                                                          |

Необязательно.                                                                                |
|          Активированный контакт (контакты)          |                                                                                                   Контакты, успешно использовавшие приглашение.                                                                                                   |
|      Максимально допустимое кол-во активаций      |                                                                                   Возможное количество активаций приглашения. Доступно только для приглашений типа "Группа".                                                                                   |
|                                       |                                                                                                                                                                                                                                                                    |
