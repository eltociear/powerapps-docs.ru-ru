# <a name="enable-multiple-language-portal-support"></a>Включение поддержки многоязычных порталов
Бизнес не ограничен одним регионом или языком. Единый портал может отображать содержимое на нескольких языках, чтобы охватывать клиентов по всему миру. Содержимое портала можно перевести на несколько языков, сохраняя единую иерархию содержимого.

![Раскрывающийся список на нескольких языках](../media/multi-language-dropdown.png "Раскрывающийся список на нескольких языках")  

Чтобы включить поддержку нескольких языков для портала, выполните следующие шаги:

1. [Включение языков в среде Common Data Service.](https://technet.microsoft.com/library/dn832148.aspx)  
2. Перейдите в раздел **Порталы** > **Веб-сайт** > **Веб-сайты**.
3. Выберите веб-сайт, для которого требуется добавить поддержку языка.
4. В разделе **Поддерживаемые языки** на вкладке **Общие** выберите **Создать язык веб-сайта**.
5. Заполните форму, включая **Язык портала** (подстановка языков, которые активированы в организации и поддерживаются порталами) и **Состояние публикации**.

   ![Добавление нового языка портала](../media/add-new-portal-language.png "Добавление нового языка портала")

   ![Задание языка по умолчанию для портала](../media/set-default-language-portal.png "Задание языка по умолчанию для портала")

> [!Note]
> При активации новых языков после подготовки портала, можно [импортировать метаданные переводов](../admin/import-metadata-translation.md), чтобы получить метаданные, переведенные на новые добавленные языки.

## <a name="create-content-in-multiple-languages"></a>Создание содержимого на нескольких языках

1. Войти на порталы Dynamics 365.
2. Перейдите к пункту **Порталы** > **Содержимое** > **Веб-страницы**, чтобы просмотреть список содержимого. Для каждой веб-страницы будет присутствовать родительская версия страницы и дочерняя версия страницы для каждого языка, активированного для портала.
3. Для добавления новой локализации страницы перейдите к базовой странице и прокрутите вниз до пункта **Локализованное содержимое**.
4. Выберите кнопку **+** в самой правой части для создания подстановки для локализованной версии.

    ![Добавление нового локализованного содержимого](../media/add-new-localized-content.png "Добавление нового локализованного содержимого")  

> [!Note]
> Поля конфигурации на домашней странице страницы содержимого не наследуются от существующих страниц содержимого. Они используются только при создании новых страниц содержимого. Конфигурации страницы содержимого необходимо обновить отдельно.

Статьи базы знаний будут отображаться только в том случае, если они были переведены на язык, заданный пользователем для отображения портала. Однако форумы и блоги обеспечивают больше возможностей управления их представлением на других языках. Указывать язык для форума или блога необязательно. Если язык не задан, форум или блог будут отображаться на основном языке организации. Если требуется, чтобы форум или блог относился к конкретному языку, необходимо создать его и назначить ему язык.

Наборы веб-ссылок являются навигационными ссылками вверху портала. Перейдя к пункту **Порталы** > **Содержимое** > **Наборы веб-ссылок**, можно управлять тем, как переведено это содержимое. Если язык активен для портала, новый набор ссылок создается для нового активированного языка.

![Активная веб-ссылка для нового языка](../media/active-weblink-new-language.png "Активная веб-ссылка для нового языка")
