# <a name="enable-multiple-language-portal-support"></a>Включить поддержку портала с несколькими языками
Бизнес не ограничивается одним регионом или языком. Один портал может отображать содержимое на нескольких языках для достижения клиентам по всему миру. Содержимое портала можно перевести на несколько языков, сохраняя единую иерархию содержимого.

![Раскрывающийся список с несколькими языками](../media/multi-language-dropdown.png "Раскрывающийся список с несколькими языками")  

Чтобы включить несколько языков для портала, выполните следующие действия.

1. [Включение языков в среде Common Data Service.](https://technet.microsoft.com/library/dn832148.aspx)  
2. Перейдите на **порталы** > **веб-сайт** ** > ** websites.
3. Выберите веб-сайт для добавления языковой поддержки.
4. В разделе **Поддерживаемые языки** на вкладке **Общие** выберите **новый язык веб-сайта**.
5. Заполните форму, включая **язык портала** (Поиск языков, активированных в Организации и поддерживаемых порталами) и **состояние публикации**.

   ![Добавление нового языка портала](../media/add-new-portal-language.png "Добавление нового языка портала")

   ![Задать язык по умолчанию для портала](../media/set-default-language-portal.png "Задать язык по умолчанию для портала")

> [!Note]
> При активации новых языков после подготовки портала можно [Импортировать переводы метаданных](../admin/import-metadata-translation.md) , чтобы получить метаданные для вновь активированных языков.

## <a name="create-content-in-multiple-languages"></a>Создание содержимого на нескольких языках

1. Войдите на порталы Dynamics 365.
2. Перейдите на **порталы** > **содержимое** > **веб-страницы** , чтобы просмотреть список содержимого. Для каждой веб-страницы будет отображаться родительская версия страницы и дочерняя версия страницы для каждого языка, активированного для портала.
3. Чтобы добавить новую локализацию страницы, перейдите на базовую страницу и прокрутите вниз до раздела **локализованное содержимое**.
4. Нажмите кнопку **+** на правой стороне, чтобы создать подстановку для локализованной версии.

    ![Добавить новое локализованное содержимое](../media/add-new-localized-content.png "Добавить новое локализованное содержимое")  

> [!Note]
> Поля конфигурации на домашней странице страницы содержимого не наследуются с существующими страницами содержимого. Они используются только при создании страниц содержимого. Необходимо обновлять конфигурации страниц содержимого по отдельности.

Статьи базы знаний будут отображаться только в том случае, если они были переведены на язык, который пользователь задает для отображения портала. Однако форумы и блоги обеспечивают более полный контроль над тем, как они представлены на других языках. Указывать язык для форума или блога необязательно. Если язык не указан, форум или блог будут отображаться на основном языке Организации. Если вы хотите использовать форум или блог для конкретного языка, необходимо создать его и присвоить ему язык.

Наборы веб-ссылок — это навигационные ссылки в верхней части портала. Перейдя к **порталам** > **содержимое** > **наборы веб-ссылок** , вы можете контролировать, как это содержимое будет переведено. Если для портала активен язык, для вновь активированного языка создаются новые наборы ссылок.

![Активная веб-ссылка для нового языка](../media/active-weblink-new-language.png "Активная веб-ссылка для нового языка")
