# <a name="create-and-manage-page-templates"></a>Создание шаблонов страниц и управление ими

Хотя веб-страницы — это узлы на портале карты портала, которые представляют содержимое, доступное пользователям портала, шаблоны страниц представляют собой фактические ASPX-страницы, которые предоставляют средства для поддержания единообразного внешнего вида и поведения на всем сайте. Шаблоны страниц создаются с помощью страниц ASP.NET, главных страниц, каскадных таблиц стилей (CSS), пользовательских элементов управления и серверных элементов управления.

При создании новой веб-страницы для сайта, независимо от того, как осуществляется публикация на стороне сервера или через интерфейс портала, необходимо выбрать шаблон страницы, который будет представлять содержимое страницы пользователям портала.

Различие между веб-страницами и шаблонами страниц, возможно, лучше понять как различие между точным URL-адресом и фактической страницей. aspx, которая выступает в качестве схемы для отображения содержимого. Каждая веб-страница представляет конкретный URL-адрес на сайте, к которому пользователи могут переходить. Когда пользователь переходит по URL-адресу, отображается содержимое, связанное с этим URL-адресом. Однако веб-страница не содержит сведений о том, как это содержимое отображается.  Это определяется шаблоном страницы, который является фактической ASPX-страницей, которая создает HTML-код, отображаемый пользователем.

При создании новой веб-страницы необходимо выбрать шаблон страницы из списка существующих шаблонов. Каждый из начальных порталов включает несколько шаблонов страниц. При использовании этих порталов в качестве основы для собственного веб-сайта эти шаблоны будут полезными в качестве основных средств демонстрации возможностей портала. Однако разработчику портала потребуется существенно изменить макет этих страниц. В большинстве случаев шаблон страницы "страница" будет использоваться в качестве шаблона для общего назначения: на веб-странице, использующей этот шаблон, будет отображаться содержимое, а также список дочерних страниц, представленных элементами навигации.

## <a name="manage-page-templates"></a>Управление шаблонами страниц

Создание нового шаблона страницы требуется только при создании новой страницы. aspx для просмотра содержимого на веб-сайте — задачи разработчика портала. На самом деле, для пурпосинг простого настройки макета сайта разработчик портала может в значительной мере изменить существующие ASPX-страницы.

1. перо [Управление приложением портала](configure-portal.md).

2. Перейдите на **порталы** > **шаблоны страниц**.

3. Чтобы создать новый шаблон страницы, нажмите кнопку **создать**.

4. Чтобы изменить существующий шаблон страницы, выберите имя шаблона страницы.

5. Введите соответствующие значения в поля.

6. Выберите **Сохранить и закрыть**.

### <a name="page-template-attributes"></a>Атрибуты шаблона страницы

|Имя |Description |

|Перезапись URL-адреса   |Путь к физической странице ASP.NET. aspx (или другому ресурсу, например ASHX), который будет использоваться для отрисовки содержимого.<br> Это поле отображается, только если в списке **тип** выбран параметр **Перезапись URL-адреса** . |
|Веб-шаблон   |Ссылка на веб-шаблон, который будет использоваться для визуализации этого шаблона.<br>Это поле отображается, только если выбран **веб-шаблон** из списка **тип** .  |
|По умолчанию   |Если значение равно "Yes", шаблон будет назначен по умолчанию раскрывающимся списком в средствах редактирования на стороне клиента.   |
|Имя сущности   |Тип сущности страницы, который должен быть отображен этим шаблоном. |Description  |Описание этого шаблона для преимуществ пользователей, редактирующих внешний вид. |
