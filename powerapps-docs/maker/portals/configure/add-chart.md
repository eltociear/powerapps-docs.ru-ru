# <a name="add-a-chart-created-in-a-model-driven-app-to-a-webpage-in-portal"></a>Добавление диаграммы, созданной в приложении, управляемом моделью, на веб-страницу на портале

Диаграмму можно добавить на веб-страницу с помощью тега жидкости с именем [Chart](../liquid/portals-entity-tags.md#chart). Вы можете добавить тег денежной ликвидности в поле **Копировать** на веб-странице или в поле **источник** в [шаблоне](../liquid/store-content-web-templates.md).
 
Например, {% Chart ID: EE3C733D-5693-DE11-97D4-00155DA3B01E%}

![Пример диаграммы](../media/dynamics365-chart-example.png "Пример диаграммы")

Можно также указать идентификатор представления (сохраненный запрос) для фильтрации запроса. Пример.

<!—Leads by Source – Open Leads -->

{% Chart ID: "EE3C733D-5693-DE11-97D4-00155DA3B01E" ViewID: "00000000-0000-0000-00AA-000010001006"%}

## <a name="get-the-id-of-a-chart"></a>Получение идентификатора диаграммы

1.  Перейдите к целевой сущности, например **Sales** > **Leads**.
2.  Разверните область **диаграммы** .
3.  Выберите нужную диаграмму.
4.  Выберите пункт **другие команды**, а затем выберите **Экспорт диаграммы**.

    ![Экспорт диаграммы](../media/export-dynamics365-chart.png "Экспорт диаграммы")

5. Откройте XML-файл экспортированной диаграммы в текстовом редакторе.
6. Скопируйте значение тега \<висуализатионид\>.

    ![Получение чартид для диаграммы](../media/dynamics365-chart-chartid.png "Получение идентификатора диаграммы для диаграммы")

7. Вставьте значение висуализатионид в объявление тега жидкостной диаграммы для параметра идентификатора диаграммы, например:

    {% Chart ID: EE3C733D-5693-DE11-97D4-00155DA3B01E%}.

## <a name="get-the-id-of-a-view"></a>Получение идентификатора представления

Необходимо открыть редактор представлений, чтобы получить идентификатор представления, который будет использоваться с тегом диаграммы ликвидности.
 
1.  Перейдите к целевой сущности, например **Sales** > **Leads**.
2.  Выберите нужное представление из раскрывающегося списка представление.
3.  На панели инструментов выберите **вид** . Откроется окно представление.

    ![Просмотр интересов в редакторе представлений](../media/dynamics365-chart-view.png "Просмотр интересов в редакторе представлений")

4. Скопируйте значение **идентификатора** из URL-адреса окна представления.

    ![Получение идентификатора представления из редактора представлений](../media/dynamics365-chart-viewid.png "Получение идентификатора представления из редактора представлений")

5. Вставьте этот идентификатор в объявление тега жидкостной диаграммы для параметра ViewID, например:

    <!—Leads by Source – Open Leads -->

    {% Chart ID: "EE3C733D-5693-DE11-97D4-00155DA3B01E" ViewID: "00000000-0000-0000-00AA-000010001006"%}

## <a name="entity-permission-requirement"></a>Требование разрешения сущности

Права на чтение утверждены для целевой сущности, запрашиваемой в диаграмме. Чтобы анонимные или прошедшие проверку подлинности пользователи могли просматривать диаграмму, необходимо убедиться, что соответствующие записи [разрешений сущности](assign-entity-permissions.md) созданы и назначены применимым [веб-ролям](create-web-roles.md). 
 
Если разрешение не предоставлено, пользователь увидит сообщение об отказе в доступе.
