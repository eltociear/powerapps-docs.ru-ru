# <a name="define-web-form-properties-for-portals"></a>Определение свойств веб-форм для порталов

Веб-форма содержит связи между страницами и начальное действие для управления инициализацией формы на портале. Связь с веб-страницей обеспечивает динамическое извлечение определения формы для данного узла страницы на веб-сайте.  

Другие параметры записи веб-формы сами управляют настройками верхнего уровня для многоэтапного процесса в целом, например, нужно ли отображать индикатор выполнения.

Чтобы просмотреть существующие веб-формы или создать новые веб-формы, откройте [приложение управления порталом](configure-portal.md) и перейдите на **порталы** > **веб-формы**.

> [!Note]
> **Веб-форма** должна быть связана со страницей для данного веб-сайта, чтобы форма была доступна для просмотра в пределах сайта.  

При создании или редактировании веб-страницы из [приложения управления порталом](configure-portal.md)в поле подстановки, предоставленном в форме « **Новая веб-страница** », можно указать **форму** .

После успешного входа пользователь будет перенаправлен на страницу, содержащую форму.                                                                               |

## <a name="geolocation-configuration-for-web-form"></a>Настройка географического расположения для веб-формы

Управляемую форму можно настроить для отображения элемента управления картой, чтобы отобразить существующее расположение в виде ПИН-кода на карте или предоставить пользователю возможность указать расположение. См. раздел [Добавление географического расположения](add-geolocation.md).

Элемент управления картой формы требует дополнительной настройки, чтобы сообщить ей, что такое идентификаторы полей расположения, чтобы присвоить им значения или извлечь из них значения. Запись шага веб-формы содержит раздел, который определяет эти сопоставления полей, для которых необходимо назначить значения. Имена полей будут зависеть от созданной схемы.

![Данные о географическом расположении в веб-форме](../media/geolocation-managed-form.png "Данные о географическом расположении в веб-форме")

> [!Note]
> Раздел «географическое положение» не отображается в облачной среде Германии независимых. Если пользователь включил географическое расположение с другой формой, он не будет отображаться во время отрисовки на портале.
