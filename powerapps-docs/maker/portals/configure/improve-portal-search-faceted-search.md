# <a name="use-faceted-search-to-improve-portal-search"></a>Использование фасетного поиска для улучшения поиска на портале

Искать содержимое портала можно с помощью фильтров, основанных на характеристиках содержимого. Фильтры, реализованные фасетным поиском, позволяют клиентам находить нужное содержимое быстрее, чем при традиционном поиске.

## <a name="enable-or-disable-faceted-search"></a>Включение и отключение фасетного поиска

Готовый фасетный поиск включен на ваших порталах. Для управления или включения его выполните следующие действия.

1. Откройте [приложение управления порталами](configure-portal.md) и выберите **Порталы** &gt; **Веб-сайт** &gt; **Параметры сайта**.
2. Выберите параметр сайта **Search/FacetedView**. 
3. Измените **Значение** на **True** для включения или на **False** для отключения фасетного поиска.

Отключение одного элемента фасетного представления:

1. Откройте [приложение управление порталами](configure-portal.md) и выберите **Порталы** &gt; **Веб-шаблоны**.
2. Выберите представление, которое требуется отключить (например, Управление знаниями — Статьи с самыми высокими оценками)
3. Выберите **Деактивировать** вверху страницы.

## <a name="group-entities-as-part-of-a-record-type-for-faceted-view"></a>Группирование сущностей как часть типа записи для фасетного представления

Параметр сайта **Search/RecordTypeFacetsEntities** позволяет группировать аналогичные сущности вместе, чтобы у пользователей был логичный способ фильтрации результатов поиска. Например, вместо отдельных параметров для форумов, записей форумов и дискуссий форумов, эти сущности группируются в типе записей "Форумы".

Перейдите в раздел **Порталы** &gt; **Веб-сайты** &gt; **Параметры сайта** и откройте параметр сайта **Search/RecordTypeFacetsEntities**. 

Обратите внимание, что различные сущности предваряются словом **Форумы:**. Это связано с тем, что первое значение является именем, с котором они сгруппированы. Это слово переводится на основе языка, который используется на портале.

## <a name="use-faceted-search-to-improve-knowledge-search-results"></a>Использование фасетного поиска для улучшения результатов поиска в базе знаний

Фасетный поиск позволяет порталам иметь фильтры поиска с самой левой стороны, в которых можно выбирать такие элементы, как форумы, блоги и статьи базы знаний. Дополнительные фильтры добавляются для определенных типов поиска. Например, статьи базы знаний можно фильтровать пои типу записей, дате изменения, оценке и продукту, чтобы клиентам было проще найти требуемое содержимое. С самой правой стороны также имеется раскрывающееся поле, которое сортирует результаты на основе выбора клиента по релевантности или количеству просмотров (относится к статьям базы знаний).
