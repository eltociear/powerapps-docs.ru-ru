# <a name="create-web-roles-for-portals"></a>Создание веб-ролей для порталов

После того как контакт настроен для использования портала, ему необходимо предоставить одну или несколько веб-ролей для выполнения специальных действий или доступа к любому защищенному содержимому на портале. Например, чтобы получить доступ к странице с ограниченным доступом, контакт должен быть назначен роли, которой чтение для этой страницы ограничено. Чтобы опубликовать новое содержимое, контакт должен быть размещен в роли, которой предоставлены разрешения на публикацию содержимого.

Чтобы создать веб-роль, сделайте следующее:

1. Откройте [приложение управления портала](configure-portal.md).

2. Перейдите на **порталы** > **веб-роли**.
    Кроме того, страницу **веб-роли** можно также открыть из панели [общий доступ](../manage-existing-portals.md#share) . 

3. Нажмите кнопку **Создать**.

4. Введите соответствующие значения в поля.

5. Щелкните **Сохранить**.

## <a name="attributes-and-relationships"></a>Атрибуты и связи

В следующей таблице объясняются атрибуты веб-роли, используемые порталами.

| Имя                     | Description                                                                                                                                                                                                                                     |
|--------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Имя                     | Описательное имя веб-роли                                                                                                                                                                                                            |
| Веб                  | Связанный веб-сайт                                                                                                                                                                                                                          |
| Description              | Объяснение назначения веб-роли. Необязательно.                                                                                                                                                                                             |
| Роль пользователей с проверкой подлинности | Логическая. Если задано значение true, это будет веб-роль по умолчанию для пользователей, прошедших проверку подлинности (см. ниже). Для данного веб-сайта должно существовать только одна веб-роль с атрибутом роли "прошедшие проверку подлинности пользователей", равным true. Это будет веб-роль по умолчанию для пользователей, прошедших проверку подлинности, которым не назначена веб-роль. |
| Анонимные пользователи, роль     | Логическая. Если задано значение true, это будет веб-роль по умолчанию для пользователей, не прошедших проверку подлинности (см. ниже). Для данного веб-сайта должно существовать только одна веб-роль с атрибутом роли "анонимные пользователи", для которой задано значение true. Это будет веб-роль по умолчанию для пользователей, не прошедших проверку подлинности. Роль анонимных пользователей будет учитывать только разрешения сущностей.| 
|| 

Теперь, когда веб-роль создана, вы сможете настроить ее в соответствии с вашими потребностями с помощью различных разрешений, правил и связей.

- **Необязательная веб-роль по умолчанию для пользователей, прошедших проверку подлинности**. включив **роль прошедшие проверку пользователи**, она станет веб-ролью по умолчанию для всех пользователей. Эта роль обычно используется для предоставления предварительно определенного доступа пользователям, которые не связаны с другими ролями. Помните, что пользователи могут иметь несколько веб-ролей, но только одна веб-роль пользователей, прошедших проверку подлинности, доступна только для пользователей, прошедших проверку подлинности.
- **Необязательная веб-роль по умолчанию для пользователей, не прошедших проверку подлинности**: **роль анонимных пользователей** предназначена для использования с разрешениями сущностей. Это не повлияет на другие правила или разрешения. Включив роль "анонимные пользователи", она станет веб-ролью по умолчанию для всех пользователей. Для пользователей, не прошедших проверку подлинности, может существовать только одна веб-роль анонимных пользователей.
