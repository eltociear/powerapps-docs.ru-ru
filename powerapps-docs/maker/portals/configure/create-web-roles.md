# <a name="create-web-roles-for-portals"></a>Создание веб-ролей для порталов

После настройки контакта для использования на портале ему необходимо задать одну или несколько веб-ролей, чтобы выполнять любые специальные действия или получить доступ к любому защищенному содержимому на портале. Например, для получения доступа к ограниченной странице контакту необходимо назначить роль, которой разрешено чтение этой страницы. Чтобы опубликовать новое содержимое, контакт необходимо поместить в роль, имеющую разрешения публикации содержимого.

Чтобы создать веб-роль:

1. Откройте [Приложение управления порталом](configure-portal.md).

2. Перейдите в раздел **Порталы** > **Веб-роли**.
    Можно также открывать страницу **Веб-роли** из панели [Общий доступ](../manage-existing-portals.md#share). 

3. Выберите **Создать**.

4. Введите соответствующие значения в поля.

5. Нажмите кнопку **Сохранить**.

## <a name="attributes-and-relationships"></a>Атрибуты и отношения

В таблице ниже разъясняются атрибуты веб-ролей, используемых порталами.

| Полное имя                     | Описание                                                                                                                                                                                                                                     |
|--------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Полное имя                     | Описательное имя веб-роли                                                                                                                                                                                                            |
| Веб-сайт                  | Связанный веб-сайт                                                                                                                                                                                                                          |
| Описание              | Объяснение назначения веб-роли. Необязательно.                                                                                                                                                                                             |
| Роль пользователей, прошедших проверку подлинности | Логическое значение. Если задано значение "истина", это будет веб-ролью по умолчанию для пользователей, прошедших проверку подлинности (см. ниже). Для каждого веб-сайта должна существовать только одна веб-роль, в которой для атрибута "Роль пользователей, прошедших проверку подлинности" задано значение "истина". Это будет веб-роль по умолчанию для пользователей, прошедших проверку подлинности, для которых не назначена веб-роль. |
| Роль анонимного пользователя     | Логическое значение. Если задано значение "истина", это будет веб-ролью по умолчанию для пользователей, не прошедших проверку подлинности (см. ниже). Для каждого веб-сайта должна существовать только одна веб-роль, в которой для атрибута "Роль анонимного пользователя" задано значение "истина". Это будет веб-ролью по умолчанию для пользователей, не прошедших проверку подлинности. Роль анонимного пользователя будет учитывать только разрешения сущности.| 
|| 

Теперь, когда веб-роль создана, вы можете настроить ее в соответствии со своими потребностями с помощью различных разрешений, ролей и связей.

- **Необязательная веб-роль по умолчанию для пользователей, прошедших проверку подлинности**: если включить **Роль пользователей, прошедших проверку подлинности**, она станет веб-ролью по умолчанию для всех пользователей. Эта роль обычно используется для предоставления предопределенного доступа для пользователей, не связанных ни с какими другими ролями. Помните, что пользователям могут быть назначены несколько веб-ролей, но может быть только одна веб-роль "Роль пользователей, прошедших проверку подлинности" для пользователей, прошедших проверку подлинности.
- **Необязательная веб-роль по умолчанию для пользователей, не прошедших проверку подлинности**: **Роль анонимного пользователя** предназначена для использования с разрешениями сущности. Она не учитывает никакие другие правила или разрешения. Если включить "Роль анонимного пользователя", она станет веб-ролью по умолчанию для всех пользователей. Может быть только одна веб-роль анонимных пользователей для пользователей, не прошедших проверку подлинности.
