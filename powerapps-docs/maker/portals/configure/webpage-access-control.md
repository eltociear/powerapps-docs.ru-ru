# <a name="control-webpage-access-for-portals"></a>Управление доступом к веб-страницам для порталов

Правила управления доступом к веб-страницам — это правила, создаваемые для сайта, чтобы управлять действиями публикации, которые веб-роль может выполнять на страницах веб-сайта, и контролировать, какие страницы видимы для веб-ролей. Сущность доступа к веб-странице имеет следующие атрибуты:

|-------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|

|   Веб   |                                                                                                           Веб-сайт, к которому применяется это правило; должен соответствовать веб-сайту страницы, к которой применяется это правило. Веб-страница фильтров.                                                                                                           |
|  Веб-страница   |                            Веб-страница, к которой применяется это правило. Правило повлияет не только на страницу, но на все дочерние страницы страницы, поэтому этот атрибут позволяет выбрать ветвь веб-сайта, к которому будет применяться правило. Если правило применяется к домашней странице, оно будет применено ко всему порталу.                            |

В этом случае потомки потомков не исключаются.</li></ul>По умолчанию выбрано все содержимое. |

После создания нового правила управления доступом свяжите его со страницей. Это приведет к тому, что оно повлияет на страницу, которой назначено правило, и на все дочерние страницы,&mdash;другими словами, всю ветвь веб-сайта.

Существует два типа правил управления доступом: предоставление изменений и ограничение чтения.

## <a name="grant-change"></a>Предоставление изменений

Предоставление изменений позволяет пользователю в веб-роли, связанной с правилом, публиковать изменения содержимого на этой странице и на всех дочерних страницах этой страницы. Предоставление изменений имеет приоритет над ограничением чтения. Например, у вас может быть раздел новостей сайта, который должен быть доступен для редактирования пользователями в веб-роли редактора новостей. Эти пользователи могут не иметь доступа ко всему сайту и, безусловно, не могут изменить весь сайт, но в этой ветви они имеют полный центр публикации содержимого. Вы создадите правило управления доступом к веб-страницам, которое называется предоставление новостей публикация в редакторы новостей.

Далее вы настроили право на предоставление изменений и веб-страницы родительской странице всей ветви новостей вашего сайта. Затем эту веб-роль следует назначить всем контактам, которые вы хотите назначить в качестве редакторов новостей. Обратите внимание, что у одного пользователя может быть несколько веб-ролей.

Правило изменения гранта всегда должно присутствовать на любом портале, который вы хотите включить для редактирования на стороне сервера. Это правило будет применяться к домашней странице сайта, тем самым делая его правилом по умолчанию для всего сайта. Это правило будет связано с веб-ролью, представляющей роль администратора для сайта. Эта роль будет назначена пользователям, которым будут предоставлены права на публикацию содержимого на стороне внешнего сервера.

## <a name="restrict-read"></a>Ограничить чтение
Правило ограничения чтения используется для ограничения просмотра страницы (и ее дочерних страниц) и ее содержимого конкретным пользователям. В то время как предоставление изменений является разрешающим правилом (оно предоставляет возможность делать что-то для пользователей), ограничение чтения является ограничивающим правилом тем, что ограничивает действие ограниченным набором пользователей. Например, у вас может быть раздел сайта, предназначенный для использования только сотрудниками. Вы можете ограничить чтение этой ветви только людьми из веб-роли сотрудника. Вы создадите новое правило с именем ограничить чтение только сотрудниками.

Затем вы задаете право на ограничение чтения и страницы на странице в верхней части ветви, которая должна быть доступна только сотрудникам. Затем вы свяжете это правило с веб-ролью Employee и назначите пользователям эту роль.

> [!Note]
> Если вы примените право **ограничения на чтение** к корневой странице домашней страницы веб-сайта и в качестве **области**выберите **исключить непосредственные дочерние веб-файлы** , то прямые дочерние веб-файлы домашней страницы будут доступны всем пользователям.
