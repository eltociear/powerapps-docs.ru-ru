# <a name="control-webpage-access-for-portals"></a>Управление доступом к веб-страницам для порталов

Правила управления доступом к веб-страницам — это правила, создаваемые для сайта как для управления действиями публикации, которые веб-роль может выполнять на страницах вашего веб-сайта, так и для управления тем, какие страницы видимы для каких веб-ролей. Сущность доступа к веб-страницам имеет следующие атрибуты:


|    Имя (название)     |                                                                                                                                                                  Описание                                                                                                                                                                   |
|-------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
|    Имя (название)     |                                                                                                                                                        Описательное имя для правила.                                                                                                                                                        |
|   Веб-сайт   |                                                                                                           Веб-сайт, к которому применяется данное правило; должен соответствовать веб-сайту страницы, к которой это правило применяется. Фильтрует веб-страницу.                                                                                                           |
|  Веб-страница   |                            Веб-страница, к которой применяется это правило. Правило будет действовать не только на эту страницу, но и на все дочерние страницы этой страницы, поэтому этот атрибут выбирает ветвь веб-сайта, к которой будет применяться правило. Если правило применяется к домашней странице, оно будет применяться ко всему порталу.                            |
|    По правому краю    |                                                                                                                                    [Предоставить изменение](#grant-change) или [Ограничить чтение](#restrict-read) ниже.                                                                                                                                     |
|    Область    | <ul><li><strong>Все содержимое</strong>: все нисходящее содержимое включается в проверку безопасности.</li><li><strong>Исключить непосредственные дочерние веб-файлы</strong>: все дочерние веб-файлы, непосредственно связанные с этой веб-страницей, будут исключены из процедуры проверки безопасности. При этом потомки дочерних элементов не исключаются.</li></ul>По умолчанию выбрано "Все содержимое". |
| Описание |                                                                                                                                                     (Необязательно) Описание правила.                                                                                                                                                      |

После создания нового правила управления доступом свяжите его со страницей. В результате правило будет действовать как на странице, которой назначено правило, так и на всех дочерних страницах &mdash; другими словами, оно будет действовать на всю ветвь веб-сайта.

Существует два типа правил управления доступом: "Предоставить изменение" и "Ограничить чтение".

## <a name="grant-change"></a>Предоставить изменение

"Предоставить изменение" позволяет пользователю веб-роли, связанной с правилом, публиковать изменения содержимого для этой страницы и всех дочерних страниц этой страницы. "Предоставить изменение" имеет приоритет над "Ограничить чтение". Например, у вас может быть раздел сайта "новости"; требуется, чтобы его могли редактировать пользователи с веб-ролью редактора новостей. Эти пользователи могут не иметь доступа ко всему сайту, и они определенно не могут изменять весь сайт, но в рамках этой ветви они имеют полные полномочия публикации содержимого. Вы создадите правило управления доступом к веб-страницам с названием "предоставить право публикации новостей редакторам новостей".

Далее вы зададите для права значение предоставления изменений, а для веб-страницы зададите родительскую странице всей ветви новостей сайта. После этого вы будете назначать эту веб-роль всем контактам, которых требуется назначить редакторами новостей. Обратите внимание, что один пользователь может иметь несколько веб-ролей.

Правило "Предоставить изменение" должно всегда присутствовать на любом портале, для которого нужно разрешить редактирование с клиентской стороны. Это правило будет применяться к домашней странице сайта, и в результате будет правилом по умолчанию для всего сайта. Это правило будет связано с веб-ролью, которая должна представлять административную роль для сайта. Пользователям, которым требуется предоставить права публикации содержимого с клиентской стороны, будет назначаться эта роль.

## <a name="restrict-read"></a>Ограничить чтение
Правило "Ограничить чтение" используется, чтобы страницу (и ее дочерние страницы) и ее содержимого могли просматривать только конкретные пользователи. Если правило "Предоставить изменение" является разрешительным (оно предоставляет своим пользователям возможность выполнять какие-то действия), то правило "Ограничить чтение" является запретительным, так как оно ограничивает действие ограниченным набором пользователей. Например, некоторый раздел сайта может быть предназначен только для сотрудников. Вы можете ограничить чтение этой ветви только пользователями в веб-ролью сотрудника. Вы создадите новое правило ограничения чтения только сотрудниками.

После этого вы зададите для права значение "Ограничить чтение", а для страницы зададите страницу в начале ветви, которая должны быть доступна для чтения только сотрудникам. После этого вы свяжите это правило с веб-ролью сотрудника и назначите пользователей с этой ролью.

> [!Note]
> Если применить параметр **Ограничить чтение** непосредственно к "домашней" странице корня веб-сайта и выбрать **Исключить непосредственные дочерние веб-файлы** в качестве значения **Область**, домашняя страница и ее прямые дочерние веб-файлы будут доступны для всех пользователей.
