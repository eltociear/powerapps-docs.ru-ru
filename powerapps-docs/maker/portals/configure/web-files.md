# <a name="create-and-manage-web-files"></a>Создание веб-файлов и управление ими

Веб-файл представляет загружаемый файл на веб-сайте порталов, используемый для хранения изображений, документов и других типов файлов.

Чтобы сохранить реальное содержимое заданного файла, порталы используют функцию вложений в примечаниях, связанных с записью веб-файла. Файл, вложенный в последнее примечание, связанное с веб-файлом, используется в качестве содержимого файла. Таким образом, размер содержимого веб-файла, который может поддерживать порталы, определяется размером вложения, поддерживаемым установкой Dynamics 365.

## <a name="manage-web-files"></a>Управление веб-файлами

Веб-файлы можно создавать, изменять и удалять на порталах PowerApps.

1. Откройте [приложение управления портала](configure-portal.md).

2. Перейдите на **порталы** > **веб-файлы**.

3. Чтобы создать новый веб-файл, нажмите кнопку **создать**.

4. Чтобы изменить существующий веб-файл, выберите имя веб-файла.

5. Введите соответствующие значения в поля.

6. Выберите **Сохранить и закрыть**.

### <a name="web-file-attributes"></a>Атрибуты веб-файла

В таблице ниже объясняется множество стандартных атрибутов веб-файлов, используемых порталами. Важно отметить, что для подготовки к просмотру большого количества атрибутов содержимого и отображения данных используется шаблон страницы, и, таким образом, разработчик портала.

| Имя                | Description               |
|---------------------|-----------------------|
|Имя |Описательное имя сущности. Это значение будет использоваться в качестве заголовка файла в большинстве шаблонов (например, для заголовков ссылок). Это обязательное поле.   |
|Веб   |Веб-сайт, к которому принадлежит сущность. Это обязательное поле.   |
|Родительская страница   |Родительская веб-страница сущности в иерархии содержимого веб-сайта. <br>Хотя для файла не требуется родительская страница — в некоторых сценариях, например, файл может иметь родительскую запись блога, а в большинстве случаев — предоставление родительской страницы.  |
|Частичный URL-адрес   |Сегмент URL-пути, используемый для создания URL-адреса портала этой страницы. <br>Одностраничная (Домашняя) страница веб-сайта — одна страница, не имеющая связанной родительской страницы, должна иметь частичное значение URL-адреса (/).<br>В качестве сегментов пути URL-адреса используются частичные значения URL-адреса. Таким образом, они не должны содержать недопустимые символы URL-пути, такие как?, #,!,%. Так как URL-адреса порталов Адксстудио создаются путем объединения частичных значений URL-адресов с косыми чертами (/), они также не должны содержать символы косой черты. Рекомендуется ограничить часть значений URL-адресов буквами, цифрами, дефисами и символами подчеркивания. Например: пресс-релеасе. PDF, Site_Header. png.  |
|Состояние публикации   |Текущее состояние рабочего процесса публикации файла, которое может определять, виден ли файл на сайте. Чаще всего эта функция используется для предоставления опубликованного или чернового элемента управления содержимым.<br>Пользователям с разрешениями на управление содержимым может быть предоставлена возможность использовать режим предварительного просмотра, который позволяет этим пользователям видеть неопубликованное содержимое (Предварительная версия).   |
| Отобразить дату        | Этот атрибут представляет собой значение даты и времени, которое может использоваться шаблоном, исключительно для целей вывода. Он не имеет функциональных последствий, но может быть полезен для таких целей, как, например, вручную указать опубликованную дату в документе пресс-релиза.    |
| Дата выпуска        | Управляет датой и временем, после чего файл будет виден на портале. Если текущая дата и время предшествует этой дате, этот файл не будет виден. (Исключением является то, что пользователям с разрешениями на управление содержимым может быть предоставлена возможность использовать режим предварительного просмотра, который позволяет пользователям просматривать (предварительно просматривать) невыпущенное содержимое.) Это полезно для управления выпуском содержимого с учетом времени, например новостей или пресс-релизов. |
| Дата окончания срока действия     | Управляет датой и временем, до которого файл будет виден на портале. Если текущая дата и время находятся после этой даты, этот файл не будет отображаться. (Исключением является то, что пользователям с разрешениями на управление содержимым может быть предоставлена возможность использовать режим предварительного просмотра, который позволяет пользователям просматривать (предварительно просматривать) содержимое с истекшим сроком действия.)                |
| Суммар             | Краткое описание файла. это значение обычно используется для добавления описания файла в элементы навигации портала, которые отображают ссылку на файл.      |
| Скрыто из карты сайта | Определяет, является ли видимым файл частью схемы узла портала. Если этот параметр установлен, файл будет по-прежнему доступен на сайте по его URL-адресу и может быть связан с, но стандартные элементы навигации (меню и т. д.) не будут включать эту страницу.      |
| Порядок вывода       | Целочисленное значение, указывающее порядок размещения файла относительно других сущностей с той же родительской страницей. Это управляет порядком расположения файлов и других сущностей карт веб-узлов, если, например, список ссылок на дочерние сущности данной страницы отображается на портале.      |
| Адрес BLOB-объекта облака  | Текстовое значение в формате `<container>/<filename>`, указывающее, что содержимое этого файла хранится в хранилище BLOB-объектов Azure.        |
| Расположение содержимого | Параметры являются встроенными или вложенными. Если указан параметр Inline, браузер должен попытаться подготовить его к просмотру в окне браузера, а если это не так, пользователю будет предложено загрузить или открыть файл. Если указано вложение, будет немедленно предложено пользователю загрузить или открыть файл, а не пытаться загрузить его в браузере, будь это возможно или нет.                                                                                        |
| Включить отслеживание     | Если параметр включен, каждый запрос для этого веб-файла будет занесен в журнал. Запись журнала веб-файла будет создана с датой & время, IP-адресом и записью контакта, если пользователь прошел проверку подлинности.      |
