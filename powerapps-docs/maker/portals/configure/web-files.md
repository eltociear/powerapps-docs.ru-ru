# <a name="create-and-manage-web-files"></a>Создание веб-файлов и управление ими

Веб-файл представляет собой загружаемый файл на веб-сайте портала, используемый для хранения изображений, документов и любых других типов файлов.

Для хранения фактического содержимого определенного файла порталы используются функцию вложения примечаний, связанных с записью веб-файла. В качестве содержимого файла используется вложенный файл новейшего примечания, связанного с веб-файлом. Поэтому размер содержимого веб-файла, который может поддерживаться порталами, определяется размером вложений примечаний, поддерживаемым вашей установкой Dynamics 365.

## <a name="manage-web-files"></a>Управление веб-файлами

Веб-файлы доступны для создания, изменения и удаления на порталах Power Apps.

1. Откройте [Приложение управления порталом](configure-portal.md).

2. Откройте **Порталы** > **Веб-файлы**.

3. Чтобы создать новый веб-файл, выберите **Создать**.

4. Чтобы изменить существующий веб-файл, выберите имя веб-файла.

5. Введите соответствующие значения в поля.

6. Выберите **Сохранить и закрыть**.

### <a name="web-file-attributes"></a>Атрибуты веб-файла

В таблице ниже разъясняются многие из стандартных атрибутов веб-файлов, используемых порталами. Важно отметить, что способ обработки многих атрибутов, ориентированных на содержимое или отображение, определяется используемым шаблоном страницы и, таким образом, разработчиком портала.

| Имя (название)                | Описание               |
|---------------------|-----------------------|
|Имя (название) |Описательное имя сущности. Это значение будет использоваться как заголовок файла в большинстве шаблонов (например, для заголовков ссылок). Это поле является обязательным.   |
|Веб-сайт   |Веб-сайт, которому принадлежит сущность. Это поле является необходимым.   |
|Родительская страница   |Родительская веб-страница сущности в иерархии содержимого веб-сайта. <br>Хотя не обязательно требуется, чтобы у файла была родительская страница — в некоторых случаях, например, вместо этого файл может иметь родительскую запись блога — в большинстве случаев рекомендуется указывать родительскую страницу.  |
|Частичный URL-адрес   |Сегмент пути URL-адреса, используемый для построения URL-адреса портала для этой страницы. <br>Одна корневая (домашняя) страница вашего веб-сайта — одна страница, у которой нет связанной родительской страницы — должна иметь значение частичного URL-адреса "/".<br>Частичные значения URL-адреса используются как сегменты пути URL-адреса. Поэтому они не должны содержать недопустимых символов пути URL-адреса, таких как ?, #, !, %. Так как URL-адреса Adxstudio Portals создаются путем соединения значений частичных URL-адресов с наклонной чертой ("/"), они также обычно не должны содержать символов наклонной черты. Рекомендуется, чтобы в значениях частичных URL-адресов использовались только буквы, цифры, дефисы и символы подчеркивания. Например: press-release.pdf, Site_Header.png.  |
|Состояние публикации   |Текущее состояние бизнес-процесса публикации файла, которое может определять, виден ли файл на сайте. Чаще всего эта функция используется для управления состоянием содержимого: "опубликовано/черновик".<br>Пользователям с разрешениями управления содержимым может быть предоставлена возможность использовать режим "предварительный просмотр", который позволяет этим пользователям видеть (предварительно просматривать) неопубликованное содержимое.   |
| Показывать дату        | Этот атрибут представляет собой значение типа "дата/время", которое может использоваться шаблоном, исключительно в целях отображения. Оно не несет никакой функциональности, но может быть полезно, например, чтобы вручную указать дату публикации в официальном сообщении для прессы.    |
| Дата выпуска        | Управляет датой и временем, после которых файл будет видимым на портале. Если текущие дата и время имеют значение до этой даты, этот файл не будет виден. (Исключение: пользователям с разрешениями управления содержимым может быть предоставлена возможность использовать режим предварительного просмотра, который позволяет этим пользователям видеть (предварительно просматривать) неопубликованное содержимое.) Это удобно для управления выпуском содержимого, зависящего от времени, например новостей или официальных сообщений для прессы. |
| Дата окончания срока действия     | Управляет датой и временем, до которых файл будет видимым на портале. Если текущие дата и время имеют значение после этой даты, этот файл не будет виден. (Исключение: пользователям с разрешениями управления содержимым может быть предоставлена возможность использовать режим предварительного просмотра, который позволяет этим пользователям видеть (предварительно просматривать) содержимое с истекшим сроком действия.)                |
| Сводка             | Краткое описание файла; это значение обычно будет использоваться для добавления описания файла в навигационные элементы портала, которые отображают ссылку на файл.      |
| Скрыто на карте сайта | Управляет отображением файла на карте сайта портала. Если этот флажок установлен, файл по-прежнему будет доступен на сайте по своему URL-адресу и на него можно задавать ссылки, но стандартные навигационные элементы (меню и т. д.) не будут включать эту станицу.      |
| Порядок отображения       | Целое значение, указывающее порядок размещения файла по отношению к другим объектам с одинаковой родительской страницей. Этот параметр определяет упорядочивание файлов и других объектов карты сайта, когда, например, список ссылок на дочерние объекты определенной страницы отображается на портале.      |
| Облачный адрес большого двоичного объекта  | Текстовое значение в формате `<container>/<filename>`, указывающее, что содержимое для этого файла хранится в хранилище больших двоичных объектов Azure.        |
| Размещение содержимого | Возможные варианты: встроено или вложение. Если указано значение "встроено", браузер должен попытаться отобразить его в окне браузера; если это невозможно, браузер предлагает пользователю загрузить или открыть файл. Если указано значение "вложение", браузер сразу же предлагает пользователю загрузить или открыть файл, не пытаясь загрузить его в браузер (независимо от того, возможно ли это).                                                                                        |
| Включить отслеживание     | Если включено, все запросы для этого веб-файла будут регистрироваться в журнале. Запись журнала веб-файлов будет создана с датой и временем, IP-адресом и записью контакта, если пользователь прошел аутентификацию.      |
