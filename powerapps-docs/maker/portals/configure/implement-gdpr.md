# <a name="implementing-general-data-protection-regulations-in-your-power-apps-portals"></a>Выполнение общего регламента по защите данных на порталах Power Apps

Общие нормы по защите данных (GDPR) — это законодательный акт европейского союза (ЕС), который защищает данные для всех пользователей в рамках ЕС. Благодаря GDPR пользователь может контролировать использование своих личных данных в Common Data Service.

Как администратор, вы можете настроить свой портал, чтобы он соответствовал стандартам GDPR. Например, несовершеннолетние должны получить разрешение родителей для использования портала. Можно также задать условия для пользователей портала. Пользователи должны согласиться с условиями для использования портала.

Нормы GDPR позволяют получать согласие у пользователей портала на использование их личных данных, выявлять несовершеннолетних пользователей и получать согласие родителей для несовершеннолетних.

## <a name="audit-logging"></a>Ведение журнала аудита

Поле **Последний успешный вход** в записи контакта на портале показывает, когда пользователь портала в последний раз выполнил вход. Эта дата выбирается аудитом записи контакта и делает эту информацию доступной в стандартном потоке аудита. Это позволяет администратору просматривать неактивных членов сообщества и удалять их записи.

> [!NOTE]
> Функция отслеживания входа устарела. Рекомендуется использовать технология аналитики, такой как аналитика приложений Azure Application Insights, для захвата этого вида сведений. Список устаревших функций см. [здесь](https://blogs.msdn.microsoft.com/crm/2018/03/20/portal-capabilities-for-dynamics-365-deprecated-features/).

## <a name="identifying-minor-portal-users-and-obtaining-parental-consent"></a>Обнаружение несовершеннолетних пользователей портала и получение согласия родителей

Нормы для определения несовершеннолетних зависят от страны или региона. Поскольку несовершеннолетний может получить доступ к порталу только с согласия родителей, можно настроить портал для определения несовершеннолетних с использованием следующих полей в записи контакта на портале:
- **Является несовершеннолетним**: указывает, что контакт считается несовершеннолетним в своей юрисдикции. По умолчанию выбрано значение **Нет**.
- **Является несовершеннолетним с полученным родительским согласием**: указывает, что контакт считается несовершеннолетним в своей юрисдикции и получил родительское согласие. По умолчанию выбрано значение **Нет**.

    ![Идентификация несовершеннолетних пользователей портала и получение согласия родителей](../media/identify-minor-in-portal.png "Идентификация несовершеннолетних пользователей портала и получение согласия родителей")

Следующие параметры сайта управляют использованием порталов несовершеннолетними пользователями и несовершеннолетними пользователями без разрешения родителей:

| Имя  | Описание   |
|-----------------------|------------------------------------------|
| Authentication/Registration/DenyMinors  | Запрещает использование портала несовершеннолетними. По умолчанию установлено значение false.                          |
| Authentication/Registration/DenyMinorsWithoutParentalConsent | Запрещает использование этого портала несовершеннолетними без согласия родителей. По умолчанию установлено значение false. |
|||

Если пользователь портала определен как несовершеннолетний, а портал настроен для блокировки несовершеннолетних, появится соответствующее сообщение и содержимое не отображается.

![Сообщение о требованиях к возрасту](../media/gdpr-age-req.png "Сообщение о требованиях к возрасту")

Если пользователь портала определен как несовершеннолетний, не имеющий согласия родителей, а портал настроен для блокировки несовершеннолетних без согласия родителей, появится соответствующее сообщение и содержимое не отображается.

![Сообщение о согласии родителей](../media/gdpr-parental-consent.png "Сообщение о согласии родителей")

Следующие фрагменты содержимого контролируют сообщения, которые появляются, когда портал используется несовершеннолетними и несовершеннолетними без родительского разрешения. Можно изменить сообщение в соответствии с вашими требованиями.

| Имя                                                        | Тип | Значение                                                                    |
|-------------------------------------------------------------|------|--------------------------------------------------------------------------|
| Account/Signin/MinorNotAllowedHeading                       | Текст | Требования к возрасту                                                         |
| Account/Signin/MinorNotAllowedCopy                          | HTML | Этот портал не предназначен для несовершеннолетних, и его использование ими не допускается. |
| Account/Signin/MinorWithoutParentalConsentNotAllowedHeading | Текст | Родительское согласие                                                         |
| Account/Signin/MinorWithoutParentalConsentNotAllowedCopy    | HTML | Для использования этого портала несовершеннолетними требуется родительское согласие.              |
|||

Когда кто-то регистрируется с использованием внешнего поставщика и портал настроен на блокировку несовершеннолетних или несовершеннолетних без родительского согласия, запись контакта не создается и аутентификация пользователя не производится.

## <a name="agreeing-to-terms-and-conditions"></a>Согласие на условия

Согласно нормам GDPR пользователи портала должны принять условия, чтобы им был разрешен маркетинг, профилирование или доступ к личной информации. Как администратор, вы можете опубликовать собственные условия для получения согласия пользователей портала, прежде чем они смогут пройти проверку подлинности на сайте.

![Условия портала](../media/gdpr-terms-agreement.png "Условия портала")

Следующие фрагменты содержимого контролируют отображение условий на экране. Можно изменить текст в соответствии с вашими требованиями.

| Имя                                           | Тип | Значение                                 |
|------------------------------------------------|------|---------------------------------------|
| Account/Signin/TermsAndConditionsHeading       | Текст | Условия                  |
| Account/Signin/TermsAndConditionsCopy          | HTML |                                       |
| Account/Signin/TermsAndConditionsAgreementText | Текст | Я принимаю эти условия |
| Account/Signin/TermsAndConditionsButtonText    | Текст | Продолжить                              |
|||

По умолчанию фрагмент содержимого `Account/Signin/TermsAndConditionsCopy` пуст. Администратор портала должен ввести условия в этот фрагмент содержимого.

Следующие параметры сайта контролируют дату публикации условий, а также отображение этих условий на портале:

| Имя  | Описание |
|------------|---------------|
| Authentication/Registration/TermsPublicationDate  | Значение даты и времени в формате GMT для представления даты вступления в силу текущий опубликованных условий. Если согласие на условия включено, пользователям портала, которые не приняли условия после этой даты, будет предложено принять их при следующем входе в систему. Если дата не указана, а согласие на условия включено, условия будут отображаться при каждом входе пользователей портала. <br> **Примечание**. Если требуется, чтобы пользователи портала давали согласие на условия при каждом входе в систему, не указывайте значение для этого параметра сайта.|
| Authentication/Registration/TermsAgreementEnabled | Значение true или false. Если задано значение true, на портале отображаются условия пользования сайтом. Пользователи должны согласиться на условия, чтобы они считались прошедшими проверку подлинности и могли использовать сайт. По умолчанию установлено значение false.        |
|||

Следующее поле добавлено в запись контакта портала для хранения даты и времени, когда пользователь портала согласился с условиями:
- **Дата согласия с условиями портала**. Указывает дату и время, когда пользователь принял условия использования портала.

    ![Дата соглашения условий портала](../media/portal-terms-agreement.png "Дата соглашения условий портала")
