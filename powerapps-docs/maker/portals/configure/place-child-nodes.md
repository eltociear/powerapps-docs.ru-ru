# <a name="place-child-nodes-by-using-shortcuts-for-portals"></a>Размещение дочерних узлов с помощью ярлыков для порталов
Используйте ярлыки, чтобы разместить дочерние узлы на портале карты портала, которые просто указывают на другие узлы, существующие на Sitemap, или на URL-адреса, внешние на портале. Другими словами, веб-страницы, Web-файлы, события и форумы могут рассматриваться как «сплошные» узлы карты узла портала: они добавляются на карту сайта, и при переходе к ним отображается фактическое содержимое этих узлов. С другой стороны, сочетания клавиш могут рассматриваться как "нематериальные" узлы. они также добавляются на карту сайта (в отличие от веб-ссылок, которые не являются), но при переходе к ним отображается содержимое целевого "сплошного" узла, на который указывает ярлык, а это содержимое отображается шаблоном страницы для этого узла.

## <a name="manage-shortcuts"></a>Управление сочетаниями клавиш

Создание, изменение и удаление ярлыков можно выполнять на порталах PowerApps.

1. Откройте [приложение управления портала](configure-portal.md).

2. Перейдите на **порталы** &gt; **ярлыки**. 

3. Чтобы создать ярлык, выберите **создать**. 

4. Чтобы изменить существующий ярлык, выберите существующий ярлык в сетке. 

5. Введите значения для указанных полей. 

6. Выберите **сохранить & закрыть** .

> [!Note]
> Ярлык должен иметь только одно из полей "целевой объект" (внешний URL-адрес, Web Page, опрос, веб-файл, событие, форум), которому назначено значение, а ярлык **будет иметь только** один целевой объект. Например, ярлык не указывает на веб-страницу и на опрос, а также на внешний URL-адрес и файл в Интернете. Если для ярлыка существует более одного целевого атрибута, сочетание клавиш просто примет первое из них, игнорируя все остальные. Порядок приоритета для выбранного целевого объекта отражается в основной форме ярлыка. Таким образом, он сначала проверяет наличие внешнего URL-адреса для ярлыка, и, если это так, целевой объект ярлыка будет внешним URL-адресом, а все остальные целевые атрибуты будут игнорироваться. Если внешний URL-адрес отсутствует, ярлык будет проверять веб-страницу, а затем — опрос, веб-файл, событие и, наконец, форум. 

## <a name="secure-shortcuts"></a>Безопасные сочетания клавиш

Безопасность для ярлыков может быть основана на родительской странице ярлыка или на целевом объекте ярлыка. Это определит, будет ли ярлык отображаться на Sitemap. Естественно, если безопасность основана на родительском объекте, доступ на запись целевого объекта ярлыка будет по-прежнему определять, будет ли функция редактирования внешнего интерфейса работать после использования ярлыка для перехода к цели ярлыка. Таким образом, безопасность ярлыка влияет только на права на перемещение и изменение для редактирования ярлыков на стороне сервера. Используемый метод безопасности зависит от ярлыка. Если оставить значение логического параметра **отключить проверку целевого объекта ярлыка** не выбрано, безопасность ярлыка будет основываться на целевом объекте. в противном случае он будет основан на родительском объекте.
