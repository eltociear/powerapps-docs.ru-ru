# <a name="place-child-nodes-by-using-shortcuts-for-portals"></a>Размещение дочерних узлов с помощью ярлыков для порталов
Используйте ярлыки, чтобы размещать дочерние узлы в любом месте карты сайта портала, которое просто указывает на другие узлы, имеющиеся на карте сайта, или на URL-адреса, внешние для портала. Другими словами, все веб-страницы, веб-файлы, события и форумы можно считать "материальными" узлами на карте портала: они добавляются на карту сайта, и при переходе к ним вы непосредственно видите фактическое содержимое этих узлов. С другой стороны, ярлыки могут рассматриваться как "нематериальные" узлы: они также добавляются на карту сайта (в отличие от веб-ссылок, которые не добавляются), но при переходе на них вы видите содержимое целевого "материального" узла, на который указывает ярлык, и содержимое отображается с помощью шаблона страницы этого узла.

## <a name="manage-shortcuts"></a>Управление ярлыками

Создание, изменение и удаление ярлыков можно выполнять в порталах Power Apps.

1. Откройте [Приложение управления порталом](configure-portal.md).

2. Перейдите к пункту **Порталы** &gt; **Ярлыки**. 

3. Чтобы создать ярлык, выберите **Создать**. 

4. Для редактирования существующего ярлыка выберите существующий ярлык в сетке. 

5. Ввод значений в указанных полях. 

6. Выберите **Сохранить и закрыть**.

> [!Note]
> Ярлык должен иметь только **одно** из "целевых" полей (внешний URL-адрес, веб-страница, опрос, веб-файл, событие, форум) с заданным значением, и ярлык будет иметь только одну цель. Например, ярлык не указывает одновременно на веб-страницу и опрос или на внешний URL-адрес и веб-файл. Если для ярлыка существует несколько атрибутов цели, ярлык принимает только первый атрибут и игнорирует все остальные. Порядок приоритетов, для которых цель будет выбираться, отражен на главной форме ярлыка. Таким образом, сначала проверяется, существует ли внешний URL-адрес для ярлыка; если существует, целью ярлыка будет внешний URL-адрес, а все прочие целевые атрибуты будут игнорироваться. Если нет внешнего URL-адреса, ярлык проверяет веб-страницу, затем опрос, веб-файл, событие и, наконец, форум. 

## <a name="secure-shortcuts"></a>Обеспечение безопасности ярлыков

Безопасность для ярлыков может быть основана на родительской странице ярлыка или на цели ярлыка. Это определяет, будет ли отображаться ярлык на карте сайта. Естественно, если безопасность основана на родительской странице, доступ для записи в целевом объекте ярлыка по-прежнему определяет, будет ли работать редактирование на клиентской стороне, после перехода на целевой объект ярлыка с помощью ярлыка. Поэтому безопасность ярлыка влияет только на навигацию и права редактирования для изменения ярлыков на клиентской стороне. Используемый метод безопасности зависит от конкретного ярлыка. Если оставить логическое значение **Отключить проверку целевого URL-адреса ярлыка** невыбранным, безопасность ярлыка будет основана на конечном объекте; в противном случае она будет основана на родительском объекте.

## <a name="navigate-with-shortcuts"></a>Навигация с помощью ярлыков

После создания сущности ярлыка он появится на вашем веб-сайте. В приведенном выше примере базовый сайт содержит две дополнительных страницы, страница один и страница два. Страница два является дочерней для страницы один, которая является дочерней для домашней страницы. Кроме того, имеется ярлык, который является дочерним для домашней страницы и указывает на страницу два.
