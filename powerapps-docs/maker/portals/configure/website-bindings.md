# <a name="create-and-manage-website-bindings"></a>Создание привязок веб-сайтов и управление ими

В портале метод по умолчанию для выбора веб-сайта заключается в поиске веб-сайта по соответствующему имени веб-сайта, определенному в файле web.config этого конкретного портала. Привязки веб-сайта предоставляют альтернативные методы выбора веб-сайта, используя имя хоста для загрузки портала или путь запроса для выбора соответствующего веб-сайта. Это исключает необходимость изменения отдельных файлов web.config для каждой версии конкретного веб-сайта. Это упрощает развертывание порталов между различными средами развертывания, средами подготовки и производственными средами. Более того, это обеспечивает общую базу данных кода порталов для работы с несколькими веб-сайтами.

## <a name="manage-website-bindings"></a>Управление привязками веб-сайтов

Привязки веб-сайтов можно создавать, изменять и удалять на порталах Power Apps. 

1. Откройте [Приложение управления порталом](configure-portal.md).

2. Перейдите в раздел **Порталы** > **Привязки к веб-сайтам**.

3. Чтобы создать новую привязку веб-сайта, выберите **Создать**.

4. Для редактирования существующей привязки веб-сайта выберите имя привязки веб-сайта.

5. Введите соответствующие значения в поля.

6. Выберите **Сохранить и закрыть**.

### <a name="website-binding-attributes"></a>Атрибуты привязки веб-сайта

Имеются атрибуты, общие для всех привязок.

|Имя|Описание|
|-----|----------|
|Имя| Заголовок для идентификации привязки веб-сайта при просмотре записей.|
|Веб-узел|[Веб-сайт](websites.md), который должен быть выбран порталом.|
|Дата выпуска|Дата, определяющая, когда разрешен выбор веб-сайта.|
|Дата окончания срока действия|Дата, определяющая, когда веб-сайта перестанет выбираться.|
|||

#### <a name="application-settings"></a>Параметры приложения

Укажите значения этих атрибутов для привязки уровня приложения. Эта привязка сопоставляет на основе веб-сайта или приложения IIS.

|Имя|Описание|
|-----|----------|
|Имя сайта|Имя веб-сайта IIS.|
|Виртуальный путь|Имя приложения IIS на этом веб-сайте.|
|||

Для веб-сайтов и облачных сервисов Azure значения имени сайта и виртуального пути определяются узлами <Site> и <VirtualApplication> файла **ServiceDefinition.csdef**.

```
<ServiceDefinition>
  <WebRole>
    <Sites>
      <Site name=Dynamics Portals>
        <VirtualApplication name=customer-portal/>
```
