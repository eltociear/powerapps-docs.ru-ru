# <a name="add-a-redirect-url-to-a-new-url-on-a-portal"></a>Добавление URL-адреса перенаправления к новому URL-адресу на портале

Клиентам часто требуется использовать простой URL-адрес, который перенаправляет на страницу, расположенную на более глубоком уровне сайта, или разрешить использование устаревшего URL-адреса на сайте и автоматическое перенаправление на новый URL-адрес на сайте. Перенаправления страниц позволяют автору содержимого указывать URL-адрес, который по запросу будет перенаправлен на постоянную или временную базу для конкретной веб-страницы или файла. Эти URL-адреса перенаправления управляются отдельно от содержимого страницы, поэтому они не обязательно должны помещаться в веб-иерархии напрямую.

## <a name="create-a-redirect"></a>Создание перенаправления

1. Откройте [приложение управления портала](configure-portal.md).

2. Перейдите на портал > **веб-сайт** **портала** > **перенаправления**.

3. Выберите **создать**.

4. Введите сведения о перенаправлении, как описано ниже.

| Имя        | Description                                                                                                                                  |
|-------------|----------------------------------------------------------------------------------------------------------------------------------------------|
| Имя        | Понятное имя перенаправления. (Может быть любым. Упрощение распознавания.)                                                              |
| Веб     | Веб-сайт, с которым связано перенаправление. (Сайт, с которого перенаправляется пользователь.)                                                         |
| Входящий URL-адрес | Частичный URL-адрес, который необходимо перенаправить. (Страница, с которой перенаправляется пользователь.)                                                            |
| Код состояния | Одно из следующих: **302 (временное перенаправление)** : Возвращает состояние временного перенаправления. Это значение по умолчанию.                                               -   **301 (постоянное перенаправление)** : Возвращает состояние постоянного перенаправления, указывающее, что ресурс перемещен навсегда.                          |
| URL-адрес         | Целевой внешний URL-адрес для перенаправления. (Используйте этот параметр, если пользователь перенаправляется по ссылке, внешней по отношению к веб-сайту, указанному выше.)                            |
| Веб-страница    | Целевая внутренняя веб-страница для перенаправления. (Используйте этот параметр, если пользователь перенаправляется на внутреннюю страницу веб-сайта, указанного выше.) |
| Маркер сайта | Целевой внутренний маркер сайта, на который будет перенаправлен.                                                                                           |

4. После ввода обязательных полей и указания значения хотя бы для одного из полей URL, веб-страницы или маркера сайта выберите **сохранить**.

    ![Перенаправление опроса клиента](../media/redirect-customer-survey.png "Перенаправление опроса клиента")  

## <a name="use-the-redirect"></a>Использование перенаправления

При запросе входящего URL-адреса браузер перенаправляется на URL-адрес целевой веб-страницы для соответствующей записи перенаправления.

Например, для значения входящего URL-адреса CS-опроса с целевой веб-страницей, заданной на странице опроса поддержки клиентов, выполните следующий запрос:

https://customerportal.contoso.com/cs-survey

в браузере будет получен запрос следующего URL-адреса:

https://customerportal.contoso.com/surveys/customer-service-survey/
