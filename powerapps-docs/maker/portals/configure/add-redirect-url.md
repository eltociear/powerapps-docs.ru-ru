# <a name="add-a-redirect-url-to-a-new-url-on-a-portal"></a>Добавление URL-адреса перенаправления в новый URL-адрес на портале

Клиентам часто требуется простой URL-адрес, который перенаправляет на страницу внутри сайта, или требуется разрешить использовать с сайтом старый URL-адрес и автоматически перенаправлять его на новый URL-адрес на сайте. Перенаправление страницы позволяет автору содержимого указать URL-адрес, при запросе которого будет выполняться перенаправление на постоянной или временной основе на определенную веб-страницу или веб-файл. Эти URL-адреса перенаправления управляются отдельно от содержимого страницы, поэтому они не должны напрямую соответствовать веб-иерархии.

## <a name="create-a-redirect"></a>Создание перенаправления

1. Откройте [Приложение управления порталом](configure-portal.md).

2. Перейдите в раздел **Порталы** > **Веб-сайт** > **Перенаправления**.

3. выберите **Создать**.

4. Введите данные перенаправления, как описано ниже.

| Имя        | Описание                                                                                                                                  |
|-------------|----------------------------------------------------------------------------------------------------------------------------------------------|
| Полное имя        | Понятное имя перенаправления. (Может быть любым. Выберите легко понятное имя.)                                                              |
| Веб-сайт     | Веб-сайт, с которым связано перенаправление. (Сайт, с которого перенаправляется пользователь.)                                                         |
| Входящий URL-адрес | Частичный URL-адрес, который перенаправляется. (Страница, с которой перенаправляется пользователь.)                                                            |
| Код состояния | Один из следующих: **302 (временное перенаправление)**: возвращает состояние временного перенаправления. Это значение по умолчанию.                                               -   **301 (постоянное перенаправление)**: возвращает состояние постоянного перенаправления, указывающее, что ресурс перемещен на постоянной основе.                          |
| URL-адрес         | Целевой внешний URL-адрес, на который производится перенаправление. (Используйте, если пользователь перенаправляется по ссылке, внешней относительно указанного выше веб-сайта.)                            |
| Веб-страница    | Целевая внутренняя веб-страница, на которую производится перенаправление. (Используйте, если пользователь перенаправляется на внутреннюю страницу указанного выше веб-сайта.) |
| Маркер сайта | Целевой внутренний маркер сайта, на который производится перенаправление.                                                                                           |

4. После ввода обязательный полей и указания значения по крайней мере одного из полей URL-адреса, веб-страницы или маркера сайта выберите **Сохранить**.

    ![Перенаправление опроса клиентов](../media/redirect-customer-survey.png "Перенаправление опроса клиентов")  

## <a name="use-the-redirect"></a>Использование перенаправления

Если запрошен входящий URL-адрес, браузер перенаправляется на URL-адрес целевой веб-страницы для соответствующей записи перенаправления.

Например, для значения входящего URL-адреса cs-survey, для которого задана целевая веб-страница опроса поддержки клиентов, следующий запрос:

https://customerportal.contoso.com/cs-survey

приводит к тому, что браузер запрашивает следующий URL-адрес:

https://customerportal.contoso.com/surveys/customer-service-survey/
