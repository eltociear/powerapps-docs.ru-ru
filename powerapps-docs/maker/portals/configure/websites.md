# <a name="manage-websites"></a>Управление веб-сайтами

Веб-сайт — это ключевая сущность приложения порталов. Приложение портала выбирает одну запись веб-сайта, и это определяет, какие сущности портала — [веб-страницы](web-page.md), [веб-файлы](web-files.md), [веб-роли](create-web-roles.md), [фрагменты содержимого](customize-content-snippets.md) и т. д. — применимы к данному приложению.

С веб-сайтом, обеспечивающим область приложения, несколько отдельных приложений портала можно подключить к одной организации.

> [!NOTE]
> Определение того, к какой записи веб-сайта привязано определенное приложение портала, обычно производится по имени веб-сайта, указанному в конфигурации развертывания портала.
Однако возможно также управлять этим с помощью префикса пути URL-адреса (см. описания родительского веб-сайта и частичного URL-адреса в разделе [атрибутов веб-сайта](#website-attributes)), или по домену, используя привязки веб-сайтов.

## <a name="manage-websites"></a>Управление веб-сайтами

Веб-сайты создаются при создании нового портала. Однако расширенное управление веб-сайтами можно выполнить из приложения "Управление порталом". 

> [!WARNING]
> При удалении записи веб-сайта также удаляются данные, относящиеся к записи веб-сайта в сущностях метаданных портала, таких как веб-страницы и веб-ссылки. Обычно это нужное поведение, так как оно означает, что весь веб-сайт и все его связанные данные могут быть удалены из организации за одну операцию.

1. Откройте [Приложение управления порталом](configure-portal.md).

2. Перейдите в раздел **Порталы** > **Веб-сайты**.

3. Для редактирования существующего веб-сайта выберите имя веб-сайта.

4. Введите или измените соответствующие значения в поля.

5. Выберите **Сохранить и закрыть**.

### <a name="website-attributes"></a>Атрибуты веб-сайта

|Имя|Описание|
|-----|----------|
|Имя|Описательное имя веб-сайта. Это поле является необходимым.|
|Основное доменное имя|Основное доменное имя портала, к которому эта запись веб-сайта будет добавлена.|
|Родительский веб-сайт|Родительский веб-сайт для веб-сайта. Обычно это поле можно игнорировать, кроме некоторых сложных конфигураций портала, в которых одно приложение портала связано с одним главным веб-сайтом по корневому пути приложения, при этом один или несколько дочерних веб-сайтов доступны по определенным подпутям.|
|Частичный URL-адрес|Сегмент пути корневого URL-адреса для всех URL-адресов, созданных для сущностей портала, связанных с данным веб-сайтом.<br>Например, если приложение портала развертывается, чтобы быть доступным в корне домена example.com, и этот атрибут не имеет значения, запрос к `http://example.com/` отображает домашнюю веб-страницу веб-сайта приложения (так как домашняя страница обязательно должна иметь частичный URL-адрес, заданный как "/").<br>Если этот атрибут настраивается на значение "my-website", домашняя веб-страница будет вместо этого иметь URL-адрес `http://example.com/my-website/`, и все страницы на веб-сайте будут иметь одинаковый префикс пути "/my-website/".<br>В большинстве конфигураций портала это поле можно игнорировать и оставить пустыми.<br>Частичные значения URL-адреса используются как сегменты пути URL-адреса. Поэтому они не должны содержать недопустимых символов пути URL-адреса, таких как "?", "#", "!", "%". Так как URL-адреса портала создаются путем соединения значений частичных URL-адресов с наклонной чертой ("/"), они также обычно не должны содержать символов наклонной черты.<br>Рекомендуется, чтобы в значениях частичных URL-адресов использовались только буквы, цифры, дефисы и символы подчеркивания. Например: "press-releases", "Users_Guide", "product1".|
