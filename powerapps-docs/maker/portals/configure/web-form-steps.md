# <a name="define-web-form-steps-for-portals"></a>Определение шагов веб-формы для порталов

Шаг веб-формы определяет логику потока взаимодействия с пользователем формы, например шаги и условное ветвление. Он также предоставляет сведения об отображении формы и дополнительном поведении.

> [!NOTE]
> Веб-формы хранят журнал шагов, посещенных пользователем, в объекте в сущности сеанса веб-формы. Если шаги веб-формы были изменены, ранее созданные данные журнала могут быть устаревшими. При любом изменении шагов рекомендуется удалять все записи сеанса веб-формы, чтобы избежать несоответствия между последовательностью шагов, зарегистрированных в журнале, и текущей последовательностью.

У каждой веб-формы, представленной на портале, имеется один или несколько шагов. Эти шаги разделяют некоторые общие свойства, кратко рассматриваемые ниже. Каждый шаг содержит указатель (поиск) на следующий шаг, за исключением конечных шагов. Конечные шаги не имеют следующего раза и, таким образом, являются последним шагом веб-формы (из-за условного ветвления может быть несколько конечных шагов)

Первый шаг не может иметь тип "Условие".|
| Следующий шаг                  | Шаг, следующий за текущим шагом. Это поле будет пустым для одиночной формы с одним шагом.                                                                                                            |
| Логическое имя целевой сущности | Логическое имя сущности, связанной с формой.                                                                                                                                               |
| Перемещение к предыдущему элементу разрешено    | Указывает, предоставляется ли пользователю возможность перейти к предыдущему шагу в веб-форме с несколькими шагами. Значение по умолчанию — "истина". Снимите флажок, чтобы пользователь не мог перейти на предыдущий шаг. |
