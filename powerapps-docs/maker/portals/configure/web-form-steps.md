# <a name="define-web-form-steps-for-portals"></a>Определение шагов веб-форм для порталов

Шаг веб-формы предоставляет логику потока пользовательского интерфейса формы, например шаги и условное ветвление. Здесь также приводятся подробные сведения о отрисовке формы и дополнительном поведении.

> [!NOTE]
> Веб-формы сохраняют историю шагов, посещенных пользователем в объекте в сущности сеанса веб-формы. Если шаги веб-формы были изменены, ранее созданные данные журнала стали устаревшими. При изменении шагов рекомендуется удалить все записи сеанса веб-формы, чтобы устранить несовпадение последовательности шагов, записанных в журнал, и текущей последовательности.

Каждая веб-форма, представленная на портале, содержит один или несколько шагов. Эти действия совместно используют некоторые общие свойства, описанные ниже. Каждый шаг содержит указатель (Уточняющий запрос) на следующий шаг, за исключением шагов терминала. Шаги терминала не имеют в следующий раз и, таким образом, последний шаг веб-формы (из-за условного ветвления, может быть несколько шагов терминала).
