# <a name="set-authentication-identity-for-a-portal"></a>Настройка удостоверения для проверки подлинности на портале

Порталы предоставляют функции проверки подлинности, построенные на [ASP.NET Identity](http://www.asp.net/identity) API. ASP.NET Identity, в свою очередь, построена на платформе [OWIN](http://www.asp.net/aspnet/overview/owin-and-katana) Framework, которая также является важным компонентом системы проверки подлинности. Предоставленные службы включают:

- Локальный вход пользователя (имя пользователя или пароль)
- Внешний вход пользователя (социального поставщика) с помощью сторонних поставщиков удостоверений
- Двухфакторная проверка подлинности с помощью электронной почты
- Подтверждение адреса электронной почты
- Восстановление пароля
- Регистрация кода приглашения для регистрации предварительно сформированных записей контактов

> [!NOTE]
> Поле **подтверждения мобильного телефона** в форме контакта на портале субъекта контакта в настоящее время не обслуживает цель. Это поле должно использоваться только при обновлении с порталов Адксстудио.

## <a name="requirements"></a>Требования

Для порталов требуется:

- Базовый портал
- Удостоверение [!INCLUDE[cc-microsoft](../../../includes/cc-microsoft.md)]
- пакеты решений рабочих процессов удостоверений [!INCLUDE[cc-microsoft](../../../includes/cc-microsoft.md)]

## <a name="authentication-overview"></a>Обзор проверки подлинности

Возврат посетителей портала может проходить проверку подлинности с помощью учетных данных локального пользователя или внешних поставщиков удостоверений. Новый посетитель может зарегистрироваться для создания учетной записи пользователя, указав имя пользователя и пароль либо войдя по внешнему поставщику. Посетители, которые отправили код приглашения (администратор портала), имеют возможность активировать код в процессе регистрации для использования новой учетной записи пользователя.

**Параметры связанного сайта:**

- `Authentication/Registration/Enabled`
- `Authentication/Registration/LocalLoginEnabled`
- `Authentication/Registration/ExternalLoginEnabled`
- `Authentication/Registration/OpenRegistrationEnabled`
- `Authentication/Registration/InvitationEnabled`
- `Authentication/Registration/RememberMeEnabled`
- `Authentication/Registration/ResetPasswordEnabled`

## <a name="forgot-password-or-password-reset"></a>Забыт пароль или сброс пароля 

Возвращение посетителей, которые нуждаются в сбросе пароля (и ранее указали адрес электронной почты в своем профиле пользователя), может запросить маркер сброса пароля для отправки в учетную запись электронной почты. Маркер сброса позволяет владельцу выбрать новый пароль. Кроме того, маркер может быть прерван, при этом исходный пароль пользователя остается без изменений.

**Параметры связанного сайта:**

- `Authentication/Registration/ResetPasswordEnabled`
- `Authentication/Registration/ResetPasswordRequiresConfirmedEmail`

**Связанный процесс:** Отправка сброса пароля контакту

1.  При необходимости настройте электронную почту в рабочем процессе.
2.  Отправьте сообщение электронной почты, чтобы вызвать процесс.
3.  Посетителю будет предложено проверить электронную почту.
4.  Посетитель получает сообщение электронной почты о сбросе пароля с инструкциями.
5.  Посетитель возвращается в форму сброса.
6.  Сброс пароля завершен.

## <a name="redeem-an-invitation"></a>Активировать приглашение

При использовании кода приглашения Регистрация посетителя может быть связана с существующей записью контакта, которая была подготовлена специально для этого посетителя. Как правило, коды приглашений отправляются по электронной почте, но вы можете использовать общую форму отправки кода, чтобы отправлять коды, хотя и другие каналы. После отправки допустимого кода приглашения выполняется обычная регистрация пользователя (регистрации) для настройки новой учетной записи пользователя.

**Параметры связанного сайта:**

`Authentication/Registration/InvitationEnabled`

**Связанный процесс:** Отправить приглашение

Сообщение электронной почты, отправленное этим рабочим процессом, необходимо настроить с помощью URL-адреса страницы активации приглашения на портале: http://portal.contoso.com/register/?returnurl=%2f&invitation={Invitation код (приглашение)}

1. Создание приглашения для нового контакта.

    ![Создание приглашения для нового контакта](../media/create-invitation.png "Создание приглашения для нового контакта")  

2. Настройте и сохраните новое приглашение.

    ![Настройка нового приглашения](../media/customize-new-invitation.png "Настройка нового приглашения")  

3. Процесс: отправить приглашение
4. Настройте электронное письмо с приглашением.
5. Сообщение электронной почты с приглашением открывает страницу активации.
6. Пользователь регистрируется с помощью отправленного кода приглашения.

    ![Регистрация с помощью кода приглашения](../media/sign-up-invitation-code.png "Регистрация с помощью кода приглашения")  

## <a name="manage-user-accounts-through-profile-pages"></a>Управление учетными записями пользователей с помощью страниц профиля

Пользователи, прошедшие проверку подлинности, управляют учетными записями пользователей с помощью панели навигации по **безопасности** на странице профиля. Пользователи не ограничиваются единственной локальной учетной записью или единственной внешней учетной записью, выбранной во время регистрации пользователя. Пользователи с внешней учетной записью могут выбрать создание локальной учетной записи, применив имя пользователя и пароль. Пользователи, которые начали работать с локальной учетной записью, могут связать несколько внешних удостоверений с их учетной записью. На странице профиля также отображается напоминание о подтверждении адреса электронной почты пользователем, запрашивая подтверждение отправки по электронной почте в учетную запись электронной почты.

**Параметры связанного сайта:**

- `Authentication/Registration/LocalLoginEnabled`
- `Authentication/Registration/ExternalLoginEnabled`
- `Authentication/Registration/TwoFactorEnabled`

## <a name="set-or-change-a-password"></a>Задание или изменение пароля

Пользователь с существующей локальной учетной записью может применить новый пароль, указав исходный пароль. Пользователь, не имеющий локальной учетной записи, может выбрать имя пользователя и пароль для настройки новой локальной учетной записи. Имя пользователя нельзя изменить после его задания.

**Параметры связанного сайта:**

`Authentication/Registration/LocalLoginEnabled`

**Связанный процесс:**
- Создайте имя пользователя и пароль.
- Изменение существующего пароля.

## <a name="confirm-an-email-address"></a>Подтвердите адрес электронной почты

Изменение адреса электронной почты (или его установка в первый раз) помещает его в неподтвержденное состояние. Пользователь может запросить отправку сообщения электронной почты с подтверждением на новый адрес электронной почты, после чего сообщение будет содержать инструкции для пользователя по завершении процесса подтверждения по электронной почте.

**Связанный процесс:** Отправка подтверждения по электронной почте контакту

1. При необходимости настройте электронную почту в рабочем процессе. 
2. Пользователь отправляет новое сообщение электронной почты, которое находится в неподтвержденном состоянии.
3. Пользователь проверяет электронную почту на предмет подтверждения.
4. Процесс: отправить подтверждение по электронной почте контакту
5. Настройте электронное письмо с подтверждением.
6. Пользователь щелкает ссылку подтверждения, чтобы завершить процесс подтверждения.

> [!NOTE]
> Убедитесь, что для контакта указан основной адрес электронной почты, так как сообщение электронной почты с подтверждением отправляется только основному адресу электронной почты (emailaddress1) контакта. Сообщение электронной почты с подтверждением не отправляется во вторичную почту (emailAddress2) или дополнительный адрес электронной почты (emailAddress3) записи контакта.

## <a name="enable-two-factor-authentication"></a>Включить двухфакторную проверку подлинности

Функция двухфакторной проверки подлинности повышает безопасность учетной записи пользователя за счет подтверждения принадлежности подтвержденного электронного письма в дополнение к стандартной локальной или внешней учетной записи входа. Пользователь, пытающийся войти в учетную запись с включенной двухфакторной проверкой подлинности, отправил код безопасности в подтвержденное электронное письмо, связанное со своей учетной записью. Для завершения процесса входа необходимо отправить код безопасности. Пользователь может запомнить браузер, который успешно прошел проверку, чтобы код безопасности не потребовался для последующих входов из того же браузера. Каждая учетная запись пользователя включает эту функцию по отдельности и требует подтверждения электронной почты.

> [!WARNING]
> Если вы создаете и включаете параметр **проверки подлинности/регистрации/мобилефонинаблед** для включения функций прежних версий, возникнет ошибка. Этот параметр сайта не предоставляется из поля и не поддерживается порталами.

**Параметры связанного сайта:**

- `Authentication/Registration/TwoFactorEnabled`
- `Authentication/Registration/RememberBrowserEnabled`

**Связанный процесс:** Отправка кода с двумя факторами по электронной почте контакту

1. Включите двухфакторную проверку подлинности.
2. Выберите, чтобы получить код безопасности по электронной почте.
3. Дождитесь сообщения электронной почты, содержащего код безопасности.
4. Процесс: отправка электронного кода с двумя факторами для контакта.
5. Двухфакторную проверку подлинности можно отключить.

## <a name="manage-external-accounts"></a>Управление внешними учетными записями

Прошедший проверку подлинности пользователь может подключиться к своей учетной записи пользователя по нескольким внешним удостоверениям, по одной из настроенных поставщиков удостоверений. После подключения удостоверений пользователь может выбрать вход с помощью любого из подключенных удостоверений. Существующие удостоверения также могут быть отключены, пока не останется одно внешнее или локальное удостоверение.

**Параметры связанного сайта:**

- `Authentication/Registration/ExternalLoginEnabled`

**Параметры сайта внешнего поставщика удостоверений**

1.  Выберите поставщика для подключения к учетной записи пользователя.

    ![Управление внешними учетными записями](../media/manage-external-accounts.png "Управление внешними учетными записями")  

2.  Войдите, используя поставщик, который вы хотите подключить.

Поставщик теперь подключен. Поставщик также может быть отключен.
