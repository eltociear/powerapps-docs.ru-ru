# <a name="configure-portal-authentication"></a>Настройка проверки подлинности на портале

В приложении портала пользователь портала, прошедший проверку подлинности, связан с пользовательским контактным или системным пользователем. Конфигурация порталов по умолчанию — на основе контактов. Для входа в систему контакту должны быть настроены соответствующие сведения для проверки подлинности веб-сайта. Пользователи портала должны быть назначены веб-ролям для получения разрешений, относящихся к пользователям, не прошедшим проверку подлинности. Чтобы настроить разрешения для веб-роли, настройте доступ к ней и правила управления доступом к веб-сайту.

Последняя версия средства проверки подлинности портала позволяет пользователям портала выполнять вход с использованием учетной записи локального контакта или внешней учетной записи на основе [ASP.NET Identity](http://www.asp.net/identity).   

- **Локальная**аутентификация. локальная проверка подлинности — это обычная проверка подлинности на основе форм, которая использует записи контактов Common Data Serviceной среды для проверки подлинности. Для создания пользовательских интерфейсов проверки подлинности разработчики могут использовать API идентификации ASP.Net, чтобы создавать пользовательские страницы входа и средства.
- **Внешняя проверка подлинности**: внешняя проверка подлинности предоставляется ASP.NET Identity API. В этом случае учетные данные учетной записи и управление паролями обрабатываются сторонним поставщиком удостоверений. К ним относятся поставщики на основе OpenID Connect, такие как Yahoo!. а также поставщики на основе Google и OAuth 2,0, такие как Twitter, Facebook и [!INCLUDE[cc-microsoft](../../../includes/cc-microsoft.md)]. Пользователи регистрируются на портале, выбрав внешний идентификатор для регистрации на портале. После регистрации внешний идентификатор имеет доступ к тем же функциям, что и локальная учетная запись. 

При регистрации как локальной, так и внешней учетной записи могут использоваться коды приглашений для регистрации, а также рабочий процесс подтверждения электронной почты. Кроме того, Администраторы портала могут включить или отключить любое сочетание параметров проверки подлинности с помощью параметров сайта портала.

> [!NOTE]
> У пользователей портала должен быть уникальный адрес электронной почты. Если две или несколько записей контактов (включая деактивированные записи контактов) имеют одинаковый адрес электронной почты, контакты не смогут пройти проверку подлинности на портале.

## <a name="account-sign-up-registration"></a>Регистрация учетной записи (регистрация)

Администраторы портала имеют несколько параметров для управления поведением регистрации учетной записи. Открытая регистрация — это наименее ограниченная конфигурация регистрации, в которой портал позволяет зарегистрировать учетную запись пользователя, просто указав удостоверение пользователя. Альтернативные конфигурации могут потребовать от пользователей указать код приглашения или действительный адрес электронной почты для регистрации на портале. Независимо от конфигурации регистрации, как локальные, так и внешние учетные записи принимаются одинаково в рабочем процессе регистрации. То есть пользователи могут выбрать тип учетной записи, которую требуется зарегистрировать.

## <a name="open-registration"></a>Открыть регистрацию

Во время регистрации пользователь может создать локальную учетную запись (указав имя пользователя и пароль) или выбрать внешний идентификатор из списка поставщиков удостоверений. Если выбрано внешнее удостоверение, пользователь должен войти с помощью выбранного поставщика удостоверений, чтобы доказать, что он владеет внешней учетной записью. В любом случае пользователь сразу же регистрируется и пройдет проверку подлинности на портале. Во время регистрации в среде Common Data Service создается новая запись контакта.

Если включена открытая регистрация, пользователям не нужно предоставлять код приглашения для завершения процесса регистрации.
