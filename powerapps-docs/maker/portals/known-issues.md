# <a name="known-issues"></a>Известные проблемы


## <a name="general-issues"></a>Общие проблемы

- **Дата изменения** приложения может быть неверной, так как это предварительно подготовленные приложения, которые могли быть подготовлены ранее.

- При создании среды вместе с начальным порталом владелец портала отображается неправильно. Он отображается как System.

- Если вы повторно используете URL-адрес недавно удаленного портала для создания нового портала, он будет иметь некоторую задержку для установки среды выполнения. Это связано с тем, что очистка предыдущих ресурсов все еще выполняется и может занять от 30 минут до 1 часа для установки нового портала в Azure. В течение этого времени портал также будет недоступен для редактирования и может показывать ошибки при запуске в Studio для редактирования.

- При переключении среды в PowerApps порталы в среде могут не отображаться сразу в списке **приложения** или **последние приложения** . Это происходит в частности в средах, которые создаются в разных регионах, чем их клиенты. Чтобы решить эту проблему, используйте обновление браузера или подождите некоторое время, чтобы портал отображался в списке приложений.

- Если вы не хотите открывать панель параметров портала на домашней странице PowerApps при сбросе портала из центра администрирования PowerApps Portal, пользователь увидит сообщение об ошибке "что пошло не так" в области параметров портала, так как портал недоступен.

- В некоторых случаях при создании портала стили на портале не применяются правильно, а веб-сайт отображается без стилей при открытии с помощью **веб-сайта обзора**. Это редко случается, и стили можно восстановить, перезапустив портал в центре администрирования PowerApps Portal.

## <a name="powerapps-portals-studio-issues"></a>Проблемы с порталами PowerApps в студии

- Если на портале имеется иерархия страниц более трех уровней, страницы из четвертого уровня не отображаются в студии PowerApps Studio.

- Выбранный размер шрифта будет отображаться только в том случае, если размер шрифта определен специально для этого текста. Если он является частью стандартных HTML-тегов, таких как p, H1, H2, H3, а так далее, PowerApps Portals Studio не будет отображать размер шрифта.

- На веб-странице с шаблоном **навигации на странице с боковой навигацией** отображаются только ссылки на страницы, которые существовали во время создания веб-страницы. Вы можете обновить ссылки в левой части страницы, изменив шаблон страницы на другой шаблон, а затем вернувшись на **страницу с навигацией на стороне**.

- При удалении веб-страницы холст не отображает обновленное меню до следующего обновления холста.

- Палитра цветов и связанные с ней строки поддерживаются только на английском языке.

- Несколько страниц шаблонов на портале самообслуживания сотрудников не могут визуализировать правильный навигатор.

- Несколько шаблонов порталов PowerApps, особенно привязанных к приложениям, управляемым моделью в Dynamics 365, не имеют элементов меню по умолчанию в соответствии с иерархией страниц. Причина в том, что порядок страниц не доступен на всех или некоторых веб-страницах. Эта проблема возникает на любом портале без порядка показа веб-страниц.

- Сообщение об ошибке отображается, если содержимое страницы (копия страницы) превышает ограничение в 65536 символов, а Страничная сводка превышает установленное по умолчанию ограничение в 2000 символов.

- Меню навигации отображается на холсте с разрешением минимальной ширины 1600px.

- Изображение, отправленное на страницу, станет дочерним по отношению к странице. Если удалить страницу и использовать изображение на другой странице, изображение не будет отображаться на портале PowerApps студии и на веб-сайте.

- В настоящее время визуализация форм в студии PowerApps не поддерживается. При добавлении формы необходимо выбрать **Обзор веб-сайта** , чтобы открыть веб-сайт и проверить форму.

- В тексте или на фоне раздела, если изменить цвет на " **без цвета**", PowerApps Portals Studio не удаляет связанные атрибуты, такие как цвет фона или цвет шрифта, а затем делает значения пустыми.

- В следующих сценариях на портале PowerApps для студии загружается и отображается сообщение об ошибке "разъединение":
    - Если домашняя страница удалена или отключена для портала.
    - Если шаблон страницы, связанный с домашней страницей или любая страница, отключен или удален.

- Студии порталов PowerApps не смогут загрузить исходный код этих фрагментов содержимого, для которых не назначен язык в Common Data Service.

- В некоторых случаях изменения для верхнего и нижнего колонтитулов в режиме WYSIWYG в студии или в редакторе кода не будут немедленно отражены.

- Если веб-странице назначен шаблон поиска в PowerApps Portals Studio, будет показана простая страница с загрузчиком. Чтобы это работало, необходимо создать соответствующий маркер сайта для этой страницы.

- Шаблон Studio по умолчанию также отображается как вариант в шаблоне страницы при создании новой страницы после ее использования в студии PowerApps Studio. Кроме того, этот шаблон вставляется только на английском языке и не поддерживает локализацию на основе Common Data Service или языка портала по умолчанию.

- Список, отображаемый как элемент управления "Календарь" или "Map", нельзя настроить через портал PowerApps в студии.

- Поле частичного URL-адреса в свойствах страницы не принимает специальные символы и разбивает отрисовку на холст на некоторое время. 

- Отправка CSS может завершиться ошибкой в сценариях, где имя файла CSS содержит специальные символы или пробел в имени файла.

- Неопубликованные веб-страницы не отображаются на холсте студии PowerApps.

- При использовании PowerApps Portal Studio, если базовый язык портала отличается от языка браузера, новые веб-страницы, созданные с помощью шаблона по умолчанию, будут иметь фиктивное содержимое, вставленное на языке браузера, а не на языке портала.

- В области **Тема** отображается только CSS, примененные к корневой странице. Хотя при попытке отправить файл CSS с тем же именем, что и любой другой CSS-файл, доступный на портале, платформа PowerApps Portals предлагает заменить этот файл.

- В настоящее время в Safari в операционной системе Mac не поддерживаются порталы PowerApps, и возникают следующие проблемы.
    - Неправильное выделение компонента и наведение указателя мыши на компонент предоставляет неверную целевую индикацию.
    - Два или три раздела столбцов не отображаются должным образом в студии PowerApps Studio, но прекрасно работают на веб-сайте.
