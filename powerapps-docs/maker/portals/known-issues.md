# <a name="known-issues"></a>Известные проблемы

[!include[cc-beta-prerelease-disclaimer](../../includes/cc-beta-prerelease-disclaimer.md)]

## <a name="general-issues"></a>Общие проблемы

- **Дата изменения** для приложения могут стать неправильна так как это предварительно подготовленные приложения, которые могли быть подготовлены ранее.

- Владелец портала не отображается правильно. Он отображается как система.

- Если вы повторно используете URL-адрес недавно удаленного портала для создания нового портала, у него будет некоторая задержка для настройки времени выполнения. Это связано с тем, что очистка предыдущих ресурсов все еще выполняется, и для настройки нового портала на Azure может потребоваться от 30 минут до 1 часа.

- При переключении среды в PowerApps порталы внутри среды могут не отображаться сразу в списке **Приложения** или **Последние приложения**. Это происходит особенно в средах созданные в другой регионе чем их клиенты. Обходной путь — использовать обновление браузера или подождать некоторое время, пока портал не появится в списке приложений.

- При успешном сбросе портала из центра администрирования порталов PowerApps появляется следующая ошибка: "Портал, к которому вы пытаетесь получить доступ, не принадлежит клиенту, в который вы в данный момент вошли. Выйдите из системы и войдите в правильный клиент." Можно посетить домашнюю страницу PowerApps и создать новый портал. 

## <a name="portal-designer-issues"></a>Проблемы конструктора портала

-   При выделении текста в текстовом поле размер шрифта выделенного текста не отображается на панели инструментов форматирования.

- Заполнение столбцов в разделе не такое, как на веб-сайте. Заполнение корректируется в соответствии с содержанием внутри разделов.

- Холст не загружает содержимое в китайском языке.

- Веб-страница, использующая шаблон **Страница с навигацией сбоку** отображает только ссылки страниц, которые существовали при создании веб-страницы. Вы можете обновить ссылки на левой стороне страницы, изменив шаблон страницы на другой шаблон, а затем вернуться к **Страница с навигацией сбоку**.

- При удалении веб-страницы холст не отображает обновленное меню до следующего обновления холста.

- При создании дочерней страницы на странице перезаписи (неподдерживаемые страницы в конструкторе портала) необходимо выбрать шаблон вручную на панели свойств, чтобы отобразить страницу.

- Если имя страницы и большое не отображаемое полностью на панели **Страницы**, кнопка **с многоточием** (...) больше не отображается. Необходимо щелкнуть правой кнопкой мыши по имени страницы, чтобы увидеть параметры страницы.

- Если вы добавили на страницу фрагмент кода деактивированного содержимого, он будет отображаться в конструкторе портала. Но фрагмент кода деактивированного содержимого будет скрыт на фактическом веб-сайте.

- Заполнители некоторых компонентов, такие как веб-ссылки, Power BI, диаграммы и т.д., недоступны для редактирования. Но можно по-прежнему изменение текста на том же. Изменения в заполнителях не будут сохранены.

- Информация и связанные действия на холсте, такие как имя компонента, в конструкторе портала поддерживаются только на английском языке.

- Палитра и связанные с ним строки поддерживаются только на английском языке.

- Некоторые страницы шаблонов на портале самообслуживания сотрудников не могут отображать правильные навигационную цепочку.

- Несколько шаблонов порталов Dynamics 365 не имеют элементы меню в соответствии с их иерархии страниц. Однако если создать новые страницы, элементы меню создаются соответственно.
