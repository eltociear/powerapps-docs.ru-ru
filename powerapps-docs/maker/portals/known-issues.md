# <a name="known-issues"></a>Известные проблемы


## <a name="general-issues"></a>Общие проблемы

- **Дата изменения** для приложения могут стать неправильна так как это предварительно подготовленные приложения, которые могли быть подготовлены ранее.

- При создании среду вместе с начальным порталом владелец портала не отображается правильно. Он отображается как система.

- Если вы повторно используете URL-адрес недавно удаленного портала для создания нового портала, у него будет некоторая задержка для настройки времени выполнения. Это связано с тем, что очистка предыдущих ресурсов все еще выполняется, и для настройки нового портала на Azure может потребоваться от 30 минут до 1 часа. Портал также не будет доступен для редактирования в этот момент и может сообщать об ошибках при запуске в студии для редактирования.

- При переключении среды в PowerApps порталы внутри среды могут не отображаться сразу в списке **Приложения** или **Последние приложения**. Это происходит особенно в средах созданные в другой регионе чем их клиенты. Обходной путь — использовать обновление браузера или подождать некоторое время, пока портал не появится в списке приложений.

- Если вы сохраните панель параметров портала открытой на домашней странице PowerApps при сброске портала из центра администрирования порталов PowerApps, пользователь увидит сообщение об ошибке "Произошла ошибка" в области параметров портала, так как портал недоступен.

- В некоторых случаях при создании портала стили не применяются правильно к порталу, и веб-сайт отображается без стилей при открытии через пункт **Обзор веб-сайта**. Это происходит редко, и стили могут быть восстановлены путем перезагрузка портала из центра администрирования порталов PowerApps.

## <a name="powerapps-portals-studio-issues"></a>Проблемы порталов PowerApps в Studio

- Если портал имеет иерархию страниц больше чем с 3 уровнями, страницы от четвертого уровня и далее не отображаются в студии порталов PowerApps.

- Выбранный размер шрифта текст отображается только в том случае, если имеется размер шрифта, определенный специально для этого текста. Если это часть стандартных тегов HTML, таких как p, H1, H2, H3 и т. д., студия порталов PowerApps не отображает размер шрифта.

- Веб-страница, использующая шаблон **Страница с навигацией сбоку** отображает только ссылки страниц, которые существовали при создании веб-страницы. Вы можете обновить ссылки на левой стороне страницы, изменив шаблон страницы на другой шаблон, а затем вернуться к **Страница с навигацией сбоку**.

- При удалении веб-страницы холст не отображает обновленное меню до следующего обновления холста.

- Палитра и связанные с ним строки поддерживаются только на английском языке.

- Некоторые страницы шаблонов на портале самообслуживания сотрудников не могут отображать правильные навигационную цепочку.

- Несколько шаблонов порталов PowerApps, особенно связанных с управляемыми моделью приложениями в Dynamics 365, не имеют элементов меню по умолчанию в соответствии с их иерархией страниц. Причина в том, что нет доступного порядка страниц во всех или нескольких веб-страницах. Любой портала без порядка отображения веб-страниц будет иметь эту проблему.

- Сообщение об ошибке отображается, когда содержимое страницы (копия страницы) превышает предел в 65536 знаков, а сводка страницы превышает предел по умолчанию 2000 знаков.

- Меню навигации видно только на холсте с разрешением минимальной ширины 1600px.

- Изображение, отправленное на страницу, становится дочерним для страницы. Если пользователь удаляет страницу и использует изображение на другой странице, оно не отображается в студии порталов PowerApps и на веб-сайте.

- Отрисовка форм в настоящее время не поддерживается в студии порталов PowerApps. При добавлении формы необходимо выбрать команду **Обзор веб-сайта**, чтобы открыть веб-сайт и проверить форму.

- Если в фоне текста или раздела изменить цвет на **Нет цвета**, то студия порталов PowerApps не удаляет связанные атрибуты, такие как цвет фона или цвет шрифта, а задает для них значения NULL.

- В следующем сценарии студия порталов PowerApps перестает загружаться и показывает ошибку "К сожалению, произошло разъединение":
    - Если домашняя страница удалена или отключена для портала.
    - Если шаблон страницы, связанный с домашней страницей или любой страницей, отключен или удален.

- Студия порталов PowerApps не сможет загрузить исходный код тех фрагментов содержимого, которые не содержат язык, назначенный в Common Data Service.

- В некоторых случаях изменения для заголовка и нижнего колонтитула, через интерфейс WYSIWYG студии порталов PowerApps или с помощью редактора кода, не будут отражены сразу.

- Если веб-странице назначен шаблон поиска в студии порталов PowerApps, отображается простая страница с загрузчиком. Чтобы это работало, необходимо создать подходящую отметку сайта для этой страницы.

- Шаблон студии по умолчанию также отображается как вариант в шаблоне страницы во время создания новой страницы, если он используется в студии порталов PowerApps. Кроме того, этот шаблон введен только в английском языке, и он не поддерживает локализацию на основе языка по умолчанию в Common Data Service или на портале.

- Список, отображаемый как элемент управления календаря или карты, не может настраиваться с помощью студии порталов PowerApps.

- Поле частичного URL-адреса в свойствах страницы не принимает специальные символы, и оно нарушает отображение на холсте в течение некоторого времени. 

- Отправка CSS может завершиться сбоем в сценариях, где имя файла CSS содержит специальные символы или пробел в имени файла.

- Неопубликованные веб-страницы не отображаются на холсте в студии порталов PowerApps.

- При использовании студии порталов PowerApps если базовый язык портала отличается от языка веб-браузера, новые веб-страницы, созданные с использованием шаблонов студии по умолчанию, будет содержать вставленное фиктивное содержимое на языке браузера, а не на языке портала.

- Только CSS, примененные на корневой странице, отображаются в области **Тема**. Однако если попытаться отправить файл CSS с тем же именем, что и любой другой файл CSS, доступный на портале, студия порталов PowerApps просит вас заменить этот файл.

- Студия порталов PowerApps в настоящее время не поддерживается в Safari в операционной системе Mac и имеет следующие проблемы:
    - Выбор компонента неправильный, и наведение указателя на компонент дает неверную индикацию цели.
    - Разделы с двумя или тремя колонками неправильно отображаются в студии порталов PowerApps, но работают нормально на веб-сайте.
