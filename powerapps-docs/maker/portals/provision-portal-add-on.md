# <a name="provision-a-portal-using-the-older-portal-add-on"></a>Подготавливает портал с помощью надстройки более старой версии портала

Если вы приобрели более раннюю надстройку портала и хотите подготавливать портал с помощью надстройки, перейдите на страницу **центра администрирования Dynamics 365** и подготавливаете портал.

> [!NOTE]
> Чтобы подготавливать портал, необходимо назначить роль системного администратора или настройщика системы Common Data Service среде, выбранной для портала. Кроме того, необходимо иметь [разрешения, необходимые](https://docs.microsoft.com/azure/active-directory/develop/howto-create-service-principal-portal#required-permissions) для создания и регистрации приложения в Azure AD. Если у вас нет необходимых разрешений, обратитесь к глобальному администратору, чтобы обновить разрешения, или попросите глобального администратора предоставить портал.

Чтобы подготавливать портал, выполните следующие действия.

1.  Перейдите на страницу **центра администрирования Dynamics 365** и перейдите на вкладку **приложения** .

2.  Выберите строку приложения с заголовком **надстройка портала**и щелкните **Управление.**

3.  В разделе **Общие параметры** введите **имя** портала. Это **имя** поможет определить портал, и его можно будет изменить позже.

4.  Поле **тип** представляет тип подписки на портал (пробная или рабочая). Это Системное поле, поэтому оно не может быть изменено пользователем. Значение изменяется в зависимости от того, является ли оно пробной подпиской или платной подпиской.

5. При необходимости в раскрывающемся списке **состояние разработки портала** выберите одно из следующих состояний разработки для портала.

    - Эскиз
    - Разработчик
    - Проверьте
    - UAT
    - Живущ

    > [!NOTE]
    > - Для существующих подготовленных порталов этот раскрывающийся список доступен на вкладке " **сведения о портале** ", а состояние "по умолчанию" не выбрано.
    > - Этот раскрывающийся список доступен только для порталов типа Production.
    > - Это поле используется корпорацией Майкрософт для понимания шаблона использования этого портала и не влияет на функциональность. Если вы используете разные имена для жизненного цикла разработки, выберите одну из них, которая ближе всего к цели. Это можно изменить позже, когда портал будет подготовлен.

5.  В поле **URL-адрес портала** введите имя поддомена, которое вы хотите использовать для портала. Можно использовать только буквы, цифры и дефисы (-); другие символы не допускаются.

    > [!NOTE]
    > - Чтобы изменить URL-адрес портала после его подготовки, см. раздел [изменение базового URL-адреса портала](admin/change-base-url.md).
    > - Сведения о связывании портала с пользовательским доменом см. в статье [Связывание портала с пользовательским](admin/add-custom-domain.md)доменом.

6.  В раскрывающемся списке **экземпляр Dynamics 365** выберите экземпляр, который нужно связать с порталом. Для этого необходимо, чтобы в выбранном экземпляре был выбран параметр Системный администратор или роль настройщика системы.

7.  В раскрывающемся списке **выберите язык портала** выберите язык по умолчанию для портала. Доступные языки будут зависеть от языков, установленных в вашем экземпляре. 

    > [!NOTE]
    > Образец данных предоставляется только на одном языке, поэтому выбор языка по умолчанию также решит, как будут переведены образцы данных. Арабский и иврит не поддерживаются и не будут отображаться в списке.

8. В раскрывающемся списке **Выбор администратора портала** выберите пользователя, который будет настраивать, настраивать и обслуживать портал. Все пользователи, имеющие роль системного администратора в Организации, будут отображаться как параметры. 

9. В разделе **аудитория на портале** выберите тип аудитории, которая будет посещать новый портал. Это определит параметры порталов, которые будут предоставлены. Можно выбрать один из следующих:

    -   Partner    
        -   Портал самообслуживания клиентов
        -   Пользовательский портал
        -   Портал партнера
        -   Служба партнерских проектов (необязательно, требуется установка решений)
        -   Служба полей партнеров (необязательно, требуется установка решений)
        -   Портал сообщества

    -   Клиент
        -   Портал самообслуживания клиентов
        -   Пользовательский портал
        -   Портал сообщества

    -   Сотрудник
        -   Портал самообслуживания сотрудников

10. В разделе **Выбор портала для развертывания** выберите тип портала, который вы хотите создать. Отображаемые параметры основаны на выбранной аудитории.

    > [!div class="mx-imgBorder"]
    > ![Настройка параметров для портала](media/configure-settings-portal.png "Настройка параметров для портала")  

11. Выберите **Отправить**и примите условия предоставления услуг.
    > [!div class="mx-imgBorder"]
    > ![Условия предоставления услуг](media/terms-of-service.png "Условия предоставления услуг")  

После принятия условий предоставления услуг портал начнет подготовку. Подготовка обычно занимает 30 минут, но может занять несколько часов в зависимости от загрузки системы. *Имя* портала на вкладке Приложение изменится на *имя*— Настройка при подготовке. Вернитесь на страницу управления порталом, чтобы убедиться, что подготовка выполнена успешно.

После подготовки портала отобразится страница **сведений о портале** с необходимыми сведениями.

> [!div class="mx-imgBorder"]
> ![Сведения о портале](media/portal-details-prov.png "Сведения о портале") 


> [!Note]
> Когда пользователь портала выполняет вход на портал в первый раз с помощью учетных данных Azure AD, страница согласия отображается для всех пользователей независимо от типа пользователя или портала.

## <a name="troubleshoot-provisioning"></a>Устранение неполадок при подготовке

Иногда может возникнуть ошибка в процессе установки пакета или при создании URL-адреса. В таких случаях процессы можно перезапускать.

Если *имя*— Настройка изменений для подготовки *имени*не удалось, необходимо перезапустить процесс подготовки.

1. Перейдите на страницу **приложения** и выберите портал.
2. Щелкните значок синего карандаша под названием **Управление**.
3. Выберите один из следующих вариантов:

   - **Перезапуск подготовки**: перезапускает процесс установки с ранее определенными конфигурациями.

   - **Изменить значения и перезапустить подготовку**. позволяет изменить некоторые значения перед перезапуском процесса подготовки.

Если установка пакета завершилась неудачно, страница "Администратор портала" откроется без каких-либо проблем, но при переходе по фактической ссылке на портал отобразится сообщение с настройкой. Чтобы подтвердить это:

1. Перейдите на страницу управления решениями на странице **центра администрирования Dynamics 365** и убедитесь, что состояние пакета — **Installed (сбой установки**). 

2. Если состояние пакета — **Installed (сбой установки**), попробуйте повторить попытку установки со страницы решения. Кроме того, убедитесь, что системный администратор устанавливает решение с языком по умолчанию в Common Data Service задайте язык, в котором должен быть установлен портал.

> [!NOTE]
> Некоторые решения имеют необходимые условия для установки, поэтому установка завершится ошибкой, если не выполнены необходимые условия. Например, чтобы установить службу "партнер" для портала партнера, необходимо установить уже установленные решения для портала партнера и службы полей. При первой попытке установить службу полей партнера установка завершится сбоем и отобразится сообщение об ошибке.
