# <a name="provision-a-portal-using-the-older-portal-add-on"></a>Подготовка портала с помощью более старой надстройки портала

Если вы приобрели более старую надстройку портала и хотите подготовить портал, используя это расширение, необходимо перейти на страницу **Центр администрирования Dynamics 365** и подготовить портал.

> [!NOTE]
> Для подготовки портала вам должна быть назначена либо роль системного администратора, либо роль настройщика системы среды Common Data Service, выбранной для портала. Также необходимо [иметь необходимыми разрешениями](https://docs.microsoft.com/azure/active-directory/develop/howto-create-service-principal-portal#required-permissions), чтобы создавать и регистрировать приложения в Azure AD. Если у вас нет необходимых разрешений, обратитесь к глобальному администратору, чтобы обновить ваши разрешения, или попросите глобального администратора подготовить портал.

Чтобы подготовить портал:

1.  Перейдите на страницу **Центр администрирования Dynamics 365**, затем выберите вкладку **Приложения**.

2.  Выберите строку приложения с заголовком **Надстройка портала**, затем выберите **Управление**.

3.  В разделе **Общие параметры** введите **Имя** для своего портала. **Имя** поможет идентифицировать портал, и потом его можно изменить.

4.  Поле **Тип** представляет тип подписки на портал (пробная или рабочая версия). Это системное поле, поэтому оно не может быть изменено пользователем. Значение изменяется в зависимости от того, какая это подписка: ознакомительная или платная.

5. При необходимости в раскрывающемся списке **Состояние разработки портала** выберите одно из следующих состояний разработки для вашего портала:

    - Прототип
    - Разработка
    - Тест
    - Приемочное тестирование пользователями
    - Live

    > [!NOTE]
    > - Для существующих подготовленных порталов этот раскрывающийся список доступен на вкладке **Сведения о портале**, и по умолчанию никакое состояние не выбрано.
    > - Этот раскрывающийся список доступен только для порталов производственного типа.
    > - Данное поле используется компанией Microsoft для изучения способов использования этого портала и не оказывает воздействия ни на какие функции. Если используются разные названия для жизненного цикла разработки, выберите то, которое ближе к вашей цели. Это можно изменить позднее после подготовки портала.

5.  В поле **URL-адрес портала** введите требуемое имя поддомена для вашего портала. Можно использовать только алфавитно-цифровые знаки или дефисы (:); другие символы не допускаются.

    > [!NOTE]
    > - Чтобы изменить базовый URL-адрес портала после его подготовки, см. раздел [Изменение базового URL-адреса портала](admin/change-base-url.md).
    > - Чтобы связать портал с настраиваемым доменом, см. раздел [Связывание портала с настраиваемым доменом](admin/add-custom-domain.md).

6.  В раскрывающемся списке **Экземпляр Dynamics 365** выберите экземпляр, с которым требуется связать портал. Для этого требуется роль системного администратора или настройщика системы в выбранном экземпляре, чтобы выбрать его.

7.  В раскрывающемся списке **Выбор языка портала** выберите язык по умолчанию для портала. Доступные языки будут зависеть от языков, установленных в экземпляре. 

    > [!NOTE]
    > Демонстрационные данные предоставляются только на одном языке, поэтому выбор языка по умолчанию также определяет перевод демонстрационных данных. Арабский и иврит не поддерживаются и не отображаются в списке.

8. В раскрывающемся списке **Выбор администратора портала** выберите пользователя, который будет конфигурировать и настраивать портал, а также управлять им. Все пользователи с ролью системного администратора в организации отображаются в качестве возможных вариантов. 

9. В разделе **Аудитория портала** выберите тип аудитории, которая будет пользоваться новым порталом. Это определит, какой перечень параметров портала будет вам предоставлен. Можно выбрать:

    -   Партнер    
        -   Портал самообслуживания клиентов
        -   Портал клиентов
        -   Портал партнеров
        -   Project Service партнеров (необязательно, требует установки решений)
        -   Field Service партнера (необязательно, требует установки решений)
        -   Портал сообщества

    -   клиент,
        -   Портал самообслуживания клиентов
        -   Портал клиентов
        -   Портал сообщества

    -   Сотрудник
        -   Портал самообслуживания сотрудников

10. В разделе **Выбор портала для развертывания** выберите тип портала, который требуется создать. Отображаемые варианты зависят от выбранной аудитории.

    > [!div class="mx-imgBorder"]
    > ![Настройка параметров для портала](media/configure-settings-portal.png "Настройка параметров для портала")  

11. Выберите **Отправить** и примите условия обслуживания.
    > [!div class="mx-imgBorder"]
    > ![Условия использования службы](media/terms-of-service.png "Условия использования службы")  

После того как вы примите условия обслуживания, начнется подготовка портала. Подготовка обычно занимает 30 минут, но может потребоваться несколько часов в зависимости от загрузки системы. Во время подготовки *Имя* портала на вкладке "Приложение" изменяется на *Имя*-настраивается. Вернитесь на страницу управления порталом, чтобы проверить, успешно ли произведена подготовка.

После подготовки портала отображается страница **Сведения о портале** с необходимыми данными.

> [!div class="mx-imgBorder"]
> ![Сведения о портале](media/portal-details-prov.png "Сведения о портале") 


> [!Note]
> Когда пользователь портала входит на портал в первый раз, используя учетные данные Azure AD, отображается страница согласия для всех пользователей независимо от типа пользователя или портала.

## <a name="troubleshoot-provisioning"></a>Устранение неполадок подготовки

Иногда процесс установки пакета или создания URL-адреса может завершиться с ошибкой. В таких случаях процессы можно перезапустить.

Если *Имя*-настройка изменяется на *Имя*-сбой подготовки, необходимо перезапустить процесс подготовки.

1. Перейдите на страницу **Приложения** и выберите портал.
2. Выберите голубую кнопку карандаша с подписью **Управление**.
3. Выберите один из следующих параметров:

   - **Перезапустить подготовку**: перезапуск процесса установки с конфигурацией, которая была определена ранее.

   - **Изменить конфигурацию и перезапустить подготовку**: позволяет изменить некоторые значения перед перезапуском процесса подготовки.

Если установка пакета завершилась с ошибкой, страница администратора портала открывается без каких-либо ошибок, но при переходе по фактическому URL-адресу портала отображается сообщение "Идет настройка". Чтобы проверить это:

1. Перейдите на страницу управление решениями страницы **Центр администрирования Dynamics 365** и проверьте, что статус пакета имеет значение **Сбой установки**. 

2. Если статус пакета **Сбой установки**, повторите попытку установки со страницы решения. Кроме того, убедитесь, что системный администратор устанавливает решение с языком по умолчанию в Common Data Service, установленным на язык, с которым должен быть установлен портал.

> [!NOTE]
> Для установки некоторых решений имеются обязательные условия, поэтому установка завершится ошибкой, если обязательные условия не выполнены. Например, для установки решения Partner Field Service для портала партнеров уже должны быть установлены решения "Портал партнеров" и Field Service. При попытке сначала установить решение Partner Field Service установка завершится сбоем и вы получите сообщение об ошибке.
