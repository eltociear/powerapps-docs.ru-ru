# <a name="clear-the-server-side-cache-for-a-portal"></a>Очистка кэша на стороне сервера для портала

Администратор портала может очистить кэш на стороне сервера для всего портала, чтобы обновленные данные из Common Data Service немедленно отражались на портале. Обновления из Common Data Service передаются на портал в асинхронном режиме, поэтому между обновлением данных времени в Common Data Service и временем отображения обновленных данных на портале может возникнуть задержка. Чтобы устранить эту задержку&mdash;например, когда она мешает работе с конфигурацией портала&mdash;можно принудительно обновить кэш на портале.

> [!NOTE]
> Соглашение об уровне обслуживания для обновления кэша (перемещение данных между Common Data Service и порталом) составляет 15 минут.

Очистка кэша на стороне сервера

1.  Войдите на портал с правами администратора.

2.  Перейдите к URL-адресу следующим образом: `<portal_path>/_services/about`

3.  Выберите **очистить кэш**. 

Кэш на стороне сервера удаляется, а данные перегружаются из Common Data Service. Обратите внимание, что очистка кэша на стороне сервера портала временно приводит к снижению производительности портала, а данные перегружаются из Common Data Service.
