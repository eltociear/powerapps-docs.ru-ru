# <a name="import-metadata-translation"></a>Импорт преобразования метаданных

При подготовке портала в организации устанавливаются решения, связанные с порталом. Во время установки решений переводы метаданных решения (например, имя поля, имя формы и имя представления) устанавливаются только для языков, активированных в настоящее время в Организации. При активации нового языка в будущем метаданные не будут автоматически устанавливаться для вновь активированного языка. Чтобы получить перевод метаданных для только что активированного языка, необходимо импортировать преобразование метаданных из центра администрирования порталов PowerApps.

## <a name="to-import-metadata-translation"></a>Импорт преобразования метаданных

1.  Откройте [центр администрирования порталов PowerApps](admin-overview.md).

2.  Перейдите к **действиям портала** > **получения последних переводов метаданных**. Появится окно подтверждения, предлагающее, следует ли обновить решения портала.

3.  Выберите **Обновить**. В решениях портала будет добавлено Последнее преобразование метаданных.

> [!Note]
> - Если доступна последняя версия пакета портала, она не обновляется. Решения на портале обновляются в той же версии. Чтобы обновить решения портала на основе последних доступных пакетов, необходимо получить доступ к центру администрирования решения.
> - Если пользователь изменил данные в Common Data Service, существующие данные не будут перезаписаны во время обновления.
> - Если устанавливаются решения на портале, обновление решения не может быть запущено.
