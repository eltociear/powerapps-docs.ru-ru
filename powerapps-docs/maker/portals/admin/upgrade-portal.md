# <a name="upgrade-a-portal"></a>Обновление портала

Этот раздел поможет понять процесс выпуска порталов Dynamics 365 для правильной подготовки к любым новым выпускам и для снижения влияния на ваших клиентов. Также обсуждаются различные компоненты, которые являются часть портала.

Портал состоит из следующих компонентов:

|Компонент|Описание|Процесс обновления|
|---------|-----------|--------------|
|Портальные решения|Решения, которые установлены в среде Common Data Service и содержат сущности метаданных для любого портала.|Обновляются самими клиентами на странице центра администрирования Dynamics 365.|
|Узел веб-сайта портала|Узел веб-узла портала — это портальный код, который формирует фактический веб-сайт.|Узел веб-сайта портала обновляется автоматически для всех порталов.<br>**Примечание**. Новая версия узла веб-сайта портала обратно совместима со всеми поддерживаемыми версиями портальных решений. Однако как только версия решения перестанет поддерживаться, ее работа с новой версией узла веб-сайта портала не гарантируется.|
|||

## <a name="impact-of-new-releases-on-a-portal-solution"></a>Влияние новых выпусков на портальное решение

Как часть любого выпуска портала, узлы веб-сайта портала обновляются автоматически на последние версии, в то время как портальные решения обновляются клиентами. Важно понимать влияние обновления каждого компонента на ваш эксплуатируемый портал, чтобы можно было планировать соответственно.

### <a name="portal-website-host-update"></a>Обновление узла веб-сайта портала

Если работает производственная версия портала (это можно посмотреть в центре администрирования порталов Power Apps), не будет никакого времени простоя введенного в эксплуатацию портала во время обновления портала. Однако если используется пробная версия портала, будет простой в течение примерно 6–10 минут, и пользователь не сможет получать доступ к порталу.

### <a name="portal-solution-update"></a>Обновление портального решения

При установке или обновления любых решений в экземпляре будет заметна некоторая нестабильность в экземпляре. Процесс обновления портального решения обновляет решения, доступные в вашем экземпляре, и повлияет на экземпляр, что, в свою очередь, также повлияет на ваш портал. Следовательно, всегда рекомендуется делать обновления решения в вашем экземпляре в темное время суток.

## <a name="get-notified-about-new-releases"></a>Получение уведомлений о новых выпусках

Каждый клиент получает уведомление о новом выпуске портала через центр сообщений Office 365 (в центре администрирования Microsoft 365). Убедитесь, что у вас есть доступ к центру сообщения Office 365 (доступ имеют глобальный администратор и администратор службы) или попросите глобального администратора или администратора службы сообщать вам обо всех новых выпусках портала.

Уведомления отправляются примерно за 2-5 рабочих днях до выпуска. Уведомления отправляются только тем клиентам, для порталов которых запланировано обновление. Каждое уведомление содержит сведения о типе обновления и дате и времени развертывания обновления вместе со ссылкой на заметки о выпуске.

## <a name="enable-a-portal-for-new-release"></a>Включение нового выпуска портала

Для портала разработчика или текстового портала можно включить ранее получение обновления до всех остальных клиентов, чтобы можно было проверить все основные сценарии на вашем тестовом портале до развертывания обновления на эксплуатационном портале. Ранние обновления развертываются по крайней мере за одну неделю до глобально развертывания любого выпуска. Кроме того, уведомления о ранних обновлениях отправляются так, как описано в разделе [Получение уведомлений о новых выпусках](#get-notified-about-new-releases).

Включение портала для раннего обновления:

1.  Открытие [Центра администрирования портала Power Apps](admin-overview.md).

2.  На вкладке **Действия портала** выберите **Включить портал для раннего обновления**.

    > [!div class="mx-imgBorder"]
    > ![Включить портал для раннего обновления](../media/upgrade-portal.png "Включить портал для раннего обновления")

> [!NOTE]
> Включить или отключить раннее обновление портала можно в любое время. Однако снимок делается для всех порталов, отмеченных для раннего доступа, за два дня до любого выпуска, и для всех порталов, отмеченных для раннего доступа после этой даты, получение раннего обновления не гарантируется.

При возникновении какой-либо проблемы на этапе раннего обновления вы можете сообщить об этом через службу поддержки Майкрософт.
