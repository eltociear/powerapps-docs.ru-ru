# <a name="set-up-power-bi-integration"></a>Настройка интеграции Power BI

Power BI — один из лучших инструментов для обеспечения простой и интерактивной визуализации. Чтобы просматривать панели мониторинга и отчеты из Power BI на веб-страницах на портале, вы должны включить визуализацию Power BI из Центра администрирования порталов Power Apps. Можно также внедрять панели мониторинга и отчеты, созданные в новой рабочей области Power BI, включив интеграцию сервиса Power BI Embedded.

> [!NOTE]
> - Необходимо иметь соответствующую лицензию Power BI.
> - Чтобы использовать сервис Power BI Embedded, необходимо иметь соответствующую лицензию Power BI Embedded. Дополнительные сведения см. в разделе [Лицензирование](https://docs.microsoft.com/power-bi/developer/embedded-faq#licensing).

## <a name="enable-power-bi-visualization"></a>Включить визуализацию Power BI

Включение визуализации Power BI позволяет встраивать панели мониторинга и отчеты в веб-страницы на портале с помощью тега powerbi Liquid.

1.  Открытие [Центра администрирования портала Power Apps](admin-overview.md).

2.  Перейдите в раздел **Настройка интеграции Power BI** > **Включить визуализацию Power BI**.

    > [!div class=mx-imgBorder]
    > ![Включить визуализацию Power BI](../media/enable-power-bi-visualization.png "Включить визуализацию Power BI")

3.  Выберите **Включить** в сообщении подтверждения. При включении визуализации Power BI портал перезапустится и станет недоступен на несколько минут. После включения визуализации Power BI отобразится сообщение.

Настройщики могут использовать тег [powerbi](../liquid/portals-entity-tags.md#powerbi) Liquid для включения панелей мониторинга Power BI и отчетов на веб-страницах портала. При встраивании контента Power BI настройщики могут использовать [параметры фильтра](https://docs.microsoft.com/power-bi/service-url-filters) для создания персонализированных представлений. Дополнительные сведения: [тег powerbi Liquid](../liquid/portals-entity-tags.md#powerbi)

### <a name="disable-power-bi-visualization"></a>Отключить визуализацию Power BI

1.  Открытие [Центра администрирования портала Power Apps](admin-overview.md).

2.  Перейдите в раздел **Настройка интеграции Power BI** > **Отключить визуализацию Power BI**.

    > [!div class=mx-imgBorder]
    > ![Отключить визуализацию Power BI](../media/disable-power-bi-visualization.png "Отключить визуализацию Power BI")

3. Выберите **Отключить** в сообщении подтверждения. При отключении визуализации Power BI портал перезапустится и станет недоступен на несколько минут. После отключения визуализации Power BI отобразится сообщение.

## <a name="enable-power-bi-embedded-service"></a>Включить сервис Power BI Embedded

Включение сервиса Power BI Embedded позволяет встраивать панели мониторинга и отчеты, созданные в новом рабочем пространстве Power BI. Панели мониторинга и отчеты внедряются на веб-страницы на портале с помощью Liquid-тега powerbi.

**Необходимые условия**: Прежде чем включать сервис Power BI Embedded, убедитесь, что вы создали ваши панели мониторинга и отчеты в новой рабочей области в Power BI. После создания рабочей области предоставьте административный доступ глобальному администратору, чтобы рабочие области отображались в центре администрирования порталов Power Apps. Дополнительные сведения о создании новых рабочих областей и добавлении доступа к ним, см. в разделе [Создание новых рабочих областей в Power BI](https://docs.microsoft.com/power-bi/service-create-the-new-workspaces).

**Ограничения сервиса Power BI Embedded**: Дополнительные сведения об ограничениях см. [в разделе Соображения и ограничения](https://docs.microsoft.com/power-bi/developer/embed-service-principal#considerations-and-limitations).

> [!NOTE]
> Убедитесь, что визуализация Power BI включена, чтобы Liquid-тег powerbi работал.

1. Открытие [Центра администрирования портала Power Apps](admin-overview.md).

2. Перейдите в раздел **Настройка интеграции Power BI** > **Включить сервис Power BI Embedded**.

    > [!div class=mx-imgBorder]
    > ![Включить сервис Power BI Embedded](../media/enable-powerbi-embedded-button.png "Включить сервис Power BI Embedded")

3. В окне **Включить интеграцию сервиса Power BI Embedded** выберите и переместите рабочие области Power BI, из которых панели мониторинга и отчеты необходимо отображать в вашем портале, в списке **Выбранные рабочие области**.

    > [!div class=mx-imgBorder]
    > ![Выбор рабочих областей Power BI](../media/enable-powerbi-embedded-window.png "Выбор рабочих областей Power BI")
    
    > [!NOTE]
    > После добавления рабочих областей в список **Выбранные рабочие области** базы данных и отчеты отображаются через несколько минут.
    

4. Щелкните **Включить**. Во время включения сервиса Power BI Embedded портал перезапускается и недоступен в течение нескольких минут. После включения сервиса Power BI Embedded отобразится сообщение.

Теперь необходимо создать группу безопасности и добавить ее в организацию Power BI. Дополнительные сведения см. в разделе [Создание группы безопасности и добавление в организацию Power BI](#create-security-group-and-add-to-power-bi-account).

### <a name="create-security-group-and-add-to-power-bi-account"></a>Создание группы безопасности и добавление ее в организацию Power BI

После включения интеграции сервиса Power BI Embedded необходимо создать группу безопасности в Azure Active Directoryy, добавить участников в нее, затем добавить эту группу безопасности в Power BI через портал администрирования Power BI. Это позволяет панелям мониторинга и отчетам, созданным в новых рабочих областях Power BI, отображаться на портале.

> [!NOTE]
> Необходимо выполнить вход с той же учетная записью глобального администратора, который вы использовали для включения сервиса Power BI Embedded.

**Шаг 1. Создание группы безопасности**

1. Войдите в [портал Azure](https://portal.azure.com) с использованием учетной записи глобального администратора для этого каталога.

2. Выберите **Azure Active Directory**, **Группы** и выберите **Создать группу**.

3. На странице **Группа** введите следующие данные:

    - **Тип группы**: безопасность.

    - **Название группы**: Портал сервиса Power BI Embedded.

    - **Описание группы**: эта группа безопасности используется для портала и интеграции сервиса Power BI Embedded.

    - **Тип членства**: Назначенный.

      > [!div class=mx-imgBorder]
      > ![Создание группы безопасности для сервиса Power BI Embedded](../media/powerbi-embed-security-group.png "Создание группы безопасности для сервиса Power BI Embedded")

4. Выберите **Создать**.

**Шаг 2. Добавление участников группы**

**Необходимое условие**. Перед добавлением участника в группу безопасности необходимо иметь идентификатор приложения портала. Идентификатор приложения портала доступен на вкладке **Сведения о портале** в [Центре администрирования порталов Power Apps](admin-overview.md).

1. Войдите в [портал Azure](https://portal.azure.com) с использованием учетной записи глобального администратора для этого каталога.

2. Выберите **Azure Active Directory**, затем выберите **Группы**.

3. Со страницы **Группы - Все группы** найдите группу **Сервис Power BI Embedded портала** и выберите ее.

    > [!div class=mx-imgBorder]
    > ![Поиск и выбор группы безопасности для службы Power BI Embedded](../media/search-security-group.png "Поиск и выбор группы безопасности для сервиса Power BI Embedded")

4. На странице **Обзор сервиса Power BI Embedded портала** выберите **Участники** в области **Управление**.

5. Выберите **Добавить участников** и введите идентификатор приложения портала в текстовое поле.

6. Выберите участника в результатах поиска, затем выберите **Выбрать**.

    > [!div class=mx-imgBorder]
    > ![Добавление участника в группу безопасности для службы Power BI Embedded](../media/add-member-powerbi-embed.png "Добавление участника в группу безопасности для сервиса Power BI Embedded")

**Шаг 3. Power BI настройка**

1. Войдите в [Power BI](https://powerbi.microsoft.com) с использованием учетной записи глобального администратора для этого каталога.

2. Выберите **Параметры** в правом верхнем углу сервиса Power BI, и выберите **Портал администрирования**.

    > [!div class=mx-imgBorder]
    > ![Выбор портала администрирования в службе Power BI](../media/select-admin-portal.png "Выбор портала администрирования в службе Power BI")

3. Выберите **Параметры клиента**.

4. В разделе **Параметры разработчика** выберите пункт **Разрешение субъектам-сервисам использовать API-интерфейсы Power BI**.

5. В поле **Отдельные группы безопасности** найдите группу **Сервис Power BI Embedded** портала и выберите ее.

    > [!div class=mx-imgBorder]
    > ![Добавление группы безопасности в портал администрирования Power BI](../media/add-sg-powerbi.png "Добавление группы безопасности в портал администрирования Power BI")

6. Выберите **Применить**.

Настройщики могут теперь использовать Liquid-тег [powerbi](../liquid/portals-entity-tags.md#powerbi) для внедрения панелей мониторинга Power BI и отчетов из новых рабочих областей Power BI на веб-страницы портала. Чтобы использовать сервис Power BI Embedded, тип проверки подлинности должен быть указан как **powerbiembedded**. При встраивании контента Power BI настройщики могут использовать [параметры фильтра](https://docs.microsoft.com/power-bi/service-url-filters) для создания персонализированных представлений. Дополнительные сведения: [Liquid-тег powerbi](../liquid/portals-entity-tags.md#powerbi).


### <a name="manage-the-power-bi-embedded-service"></a>Управление сервисом Power BI Embedded

1. Открытие [Центра администрирования портала Power Apps](admin-overview.md).

2. Перейдите в раздел **Настройка интеграции Power BI** > **Управление сервисом Power BI Embedded**.

    > [!div class=mx-imgBorder]
    > ![Управление сервисом Power BI Embedded](../media/manage-powerbi-embedded-button.png "Управление службой Power BI Embedded")

3. В окне **Управление интеграцией сервиса Power BI Embedded** удалите и переместите рабочие области Power BI, из которых панели мониторинга и отчеты необходимо отображать в вашем портале, в списке **Выбранные рабочие области**.

    > [!div class=mx-imgBorder]
    > ![Выбор рабочих областей Power BI](../media/manage-powerbi-embedded-window.png "Выбор рабочих областей Power BI")
    
    > [!NOTE]
    > После удаления рабочих областей из списка **Выбранные рабочие области** для отображения изменений может потребоваться до 1 часа. До этого базы данных и отчеты отображаются на портале без каких-либо проблем.

4. Нажмите кнопку **Сохранить**.

### <a name="disable-the-power-bi-embedded-service"></a>Отключение сервиса Power BI Embedded

1.  Открытие [Центра администрирования портала Power Apps](admin-overview.md).

2.  Перейдите в раздел **Настройка интеграции Power BI** > **Управление сервисом Power BI Embedded**.

    > [!div class=mx-imgBorder]
    > ![Управление сервисом Power BI Embedded](../media/manage-powerbi-embedded-button.png "Управление службой Power BI Embedded")

3. В окне **Управление интеграцией сервиса Power BI Embedded** выберите **Отключить интеграцию сервиса Power BI Embedded**.

    > [!div class=mx-imgBorder]
    > ![Отключение сервиса Power BI Embedded](../media/disable-powerbi-embedded-window.png "Отключение сервиса Power BI Embedded")

4. Нажмите кнопку **Сохранить**.

5. Выберите **ОК** в сообщении подтверждения. Во время отключения сервиса Power BI Embedded портал перезапускается и недоступен в течение нескольких минут. После отключения сервиса Power BI Embedded отобразится сообщение.
