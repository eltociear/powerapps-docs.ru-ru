# <a name="set-up-power-bi-integration"></a>Настройка интеграции с Power BI

Power BI является одним из лучших средств для предоставления ценной информации с помощью простой и интерактивной визуализации. Чтобы просмотреть панели мониторинга и отчеты Power BI на веб-страницах портала, необходимо включить визуализацию Power BI из центра администрирования порталов PowerApps. Вы также можете внедрить панели мониторинга и отчеты, созданные в новой рабочей области Power BI, включив интеграцию со службой Power BI Embedded.

> [!NOTE]
> - Необходима соответствующая лицензия Power BI.
> - Чтобы использовать службу Power BI Embedded, необходима соответствующая Power BI Embedded лицензия. Дополнительные сведения см. в разделе [Лицензирование](https://docs.microsoft.com/power-bi/developer/embedded-faq#licensing).

## <a name="enable-power-bi-visualization"></a>Включить визуализацию Power BI

Включение визуализации Power BI позволяет внедрять панели мониторинга и отчеты на веб-страницах портала с помощью тега жидкостя powerbi.

1.  Откройте [центр администрирования порталов PowerApps](admin-overview.md).

2.  Перейдите к разделу **Настройка интеграции Power BI** > **включить визуализацию Power BI**.

    > [!div class=mx-imgBorder]
    > ![Включение визуализации Power BI](../media/enable-power-bi-visualization.png "Включение визуализации Power BI")

3.  Выберите **включить** в сообщении с подтверждением. Во время включения визуализации Power BI портал перезапускается и будет недоступен в течение нескольких минут. При включении визуализации Power BI появляется сообщение.

Средства настройки могут использовать тег жидкостей [powerbi](../liquid/portals-entity-tags.md#powerbi) для внедрения Power BI панелей мониторинга и отчетов на веб-страницах портала. При внедрении содержимого Power BI пользовательские настройки могут использовать [Параметры фильтра](https://docs.microsoft.com/en-us/power-bi/service-url-filters) для создания персонализированных представлений. Дополнительные сведения: [тег ликвидя powerbi](../liquid/portals-entity-tags.md#powerbi)

### <a name="disable-power-bi-visualization"></a>Отключить визуализацию Power BI

1.  Откройте [центр администрирования порталов PowerApps](admin-overview.md).

2.  Перейдите к разделу **Настройка интеграции Power BI** > **отключить визуализацию Power BI**.

    > [!div class=mx-imgBorder]
    > ![Отключить визуализацию Power BI визуализации](../media/disable-power-bi-visualization.png "Power BI")

3. В сообщении с подтверждением выберите **Отключить** . При отключении Power BI визуализации портал перезапускается и будет недоступен в течение нескольких минут. При отключении Power BI визуализации появляется сообщение.

## <a name="enable-power-bi-embedded-service"></a>Включение службы Power BI Embedded

Включение службы Power BI Embedded позволяет внедрять панели мониторинга и отчеты, созданные в новой рабочей области Power BI. Панели мониторинга и отчеты внедряются на веб-страницы на портале с помощью тега жидкости powerbi.

**Предварительные требования**. перед включением службы Power BI Embedded убедитесь, что в новой рабочей области в Power BI созданы информационные панели и отчеты. После создания рабочей области предоставьте административный доступ к глобальному администратору, чтобы рабочие области отображались в центре администрирования порталов PowerApps. Дополнительные сведения о создании новых рабочих областей и добавлении к ним доступа см. [в разделе Создание новых рабочих областей в Power BI](https://docs.microsoft.com/en-us/power-bi/service-create-the-new-workspaces).

**Ограничения службы Power BI Embedded**. сведения об ограничениях см. в статье [рекомендации и ограничения](https://docs.microsoft.com/power-bi/developer/embed-service-principal#considerations-and-limitations).

> [!NOTE]
> Убедитесь, что для работы тега жидкостного жидкости Power BI включена визуализация Power BI.

1. Откройте [центр администрирования порталов PowerApps](admin-overview.md).

2. Перейдите к разделу **Настройка интеграции Power BI** > **Enable Power BI Embedded Service**.

    > [!div class=mx-imgBorder]
    > ![Включение](../media/enable-powerbi-embedded-button.png "службы Power BI Embedded включить службу") Power BI Embedded

3. В окне **Включение интеграции со службой Power BI Embedded** выберите и переместите Power BI рабочие области, из которых панели мониторинга и отчеты должны отображаться на портале в списке **Выбранные рабочие области** .

    > [!div class=mx-imgBorder]
    > ![Выбор Power BI рабочих областей](../media/enable-powerbi-embedded-window.png "Выбор Power BI рабочих областей")
    
    > [!NOTE]
    > После добавления рабочих областей в список **Выбранные рабочие области** базы данных и отчеты подготавливаются к просмотру через несколько минут.
    

4. Выберите **включить**. Во время включения Power BI Embedded службы портал перезапускается и недоступен в течение нескольких минут. При включении службы Power BI Embedded появляется сообщение.

Теперь необходимо создать группу безопасности и добавить ее в учетную запись Power BI. Дополнительные сведения см. в статьях [Создание группы безопасности и добавление в учетную запись Power BI](#create-security-group-and-add-to-power-bi-account).

### <a name="create-security-group-and-add-to-power-bi-account"></a>Создание группы безопасности и добавление в учетную запись Power BI

После включения интеграции со службой Power BI Embedded необходимо создать группу безопасности в Azure Active Directory, добавить в нее участника, а затем добавить группу безопасности в Power BI через портал администрирования Power BI. Это позволяет отображать панели мониторинга и отчеты, созданные в новых Power BI рабочих областях, на портале.

> [!NOTE]
> Необходимо войти в систему с помощью той же учетной записи глобального администратора, которая использовалась для включения службы Power BI Embedded.

**Шаг 1. Создание группы безопасности**

1. Войдите в [портал Azure](https://portal.azure.com) с помощью учетной записи глобального администратора для каталога.

2. Выберите **Azure Active Directory**, **группы**, а затем выберите **создать группу**.

3. На странице **Группа** введите следующие сведения.

    - **Тип группы**: безопасность.

    - **Имя группы**: портал Power BI Embedded служба.

    - **Описание группы**: Эта группа безопасности используется для интеграции портала и службы Power BI Embedded.

    - **Тип членства**: назначено.

      > [!div class=mx-imgBorder]
      > ![Создание группы безопасности для службы Power BI Embedded](../media/powerbi-embed-security-group.png "Создание группы безопасности для службы Power BI Embedded")

4. Выберите **Создать**.

**Шаг 2. Добавление члена группы**

**Предварительные требования**. перед добавлением члена в группу безопасности необходимо иметь идентификатор приложения портала. Идентификатор приложения портала доступен на вкладке **сведения о портале** в [центре администрирования PowerApps порталов](admin-overview.md).

1. Войдите в [портал Azure](https://portal.azure.com) с помощью учетной записи глобального администратора для каталога.

2. Выберите **Azure Active Directory**, а затем выберите **группы**.

3. На странице **группы — все группы** найдите группу **служб Power BI Embedded портала** и выберите ее.

    > [!div class=mx-imgBorder]
    > ![Найдите и выберите группу безопасности Power BI Embedded Поиск службы](../media/search-security-group.png "и выберите группу безопасности для службы Power BI Embedded") .

4. На странице **Общие сведения о службе Power BI Embedded портала** выберите **члены** в области **Управление** .

5. Выберите **Добавить участников**и введите в ТЕКСТОВОМ поле идентификатор приложения портала.

6. Выберите элемент в результатах поиска и нажмите **кнопку Выбрать**.

    > [!div class=mx-imgBorder]
    > ![Добавление участника в группу безопасности для службы Power BI Embedded](../media/add-member-powerbi-embed.png "Добавить участника в группу безопасности для службы Power BI Embedded")

**Шаг 3. Настройка Power BI**

1. Войдите в [Power BI](https://powerbi.microsoft.com) , используя учетную запись глобального администратора для каталога.

2. Выберите **Параметры** в правом верхнем углу служба Power BI и щелкните **портал администрирования**.

    > [!div class=mx-imgBorder]
    > ![Выберите портал администрирования в Служба Power BI](../media/select-admin-portal.png "выберите портал администрирования в Служба Power BI")

3. Выберите **Параметры клиента**.

4. В разделе **Параметры разработчика** выберите **Разрешить субъектам-службам использовать Power BI API**.

5. В поле **определенные группы безопасности** найдите группу **служб Power BI Embedded портала** и выберите ее.

    > [!div class=mx-imgBorder]
    > ![Добавление группы безопасности на портале администрирования Power BI](../media/add-sg-powerbi.png "Добавление группы безопасности на портале администрирования Power BI")

6. Нажмите кнопку **Применить**.

Теперь средства настройки могут использовать тег жидкостей [powerbi](../liquid/portals-entity-tags.md#powerbi) для внедрения Power BI панелей мониторинга и отчетов из новых Power BI рабочих областей на веб-страницах портала. Чтобы использовать службу Power BI Embedded, тип проверки подлинности должен быть указан как **повербиембеддед**. При внедрении содержимого Power BI пользовательские настройки могут использовать [Параметры фильтра](https://docs.microsoft.com/en-us/power-bi/service-url-filters) для создания персонализированных представлений. Дополнительные сведения см. в [пометке жидкость powerbi](../liquid/portals-entity-tags.md#powerbi).


### <a name="manage-the-power-bi-embedded-service"></a>Управление службой Power BI Embedded

1. Откройте [центр администрирования порталов PowerApps](admin-overview.md).

2. Перейдите к разделу **Настройка интеграции Power BI** > **Управление службой Power BI Embedded**.

    > [!div class=mx-imgBorder]
    > ![Управление](../media/manage-powerbi-embedded-button.png "службой Power BI Embedded управление") службами Power BI Embedded

3. В окне **Управление интеграцией Power BI Embeddedной службы** удалите или переместите Power BI рабочие области, из которых панели мониторинга и отчеты должны отображаться на портале, в список **Выбранные рабочие области** .

    > [!div class=mx-imgBorder]
    > ![Выбор Power BI рабочих областей](../media/manage-powerbi-embedded-window.png "Выбор Power BI рабочих областей")
    
    > [!NOTE]
    > После удаления рабочих областей из списка **Выбранные рабочие области** для отражения изменений может потребоваться до 1 часа. До тех пор базы данных и отчеты подготавливаются к просмотру на портале без каких бы то ни было проблем.

4. Щелкните **Сохранить**.

### <a name="disable-the-power-bi-embedded-service"></a>Отключение службы Power BI Embedded

1.  Откройте [центр администрирования порталов PowerApps](admin-overview.md).

2.  Перейдите к разделу **Настройка интеграции Power BI** > **Управление службой Power BI Embedded**.

    > [!div class=mx-imgBorder]
    > ![Управление](../media/manage-powerbi-embedded-button.png "службой Power BI Embedded управление") службами Power BI Embedded

3. В окне " **Управление интеграцией Power BI Embeddedной службы** " выберите **отключить интеграцию со службой Power BI Embedded**.

    > [!div class=mx-imgBorder]
    > ![Отключение службы Power BI Embedded](../media/disable-powerbi-embedded-window.png "Отключить Power BI Embedded")

4. Щелкните **Сохранить**.

5. Нажмите кнопку **ОК** в сообщении с подтверждением. Во время отключения Power BI Embeddedной службы портал перезапускается и недоступен в течение нескольких минут. При отключении службы Power BI Embedded появляется сообщение.
