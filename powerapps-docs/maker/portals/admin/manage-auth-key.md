# <a name="connect-to-a-dynamics-365-online-organization-using-a-portal"></a>Подключение к организации Dynamics 365 Online с помощью портала

[!include[cc-beta-prerelease-disclaimer](../../../includes/cc-beta-prerelease-disclaimer.md)]

Портал подключается к организации Dynamics 365 (локальная версия) с помощью приложения Azure Active Directory. Приложение создается в том же клиенте, в котором подготовлен портал. Приложение регистрируется в организации Dynamics 365 в процессе подготовки портала.

![Подключение портала к организации Dynamics 365](../media/connect-with-dynamics.png "Подключение портала к организации Dynamics 365")

С каждым порталом связано отдельное приложение Azure Active Directory, независимо от того, подключены ли они к одной организации Dynamics 365 или нет. Поставщик проверки подлинности Azure Active Directory по умолчанию, созданный для портала, использует то же приложение Azure Active Directory для проверки подлинности портала. Проверка подлинности принудительно выполняется веб-ролями, назначенными пользователю, который выполняет доступ к порталу.

Связанное приложение портала отображается в Azure Active Directory. Имя этого приложения будет Microsoft CRM Portals, и код портала указан в поле **URI кода приложения** в приложении Azure Active Directory. Пользователь, который подготавливает портал, является владельцем данного приложения. Не следует удалять или изменять это приложение, в противном случае работоспособность портала может быть нарушена. Вы должны быть ответственным за приложение, чтобы управлять порталом из центра администрирования портала.

## <a name="authentication-key"></a>Ключ проверки подлинности

Портал для подключения к Dynamics 365 с использованием приложения Azure Active Directory, он нужен ключом проверки подлинности подключенного с приложением Azure Active Directory. Этот ключ создается при подготовке портала, и открытая часть этого ключа автоматически отправляется в приложение Azure Active Directory.

> [!IMPORTANT]
> Срок действия ключа проверки подлинности истекает через два года. Его необходимо обновлять каждые два года, чтобы портал был по прежнему подключен к организации Dynamics 365. Если не обновить ключ, портал перестанет работать.  

### <a name="authentication-key-details"></a>Сведения о ключе проверки подлинности

Сведения о ключе проверки подлинности отображаются в центре администрирования портала и на портале.

**Центр администрирования портала**

1. Открытие [Центра администрирования портала PowerApps](admin-overview.md).

2. Выберите **Управление ключом проверки подлинности портала**. Ключ проверки подлинности отображается вместе с датой окончания срока действия и отпечатком.

   > [!div class=mx-imgBorder]
   > ![Сведения о ключе проверки подлинности в центре администрирования портала](../media/manage-auth-key.png "Сведения о ключе проверки подлинности в центре администрирования портала")

**Портал**

1. Войдите на портал как администратор.

2. Перейдите по URL-адресу <путь_к_порталу>/_services/about. Отображается дата окончания срока действия ключа проверки подлинности. 

   > [!div class=mx-imgBorder]
   > ![Страницы службы портала](../media/portal-services-page.png "Страницы службы портала")

> [!NOTE]
> Для просмотра сведений о ключе проверки подлинности необходимо выполнить вход в портал в рамках того же сеанса веб-браузера, и необходимо обладать всеми разрешениями на доступ к веб-сайту.

### <a name="authentication-key-expiration-notification"></a>Уведомление окончания действия ключа проверки подлинности

Перед истечением срока действия ключа проверки подлинности вы получите уведомление по электронной почте, в центре администрирования портала и на портале.

**Электронная почта**

Сообщение электронной почты будет отправлено адресатам, подписанным на уведомления электронной почты для организации, подключенной к их порталу. Подробнее о подписке на уведомления электронной почты: [Управление уведомлениями электронной почты для администраторов](https://docs.microsoft.com/en-us/dynamics365/customer-engagement/admin/manage-email-notifications)

Уведомления электронной почты отправляются со следующими интервалами: 
- 90 дня 
- 60 дня 
- 30 дня 
- 15 дней 
- 7 дней 
- 6 дня 
- 5 дней 
- 4 дня 
- 3 дня 
- 2 дня 
- 1 день 
- 12 часов 
- 6 часов 
- 3 часов

Также вы будете получать уведомление после истечения срока действия ключа каждый день в течение 1 недели после окончания срока действия ключа.

> [!NOTE]
> - Интервалы рассчитываются в формате времени UTC от даты окончания срока действия ключа.
> - Не гарантируется, что сообщения электронной почты будут приходит точно через интервалы, указанные выше. Уведомление электронной почты может быть задержано или может отсутствовать. Обязательно проверяйте дату окончания срока действия ключа в Интернете.

**Центр администрирования портала**

Сообщение об окончании срока действия ключа отображается вверху страницы.

> [!div class=mx-imgBorder]
> ![Уведомление о ключе проверки подлинности в центре администрирования портала](../media/portal-admin-center-auth-notif.png "Уведомление о ключе проверки подлинности в центре администрирования портала")

**Портал**

При переходе по URL-адресу <путь_к_порталу>/_services/about уведомление об истечении срока действия ключа отображается внизу страницы.

> [!NOTE]
> Необходимо выполнить вход в портал в рамках того же сеанса веб-браузера, и необходимо обладать всеми разрешениями на доступ к веб-сайту.

> [!div class=mx-imgBorder]
> ![Уведомление о ключе проверки подлинности на портале](../media/portal-service-page-auth-notif.png "Уведомление о ключе проверки подлинности на портале")

## <a name="renew-portal-authentication-key"></a>Обновление ключа проверки подлинности портала

Необходимо обновлять ключ каждые два года, чтобы портал мог подключаться к организации Dynamics 365.

> [!NOTE]
> Чтобы обновить ключ, необходимо иметь разрешения на управление порталом.

1. Открытие [Центра администрирования портала PowerApps](admin-overview.md).

2. Выберите **Управление ключом проверки подлинности портала**. Ключ проверки подлинности отображается вместе с датой окончания срока действия и отпечатком.

    > [!div class=mx-imgBorder]
    > ![Управление ключом проверки подлинности портала](../media/manage-portal-auth-key.png "Управление ключом проверки подлинности портала")

3. Выберите **Обновить ключ**.

4. Выберите **Обновить** в сообщении. Начинается процесс обновления, и отображается сообщение.

> [!NOTE]
> - Пока этот процесс выполняется в фоновом режиме, портал будет один раз перезапущен.
> - При обновлении ключа он обновляется на два года.
> - Эта процедура займет от пяти до семи минут.

### <a name="troubleshooting"></a>Устранение неполадок

Если обновление ключа завершается сбоем, отображается сообщение об ошибке вместе со следующим действием:

- **Повторить попытку обновления ключа проверки подлинности**. Это действие позволяет перезапустить процесс обновления ключа проверки подлинности. Если обновление завершается ошибкой несколько раз, обратитесь в службу поддержки Майкрософт.

    > [!div class=mx-imgBorder]
    > ![Повторить попытку обновления ключа проверки подлинности портала](../media/retry-auth-key-update.png "Повторить попытку обновления ключа проверки подлинности портала")
