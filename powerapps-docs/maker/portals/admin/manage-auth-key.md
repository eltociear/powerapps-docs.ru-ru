# <a name="connect-to-a-common-data-service-environment-using-a-portal"></a>Подключение к среде Common Data Service с помощью портала

Портал подключается к среде Common Data Service с помощью приложения Azure Active Directory. Приложение создается в том же клиенте, где подготовлен портал. Приложение регистрируется в среде Common Data Service в процессе подготовки портала.

![Подключение портала с помощью среды Common Data Service](../media/connect-with-dynamics.png "Подключение портала к среде Common Data Service")

С каждым порталом связано отдельное Azure Active Directoryное приложение, независимо от того, подключено ли оно к той же среде Common Data Service. Поставщик проверки подлинности по умолчанию Azure Active Directory, созданный для портала, использует то же приложение Azure Active Directory для проверки подлинности портала. Авторизация обеспечивается веб-ролями, назначенными пользователю, обращающимся к порталу.

Связанное приложение портала можно просмотреть в Azure Active Directory. Имя этого приложения будет порталом Microsoft CRM, а идентификатор портала — в поле **URI идентификатора** приложения в Azure Active Directory приложении. Это приложение принадлежит пользователю, который подготавливает портал. Не следует удалять или изменять это приложение, а также прерывать работу портала. Для управления порталом из центра администрирования порталов PowerApps необходимо быть владельцем приложения.

## <a name="authentication-key"></a>Ключ проверки подлинности

Для подключения портала к Common Data Service с помощью Azure Active Directory приложения требуется ключ проверки подлинности, подключенный к приложению Azure Active Directory. Этот ключ создается при подготовке портала, а общедоступная часть этого ключа автоматически передается в приложение Azure Active Directory.

> [!IMPORTANT]
> Срок действия ключа проверки подлинности истекает через два года. Его необходимо продлить каждые два года, чтобы убедиться, что портал будет продолжать подключаться к среде Common Data Service. Если не обновить ключ, портал перестанет работать.  

### <a name="authentication-key-details"></a>Сведения о ключе проверки подлинности

Сведения о ключе проверки подлинности отображаются в центре администрирования и на портале PowerApps Portal.

**Центр администрирования порталов PowerApps**

1. Откройте [центр администрирования порталов PowerApps](admin-overview.md).

2. Выберите **Управление ключами проверки подлинности портала**. Ключ проверки подлинности отображается вместе с датой окончания срока действия и отпечатком.

   > [!div class=mx-imgBorder]
   > ![Сведения о ключе аутентификации в центре управления порталов powerapps](../media/manage-auth-key.png "сведения о ключе проверки подлинности в центре администрирования powerapps порталов")

**Портала**

1. Войдите на портал с правами администратора.

2. Перейдите по URL-адресу < portal_path >/_services/About. Отображается дата истечения срока действия ключа проверки подлинности. 

   > [!div class=mx-imgBorder]
   > ![](../media/portal-services-page.png "Страница службы") портала страницы службы портала

> [!NOTE]
> Чтобы просмотреть сведения о ключе проверки подлинности, необходимо войти на портал в том же сеансе браузера и получить все разрешения на доступ к веб-сайту.

Дополнительные сведения о регистрации для уведомления по электронной почте: [Управление уведомлениями по электронной почте администраторам](https://docs.microsoft.com/dynamics365/customer-engagement/admin/manage-email-notifications)

- 15 дней

- 6 дней 
- 5 дней 
- 4 дня 
- 3 дня 
- 2 дня

- 12 часов 
- 6 часов 
- 3 часа

## <a name="renew-portal-authentication-key"></a>Обновление ключа проверки подлинности на портале

Необходимо продлить ключ каждые два года, чтобы обеспечить возможность подключения портала к среде Common Data Service.

> [!NOTE]
> Чтобы обновить ключ, необходимо иметь разрешения на управление порталом.

1. Откройте [центр администрирования порталов PowerApps](admin-overview.md).

2. Выберите **Управление ключами проверки подлинности портала**. Ключ проверки подлинности отображается вместе с датой окончания срока действия и отпечатком.

    > [!div class=mx-imgBorder]
    > ![Управление]ключом аутентификации на портале(../media/manage-portal-auth-key.png "Управление ключами проверки подлинности портала")

3. Выберите **обновить ключ**.

4. В сообщении выберите **Обновить** . Начнется процесс обновления, и отобразится сообщение.

> [!NOTE]
> - Хотя этот процесс выполняется в фоновом режиме, портал будет перезапущен один раз.
> - При обновлении ключа он обновляется в течение двух лет.
> - Этот процесс займет от пяти до семи минут.

### <a name="troubleshooting"></a>Устранение неполадок

В случае сбоя обновления ключа отображается сообщение об ошибке, а также следующее действие:

- **Повторите обновление ключа проверки подлинности**. Это действие позволяет перезапустить процесс обновления ключа проверки подлинности на портале. Если обновление не удается выполнить несколько раз, обратитесь в службу поддержки Майкрософт.

    > [!div class=mx-imgBorder]
    > ![Повторить попытку обновления ключа аутентификации на портале](../media/retry-auth-key-update.png "повторить попытку обновления ключа проверки подлинности")
