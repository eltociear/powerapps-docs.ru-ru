# <a name="portal-checker"></a>Средство проверки портала

Средство проверки портала является средством самостоятельной диагностики, которое может использоваться администраторами портала для обнаружения распространенных проблем на портале. Средство проверки портала помогает выявление проблем с порталом с помощью различных параметров конфигурации и предлагает рекомендации по их устранению.

При запуске средства проверки портала результаты отображаются в разделе **Результаты диагностики** в формате сетки. Сетка результатов содержит следующие столбцы.

- **Ошибка**: отображает ошибку верхнего уровня, возникшую у клиента; Например, проблемы с производительностью.
- **Категория**: отображает область верхнего уровня, где можно классифицировать проблемы. Например, подготовка, обновление решения и т. д.
- **Результат**: отображает состояние проблемы; Например, ошибка, предупреждение и т. д.

По умолчанию данные в сетке сортируются по столбцу **результатов** в следующем порядке: Error, Warning и Pass.

> [!div class=mx-imgBorder]
> (../media/diagnostic-results.png "Результаты") диагностики ![результатов диагностики]

Вы можете развернуть ошибку, чтобы просмотреть подробные сведения и шаги по их устранению. Если для устранения рисков требуются какие-либо действия, вы увидите кнопку, которая будет выполнять действие. Вы также можете оставить отзыв о том, было ли полезно решение об устранении рисков.

> [!div class=mx-imgBorder]
> ![Расширение проблемы в результатах диагностики](../media/diagnostic-results-issue-expand.png "расширение проблемы в результатах диагностики")

При необходимости можно повторно выполнить проверки диагностики, которые будут обновлять результаты обновленными данными.

> [!NOTE]
> Если портал отключен или включена фильтрация IP-адресов, некоторые диагностические проверки не будут выполняться на портале.

Список распространенных проблем, устраняемых средством проверки портала, см. в разделе [Общие проблемы с порталом, которые можно диагностировать с помощью средства проверки портала и](https://docs.microsoft.com/dynamics365/customer-engagement/portals/portal-faq)рекомендации.

Чтобы запустить средство проверки портала, выполните следующие действия.

1.  Откройте [центр администрирования порталов PowerApps](admin-overview.md).

2.  Перейдите к разделу **Запуск средства проверки портала**.

    > [!div class=mx-imgBorder]
    > ![Запустить]средство проверки портала запустить средство проверки(../media/run-diagnostics.png "портала")

3.  Выберите **запустить средство проверки портала**. Начнется сеанс диагностики, и будут собраны данные о проблемах клиента. Результаты отображаются в разделе **Результаты диагностики** .

    > [!div class=mx-imgBorder]
    > (../media/diagnostic-results.png "Результаты") диагностики ![результатов диагностики]

4.  Чтобы повторно запустить проверки диагностики, выберите **Обновить результаты**.

    > [!div class=mx-imgBorder]
    > Обновить результаты ![диагностики](../media/diagnostic-results-refresh.png "обновление результатов диагностики")
