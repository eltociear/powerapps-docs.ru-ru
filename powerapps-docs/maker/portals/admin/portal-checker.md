# <a name="portal-checker"></a>Средство проверки портала

Средство проверки портала является средством самодиагностики, которое может использоваться администраторами портала для выявления общих проблем на своем портале. Средство проверки порталов поможет обнаружению проблем с порталом, проверяя различные параметры конфигурации и приводя предложения о том, как их исправить.

При запуске средства проверки портала результаты отображаются в разделе **Диагностические результаты** в формате таблицы. Сетка результатов имеет следующие столбцы:

- **Проблема**: отображает проблему верхнего уровня, с которой столкнулся клиент; например, проблема производительности.
- **Категория**: отображается область верхнего уровня, по которой проблему можно классифицировать; например, подготовка, обновление решения и т. п.
- **Результат**: отображает состояние проблемы; например, ошибка, предупреждение и т. п.

По умолчанию информация в сетке сортируется по столбцу **Результаты** в таком порядке: ошибка, предупреждение и успешно пройденная проверка.

> [!div class=mx-imgBorder]
> ![Результаты диагностики](../media/diagnostic-results.png "Результаты диагностики")

Проблему можно разворачивать для просмотра детальных сведений и мер по устранению. Если меры по устранению требуют выполнение какого-либо действия, отображается кнопка, которая выполняет действие. Можно также предоставить отзыв о том, была ли мера устранения полезной.

> [!div class=mx-imgBorder]
> ![Развертывание проблемы в результатах диагностики](../media/diagnostic-results-issue-expand.png "Развертывание проблемы в результатах диагностики")

Если требуется, можно повторно выполнить диагностические проверки, и результаты будут обновлены с учетом обновленных данных.

> [!NOTE]
> Если портал выключен или включена фильтрация IP-адресов, некоторые диагностические проверки не будут выполнены на вашем портале.

Список общих проблем, диагностируемых средством проверки портала, см. в разделе [Общие проблемы портала, диагностируемые средством проверки портала, и рекомендации](https://docs.microsoft.com/dynamics365/customer-engagement/portals/portal-faq).

Для запуска проверки портала:

1.  Открытие [Центра администрирования портала Power Apps](admin-overview.md).

2.  Перейдите к пункту **Запустить проверку портала**.

    > [!div class=mx-imgBorder]
    > ![Запустить проверку портала](../media/run-diagnostics.png "Запустить проверку портала")

3.  Выберите **Запустить проверку портала**. Сеанс диагностики начинается и собирает данные о проблемах клиента. Результаты отображаются в разделе **Результаты диагностики**.

    > [!div class=mx-imgBorder]
    > ![Результаты диагностики](../media/diagnostic-results.png "Результаты диагностики")

4.  Для повторного выполнения диагностических проверок выберите пункт **Обновить результаты**.

    > [!div class=mx-imgBorder]
    > ![Обновление результатов диагностики](../media/diagnostic-results-refresh.png "Обновление результатов диагностики")
