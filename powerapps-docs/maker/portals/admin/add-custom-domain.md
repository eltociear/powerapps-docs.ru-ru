# <a name="add-a-custom-domain-name"></a>Добавить пользовательское доменное имя

Настраиваемый домен помогает клиентам искать ваши ресурсы поддержки и повышает репутацию вашей марки. К порталу можно добавить только одно настраиваемое имя домена. После подготовки портала и получения имени домена потребуется сертификат SSL, чтобы задать настраиваемое имя узла. Приобретенный сертификат SSL можно использовать для своего домена, чтобы связать ваш портал с пользовательским доменом с помощью мастера.

1. Открытие [Центра администрирования портала Power Apps](admin-overview.md).

2. Откройте **Действия портала** > **Добавить пользовательское доменное имя**. Открывается мастер для выбора сертификата SSL.

3. На странице **Выберите SSL-сертификат** выберите один из следующих параметров:
   - **Отправить новый сертификат**: выберите этот вариант, чтобы отправить файл PFX, если вы еще не отправили его в организацию. Выберите кнопку отправки под пунктом **Файл**, чтобы выбрать файл PFX. После выбора файла введите пароль для вашего сертификата SSL в поле **Пароль**.
   - **Использовать имеющийся сертификат**: выберите этот вариант, чтобы выбрать правильный сертификат из раскрывающегося списка.

     > [!Note]
     > Сертификат SSL должен удовлетворять всем следующим требованиям:
     > - Подписан доверенным центром сертификации
     > - Экспортирован как защищенный паролем файл PFX
     > - Содержит закрытый ключ длиной не менее 2048 битов
     > - Содержит все промежуточные сертификаты в цепочке сертификатов
     > - Должно быть включено SHA2; поддержка SHA1 в настоящее время удаляется из популярных браузеров

4. Выберите **Далее**.

5. На странице **Выберите имя узла** выберите один из следующих параметров:
    - **Добавить новое имя узла**: выберите этот вариант, чтобы создать новый пользовательский домен. Введите требуемое имя домена в поле **Имя домена**.
    - **Использовать существующее имя узла**: выберите этот вариант, чтобы выбрать имя узла из раскрывающегося списка. 
   
   > [!Note]
   > - У портала может быть только одно пользовательское имя домена. 
   > - Для создания настраиваемого имени узла необходимо создать CNAME с вашим поставщиком домена, которое направляет ваш домен на URL-адрес портала. Если вы только что добавили CNAME у вашего поставщика домена, потребуется некоторое время для его распространения на все DNS-серверы. Если имя не распространено и вы добавите его здесь, отображается следующее сообщение об ошибке: "Добавьте запись CNAME к данному имени домена". Повторите попытку через некоторое время.

6. Просмотрите введенные сведения, затем выберите **Далее**, чтобы начать создание привязки SSL. Должно появиться сообщение "Для этого портала успешно настроено пользовательское доменное имя. Чтобы открыть портал, перейдите по адресу {Пользовательское имя домена}. {Пользовательское доменное имя} будет гиперссылкой на только что настроенный пользовательский URL-адрес портала.

7. Выберите **Готово**, чтобы закрыть мастер.

    > [!Note]
    > Если необходимо изменить существующее настраиваемое имя домена, необходимо отправить новый сертификат SSL и выполнить шаги, как указано в этом разделе.
