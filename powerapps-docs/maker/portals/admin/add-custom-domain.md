# <a name="add-a-custom-domain-name"></a>Добавление имени пользовательского домена

Личный домен позволяет клиентам быстрее находить ваши ресурсы по поддержке и улучшать вашу торговую марку. На портал можно добавить только одно пользовательское доменное имя. После подготовки портала и получения имени домена вам потребуется SSL-сертификат для настройки пользовательского имени узла. Вы можете использовать приобретенный SSL-сертификат для своего домена, чтобы связать портал с пользовательским доменом с помощью мастера.

1. Откройте [центр администрирования порталов PowerApps](admin-overview.md).

2. Перейдите к **действиям портала** > **Добавить пользовательское доменное имя**. Откроется мастер, чтобы выбрать SSL-сертификат.

3. На странице **Выбор SSL-сертификата** выберите один из следующих вариантов.
   - **Отправить новый сертификат**. Выберите этот параметр, чтобы отправить PFX-файл, если он еще не передан в организацию. Чтобы выбрать PFX-файл, нажмите кнопку отправить под **файлом** . После выбора файла введите пароль для SSL-сертификата в поле **Password (пароль** ).
   - **Использовать существующий сертификат**: Выберите этот параметр, чтобы выбрать правильный сертификат из раскрывающегося списка.

     > [!Note]
     > SSL-сертификат должен соответствовать следующим требованиям:
     > - Подписано доверенным центром сертификации
     > - Экспортировано как защищенный паролем PFX-файл
     > - Содержит закрытый ключ длиной не менее 2048 бит
     > - Содержит все промежуточные сертификаты в цепочке сертификатов
     > - Необходимо включить SHA2. Поддержка SHA1 удаляется из популярных браузеров

4. Нажмите кнопку **Далее**.

5. На странице **Выбор имени узла** выберите один из следующих параметров.
    - **Добавить новое имя узла**: Выберите этот параметр, чтобы создать новый личный домен. В поле **доменное** имя введите нужное доменное имя.
    - **Использовать существующее имя узла**: Выберите этот параметр, чтобы выбрать имя узла из раскрывающегося списка. 
   
   > [!Note]
   > - Для портала можно использовать только одно пользовательское доменное имя. 
   > - Чтобы создать пользовательское имя узла, необходимо создать запись CNAME с поставщиком домена, которая указывает домену на URL-адрес портала. Если вы только что добавили запись CNAME в поставщик домена, она займет некоторое время для распространения на все DNS-серверы. Если имя не распространено и вы добавите его сюда, появится следующее сообщение об ошибке: добавьте запись CNAME к этому доменному имени. Повторите попытку через некоторое время.

6. Просмотрите введенные сведения и нажмите кнопку **Далее** , чтобы начать создание привязки SSL. Вы увидите, что имя личного домена сообщения успешно настроено для этого портала. Теперь для доступа к этому порталу можно перейти на страницу {пользовательское имя домена}. {Имя личного домена} будет гиперссылкой на пользовательский URL-адрес настраиваемого портала, который был только что настроен.

7. Нажмите кнопку **Готово** , чтобы закрыть мастер.

    > [!Note]
    > Если вы хотите изменить существующее имя личного домена, необходимо отправить новый SSL-сертификат и выполнить действия, описанные в мастере, как описано [здесь](#link-your-portal-to-a-custom-domain).
