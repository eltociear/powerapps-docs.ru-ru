# <a name="restrict-portal-access-by-ip-address"></a>Ограничение доступа к порталу по IP-адресу

Портал является общедоступным при подготовке к доступу любому пользователю с любого компьютера. Теперь вы можете ограничить доступ к порталу из списка IP-адресов. Например, правительственная организация может попытаться использовать содержимое только в корпоративной сети. Коммерческой организации может потребоваться отображать портал только при его публикации, а не во время разработки, чтобы избежать утечки данных.

Когда запрос на портал создается от любого пользователя, его IP-адрес оценивается по списку разрешений. Если IP-адрес отсутствует в списке, на портале отобразится веб-страница с кодом состояния HTTP 403.

Чтобы добавить или удалить IP-адреса, необходимо назначить одну из следующих ролей:
- Глобальный администратор Office 365 
- Администратор службы. Дополнительные сведения: [Использование роли администратора службы для управления клиентом](https://technet.microsoft.com/en-us/library/mt793847.aspx)  
- Системный администратор среды Common Data Service, выбранной для портала

## <a name="add-an-ip-address"></a>Добавление IP-адреса

Чтобы разрешить доступ к порталу с IP-адреса или набора IP-адресов, можно добавить IP-адреса в список. Это позволяет получить доступ к порталу только из списка добавленных IP-адресов. Если вы не добавите IP-адрес, портал будет доступен со всех IP-адресов.

После добавления IP-адреса в список ограничений портал будет доступен только для указанного IP-адреса. При попытке доступа к порталу с любого другого IP-адреса будет отказано в доступе, и отобразится веб-страница с кодом состояния HTTP 403. Содержимое этой веб-страницы является статическим и не может быть изменено.

> [!div class=mx-imgBorder]
> ![Html 403 ошибка](../media/ip-address-page-error.png "HTML 403 ошибка")  

> [!NOTE]
> Необходимо указать общедоступный IP-адрес, к которому может получить доступ портал. Портал не может получить доступ к частному IP-адресу.

1.  Откройте [центр администрирования порталов PowerApps](admin-overview.md).

2.  Перейдите к разделу **Настройка ограничения IP-адресов**. Отобразится список IP-адресов и их типов.

    > [!div class=mx-imgBorder]
    > ![Настройка ограничения IP-адресов](../media/set-up-ip-address-restrict.png "Настройка ограничения IP-адресов")

3.  На странице Настройка ограничений IP-адресов выберите **Добавить новый**.

4.  В окне Добавление IP-адреса введите следующие значения.

    - **Выберите тип IP-адреса**: укажите, является ли IP-адрес IPv4 или IPv6.

    - **Укажите IP-адрес в нотации CIDR**: укажите IP-адрес в нотации CIDR. Дополнительные сведения: [междоменная маршрутизация в любых классах](https://en.wikipedia.org/wiki/Classless_Inter-Domain_Routing)

      > [!div class=mx-imgBorder]
      > ![Добавление IP-адреса](../media/add-ip-address.png "Добавление IP-адреса")    

5.  Выберите **настроить**.

## <a name="remove-an-ip-address"></a>Удаление IP-адреса

Чтобы удалить доступ к порталу с ранее разрешенного IP-адреса, можно удалить IP-адрес из списка. Если удалить все IP-адреса, портал будет доступен со всех IP-адресов.

1.  Откройте [центр администрирования порталов PowerApps](admin-overview.md).

2.  Перейдите к разделу **Настройка ограничения IP-адресов**. Отобразится список IP-адресов и их типов.

    > [!div class=mx-imgBorder]
    > ![Настройка ограничения IP-адресов](../media/set-up-ip-address-restrict.png "Настройка ограничения IP-адресов")

3.  Выберите **удалить IP-адрес (x)** рядом с IP-адресом, который необходимо удалить.

4.  Выберите **Удалить** в сообщении с подтверждением.
