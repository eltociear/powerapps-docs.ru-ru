# <a name="manage-existing-portals-in-powerapps"></a>Управление существующими порталами в PowerApps

После создания портала он отображается в разделе **недавние приложения** на домашней странице PowerApps.

> [!div class=mx-imgBorder]
> ![последние приложения](media/recent-apps.png "Последние приложения")  

Для управления приложением выберите **Дополнительные команды** ( **...** ) для портала и выберите действие в контекстном меню.

> [!div class=mx-imgBorder]
> ![Параметры приложения портала](media/portal-app-options.png "Параметры приложения портала")  

## <a name="edit"></a>Редактор

Открывает [порталы PowerApps в студии](portal-designer-anatomy.md) для изменения содержимого и компонентов портала.  

> [!div class=mx-imgBorder]
> ![Портал портала](media/portal-maker.png "Портал портала")  

## <a name="browse"></a>Ресурсов

Открывает портал для просмотра веб-сайта. Это поможет вам просмотреть портал так, как он будет выглядеть для клиентов.

> [!div class=mx-imgBorder]
> ![веб-сайт портала](media/portal-website.png "Веб-сайт портала")  

Кроме того, можно также открыть портал для просмотра веб-сайта, нажав кнопку **Обзор веб-сайта** в [студии порталов PowerApps](portal-designer-anatomy.md) , чтобы просмотреть изменения, внесенные на веб-сайт. Веб-сайт откроется на новой вкладке с URL-адресом веб-сайта.

## <a name="share"></a>Предоставить общий доступ

Совместное использование портала с внутренними или внешними пользователями. Выполните действия, указанные на панели **общий доступ к этому порталу** .

> [!div class=mx-imgBorder]
> ![Общий доступ к порталу](media/share-portal.png "Общий доступ к порталу")  

### <a name="share-with-internal-users"></a>Общий доступ для внутренних пользователей

Чтобы предоставить доступ к порталу внутренним пользователям, сначала необходимо создать роль безопасности, а затем назначить пользователей роли безопасности, чтобы они могли использовать портал.

> [!NOTE]
> В Common Data Service, если у вас нет соответствующих прав доступа к сущностям портала, могут появиться такие ошибки, как "у вас нет доступа для просмотра решений в этой среде". или "у вас нет доступа для просмотра веб-сайта в этой среде". Рекомендуется быть членом роли безопасности системного администратора в соответствующей Common Data Service базе данных.

#### <a name="step-1-create-a-security-role"></a>Шаг 1. Создание роли безопасности

1.  В области **общий доступ к этому порталу** в разделе **Создание роли безопасности**выберите **роли безопасности**. Отобразится список всех настроенных ролей безопасности.

2.  На панели инструментов действия нажмите кнопку **создать**.

3.  В окне **Новая роль безопасности** введите имя роли.

4.  Задайте привилегии для всех сущностей, используемых на портале.

5.  Завершив настройку роли безопасности, на панели инструментов нажмите кнопку **сохранить и закрыть**.

Сведения о ролях и привилегиях безопасности см. в разделе [роли и привилегии безопасности](https://docs.microsoft.com/power-platform/admin/security-roles-privileges).

#### <a name="step-2-assign-users-to-the-security-role"></a>Шаг 2. Назначение пользователей для роли безопасности

1.  В области **общий доступ к этому порталу** в разделе **Назначение пользователей роли безопасности**выберите **Пользователи**. Отобразится список всех пользователей.

2.  Выберите пользователя, которому требуется назначить роль безопасности.

3.  Выберите элемент **Управление ролями**.

    > [!NOTE]
    > Если не удается увидеть кнопку **Управление ролями** на панели команд, необходимо изменить клиент, установив для форцеуЦи значение 0 в URL-адресе. Например, https://&lt;org\_URL&gt;/Маин.аспкс? pageType = ентитилист & ЕТН = SystemUser & ФорцеуЦи = 0

4.  В диалоговом окне **Управление ролями пользователей** выберите созданную ранее роль безопасности и нажмите кнопку **ОК**.

### <a name="share-with-external-users"></a>Поделиться с внешними пользователями

Портал должен работать анонимно и должен быть доступен внешним пользователям. Если вы хотите ознакомиться с расширенными возможностями управления ролями и разрешениями для внешних пользователей, см. статью [Настройка контакта для использования на портале](configure/configure-contacts.md), [приглашение контактов на](configure/invite-contacts.md)портал, [Создание веб-ролей для порталов](configure/create-web-roles.md), [Назначение разрешений для сущностей. ](configure/assign-entity-permissions.md).

## <a name="delete"></a>Удален

Удаляет портал и размещенные ресурсы. При удалении портала его URL-адрес становится недоступным. Удаление портала не влияет на конфигурации портала или решения, имеющиеся в вашей среде, и остается неизменной.
Чтобы полностью удалить конфигурации портала из среды, необходимо вручную удалить конфигурации портала. Для этого используйте приложение управления портала и удалите соответствующую запись веб-сайта для портала.

> [!NOTE]
> Если у вас недостаточно прав для удаления портала, отображается сообщение об ошибке. Для удаления портала необходимо иметь роль системного администратора. Кроме того, вы должны быть владельцем приложения портала в Azure Active Directory. Пользователь, создающий портал, по умолчанию является владельцем и может удалить портал. Сведения о добавлении себя в качестве владельца см. в статье [Добавление себя в качестве владельца приложения Azure AD](admin/admin-overview.md#add-yourself-as-an-owner-of-the-azure-ad-application).
