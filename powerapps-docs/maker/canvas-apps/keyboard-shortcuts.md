# <a name="keyboard-shortcuts-for-canvas-apps"></a>Сочетания клавиш для приложений Canvas

> [!NOTE]
> Сочетания клавиш могут различаться в зависимости от раскладки клавиатуры.

## <a name="file"></a>Файл

| Сочетания | Действие |
|--|--|
| Ctrl + O (или ALT + F) | Откройте файл. |
| Ctrl + Shift + S (или Alt + P) | Сохраните приложение с другим именем. |
| CTRL+S | Сохраните приложение с тем же именем или в первый раз. |
| CTRL+SHIFT+P | Сохраните приложение и активируйте диалоговое окно публикации. |
| F12 | Скачайте файл приложения (. MSApp). |
| Alt+F | Откройте меню **файл** . |

## <a name="alternate-behavior"></a>Альтернативное поведение

| Сочетания | Действие |
|--|--|
| Alt или CTRL + SHIFT | 1. перед выбором элемента управления скройте элементы дизайна, чтобы можно было взаимодействовать с элементами управления в качестве пользователя приложения.<br>2. После инициации изменения размера или расположения элемента управления, удерживая нажатой клавишу, все точки привязки переопределяются. |

Как и в случае с электронной таблицей Excel, приложения Canvas являются активными и работают даже при редактировании.  Нет необходимости в переходе в режим предварительного просмотра для запуска приложения, что значительно ускоряет цикл изменения и тестирования.  Но это создает проблему: как различить выбор элемента управления "Кнопка", чтобы его можно было изменить из выбора элемента управления "Кнопка" для выполнения логики в нашем приложении?

В режиме конструктора по умолчанию выбирается объект для редактирования: перемещение, изменение размера, изменение свойств и иным образом Настройка объекта.  Это значение по умолчанию можно переопределить, удерживая нажатыми клавиши ALT или Ctrl + Shift перед тем, как *начать* выбор, который рассматривает выбор так, как будто пользователь приложения сделал это.  

В следующей анимации элемент управления "Кнопка" сначала выбирается для редактирования.  Графические элементы отображаются вокруг элемента управления, а в строке формул отображается свойство **OnSelect** , готовое к редактированию.  Затем кнопка освобождается.  *При первом*нажатии клавиши ALT элемент управления "Кнопка" снова выбирается, но на этот раз вычисляется свойство **OnSelect** и отображается уведомление, как если бы кнопка была выбрана в работающем приложении.  

![Анимация, показывающая эффект начала, удерживая нажатой клавишу Alt, выберите элемент управления "Кнопка"](media/keyboard-shortcuts/alt-select.gif)

Клавиша Alt также может использоваться *после* выбора элемента управления для переопределения точек привязки для перемещения и изменения размера.  Следующая анимация показывает изменение размера карточки данных в элементе управления [**формы редактирования**](controls/control-form-detail.md) .  Изначально изменение размера ограничено конкретными точками привязки.  В дальнейшем, *не отпуская кнопку мыши*, клавиша Alt будет нажата в дополнение к кнопке мыши. Добавление клавиши ALT переопределяет точки привязки, и любая ширина может быть получена с помощью мыши. 

![Анимация, демонстрирующая эффект добавления клавиши ALT к изменению размера карточки данных](media/keyboard-shortcuts/alt-fine-control.gif)
