# <a name="accessibility-properties-for-canvas-apps"></a>Свойства специальных возможностей для приложений Canvas

Настройка свойств, представляющих альтернативные способы взаимодействия с элементами управления, подходящие для пользователей с физическими нарушениями.

## <a name="properties"></a>Свойства

**AccessibleLabel** — метка для средств чтения с экрана. Пустое значение для элементов управления "Изображение", "Значок" и "Форма" делает невидимыми эти элементы управления для программ чтения с экрана (они распознаются как оформление).

**Live** — как средства чтения с экрана должны объявлять об изменениях в содержимом. Доступно только в элементе управления " **[Метка](control-text-box.md)** ".

* Если задано значение **Off**, средство чтения с экрана не объявляет об изменениях.
* Если задано значение " **вежливо**", средство чтения с экрана завершает работу, прежде чем сообщать об изменениях, произошедших в процессе чтения с экрана.
* Если задано значение " **Assert**", средство чтения с экрана прерывает работу, чтобы объявить об изменениях, произошедших в процессе чтения с экрана.

Узнайте, как [объявлять динамические изменения в динамических регионах](../accessible-apps-live-regions.md).

**TabIndex** — определяет, участвует ли элемент управления при навигации с помощью клавиатуры.

Навигация с помощью клавиатуры является важным аспектом любого приложения.  Многие сочетания клавиш более эффективны по сравнению с использованием сенсорного ввода или мыши и позволяют средствам чтения с экрана для визуального нарушения.  Порядок навигации должен:
- Зеркальное отображение отображаемых визуальных данных.
- Только позиция табуляции в элементах управления, которые являются интерактивными.
- Следуйте интуитивно понятным, а затем вниз по порядку "Z" или вниз, а затем по порядку "Обратная N".

Указанные выше требования будут выполнены со значениями **TabIndex** по умолчанию, и мы рекомендуем не изменять их.  По умолчанию большинство пользователей заполагалось визуально и хорошо работает с программой чтения с экрана.  Но в некоторых случаях потребуется переопределить значение по умолчанию.  Используйте свойство **TabIndex** и [ **Расширенный** элемент управления группой](https://powerapps.microsoft.com/en-us/blog/enhanced-group-experimental-control-with-layout-control-and-nesting/) (экспериментальный) для внесения изменений в порядок навигации.  

Свойство **TabIndex** имеет два рекомендуемых значения:

| Значение TabIndex | Поведение | По умолчанию для |
|----------------|----------|-------------|
| 0 | Элемент управления участвует в навигации с помощью клавиатуры. | [**Кнопка**](control-button.md), [**Ввод текста**](control-text-input.md), [**поле со списком**](control-combo-box.md)и другие, как правило, интерактивные элементы управления. |
| &minus;1 | Элемент управления не участвует в навигации с помощью клавиатуры. | [**Метка**](control-text-box.md), [**изображение**](control-image.md), [**значок**](control-shapes-icons.md)и другие обычно неинтерактивные элементы управления. |

Порядок навигации обычно проходит слева направо, а затем сверху вниз в шаблоне "Z". Порядок основан на значениях свойств **X** и **Y** элементов управления. Если элементы управления динамически перемещаются на экран, например с помощью формулы для **X** или **Y** на основе таймера или другого элемента управления, порядок переходов также изменится динамически.

Используйте [ **Расширенный** элемент управления группой](https://powerapps.microsoft.com/en-us/blog/enhanced-group-experimental-control-with-layout-control-and-nesting/) (экспериментальный) для объединения элементов управления, которые должны быть перемещены вместе, или для создания столбцов в шаблоне "Reverse-N".  В верхней части следующего примера поля имен содержатся в расширенном элементе управления группы, что приводит к продолжению навигации, прежде чем перемещаться между ними.  В нижней части примера элементы управления группы не используются, и переходы выполняются в, а затем вниз, как обычные, что не является интуитивно понятным для групп элементов управления. 

![Анимация, показывающая Улучшенный элемент управления группы, которая приводит к переходу в пределах группы, прежде чем перемещаться по](media/properties-accessibility/enhanced-group.gif)

Аналогичным образом, переходы по контейнерам, таким как элементы управления [**формы**](control-form-detail.md) и [**коллекции**](control-gallery.md) , будут перемещаться по всем элементам контейнера перед переходом к следующему элементу управления за пределами контейнера.  

Элементы управления, которые имеют значение *false* или **значение свойства** **DisplayMode** **disabled** , не включаются в навигацию.  

При использовании браузера переход от последнего элемента управления экрана к встроенным элементам управления обозревателя, таким как URL-адрес, будет перемещен.  

> [!WARNING]
> Избегайте значений **TabIndex** больше 0. В конечном итоге элементы управления подготавливаются к просмотру в формате HTML, где даже в [консорциуме W3C](https://www.w3.org/TR/wai-aria-practices/#kbd_general_between) автору настоятельно рекомендуется не использовать эти значения. " Многие средства HTML предупреждают о значениях больше 0, как если бы [средство проверки приложений](../accessibility-checker.md) при отходе отчета "проверить порядок элементов экрана".  По крайней мере: использование **TabIndex** таким образом может быть очень трудным, что может привести к невозможности использования средств чтения с экрана.
> 
> Если элементы управления с **TabIndex** больше 0, пользователи будут переходить к элементам управления, увеличивая значения **TabIndex** (1, затем 2 и т. д.). Когда пользователи проходят все элементы управления с положительными значениями **TabIndex** , они будут переходить к элементам управления с **TabIndex** 0, включая встроенные элементы управления браузера. Если имеется несколько элементов управления с одинаковым **TabIndex**, их координаты **X** и **Y** определяют их относительный порядок.
