# <a name="camera-control-in-power-apps"></a>Элемент управления "Камера" в Power Apps

Элемент управления, позволяющий пользователям получать изображения с помощью камеры на устройстве.

## <a name="description"></a>Описание

Используйте элемент управления **камерой** для записи изображений с помощью камеры устройства.  Устройство должно иметь камеру, и пользователь должен авторизовать приложение для использования камеры. Элемент управления "Камера" поддерживается при работе в веб-браузере.

Последняя Записанная картинка доступна через свойство **Photo** . С помощью этого свойства изображения могут быть:

- **Просматривается с помощью элемента управления изображения.** Используйте элемент управления [изображение](control-image.md) для просмотра захваченного образа. Дополнительные сведения см. в [примерах](#examples).  
- **Временное помещение в переменную или коллекцию.**  Используйте функции [Set](../functions/function-set.md) или [собирать](../functions/function-clear-collect-clearcollect.md) для хранения изображений в переменной или коллекции.  Будьте внимательны при использовании нескольких образов в коллекции одновременно с ограниченным объемом памяти устройства. Используйте функции [SaveData](../functions/function-savedata-loaddata.md) и [LoadData](../functions/function-savedata-loaddata.md) , чтобы перемещать изображения в локальное хранилище на устройстве и в [автономные сценарии](../offline-apps.md).
- **Хранится в базе данных.**  Используйте функцию [Patch](../functions/function-patch.md) для хранения изображений в базе данных.
- **Передается в виде текстовой строки в кодировке Base64.**  Используйте функцию [JSON](../functions/function-json.md) для кодирования изображений в Base64.

Используйте свойства **Stream**, **Стреамрате**и **onstream** для автоматической записи образов в таймере, например для привязки изображения каждую минуту для создания последовательности временных интервалов.

На захваченный носитель ссылается URI текстовой строки. Дополнительные сведения см. в [документации по типам данных](../functions/data-types.md#uris-for-images-and-other-media).

Изображения, созданные с помощью элемента управления "Камера", обычно не имеют полного разрешения камеры. Если требуются образы с полным разрешением, используйте элемент управления [Добавить рисунок](control-add-picture.md) .

## <a name="key-properties"></a>Основные свойства

**Camera** — на устройстве с несколькими камерами это число идентифицирует камеру, используемую приложением.

**OnStream** — поведение приложения при обновлении свойства **Stream**.

**Фото** — изображение, записываемое, когда пользователь получает изображение. 

**Stream** — изображение, автоматически обновляемое согласно свойству **StreamRate**.

**StreamRate** — интервал обновления изображения, определенного с помощью свойства **Stream**, в миллисекундах. Допустимые значения — от 100 (1/10 секунды) до 3 600 000 (1 час).

## <a name="additional-properties"></a>Дополнительные свойства

[AccessibleLabel](properties-accessibility.md) — метка для средств чтения с экрана. Необходимо описать назначение съемки.

[BorderColor](properties-color-border.md) — цвет границы элемента управления.

[BorderStyle](properties-color-border.md) — стиль границы элемента управления: **Solid**, **Dashed**, **Dotted** или **None**.

[BorderThickness](properties-color-border.md) — толщина границы элемента управления.

**Brightness** — яркость изображения.

**Contrast** — контрастность, или легкость различения похожих цветов в изображении.

[DisplayMode](properties-core.md) — указывает, допускает ли элемент управления ввод данных пользователем (**изменить**), отображает только данные (**представление**) или отключено (**отключено**).

[Фокуседбордерколор](properties-color-border.md) — цвет границы элемента управления, когда элемент управления имеет особое значение.

[Фокуседбордерсиккнесс](properties-color-border.md) — Толщина границы элемента управления, когда элемент управления имеет особое значение.

[Height](properties-size-location.md) — расстояние между верхним и нижним краями элемента управления.

[OnSelect](properties-core.md) — поведение приложения, когда пользователь щелкает элемент управления или касается его.

[TabIndex](properties-accessibility.md) — порядок навигации по клавиатуре по сравнению с другими элементами управления.

[Tooltip](properties-core.md) — пояснительный текст, отображаемый при наведении указателя мыши на элемент управления.

[Visible](properties-core.md) — определяет, отображается элемент управления или он скрыт.

[Width](properties-size-location.md) — расстояние между левым и правым краями элемента управления.

[X](properties-size-location.md) — расстояние между левым краями элемента управления и левой границей родительского контейнера или экрана.

[Y](properties-size-location.md) — расстояние между верхним краями элемента управления и верхней границей родительского контейнера или экрана.

## <a name="examples"></a>Примеры

В этих примерах вам потребуется устройство с камерой. Чтобы протестировать приложение, используйте веб-камера, доступную в браузере. Или, сохранив приложение и загрузив его на устройство iOS или Android с помощью камеры.

### <a name="simple-display-of-a-captured-picture"></a>Простое отображение захваченного изображения

1. [Добавьте](../add-configure-controls.md) элемент управления " **Камера** ".

1. Разрешите приложению использовать камеру устройства при появлении соответствующего запроса.

1. Добавление элемента управления **Image** .

1. Задайте для свойства **Image** элемента управления **Image** следующую формулу:

    ```powerapps-dot
    Camera1.Photo
    ```

    > [!NOTE]
    > Замените имя элемента управления Camera *Camera1* соответствующим образом.

1. Нажмите клавишу F5, чтобы просмотреть приложение.

1. Сделайте снимок, выбрав или коснувшись элемента управления камерой.  Результат должен отображаться в элементе управления изображения.

1. Нажмите клавишу F5, а затем сделайте снимок, выбрав или коснувшись **микамера**.

1. Добавление [вертикального](control-gallery.md) элемента управления "Коллекция". А затем измените размер элемента управления [изображения](control-image.md) , его шаблона и элемента управления " **коллекция изображений** " для размещения на экране.

## <a name="accessibility-guidelines"></a>Руководство по настройке специальных возможностей

Элемент управления "Камера" отображает канал камеры, а также функции, которые принимают изображение. Таким образом, существуют аналогичные рекомендации по специальным возможностям, такие как с кнопками.

### <a name="video-alternatives"></a>Варианты для видео

Попробуйте добавить альтернативную форму ввода данных для пользователей с ослабленным зрением. Например, [Добавьте рисунок](control-add-picture.md) , чтобы разрешить пользователям отправлять изображение с устройства.

### <a name="color-contrast"></a>Контрастность

Между [фокуседбордерколор](properties-color-border.md) и внешним цветом должен быть достаточной цветовой контраст.

### <a name="screen-reader-support"></a>Поддержка средства чтения с экрана

[Акцессиблелабел](properties-accessibility.md) должен присутствовать.

### <a name="keyboard-support"></a>Поддержка клавиатуры

- Значение [TabIndex](properties-accessibility.md) должно быть больше или равно нулю, чтобы пользователи с клавиатуры могли переходить к нему.

- Индикаторы фокуса должны быть хорошо видны. Используйте [фокуседбордерколор](properties-color-border.md) и [фокуседбордерсиккнесс](properties-color-border.md) для обновления видимости индикаторов фокуса.
