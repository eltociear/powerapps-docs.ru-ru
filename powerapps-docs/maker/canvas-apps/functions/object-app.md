# <a name="app-object-in-powerapps"></a>Объект приложения в PowerApps

Сведения о текущих и контроля над поведением приложения.

## <a name="description"></a>Описание

Элемент управления, такие как **приложения** объект предоставляет свойства, которые задают, какой экран отображается, и которые предлагают пользователю сохранить изменения, чтобы они не удаляются. Каждое приложение содержит **приложения** объекта.

Можно записать формулы для некоторых свойств **приложения** объекта. В верхней части **представление в виде дерева** области выберите **приложения** объекта от вашего имени будет любой другой элемент управления или экрана. Просмотр и изменение одного из свойств объекта, выбрав его в раскрывающемся списке слева от строки формул.

> [!div class="mx-imgBorder"]
> ![Объект приложения в области представления дерева](media/object-app/appobject.png)

## <a name="activescreen-property"></a>Свойство ActiveScreen

**ActiveScreen** свойство идентифицирует экрана, на которой отображаются.

Это свойство возвращает объект экрана, который можно использовать для ссылки на свойства экрана или сравнивать на другой экран, чтобы определить, какой экран отображается. Можно также использовать выражение **App.ActiveScreen.Name** для извлечения имени части экрана, на которой отображаются.

Используйте **[обратно](function-navigate.md)** или **[Navigate](function-navigate.md)** сменяющей экран, который отображается.

## <a name="onstart-property"></a>Свойство OnStart

**OnStart** свойство выполняется, когда пользователь запускает приложение. Создатели приложений это свойство часто используется для выполнения следующих задач:

- Получать и кэшировать данные в коллекции с помощью **[собирать](function-clear-collect-clearcollect.md)** функции.
- Настройка глобальных переменных с помощью **[задать](function-set.md)** функции.
- Перейдите на начальный экран, с помощью **[Navigate](function-navigate.md)** функции.

Эта формула вычисляется перед появлением первого экрана. Экран не загружается, поэтому нельзя задать переменные контекста с **[UpdateContext](function-updatecontext.md)** функции. Тем не менее, можно передавать переменные контекста с **Navigate** функции.

После изменения **OnStart** свойство, проверьте его, при наведении **приложения** объекта в **представление в виде дерева** панель, выбрав кнопку с многоточием (...), отображаемый, а затем выбрав **Запуска OnStart**. В отличие от, когда приложение загружается в первый раз, существующие коллекции и переменные будут уже заданы. Для начала пустые коллекции, используйте **[ClearCollect](function-clear-collect-clearcollect.md)** вместо функции **собирать** функции.

> [!div class="mx-imgBorder"]
> ![Приложение item контекстное меню для запуска OnStart](media/object-app/appobject-runonstart.png)

## <a name="confirmexit-properties"></a>Свойства ConfirmExit

Никто не хочет отменить несохраненные изменения. Используйте **ConfirmExit** и **ConfirmExitMessage** свойства, чтобы предупредить пользователя, прежде чем их закрыть приложение.

> [!NOTE]
> **ConfirmExit** не работает в приложениях, которые внедряются в, например, Power BI и SharePoint.

> [!NOTE]
> В настоящее время, эти свойства могут ссылаться на элементы управления на первом экране только если **нагрузки Delayed** (который по умолчанию для новых приложений он разрешен) включена функция предварительной версии. Если были сделаны ссылки, PowerApps Studio не отображает ошибку, но не открывается опубликованное приложение в PowerApps Mobile или браузер. Мы активно работаем над устранением этого ограничения. В то же время вы можете отключить **нагрузки Delayed** в **файл** > **параметры приложения** > **Дополнительные параметры**(в разделе **функции предварительной версии**).

### <a name="confirmexit"></a>ConfirmExit

**ConfirmExit** — это логическое свойство, если *true*, откроется диалоговое окно подтверждения перед закрытием приложения. По умолчанию это свойство имеет *false*, и диалоговое окно не появляется.

Это свойство используется для отображения диалогового окна подтверждения, если пользователь внесены изменения, но не сохранены. Использовать формулу, которая можно проверить переменные и свойства элемента управления (например, **Unsaved** свойство [ **форма редактирования** ](../controls/control-form-detail.md) управления).

Диалоговое окно подтверждения появится в любой ситуации, где данные могут быть потеряны, как в следующих примерах:

- Под управлением [ **выхода** ](function-exit.md) функции.
- Если приложение выполняется в браузере:
  - Закрытие браузера или вкладку браузера, в котором выполняется приложение.
  - Выбор кнопки "Назад" браузера.
- Если приложение выполняется в PowerApps Mobile (iOS или Android):
  - Под управлением [ **запуска** ](function-param.md) функции.<br>**Запуска** функция не приводит к возникновению диалоговое окно в браузере, так как другой вкладке откроется таким образом, чтобы данные не будут потеряны.
  - Проведение пальцем по экрану, переключиться на другое приложение в PowerApps Mobile.
  - Нажав кнопку "Назад" на устройстве Android.

Точное представление диалоговое окно подтверждения может отличаться на устройствах и версиях PowerApps.

Диалоговое окно подтверждения не отображается в PowerApps Studio.

### <a name="confirmexitmessage"></a>ConfirmExitMessage

По умолчанию диалоговое окно подтверждения показано универсальное сообщение, такие как **«Может имеются несохраненные изменения.»** на языке пользователя.

Используйте **ConfirmExitMessage** для предоставления пользовательского сообщения в диалоговом окне подтверждения. Если это свойство имеет *пустой*, используется значение по умолчанию. Пользовательские сообщения, усекаются при необходимости в диалоговом окне подтверждения, поэтому сообщение на несколько строк не более.

В браузере диалоговое окно подтверждения может отображается универсальное сообщение из браузера.

### <a name="example"></a>Пример

1. Создать приложение, которое содержит два элемента управления формы, **AccountForm** и **ContactForm**.

1. Задайте **приложения** объекта **ConfirmExit** свойство следующее выражение:

    ```powerapps-comma
    AccountForm.Unsaved Or ContactForm.Unsaved
    ```

    Это диалоговое окно отображается, если пользователь изменяет данные в любой форме, а затем предпринимается попытка закрыть приложение без сохранения этих изменений.

    > [!div class="mx-imgBorder"]
    > ![Диалоговое окно подтверждения, универсальные](media/object-app/confirm-native.png)

1. Задайте **приложения** объекта **ConfirmExitMessage** следующую формулу:

    ```powerapps-comma
    If( AccountsForm.Unsaved;
        "Accounts form has unsaved changes.";
        "Contacts form has unsaved changes."
    )
    ```

    Это диалоговое окно отображается, если пользователь изменяет данные в форме учетной записи, а затем предпринимается попытка закрыть приложение без сохранения этих изменений.

    > [!div class="mx-imgBorder"]
    > ![Диалоговое окно подтверждения формами](media/object-app/confirm-native-custom.png)
