# <a name="back-and-navigate-functions-in-powerapps"></a>Функции Back и Navigate в PowerApps

Эти функции изменяют отображаемый экран.

## <a name="overview"></a>Обзор

Большинство приложений содержат несколько экранов.  Чтобы изменить отображаемый экран, используйте функции **Back** и **Navigate**. Например, если вы хотите, чтобы при нажатии этой кнопки отображался другой экран, можно задать в качестве значения свойства **[OnSelect](../controls/properties-core.md)** для кнопки формулу с функцией **Navigate**. Чтобы управлять переключениями одного экрана на другой, в этой формуле можно указать визуальный переход, например **Fade**.  

Функции **Back** и **Navigate** изменяют только отображаемый экран. Экраны, которые в настоящее время не отображаются, продолжают работать в фоновом режиме. Вы можете создавать формулы, ссылающиеся на свойства элементов управления на других экранах. К примеру пользователь может изменить значение ползунка на одном экране, перейдите на другой экран, который использует это значение в формуле и выяснить, как изменения повлияют на работу нового экрана. Пользователь может затем вернитесь к исходному экрану и убедитесь, что ползунка было сохранено его значение.

[Переменные контекста](../working-with-variables.md#use-a-context-variable) также сохраняются во время перехода между экранами. Чтобы задать одну или несколько переменных контекста для экрана, который будет отображаться формулой, используйте функцию **Navigate**. Это единственный способ задать переменную контекста вне экрана. Этот подход можно использовать для передачи параметров на экран. Если вы используете другое средство программирования, этот подход аналогичен переносу параметров в процедуры.

Ее можно использовать любой из этих функций только в [формуле поведения](../working-with-formulas-in-depth.md).

## <a name="navigate"></a>Navigate

Для первого аргумента укажите имя отображаемого экрана.  

 Для второго аргумента укажите способ изменения старого экрана на новый.

| Аргумент перехода | Описание | Демонстрация |
| --- | --- | --- |
| **ScreenTransition.Cover** |Новый экран слайды в представление, перемещение справа налево, на текущий экран. | ![анимации титульной перехода экрана](media/function-navigate/cover.gif) |
| **ScreenTransition.CoverRight** |Новый экран слайды в представление, перемещение слева направо, чтобы охватить текущего экрана. | ![анимации правой титульной перехода экрана](media/function-navigate/coverright.gif) |
| **ScreenTransition.Fade** |Текущий экран затухание для того, чтобы показать новый экран. | ![анимация исчезания перехода экрана](media/function-navigate/fade.gif) |
| **ScreenTransition.None** (по умолчанию) |Новый экран быстро заменяет текущий экран. | ![Переход экрана none анимации](media/function-navigate/none.gif) |
| **ScreenTransition.UnCover** | Текущий экран уходит из поля зрения, перемещение справа налево, для анализа данных в новый экран. | ![Переход экрана Открывание анимации](media/function-navigate/uncover.gif) |
| **ScreenTransition.UnCoverRight** | Текущий экран слайды исчезновение, перемещение слева направо, для анализа данных в новый экран. | ![Переход экрана Открывание правой анимации](media/function-navigate/uncoverright.gif) |

Вы можете использовать функцию **Navigate**, чтобы создать или обновить переменные контекста нового экрана. В виде необязательного третьего аргумента передайте [запись](../working-with-tables.md#records), содержащую имя переменной контекста как имя [столбца](../working-with-tables.md#columns), и новое значение для переменной контекста.  Эта запись аналогична записи, которую вы использовали с функцией **[UpdateContext](function-updatecontext.md)**.

Для старого экрана задайте свойство **[OnHidden](../controls/control-screen.md)**, для нового — **[OnVisible](../controls/control-screen.md)** или задайте оба свойства, чтобы внести дополнительные изменения в процедуру перехода. При этом будет обновлено свойство **App.ActiveScreen**.

**Перейдите** обычно возвращает **true** , но возвратит **false** Если возникает ошибка.

## <a name="back"></a>Back

**Обратно** функция возвращает на экран, который был показан последним.

Для каждого **Navigate** вызова, приложение отслеживает на экране, которое было открыто и перехода. Можно использовать последовательные **обратно** вызовы вплоть до экран, который отображается, когда пользователь возвращаться в работу приложения.

Когда **обратно** выполняется функция, обратный переход используется по умолчанию. Например, если экран отображался через **CoverRight** перехода, **обратно** использует **UnCover** (который находится слева) для возврата.  **Исчезание** и **None** являются собственные обратными величинами. Передайте дополнительного аргумента **обратно** для определенного перехода.

**Обратно** обычно возвращает **true** , но возвращает **false** Если пользователь еще не перешли на другой экран с момента запуска приложения.

## <a name="syntax"></a>Синтаксис

**Обратно**([ *перехода* ])

* *Переход* — необязательный. Визуальный переход между текущим экраном и предыдущий экран. Ознакомьтесь со списком допустимых значений для этого аргумента ранее в этом разделе. По умолчанию переход, через который возвращает экрана обратна перехода, через который момента ее появления.

**Перейдите**( *экрана* [; *перехода* [; *UpdateContextRecord* ]])

* *Screen* — обязательный аргумент. Отображаемый экран.
* *Переход* — необязательный.  Визуальный переход, используемый при переходе между текущим экраном и следующим. Допустимые значения для этого аргумента см. в предыдущем разделе. Значение по умолчанию — **None**.
* *UpdateContextRecord* — необязательный аргумент.  Запись, содержащая имя как минимум одного столбца и значение каждого из них. Эта запись обновляет переменные контекста нового экрана таким же образом, как при передаче их функции **[UpdateContext](function-updatecontext.md)**.

| **Navigate( Details; ScreenTransition.Fade )** |Отображается экран **Details** с переходом типа **Fade**.  Значение переменной контекста не меняется. |Текущий экран исчезает, и отображается экран **Details**. |

| **Navigate( Details; ScreenTransition.Fade; {&nbsp;ID:&nbsp;12&nbsp;;&nbsp;Shade:&nbsp;Color.Red&nbsp;} )** |Отображается экран **Details** с переходом типа **Fade**. Значение переменной контекста **ID** изменяется на **12**, а значение переменной контекста **Shade** — на **Color.Red**. |Текущий экран исчезает, и отображается экран **Details**. Для переменной контекста **ID** на экране **Details** задается значение **12**, а для переменной контекста **Shade** — значение **Color.Red**. Если на экране **Details** задать для свойства **Fill** элемента управления значение **Shade**, этот элемент управления будет отображаться красным цветом. |
| **Back()** | Отображает предыдущий экран с переходом возвращаемого значения по умолчанию. | Отображает предыдущий экран через обратный переход перехода, через который отображается текущий экран. |
| **Назад (ScreenTransition.Cover)** |  Отображает предыдущий экран с **охватывают** перехода. | Отображает предыдущий экран через **охватывают** переход, независимо от того, переход, через который отображается текущий экран. |

### <a name="step-by-step"></a>Пошаговое руководство

1. Создайте пустое приложение.

1. Добавление второго экрана к нему.

    Приложение содержит два пустых экрана: **Screen1** и **Screen2**.

1. Задайте **заполнения** свойство **Screen2** значение `Gray`.

1. На **Screen2**, добавьте кнопку и задайте его **[OnSelect](../controls/properties-core.md)** следующую формулу:

    ```powerapps-comma
    Navigate( Screen1; ScreenTransition.Cover )
    ```

1. Удерживая нажатой **Alt** ключ, нажмите кнопку.

    **Screen1** отображается на белом фоне через переход, который охватывает слева.

1. На **Screen1**, добавьте кнопку и задайте его **OnSelect** следующую формулу:

    ```powerapps-comma
    Back()
    ```

1. Удерживая нажатой **Alt** ключ, нажмите кнопку.

    Второй экран отображается на сером фоне через переход, который получает данные справа (обратное **охватывают**).

1. Нажмите кнопку на каждом экране несколько раз, чтобы переходить вперед и назад.
