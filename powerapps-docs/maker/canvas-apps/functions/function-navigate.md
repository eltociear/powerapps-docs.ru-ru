# <a name="back-and-navigate-functions-in-power-apps"></a>Назад и Навигация по функциям в Power Apps

Эти функции изменяют отображаемый экран.

## <a name="overview"></a>Обзор

Большинство приложений содержат несколько экранов.  Чтобы изменить отображаемый экран, используйте функции **Back** и **Navigate**. Например, если вы хотите, чтобы при нажатии этой кнопки отображался другой экран, можно задать в качестве значения свойства **[OnSelect](../controls/properties-core.md)** для кнопки формулу с функцией **Navigate**. Чтобы управлять переключениями одного экрана на другой, в этой формуле можно указать визуальный переход, например **Fade**.  

Функции **Back** и **Navigate** изменяют только отображаемый экран. Экраны, которые в настоящее время не отображаются, продолжают работать в фоновом режиме. Можно создавать формулы, которые ссылаются на свойства элементов управления на других экранах. Например, пользователь может изменить значение ползунка на одном экране, перейти к другому экрану, использующему это значение в формуле, и выяснить, как это влияет на то, что происходит на новом экране. Затем пользователь может вернуться к исходному экрану и подтвердить, что ползунок сохранил свое значение.

[Переменные контекста](../working-with-variables.md#use-a-context-variable) также сохраняются во время перехода между экранами. Чтобы задать одну или несколько переменных контекста для экрана, который будет отображаться формулой, используйте функцию **Navigate**. Это единственный способ задать переменную контекста вне экрана. Этот подход можно использовать для передачи параметров на экран. Если вы используете другое средство программирования, этот подход аналогичен переносу параметров в процедуры.

Можно использовать любую функцию только в [формуле поведения](../working-with-formulas-in-depth.md).

## <a name="navigate"></a>Navigate

Для первого аргумента укажите имя отображаемого экрана.  

 Для второго аргумента укажите способ изменения старого экрана на новый.

| Аргумент перехода | Description | Примера |
| --- | --- | --- |
| **ScreenTransition.Cover** |Новый экран в виде слайдов, перемещение справа налево для покрытия текущего экрана. | ![анимация экрана перехода на экран](media/function-navigate/cover.gif) |
| **Скринтранситион. Коверригхт** |Новый экран, продвигаясь влево и вправо для покрытия текущего экрана. | ![Переход на экран, посвященный правильной анимации](media/function-navigate/coverright.gif) |
| **ScreenTransition.Fade** |Текущий экран плавно исчезает, чтобы отобразить новый экран. | ![анимация затухания перехода на экран](media/function-navigate/fade.gif) |
| **Скринтранситион. None** (по умолчанию) |Новый экран быстро заменяет текущий экран. | ![анимация экрана "нет анимации"](media/function-navigate/none.gif) |
| **ScreenTransition.UnCover** | Текущий экран слайды не отображаются, перемещение справа налево для обнаружения нового экрана. | ![анимация при переходе на экран](media/function-navigate/uncover.gif) |
| **Скринтранситион. Унковерригхт** | Текущий экран слайды не отображаются, перемещение слева направо для обнаружения нового экрана. | ![Переход на экран при выявлении правой анимации](media/function-navigate/uncoverright.gif) |

Вы можете использовать функцию **Navigate**, чтобы создать или обновить переменные контекста нового экрана. В виде необязательного третьего аргумента передайте [запись](../working-with-tables.md#records), содержащую имя переменной контекста как имя [столбца](../working-with-tables.md#columns), и новое значение для переменной контекста.  Эта запись аналогична записи, которую вы использовали с функцией **[UpdateContext](function-updatecontext.md)** .

Для старого экрана задайте свойство **[OnHidden](../controls/control-screen.md)** , для нового — **[OnVisible](../controls/control-screen.md)** или задайте оба свойства, чтобы внести дополнительные изменения в процедуру перехода. При этом будет обновлено свойство **App.ActiveScreen**.

Функция **navigate** обычно возвращает **значение true** , но при обнаружении ошибки возвращает **значение false** .

## <a name="back"></a>Back

Функция **Back** возвращается к последнему отображенному экрану.

Для каждого вызова **навигации** приложение отслеживает появления экрана и переход. Можно использовать последовательные вызовы **,** чтобы вернуть весь экран, который отображался при запуске приложения пользователем.

При выполнении функции **Back** по умолчанию используется обратный переход. Например, если экран появлялся через переход **коверригхт** , то функция **Back** использует для возврата **Непокрытые** (слева).  **Выцветание** и **ничего** не являются собственными обратными. Передайте необязательный аргумент **, чтобы принудительно выполнить конкретный** переход.

**Обычно возвращает** **значение true** , но возвращает **значение false** , если пользователь не перешел на другой экран с момента запуска приложения.

## <a name="syntax"></a>Синтаксис

**Назад**([ *Переход* ])

* *Переход* — необязательный. Визуальный переход для использования между текущим и предыдущим экранами. См. список допустимых значений этого аргумента ранее в этом разделе. По умолчанию переход, по которому возвращается экран, является обратным по отношению к переходу, с помощью которого он отображался.

**Навигация**( *экран* [, *Переход* [, *упдатеконтекстрекорд* ]])

* *Screen* — обязательный аргумент. Отображаемый экран.
* *Переход* — необязательный.  Визуальный переход, используемый при переходе между текущим экраном и следующим. Допустимые значения для этого аргумента см. в предыдущем разделе. Значение по умолчанию — **None**.
* *UpdateContextRecord* — необязательный аргумент.  Запись, содержащая имя как минимум одного столбца и значение каждого из них. Эта запись обновляет переменные контекста нового экрана таким же образом, как при передаче их функции **[UpdateContext](function-updatecontext.md)** .

## <a name="examples"></a>Примеры

| Формула | Description | Возвращаемый результат |
| --- | --- | --- |
| **Навигация (сведения)** |Отображается экран **Details** без эффекта перехода или изменения значения переменной контекста. |Экран **Details** отображается мгновенно. |
| **Navigate( Details, ScreenTransition.Fade )** |Отображается экран **Details** с переходом типа **Fade**.  Значение переменной контекста не меняется. |Текущий экран исчезает, и отображается экран **Details**. |
| **Navigate( Details, ScreenTransition.Fade, {&nbsp;ID:&nbsp;12&nbsp;} )** |Отображается экран **Details** с переходом типа **Fade**, и значение переменной контекста **ID** меняется на **12**. |Текущий экран исчезает, и отображается экран **Details**, а для переменной контекста **ID** задается значение **12**. |
| **Navigate( Details, ScreenTransition.Fade, {&nbsp;ID:&nbsp;12&nbsp;,&nbsp;Shade:&nbsp;Color.Red&nbsp;} )** |Отображается экран **Details** с переходом типа **Fade**. Значение переменной контекста **ID** изменяется на **12**, а значение переменной контекста **Shade** — на **Color.Red**. |Текущий экран исчезает, и отображается экран **Details**. Для переменной контекста **ID** на экране **Details** задается значение **12**, а для переменной контекста **Shade** — значение **Color.Red**. Если на экране **Details** задать для свойства **Fill** элемента управления значение **Shade**, этот элемент управления будет отображаться красным цветом. |
| **Back()** | Отображает предыдущий экран с возвращаемым переходом по умолчанию. | Отображает предыдущий экран с обратным переходом перехода, через который появлялся текущий экран. |
| **Назад (Скринтранситион. покрытие)** |  Отображает предыдущий экран с переходом на **обложку** . | Отображает предыдущий экран с помощью перехода на **обложку** , независимо от перехода, в котором появился текущий экран. |

### <a name="step-by-step"></a>Пошаговое руководство

1. Создайте пустое приложение.

1. Добавьте в него второй экран.

    Приложение содержит два пустых экрана: **Screen1** и **Screen2**.

1. Задайте для свойства **Fill** объекта **Screen2** значение `Gray`.

1. В **Screen2**добавьте кнопку и задайте для ее свойства **[OnSelect](../controls/properties-core.md)** значение этой формулы:

    ```powerapps-dot
    Navigate( Screen1, ScreenTransition.Cover )
    ```

1. Удерживая нажатой клавишу **ALT** , нажмите кнопку.

    **Screen1** отображается с белым фоном с помощью перехода, охватывающего левую.

1. В **Screen1**добавьте кнопку и задайте для ее свойства **OnSelect** значение этой формулы:

    ```powerapps-dot
    Back()
    ```

1. Удерживая нажатой клавишу **ALT** , нажмите кнопку.

    Второй экран отображается с серым фоном с помощью перехода, которое обнаруживается справа (обратная **Обложка**).

1. Несколько раз нажмите кнопку на каждом экране, чтобы вернуться вперед и назад.
