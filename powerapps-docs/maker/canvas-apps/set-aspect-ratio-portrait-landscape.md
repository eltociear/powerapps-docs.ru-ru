# <a name="change-screen-size-and-orientation-of-a-canvas-app-in-powerapps"></a>Изменение размера и ориентации экрана в приложении на основе холста в PowerApps
Настройте приложение на основе холста, изменив размер и ориентацию его экрана.

## <a name="prerequisites"></a>Технические условия

Создание приложения или откройте существующее для редактирования, затем установите **параметры приложения** на **файл** меню.

## <a name="change-screen-size-and-orientation"></a>Изменение размера и ориентации экрана
1. В разделе **App settings** (Параметры приложения) выберите пункт **Screen size + orientation** (Размер и ориентация экрана).

    ![Параметр изменения размера и ориентации экрана приложения](./media/set-aspect-ratio-portrait-landscape/size-orientation.png)

1. В **ориентации** выберите **книжной** или **Альбомная**.

1. (Только для планшетных приложений) В разделе **пропорции**, выполните одно из следующих действий:

    - Выберите соотношение сторон целевого устройства для этого приложения.
    - Выберите **Custom** можно задать собственный размер и укажите ширины в диапазоне от 50 3840 и высотой между 50 2160.

    ![Изменение пропорций для планшетного приложения](./media/set-aspect-ratio-portrait-landscape/aspect-tablet.png)
    
1. В разделе **масштабирования в соответствии с**, либо указать **на** или **Off**.

    Этот параметр включен по умолчанию, чтобы изменить размеры экранов приложения для его вписывается в доступное пространство на устройстве. Если этот параметр включен, приложение **ширины** свойство совпадения его **DesignWidth**и приложения **высота** соответствует его **DesignHeight**.

    Если вы отключите этот параметр, приложение изменяется так, чтобы соотношение сторон устройства, на котором она запущена и будет занимать все доступное пространство. Приложение не масштабируется, и, таким образом, экраны могут отображаться Дополнительные сведения.

    Если этот параметр отключен, **зафиксировать пропорции** автоматически отключается и отключена. Кроме того **ширины** всех экранов свойству `Max(App.Width; App.DesignWidth)`и их **высота** свойству `Max(App.Height; App.DesignHeight)` таким образом, чтобы они отслеживают размеры окна, в котором выполняется приложение. Благодаря этому изменению можно создать приложения, реагирующие на различные устройства и размеры окна. Дополнительные сведения: [Создать гибкий макет](create-responsive-layout.md)

1. Для параметра **Lock aspect ratio** (Заблокировать пропорции) укажите значение **On** (Вкл.) или **Off** (Выкл.).

    Если этот параметр включен, приложение сохраняет ориентацию экрана и пропорции, заданные на шагах 2 и 3, независимо от того, устройство. Например приложение для телефона, на котором выполняется в веб-браузер сохраняет отношение для телефона, показывающий темно-панели на каждой стороне вместо заполнения окна.

    Если этот параметр отключен, приложение изменяется так, чтобы соотношение сторон устройства, на котором он выполняется (и искажение пользовательского интерфейса, при необходимости).

1. Для параметра **Lock orientation** (Заблокировать ориентацию) укажите значение **On** (Вкл.) или **Off** (Выкл.).

    Если заблокировать ориентацию приложения, приложение сохраняет указанную вами ориентацию. Если приложение выполняется на устройство, для которой с другой ориентацией экрана, приложение отображает неправильно и может показывать нежелательные результаты. Если разблокировать ориентацию, он корректирует ориентацию экрана устройства, на котором он выполняется.

    Можно также изменить ориентацию, включив **встраивание пользовательского интерфейса приложения Enable** в **Дополнительные параметры**. Эта функция верхнего левого выравнивает приложения, когда он внедрен и изменяет цвет фона размещения полотна к белому.

1. Нажмите кнопку **Apply** (Применить), чтобы сохранить изменения.

## <a name="next-step"></a>Дальнейшие действия
В меню **Файл** выберите пункт **Сохранить**, чтобы повторно опубликовать приложение с новыми параметрами.
