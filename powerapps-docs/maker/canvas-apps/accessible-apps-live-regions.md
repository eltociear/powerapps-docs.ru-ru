# <a name="announce-dynamic-changes-with-live-regions-for-canvas-apps"></a>Сообщаем о динамических изменениях, с помощью динамических областей для приложения на основе холста

Динамические изменения представлять трудности с недостатками зрения. Пользователям, обращающимся приложения через средство чтения с экрана ориентированы на одну часть приложения. Если изменение происходит в другом месте, эти пользователи не будут учитывать его.

Эту проблему можно решить, добавив динамических областей, которые следят за средства чтения с экрана. При изменении содержимого динамической области, средство чтения с экрана будут сообщать это изменение.

Базовый механизм для динамических областей являются [aria-live регионах](https://www.w3.org/TR/wai-aria-1.1/#dfn-live-region), поэтому действуют те же правила.

## <a name="example-uses-of-live-regions"></a>Примеры использования динамических областей

Динамические области позволяет уведомлять пользователей о таких событиях:

* В форме, возникает ошибка проверки.
* Действие по щелчку кнопки успешна. Например пользователь может выбрать кнопку, чтобы добавить элемент в коллекцию и динамической области удалось показать сообщение «Добавлен объект».
* Пользователь выбрал другую вкладку.
* Таймер фонового обновление канала новостей.

## <a name="create-and-configure-a-live-region"></a>Создание и настройка динамической области

Можно настроить только **[метка](controls/control-text-box.md)** управления в качестве динамической области. Его **Live** свойство определяет, какой тип динамической области.

* **Отключение**: Динамической области. Средства чтения с экрана не объявляют об изменениях.
* **Мягкое**: Чтения с экрана сообщать изменения после завершения кстати. Используйте это значение для некритические уведомления, не требующих немедленного вмешательства.
* **Assertive**: Средства чтения с экрана прерываний сами рады сообщить о изменения немедленно. Используйте это для критически важных уведомлений, которые требуют немедленного внимания.

При изменении текстового содержимого динамической области чтения с экрана будут сообщать все текстовое содержимое, а не только измененные части. Если значение **[текст](controls/properties-core.md)** свойству присваивается пустая строка **«»** , ничего не было объявлять средство чтения с экрана.

Чтобы повторять сообщение, очистите текстовое содержимое, задав значение **[текст](controls/properties-core.md)** пустую строку **«»** и затем задайте значение в сообщение.

## <a name="best-practices"></a>Советы и рекомендации

* Всегда значение **[Visible](controls/properties-core.md)** значение true. Некоторые средства чтения с экрана не определять динамических областей, которые исчезают и на экране.
* Старайтесь не изменять значение  **[Live](controls/properties-accessibility.md)** . Некоторые экрана, на котором читатели не определить, когда область не live станет динамической и наоборот.
* Поместите динамической области в логическое положение в приложении, даже если он не отображается. Убедитесь, что его содержимое разумные в контексте с элементами до и после него. Пользователям динамической области в любое время через регулярные переход с помощью чтения с экрана, не только в том случае, когда изменения вступают в силу.
