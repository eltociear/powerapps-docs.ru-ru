# <a name="what-is-an-on-premises-data-gateway"></a>Что такое локальный шлюз данных?

Локальный шлюз данных выступает в качестве моста для обеспечения быстрой и безопасной передачи данных между локальными данными (данными, которые не находятся в облаке) и несколькими облачными службами Майкрософт. К этим облачным службам относятся Power BI, Power Apps, Автоматизация питания, Azure Analysis Services и Azure Logic Apps. С помощью шлюза организации могут хранить базы данных и другие источники данных в своих локальных сетях, но при этом безопасно использовать локальные данные в облачных службах.

## <a name="types-of-gateways"></a>Типы шлюзов

Существует два разных типа шлюзов, каждый из которых используется для различных сценариев:

- **Локальный шлюз данных** позволяет нескольким пользователям подключаться к нескольким локальным источникам данных. Вы можете использовать локальный шлюз данных со всеми поддерживаемыми службами с одной установкой шлюза. Этот шлюз хорошо подходит для сложных сценариев, в которых несколько пользователей обращаются к нескольким источникам данных.

- **Локальный шлюз данных (персональный режим)** позволяет одному пользователю подключаться к источникам и не может использоваться совместно с другими пользователями. Локальный шлюз данных (персональный режим) можно использовать только с Power BI. Этот шлюз хорошо подходит для сценариев, где вы единственный пользователь, создающий отчеты, и вам не нужно предоставлять общий доступ к каким-либо источникам данных другим пользователям.

## <a name="use-a-gateway"></a>Использование шлюза

Для использования шлюза существует четыре основных шага.

1. [Скачайте и установите шлюз](/data-integration/gateway/service-gateway-install) на локальном компьютере.
2. [Настройте](/data-integration/gateway/service-gateway-app) шлюз на основе брандмауэра и других сетевых требований.
3. [Добавьте администраторов шлюзов](/data-integration/gateway/service-gateway-manage) , которые также могут управлять другими требованиями к сети и администрировать их.
4. [Устранение неполадок](/data-integration/gateway/service-gateway-tshoot) шлюза в случае возникновения ошибок.
