# <a name="what-is-power-apps"></a>Что такое Power Apps?

Power Apps — это набор приложений, служб, соединителей и платформ данных, обеспечивающий быструю среду разработки приложений для создания пользовательских приложений в соответствии с потребностями бизнеса. С помощью Power Apps можно быстро создавать пользовательские бизнес-приложения, которые подключаются к бизнес-данным, хранящимся *либо* в базовой платформе данных ([Common Data Service](/powerapps/maker/common-data-service/data-platform-intro)), *либо* в различных сетевых и локальных источниках данных (SharePoint, Excel, Office 365, Dynamics 365, SQL Server и т. д.). 

Приложения, созданные с помощью Power Apps, предоставляют широкие возможности бизнес-логики и рабочих процессов, позволяющие преобразовывать бизнес-процессы вручную в цифровые автоматизированные процессы. Кроме того, приложения, созданные с помощью приложений Power Apps, имеют реагирование на разработку и могут легко работать в браузере или на мобильных устройствах (телефоне или планшете). Power Apps "более демократичным" — это пользовательский процесс создания бизнес-приложений, позволяющий пользователям создавать полнофункциональные пользовательские бизнес-приложения без написания кода.

Power Apps также предоставляет расширяемую платформу, которая позволяет разработчикам программного обеспечения взаимодействовать с данными и метаданными, применять бизнес-логику, создавать пользовательские соединители и интегрировать их с внешними данными.

Дополнительные сведения см. в статье [канал Power Apps](https://www.youtube.com/channel/UCGfWR2ekfRFckLjev6eQYLg) на YouTube.

## <a name="power-apps-for-app-makerscreators"></a>Power Apps для руководителей и создателей приложений

С помощью Power Apps можно создавать приложения трех типов: **холст**, **управляемые моделями**и **портал**. Дополнительные сведения см. [в статье о создании приложений в Power Apps](maker/index.md).

Чтобы создать приложение, начните с [make.powerapps.com](https://make.powerapps.com).

- **Power Apps Studio** — это конструктор приложений, используемый для создания приложений Canvas. В конструкторе приложений создание приложений напоминает создание слайдов в Microsoft PowerPoint. Дополнительные сведения: [Создание приложения на основе данных](/powerapps/maker/canvas-apps/data-platform-create-app)  

- **Конструктор приложений** для приложений на основе модели позволяет определять карту сайта и добавлять компоненты для создания приложения на основе модели. Дополнительные сведения: [проектирование приложений, управляемых моделями, с помощью конструктора приложений](maker/model-driven-apps/design-custom-business-apps-using-app-designer.md)

## <a name="power-apps-for-app-users"></a>Приложения Power Apps для пользователей приложений

Вы можете запускать собственные приложения и приложения других разработчиков, к которым вам предоставили доступ, в браузере или на мобильном устройстве (на телефоне или планшете). Дополнительные сведения: [Поиск и запуск приложений](user/index.md)

## <a name="power-apps-for-admins"></a>Power Apps для администраторов

Администраторы Power Apps могут использовать:

- **Центр администрирования Power Apps** ([Admin.powerapps.com](https://admin.powerapps.com)) для создания сред, пользователей, ролей и политик защиты от потери данных и управления ими. 

- **Центр администрирования Power Platform Center** ([Admin.powerplatform.Microsoft.com](https://admin.powerplatform.microsoft.com)) для управления средами, получения рекомендаций для самостоятельного устранения проблем и поддержки Power Apps и Power автоматизиру, а затем просмотрите Common Data Service Analytics. 

Дополнительные сведения: [Администрирование Power Apps](/power-platform/admin/admin-guide)

## <a name="power-apps-for-developers"></a>Power Apps для разработчиков

Разработчики — это создатели приложений, которые могут создавать код для расширения возможностей создания и настройки бизнес-приложений. Разработчики могут использовать код для создания данных и метаданных, применять логику на стороне сервера с помощью функций Azure, подключаемых модулей и расширений рабочего процесса, применить логику на стороне клиента с помощью JavaScript, осуществлять интеграцию с внешними данными с помощью виртуальных сущностей и веб-перехватчиков, создавать пользовательские соединители и внедрять приложения в интерфейс веб-сайта для создания интегрированных решений. Дополнительные сведения: [Power Apps для разработчиков](/powerapps/#pivot=home&panel=developer)

## <a name="power-apps-and-dynamics-365"></a>Power Apps и Dynamics 365

Приложения Dynamics 365, такие как Dynamics 365 Sales, Dynamics 365 Customer Service, Dynamics 365 Marketing, также используют базовую платформу данных (Common Data Service), используемую приложениями Power Apps для хранения и защиты данных. Это позволяет создавать приложения с помощью Power Apps и Common Data Service непосредственно для основных бизнес-данных, уже используемых в Dynamics 365 без необходимости интеграции. Дополнительные сведения: [Dynamics 365 и Common Data Service](maker/common-data-service/data-platform-intro.md#dynamics-365-and-common-data-service)
