# <a name="what-is-power-apps"></a>Что такое Power Apps?

Power Apps — это набор приложений, служб и соединителей, а также платформа данных, которая предоставляет среду разработки для эффективного создания пользовательских приложений для бизнеса. С помощью Power Apps можно быстро создавать пользовательские бизнес-приложения, которые подключаются к бизнес-данным, хранящимся в *базовой платформе данных* ([Common Data Service](/powerapps/maker/common-data-service/data-platform-intro)) *или* в различных облачных и локальных источниках данных (SharePoint, Excel, Office 365, Dynamics 365, SQL Server и т. д.). 

Приложения, созданные с помощью Power Apps, предоставляют широкие возможности бизнес-логики и рабочих процессов для преобразования ручных бизнес-процессов в цифровые, автоматизированные процессы. Кроме того, приложения, созданные с помощью Power Apps, имеют адаптивный дизайн и могут выполняться в браузере или на мобильных устройствах (телефонах или планшетах). Платформа Power Apps упрощает разработку пользовательских бизнес-приложений, позволяя пользователям создавать многофункциональные бизнес-приложения без написания кода.

Система Power Apps также предоставляет расширяемую платформу, которая позволяет профессиональным разработчикам программно взаимодействовать с данными и метаданными, применять бизнес-логику, создавать настраиваемые соединители и осуществлять интеграцию с внешними данными.

Дополнительные сведения см. на [канале, посвященном Power Apps](https://www.youtube.com/channel/UCGfWR2ekfRFckLjev6eQYLg), на YouTube.

## <a name="power-apps-for-app-makerscreators"></a>Power Apps для разработчиков и создателей приложений

Power Apps позволяет создавать приложения трех типов: **холст**, **на основе модели** и **портал**. Дополнительные сведения: [Общие сведения о создании приложений в Power Apps](maker/index.md).

Создание приложения начинается на странице [make.powerapps.com](https://make.powerapps.com).

- **Power Apps Studio** — это конструктор приложений, используемый для создания приложений на основе холста. В конструкторе приложений создание приложений напоминает создание слайдов в Microsoft PowerPoint. Дополнительные сведения: [Создание приложения на основе данных](/powerapps/maker/canvas-apps/data-platform-create-app)  

- **Конструктор приложений** для приложений на основе модели позволяет определять карту сайта и добавлять компоненты для создания приложения на основе модели. Дополнительные сведения: [Проектирование приложений на основе моделей с помощью конструктора приложений](maker/model-driven-apps/design-custom-business-apps-using-app-designer.md)

## <a name="power-apps-for-app-users"></a>Power Apps для пользователей приложения

Вы можете запускать собственные приложения и приложения других разработчиков, к которым вам предоставили доступ, в браузере или на мобильном устройстве (на телефоне или планшете). Дополнительные сведения: [Поиск и запуск приложений](user/index.md)

## <a name="power-apps-for-admins"></a>Power Apps для администраторов

Администраторы Power Apps могут использовать следующее.

- **Центр администрирования Power Apps** ([admin.powerapps.com](https://admin.powerapps.com)) для создания сред, пользователей, ролей и политик защиты от потери данных и управления ими. 

- **Центр администрирования Power Platform** ([admin.powerplatform.microsoft.com](https://admin.powerplatform.microsoft.com)) для управления средами, оперативного получения рекомендаций по самостоятельному устранению неполадок и поддержки по приложениям Power Apps и Power Automate, а также для просмотра аналитики Common Data Service. 

Дополнительные сведения: [Администрирование Power Apps](/power-platform/admin/admin-guide)

## <a name="power-apps-for-developers"></a>Power Apps для разработчиков

Разработчики — это создатели приложений, которые могут создавать код для расширения возможностей создания и настройки бизнес-приложений. Разработчики могут использовать код для создания данных и метаданных, применять логику на стороне сервера с помощью функций Azure, подключаемых модулей и расширений рабочего процесса, применить логику на стороне клиента с помощью JavaScript, осуществлять интеграцию с внешними данными с помощью виртуальных сущностей и веб-перехватчиков, создавать пользовательские соединители и внедрять приложения в интерфейс веб-сайта для создания интегрированных решений. Дополнительные сведения: [Power Apps для разработчиков](/powerapps/#pivot=home&panel=developer)

## <a name="power-apps-and-dynamics-365"></a>Power Apps и Dynamics 365

Такие приложения Dynamics 365, как Dynamics 365 Sales, Dynamics 365 Customer Service, Dynamics 365 Marketing, также работают с базовой платформой данных (Common Data Service), используемой Power Apps для хранения и защиты данных. С помощью Power Apps и Common Data Service можно создавать приложения непосредственно на основе базовых бизнес-данных, которые уже используются в Dynamics 365. При этом не нужно выполнять интеграцию. Дополнительные сведения: [Использование Dynamics 365 и Common Data Service](maker/common-data-service/data-platform-intro.md#dynamics-365-and-common-data-service)
