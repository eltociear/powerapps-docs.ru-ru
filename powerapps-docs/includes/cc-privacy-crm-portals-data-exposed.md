Если включить возможности портала для [!INCLUDE[pn_microsoftcrm](pn-microsoftcrm.md)], доступ к данным [!INCLUDE[pn_dynamics_crm](pn-dynamics-crm.md)], таким как имя клиента, название продукта, номер обращения и любые настраиваемые данные сущности, можно предоставлять через доступный извне портал [!INCLUDE[pn_dynamics_crm](pn-dynamics-crm.md)]. Все данные, доступ к которым предоставляется через портал, хранятся в памяти в веб-приложениях Microsoft [!INCLUDE[pn_azure_shortest](pn-azure-shortest.md)] для кэширования, а также в виде файлов на локальном жестком диске для обеспечения возможности поиска на портале.

Администратор клиента включает порталы [!INCLUDE[pn_dynamics_crm](pn-dynamics-crm.md)], настраивая их с помощью [!INCLUDE[pn_dyn_365_admin_center](../includes/pn-dyn-365-admin-center.md)]; при этом в выбранный экземпляр [!INCLUDE[pn_dynamics_crm](pn-dynamics-crm.md)] также устанавливается пакет (с решениями и данными). Администратор клиента или пользователь [!INCLUDE[pn_dynamics_crm](pn-dynamics-crm.md)], настроенный как администратор портала, может затем указать данные, к которым будет предоставлен доступ через портал. Если в будущем потребуется отключить возможности портала, администратор клиента может отменить подписку на надстройку портала в [!INCLUDE[pn_Office_365](pn-office-365.md)].

Для реализации возможностей портала необходимы следующие важные компоненты и службы [!INCLUDE[pn_azure_shortest](pn-azure-shortest.md)]:
- [Веб-приложения Azure](https://azure.microsoft.com/services/app-service/web/): веб-приложения [!INCLUDE[pn_azure_shortest](pn-azure-shortest.md)] используются для размещения портала в [!INCLUDE[pn_azure_shortest](pn-azure-shortest.md)].
- [Диспетчер трафика Azure](https://azure.microsoft.com/services/traffic-manager/): диспетчер трафика [!INCLUDE[pn_azure_shortest](pn-azure-shortest.md)] используется для обеспечения высокой доступности службы путем направления пользователя на работающий экземпляр веб-приложений. 
- [Служебная шина Azure](https://azure.microsoft.com/services/service-bus/): служебная шина [!INCLUDE[pn_azure_service_bus](pn-azure-service-bus.md)] (темы/подписки) используется для обработки недействительности кэша порталов. [!INCLUDE[pn_azure_service_bus](pn-azure-service-bus.md)] временно хранит сообщения, создаваемые, когда какая-либо связанная с порталом запись изменяется в [!INCLUDE[pn_dynamics_crm](pn-dynamics-crm.md)]; эти сообщения передаются в веб-приложения для обработки недействительности кэша. 
- [Azure Key Vault](https://azure.microsoft.com/services/key-vault/): все службы хранят данные о конфигурации в службе [!INCLUDE[pn_azure_key_vault](pn_azure_key_vault.md)].
- [Служба хранилища Azure](https://azure.microsoft.com/services/storage/): в хранилище [!INCLUDE[pn_azure_shortest](pn-azure-shortest.md)] хранятся данные, относящиеся к организации, клиенту и порталу.
- [Azure Active Directory](https://azure.microsoft.com/services/active-directory/): все веб-службы используют [!INCLUDE[pn_azure_active_directory](pn-azure-active-directory.md)] для проверки подлинности.
