> [!IMPORTANT]
> Во многих странах и регионах (кроме ЕС) требуют получить согласие, прежде чем устанавливать файл cookie на компьютер пользователя. Обязанностью вашей организации является осознание и соответствие всем соответствующим законам и правилам на рынках, на которых вы работаете, включая согласие на установку файлов cookie. Дополнительные сведения о нормах ЕС см. в [ec.europa.eu/ipg/basics/legal/cookies/](http://ec.europa.eu/ipg/basics/legal/cookies/). [!INCLUDE[pn-marketing-business-app-module-name](../includes/pn-marketing-business-app-module-name.md)] использует файлы cookie одним из следующих способов.
> - Маркетинговые страницы, созданные и размещенные Dynamics 365 for Marketing, используют файлы cookie для включения функции предварительного заполнения для известных контактов и для регистрации, когда любой контакт открывает маркетинговую страницу. Маркетинговые страницы и шаблоны страниц Dynamics 365 не содержат функцию согласия файлов cookie, поэтому его необходимо добавить. Один из способов сделать это — добавить код JavaScript для раздела `<head>` страницы с помощью редактора конструктора страниц HTML. Можно разработать собственное решение JavaScript или использовать одно из уже доступных бесплатных или лицензированных. Также можно использовать встроенное заявление о конфиденциальности для маркетинговых страниц, чтобы объявить об использовании файлов cookie. Однако этого может быть недостаточно во всех юрисдикциях, поскольку это не остановит установку cookie &mdash;, а просто информирует посетителей о том, что эти файлы используются (дополнительную информацию об этой функции см. в разделе [Настройка маркетинговых страниц](../marketing/marketing-settings.md#config-mkt-pages)). [!INCLUDE[proc-more-information](../includes/proc-more-information.md)] [Создание и развертывание маркетинговых страниц](../marketing/create-deploy-marketing-pages.md)
> - Компоненты _веб-сайтов_ используют файлы cookie для регистрации посещений на любой веб-странице, где вы размещаете скрипт отслеживание. Файлы cookie используются совместно с компонентами маркетинговых страниц для определения контактов с совпадающими идентификаторами посетителя для отправки целевой страницы. Если веб-сайт уже использует файлы cookie и включает функцию согласия, он, вероятно, охватывает сценарий веб-сайта Dynamics 365 for Marketing. Однако, если у вас нет согласия cookie, встроенного в сайт, нужно добавить его перед добавлением сценария веб-сайта на страницы, которые продаются жителям соответствующих стран или регионов. > - Портал событий использует файл cookie сеанса, чтобы позволить контактам заходить и регистрироваться для событий. [!INCLUDE[proc-more-information](../includes/proc-more-information.md)] [Настройка портала событий](../marketing/set-up-event-portal.md)
