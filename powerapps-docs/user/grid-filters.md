# <a name="use-grid-filters"></a>Использование фильтров сетки 

Сетки в едином интерфейсе были улучшены для увеличения объема данных, которые выводятся на экране. Теперь можно выбирать среди множества различных параметров фильтрации столбца. Тип данных в столбце определяет, какие параметры фильтра будут при этом доступны. Например, столбец **Полное имя** в сетке **Контакты** имеет параметры фильтрации, отличные от параметров столбца **Тип действия** в сетке **Действия**.


   > [!div class="mx-imgBorder"]
   > ![Фильтрация сетки](media/filter-options.png "Фильтрация сетки")
   

## <a name="grid-and-filter-navigation"></a>Навигация по сетке и фильтрам

При фильтрации данных в сетке главная страница сетки запоминает фильтр, порядок сортировки и состояние страницы при переходе на другую страницу и обратно. Это также работает при фильтрации данных по быстрому поиску, столбцам, номеру страницы и т. д. 

   > [!div class="mx-imgBorder"]
   > ![Переход назад к странице открывает ее в том же состоянии](media/grid-remember-state-on-back-navigate.gif "Переход назад к странице открывает ее в том же состоянии")

На панели переходов страницы используется первое поле сортировки. Если порядок сортировки не был изменен, панель переходов использует первичное поле.

   > [!div class="mx-imgBorder"]
   > ![Выберите фильтр на панели переходов](media/jumpbar-filter-on-sorted-column.gif "Выберите фильтр на панели переходов")
  
При выборе значка иерархии выполняется переход к иерархическому представлению.

   > [!div class="mx-imgBorder"]
   > ![Значок иерархии](media/grid-row-hierarchy-icon.png "Значок иерархии")

Можно также открыть основное поле и поля подстановок в новой вкладке или окне.

   > [!div class="mx-imgBorder"]
   > ![Открыть в новом окне](media/newtab.png "Открыть в новом окне")
  
  
## <a name="lookup-field-column"></a>Столбец поля подстановки

При фильтрации по столбцу подстановки можно сделать выбор в списке записей для фильтрации, а не вводить данные вручную. Например, в столбце **Первичный контакт** уточняющего запроса можно выбрать имя контакта из списка записей для фильтрации.

   > [!div class="mx-imgBorder"]
   > ![Фильтрация при подстановке](media/lookup-filter.png "Фильтрация при подстановке")

## <a name="date-filter"></a>Фильтр дат

Надежный фильтр **Даты** содержит множество различных значений, которые можно выбрать, например **Дата** для поиска по точной дате, **Следующий финансовый год** или **В финансовом периоде** для поиска по году или кварталу.

   > [!div class="mx-imgBorder"]
   > ![Фильтрация дат](media/date-filter.png "Фильтрация дат")

## <a name="filter-the-list-of-activities"></a>Фильтрация списка действий

Список действий можно отфильтровать, чтобы просмотреть только те, которые вас интересуют. Например, можно дополнительно ограничить действия, которые вы видите в представлении, с помощью фильтра типа действий. Фильтр типа действий позволяет фильтровать действия на основе типа, например электронной почты, задачи, телефонного звонка и т. д.


   > [!div class="mx-imgBorder"]
   > ![Фильтр действий](media/activity_filter.png "Фильтр действий")


### <a name="known-issue"></a>Известная проблема 

Если изменить формат отображения числа, валюты, времени или даты, а затем отфильтровать данные по сетке, то фильтр не будет использовать выбранный формат отображения. Фильтры по-прежнему будут отображаться в системном формате по умолчанию, а в некоторых случаях фильтрация может вообще не работать. 

Чтобы устранить эту проблему, задайте для параметра форматов представления чисел, валюты, времени и дат значения по умолчанию. 

1. В правом верхнем углу нажмите значок шестеренки ![Шестеренка](media/selection-rule-gear-button.png) и выберите **Параметры персонализации**.

2. На вкладке **Форматы** измените значение параметра чисел, валюты, времени и дат на значение по умолчанию.

    > [!div class="mx-imgBorder"] 
    > ![Параметры формата](media/default-format.png "Параметры формата")
    
Мы работаем над этой проблемой; вернитесь сюда позже, чтобы проверить, вышло ли исправление.

  
## <a name="use-search-on-a-grid"></a>Использование поиска в сетке

При использовании команды **Искать в этом представлении** на странице сетки система ведет поиск данных в представлении, в котором вы находитесь. В следующем примере выполняется поиск в сетке **Контакты**.

1. Перейдите в сетку **Контакты**, а затем выберите **Мои активные контакты** из списка представлений.

    > [!div class="mx-imgBorder"]
    > ![Представление Мои активные контакты](media/myactive-contacts-view.png "Представление Мои активные контакты")

2. Выберите команды **Искать в этом представлении** для поиска данных в представлении.

    > [!div class="mx-imgBorder"]
    > ![Поиск в представлении](media/search-view.png "Поиск в этом представлении")

Система ведет поиск данных в представлении **Мои активные контакты** и отображает результаты поиска, используя тот же набор столбцов, который используется в текущем представлении.

   > [!div class="mx-imgBorder"]
   > ![Поиск в представлении](media/search-view2.png "Результаты поиска в команде Поиск в этом представлении")


## <a name="use-the-quick-find-search-experience"></a>Использование интерфейса быстрого поиска

Чтобы вернуться к старому интерфейсу быстрого поиска, который использует определение представления быстрого поиска сущности для выполнения поиска, необходимы разрешения администратора.

1. В правом верхнем углу щелкните значок шестеренки ![значок шестеренки](media/selection-rule-gear-button.png) и выберите пункт **Дополнительные параметры**.

2. Последовательно выберите **Параметры** > **Администрирование** > **Параметры системы**.

3. На вкладке **Общие** в разделе **Настройка быстрого поиска**выберите **Да** в поле **Использовать представление быстрого поиска сущности для поиска в таблицах и вложенных сетках**.
