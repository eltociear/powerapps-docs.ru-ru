# <a name="use-grid-filters"></a>Использование фильтров сетки 

Сетки в унифицированном интерфейсе были улучшены для увеличения объема данных, которые можно увидеть на экране. Теперь можно выбрать из множества различных параметров фильтрации для столбца. тип данных в столбце определяет, какие параметры фильтра доступны. Например, столбец « **полное имя** » в сетке « **Контакты** » имеет разные параметры фильтрации, чем столбец « **тип действия** » в сетке « **действия** ».


   > [!div class="mx-imgBorder"]
   > ![Фильтрация сетки](media/filter-options.png "Фильтрация сетки")
   

## <a name="grid-and-filter-navigation"></a>Навигация по сетке и фильтрам

При фильтрации данных в сетке Главная страница сетки запоминает фильтр, порядок сортировки и состояние страницы при переходе на другую страницу и обратно. Это работает при фильтрации данных по быстрому поиску, фильтрации столбцов, номеру страницы и т. д. 

   > [!div class="mx-imgBorder"]
   > ![Переход назад к странице открывает его в том же состоянии](media/grid-remember-state-on-back-navigate.gif "Переход назад к странице открывает его в том же состоянии")

На панели переходов страницы используется первое поле сортировки. Если в порядок сортировки не было изменено, панель переходов использует первичное поле.

   > [!div class="mx-imgBorder"]
   > ![Выберите фильтр на панели переходов](media/jumpbar-filter-on-sorted-column.gif "Выберите фильтр на панели переходов")
  
При выборе значка иерархия выполняется переход к представлению иерархия.

   > [!div class="mx-imgBorder"]
   > ![Значок иерархии](media/grid-row-hierarchy-icon.png "Значок «Иерархия»")

Можно также открыть основное поле и поля подстановок в новой вкладке или окне.

   > [!div class="mx-imgBorder"]
   > ![Открыть в новом окне](media/newtab.png "[Открыть в новом окне")
   
   
### <a name="known-issue"></a>Известная проблема

Если изменить формат отображения числа, валюты, времени или даты, а затем отфильтровать данные по сетке, то фильтр не будет показывать выбранный формат отображения. Фильтры по-прежнему будут отображаться в системном формате по умолчанию, и в некоторых случаях фильтрация может вообще не работать. 

Чтобы устранить эту проблему, задайте для параметра формат представления число, валюта, время и дата значение по умолчанию. 

1. В правом верхнем углу щелкните значок шестеренки ![значок шестеренки](media/selection-rule-gear-button.png), а затем выберите **параметры персонализации**.

2. На вкладке **форматы** измените значение параметра число, валюта, время и дата до значения по умолчанию.

    > [!div class="mx-imgBorder"] 
    > ![Параметры формата](media/default-format.png "параметры формата")
    
    
  Мы работаем над проблемой, проверьте доступность.

### <a name="lookup-field-column"></a>Столбец поля подстановки

При фильтрации по столбцу подстановки можно выбрать из списка записей для фильтрации, а не вводить данные вручную. Например, в столбце подстановки **первичного контакта** можно выбрать Контактное имя из списка записей для фильтрации.

   > [!div class="mx-imgBorder"]
   > ![Фильтрация поиска](media/lookup-filter.png "Фильтрация поиска")

### <a name="date-filter"></a>Фильтр дат

Надежный фильтр **дат** включает множество различных значений, которые можно выбрать, например, **для поиска** по точному времени или **следующему финансовому году** или **в финансовом периоде** для поиска по годам или кварталам.

   > [!div class="mx-imgBorder"]
   > ![Фильтрация по дате](media/date-filter.png "Фильтрация по дате")

### <a name="filter-the-list-of-activities"></a>Фильтрация списка действий

Список действий можно отфильтровать, чтобы просмотреть только те, которые вас интересуют. Например, можно дополнительно ограничить действия, которые вы видите в представлении, с помощью фильтра типа действия. Фильтр типа действия позволяет фильтровать действия на основе типа, например электронной почты, задачи, телефонного звонка и т. д.


   > [!div class="mx-imgBorder"]
   > ![Фильтр действий](media/activity_filter.png "Фильтр действий")

### <a name="use-search-on-a-grid"></a>Использование поиска в сетке

При использовании параметра **Искать в этом представлении** на странице сетки система выполняет поиск данных в представлении, в котором Вы находитесь. В следующем примере выполняется поиск в сетке **Контакты** .

1. Перейдите в сетку **Контакты** , а затем выберите **Мои активные контакты** в списке представлений.

    > [!div class="mx-imgBorder"]
    > ![Мое активное представление контактов](media/myactive-contacts-view.png "Мое активное представление контактов")

2. Выберите **Искать в этом представлении** , чтобы найти данные в представлении.

    > [!div class="mx-imgBorder"]
    > ![Представление поиска](media/search-view.png "Поиск в этом представлении")

Система выполняет поиск данных в представлении " **Мои активные контакты** " и отображает результаты поиска, используя тот же набор столбцов, который используется в текущем представлении.

   > [!div class="mx-imgBorder"]
   > ![Представление поиска](media/search-view2.png "Результаты поиска в команде поиска в этом представлении")


#### <a name="use-the-quick-find-search-experience"></a>Использование интерфейса быстрого поиска

Чтобы вернуться к старому интерфейсу поиска с быстрым поиском, который использует определение представления быстрого поиска для выполнения поиска, необходимы разрешения администратора.

1. В правом верхнем углу щелкните значок шестеренки ![значок шестеренки](media/selection-rule-gear-button.png), а затем выберите **Дополнительные параметры**.

2. Последовательно выберите **параметры** > **Администрирование** > **Параметры системы**.

3. На вкладке **Общие** в разделе **Настройка быстрого поиска**выберите **Да** для параметра **использовать представление быстрого поиска сущности для поиска в сетках и вложенных сетках**.
