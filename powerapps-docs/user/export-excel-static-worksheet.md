# <a name="export-to-an-excel-static-worksheet"></a>Экспорт в статический лист Excel

Если требуется предоставить сведения о данных в приложении для пользователя, не имеющего доступа к приложению, или данных, которые не меняются часто, рассмотрите возможность экспорта данных приложения на статический лист Office Excel.

В каждый момент времени можно экспортировать до 100 000 записей. По умолчанию приложение, управляемое моделью, содержит до 50 записей на страницу. Щелкните стрелки **страницы** в нижней части списка, чтобы просмотреть дополнительные страницы.  
  
## <a name="export-data-to-an-excel-static-worksheet"></a>Экспорт данных в статический лист Excel  
Возможно, у вас есть возможность экспортировать данные в статический лист Excel во всех типах записей. Однако в некоторых случаях формат может быть устаревшим, или данные могут быть не отфильтрованы по отображаемым в приложении.  
  
1. Откройте список записей в приложении, щелкните стрелку справа от параметра **Экспорт в Excel**, а затем выберите **статический лист (только страница)** .  
  
2. По умолчанию экспортированный лист содержит поля, отображаемые в списке, с использованием того же порядка полей, сортировки и ширины полей. Чтобы внести изменения в столбцы в представлении расширенного поиска, выберите **изменить столбцы**. 
  
3. Выберите **сохранить** , а затем сохраните XLSX файл. Запишите место, где был сохранен файл.  
  
   > [!NOTE]
   > Если вы собираетесь изменить файл данных позже, рекомендуется сохранить файл перед его открытием. В противном случае может появиться следующее сообщение об ошибке: **Excel не может открыть или сохранить больше документов из-за нехватки свободной памяти или места на диске.**  
   > 
   > Чтобы устранить эту проблему, выполните следующие действия.  
   > 
   > 1. Откройте Excel и выберите **файл** > **Параметры** > **центр управления** > безопасностью центр**настройки** > **защищенное представление**.  
   > 2.  В **режиме защищенного просмотра**очистите все три элемента.  
   > 3.  Нажмите кнопку **ОК** > **ОК**.  
   > 
   > Мы настоятельно рекомендуем сохранить и открыть файл данных, а не отключать защищенное представление, что может поставить под угрозу компьютер.  


4. Откройте Excel, а затем откройте XLSX файл, сохраненный на предыдущем шаге.  
  
   По умолчанию экспортированный лист содержит поля, отображаемые в списке, с использованием того же порядка полей, сортировки и ширины полей.  
  
## <a name="tips"></a>Советы  
  
- Можно отправить по электронной почте статический экспортированный лист любому пользователю или сохранить его в общем файле. Любой пользователь, открывающий файл, увидит все данные в файле.
  
- Нельзя изменить столбцы для системного представления, например для **всех активных учетных записей**. Необходимо либо настроить представление, для которого требуется системный администратор или роль безопасности System Настройщик, либо воспользоваться расширенным поиском, чтобы создать собственное представление на основе текущего представления.  
    
- В приложениях на основе модели значения валют экспортируются в Excel как числа. После экспорт, см. раздел справки Excel "Отображение чисел как валюты", чтобы отформатировать данные как валюту.
  
- Значения даты и времени, отображаемые в приложении, отображаются только как Дата при экспорте файла в Excel, но в ячейке фактически отображается дата и время.  
  
- Если вы собираетесь внести изменения и импортировать файл данных обратно в приложение, помните, что защищенные, вычисляемые и составные поля (такие как полное имя) доступны только для чтения и не могут быть импортированы в приложение. Вы сможете изменять эти поля в Excel, но при импорте данных обратно в приложение эти поля не будут обновлены. Если вы хотите обновить эти поля, например имя контакта, рекомендуется использовать это представление для экспорта данных, обновления в Excel и импорта их обратно в приложение для внесения изменений.
