# <a name="troubleshoot-problems-with-data-not-displaying-in-a-report"></a>Устранение проблем с данными, не отображаемыми в отчете

Существует несколько причин, по которым данные, которые должны находиться в отчете, не отображаются.  
  
- **Недостаточно разрешений безопасности**. Если в Common Data Service у вас нет разрешения на просмотр записи, она не будет отображаться в отчете.  
  
- **Данные не введены.** Пользователь, который вводит данные, мог оставить поля пустыми.  
  
- **Данные не соответствуют критериям отчета.** Многие отчеты включают фильтр по умолчанию, который отображает только активные записи. Или вы могли выбрать критерии, которым не соответствуют записи. Попробуйте изменить фильтр отчета. Дополнительные сведения см. в разделе [Изменение фильтра отчета по умолчанию](edit-report-filter.md)  
  
- **Возможно, вы просматриваете кэшированную копию отчета.** По умолчанию данные в отчетах Common Data Service извлекаются из базы данных при каждом запуске отчета. Однако системный администратор мог изменить отчет для запуска из кэша. Если недавно введенные данные не включаются в отчет, то у вас может быть более старая версия отчета из кэша. Чтобы обновить отчет, на панели инструментов отчета нажмите кнопку **Обновить**.  
  
- **Возможно, у вас нет разрешения на чтение записей во вложенном отчете.** Если у вас нет разрешения на чтение типов записей, которые включены во вложенный отчет, появится сообщение об ошибке, уведомляющее, что не удалось отобразить вложенный отчет.  
  
- **Параметры конфиденциальности Microsoft Internet Explorer могут блокировать необходимые файлы cookie.** Если в отчетах с диаграммами вместо диаграммы отображается красная буква X, параметры конфиденциальности могут блокировать файл cookie, необходимый для элемента управления "Диаграмма". Чтобы устранить эту проблему, в браузере включите файлы cookie для сервера, на котором выполняются службы Reporting Services.
