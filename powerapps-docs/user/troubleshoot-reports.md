# <a name="troubleshoot-problems-with-data-not-displaying-in-a-report"></a>Устранение проблем с данными, не отображаемыми в отчете

Существуют несколько причин, по которым данные, которые должны находиться в отчете, не отображаются.  
  
- **Недостаточные разрешения безопасности**. Если у вас нет разрешения на Common Data Service для просмотра записи, он не будет отображаться в отчете.  
  
- **Данные не указаны.** Пользователь, который вводит данные, может оставить поля пустыми.  
  
- **Данные не соответствуют критериям отчета.** Многие отчеты включают фильтр по умолчанию, который отображает только активные записи, или Выбранные критерии, у которых нет совпадающих записей. Попробуйте изменить фильтр отчета. Дополнительные сведения см. [в разделе Изменение фильтра по умолчанию для отчета](edit-report-filter.md) .  
  
- **Возможно, вы просматриваете кэшированную копию отчета.** По умолчанию данные в Common Data Service отчеты извлекаются из базы данных при каждом запуске отчета. Однако системный администратор мог изменить отчет для запуска из кэша. Если недавно введенные данные не включаются в отчет, то у вас может быть более старая версия отчета из кэша. Чтобы обновить отчет, на панели инструментов отчета нажмите кнопку **Обновить** .  
  
- **Возможно, у вас нет разрешения на чтение записей во вложенном отчете.** Если у вас нет разрешения на чтение типов записей, которые включены во вложенный отчет, появится сообщение об ошибке, сообщающее, что не удалось отобразить вложенный отчет.  
  
- **Параметры конфиденциальности Microsoft Internet Explorer могут блокировать необходимые файлы cookie.** Для отчетов с диаграммами, если вместо диаграммы отображается красная буква X, параметры конфиденциальности могут блокировать файл cookie, необходимый для элемента управления диаграммы. Чтобы устранить эту проблему, в браузере Включите файлы cookie для сервера, на котором выполняется Reporting Services.
