# <a name="add-a-connection-role-to-link-records-to-each-other"></a>Добавление роли подключения для связывания записей друг с другом

[!INCLUDE [cc-beta-prerelease-disclaimer](../includes/cc-beta-prerelease-disclaimer.md)]

Подключения позволяют легко связывать пользователей, контакты, предложения, заказы на продажу и многие другие записи сущностей. Записям в ассоциации можно назначить определенные роли, которые помогают определить назначение связи.

Это быстрый способ создания нескольких подключений и ролей для конкретной записи. Например, контакт может иметь много отношений с другими контактами, счетами или контрактами. В каждой связи контакт может играть другую роль.

Роли подключения напрямую связаны с подключением. Чтобы использовать роль подключения, необходимо сначала добавить подключение к записи.

Прежде чем можно будет добавить роли подключения, ее необходимо включить администратором. Дополнительные сведения см. в разделе [Настройка ролей подключения](https://docs.microsoft.com/en-us/powerapps/maker/common-data-service/configure-connection-roles).

1. Чтобы добавить подключения или управлять ими, выберите запись, которая будет управляться как возможность.  
2. Перейдите на вкладку **связанные** и выберите **Подключение**. Откроется сетка соединений со списком соединений для записи.

    > [!div class="mx-imgBorder"]
    > ![Добавление новой роли подключения](media/connection1.png "Добавление новой роли подключения") 

3. Выберите **подключить** , а затем выберите **другой** или **для меня**.

    > [!div class="mx-imgBorder"]
    > ![Выбор типа подключения](media/connection2.png "Выбор типа подключения") 
  
4. В поле **имя** введите или найдите имя записи для соединения.

5. В поле в **качестве роли** выберите значок поиска, а затем выберите **создать роль подключения**. Или используйте поиск, чтобы найти существующую роль, которую необходимо связать с подключением, а затем нажмите кнопку **сохранить**.

    > [!div class="mx-imgBorder"]
    > ![Выбор новой роли подключения](media/connection3.png "Выбор новой роли подключения")  

    > [!NOTE]
    > Если вы указали сведения перед созданием новой роли подключения, отобразится диалоговое окно с предупреждением о том, нужно ли отменить и продолжить работу с подключением, либо оставить текущую запись, над которой вы работаете.

6. Чтобы создать новую роль подключения, на экране **Создание роли подключения** введите имя и выберите **категорию роль подключения**.

    > [!div class="mx-imgBorder"]
    >  ![Добавить категорию роли подключения](media/connection4.png "Добавить категорию роли подключения") 

7. По завершении нажмите кнопку **сохранить & закрыть**.

  
## <a name="manage-connection-roles"></a>Управление ролями подключения

Для управления ролью подключения выберите роль подключения из сущности соединения. Откроется запись сущности "роль подключения".  Можно изменить имя, выбрать категорию роли подключения и добавить описание.


   > [!div class="mx-imgBorder"]
   > ![Изменить роль подключения](media/connection7.png "Роль едитконнектион") 
  
Можно также управлять типами ролей подключения, которые должны быть связаны с ролью подключения.

1. Откройте роль подключение, а затем выберите **Управление типом записи** в команде. 

    > [!div class="mx-imgBorder"]
    > ![Изменить роль подключения](media/connection5.png "Роль едитконнектион") 
  

2. Откроется список типов ролей подключения, которые можно добавить или удалить для этой роли подключения.

    > [!div class="mx-imgBorder"]
    > ![Управление типом записи](media/connection6.png "Управление типом записи")
