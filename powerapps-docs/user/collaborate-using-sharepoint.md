# <a name="collaborate-using-sharepoint"></a>Совместная работа с помощью SharePoint 

С помощью Common Data Service можно хранить документы в SharePoint и управлять ими в приложении. Документы, создаваемые в приложении, сохраняются в SharePoint и автоматически синхронизируются с компьютером и мобильными устройствами.

Чтобы можно было использовать SharePoint для хранения документов, системный администратор должен включить эту возможность. Дополнительные сведения:

-   [Обращение к администратору или в службу поддержки](find-admin.md)  

-   [Управление документами с помощью SharePoint](https://docs.microsoft.com/power-platform/admin/manage-documents-using-sharepoint)  

## <a name="where-do-you-access-the-documents-from"></a>Где можно получить доступ к документам?

1. Для типов записей, поддерживающих управление документами, откройте запись, выберите вкладку **Связанные**, а затем выберите **Документы**.

   > [!div class="mx-imgBorder"]
   > ![Открытие вкладки "Документы" в записи](media/onedrive_nav.png "Открытие вкладки "Документы" в записи")

2. Выберите **Расположение документа** > **Документы на сайте по умолчанию 1**. Когда система SharePoint включена, по умолчанию для расположения задано значение **Документы на сайте по умолчанию 1**.

   > [!div class="mx-imgBorder"]
   > ![Расположение по умолчанию](media/sharepoint_defualtsite.png "Расположение по умолчанию")


## <a name="create-a-new-document-and-save-it-to-sharepoint"></a>Создание документа и его сохранение в SharePoint

1. Откройте запись и перейдите в представление **Сетка, связанная с документом**. Например, откройте запись контакта.

2. В открытой записи перейдите на вкладку **Связанные** и выберите **Документы**.
 
    > [!div class="mx-imgBorder"]
    > ![Открытие вкладки "Документы" в записи](media/onedrive_nav.png "Открытие вкладки "Документы" в записи")

2. Выберите **Расположение документа** и измените расположение на **Документы на сайте по умолчанию 1**.

3. Выберите **Создать**, а затем выберите тип документа, например Word, Excel или PowerPoint.

    > [!div class="mx-imgBorder"]
    > ![Создание документа](media/onedrive_new_doc.png "Создание документа")

4. Введите имя документа и нажмите кнопку **Сохранить**.  

## <a name="create-a-new-folder-in-the-default-sharepoint-site-location"></a>Создание папки в расположении сайта SharePoint по умолчанию

1. Откройте запись и перейдите в представление **Сетка, связанная с документом**. Например, откройте запись контакта.

2. В открытой записи перейдите на вкладку **Связанные** и выберите **Документы**.
 
    > [!div class="mx-imgBorder"]
    > ![Открытие вкладки "Документы" в записи](media/onedrive_nav.png "Открытие вкладки "Документы" в записи")

2. Выберите **Расположение документа** и измените расположение на **Документы на сайте по умолчанию 1**.

3. Щелкните **Создать**, а затем выберите пункт **Папка**.

    > [!div class="mx-imgBorder"]
    > ![Создание папки](media/Sharepoint_new_folder.png "Создание папки")
    
 4. Введите имя папки и нажмите кнопку **Сохранить**.  
 
 
 ## <a name="upload-an-existing-document-to-sharepoint-from-your-app"></a>Отправка существующего документа в SharePoint из приложения

1. Перейдите к записи, для которой нужно создать документ, выберите вкладку **Связанные**, а затем выберите **Документы**.
 
2. Выберите команду **Отправить**.

   > [!div class="mx-imgBorder"]
   > ![Отправка документов](media/upload_doc.png "Отправка документов")

3. Выберите файл, который требуется отправить. За раз можно выбрать только один файл.

   Документ создается в текущем расположении документа.

   > [!Note]
   > Отправить можно файл размером до 50 МБ. Если подключение к Интернету медленное, при отправке больших файлов может происходить ошибка.

4. Если файл с таким именем уже есть в SharePoint, укажите, следует ли перезаписать его.

5. Выберите **ОК**.

## <a name="manage-sharepoint-locations"></a>Управление расположениями SharePoint

Вы можете создавать новые или изменять существующие расположения SharePoint из приложения в Common Data Service.

### <a name="edit-a-location"></a>Изменение расположения

1. Откройте запись, перейдите на вкладку **Связанные** и выберите **Документы**.

2. Выберите **Изменить расположение**, а затем выберите расположение сайта SharePoint.

   Отображается диалоговое окно **Изменение расположения**.

   > [!div class="mx-imgBorder"]
   > ![Изменение расположения](media/edit_location.png "Изменение расположения")

3. Отображаемое имя, родительский сайт и имя папки заполняются автоматически. Введите сведения о новом расположении, а затем выберите команду **Сохранить**.

### <a name="add-a-new-location"></a>Добавление расположения

1. Откройте запись, перейдите на вкладку **Связанные** и выберите **Документы**.

2. Выберите **Добавить расположение**. 

   Откроется диалоговое окно **Добавление расположения**.

   > [!div class="mx-imgBorder"]
   > ![Добавление расположения](media/add_location.png "Добавление расположения")

3. Отображаемое имя, родительский сайт и имя папки заполняются автоматически. При необходимости измените сведения, а затем выберите команду **Сохранить**.

## <a name="files-tab-faq"></a>Часто задаваемые вопросы о вкладке "Файлы"

*Почему расположение для доступа к документам было перемещено?* 
- Мы перенесли команду, чтобы упростить поиск документов, сократив при этом число щелчков мышью.

*Исчезла ли вкладка "Документы"?*
- Нет, он никуда не делась. Пользователи по-прежнему могут получить доступ к документам, связанным с соответствующей записью, просто открыв меню **Связанные**, а затем перейдя по ссылке **Документы**.

*Будут ли автоматически создаваться подпапки в SharePoint после этого изменения?*
- Да. Поведение аналогично поведению ссылки **Документы** в меню **Связанные**. Когда пользователь впервые выбирает вкладку **Файлы**, система создает соответствующую вложенную папку SharePoint. 

*Есть ли способ добавить вкладку "Файлы" в другие сущности или удалить ее?*
- Да. Чтобы добавить или удалить вкладку **Файлы**, выполните действия, описанные в следующей статье: [Добавление вкладки "Документы SharePoint" в главную форму сущности](../maker/model-driven-apps/add-documents-tab-entity-main-form.md)
