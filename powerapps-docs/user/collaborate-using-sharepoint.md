# <a name="collaborate-using-sharepoint"></a>Совместная работа с помощью SharePoint 

Управление общими типами документов, такими как Word, Excel и PowerPoint, и создание папок для сохранения и упорядочения документов, которые прозрачно хранятся в SharePoint, из приложения на основе модели. 

> [!NOTE]
> Эта функция требует, чтобы системный администратор включил управление документами SharePoint. Дополнительные сведения: [Управление документами с помощью SharePoint](/power-platform/admin/manage-documents-using-sharepoint)

Для записей типов "Учетная запись" и "Контакт" папка расположения документов по умолчанию автоматически создается в SharePoint при первом переходе на вкладку **Файлы**. Для других стандартных или пользовательских записей сущностей перейдите на вкладку **Связанные** > **Документы**. Имя расположения документа имеет следующий формат: <имя_записи>_<идентификатор_записи>.

По умолчанию для расположения задано значение "Документы на сайте по умолчанию 1".

## <a name="add-a-document"></a>Добавление документа
1.  Откройте запись учетной записи или контакта и выберите вкладку **Файлы**. Для других стандартных или настраиваемых сущностей, у которых включено управление документами, выберите вкладку **Связанные**, а затем выберите **Документы**.
2.  Выберите один из следующих параметров. 
    - Чтобы создать новый документ, выберите **Создать**, выберите нужный тип документа, например Word, Excel или OneNote, а затем введите имя. Нажмите кнопку **Сохранить**. На новой вкладке откроется пустой документ. 
    - Чтобы добавить существующий документ, выберите команду **Отправить**, пункт **Выбрать файл**, перейдите в нужное место и выберите файл, а затем выберите команду **Открыть**. Выберите **ОК**. 

Файл документа появится в представлении **Сетка, связанная с документом**. 

> [!div class="mx-imgBorder"] 
> ![](media/add-doc-sharepoint.png "Add document to SharePoint")

Документ также появится в расположении папки сайта SharePoint. 

> [!div class="mx-imgBorder"] 
> ![](media/doc-on-sharepoint.png "Document on SharePoint")

## <a name="manage-sharepoint-locations"></a>Управление расположениями SharePoint
Вы можете создавать новые или изменять существующие расположения SharePoint из приложения на основе модели.

1. В списке **Файлы** на панели команд выберите **Открыть расположение**, а затем выберите расположение.
2. Чтобы изменить расположение, на панели команд выберите команду **Изменить расположение** <location name>.
Отображается диалоговое окно **Изменение расположения**.
3. Отображаемое имя, родительский сайт и имя папки заполняются автоматически. Введите сведения о новом расположении, а затем выберите команду **Сохранить**.
4. Чтобы добавить расположение, на панели команд выберите команду **Добавить расположение**.
5. Откроется диалоговое окно **Добавление расположения**.

    > [!div class="mx-imgBorder"] 
    > ![](media/add-location-dialog-box.png "Add location dialog box")
6. Отображаемое имя, родительский сайт и имя папки заполняются автоматически. При необходимости измените сведения, а затем выберите команду **Сохранить**.

## <a name="actions-on-documents"></a>Операции с документами
При выборе одного документа или нескольких в списке "Документы" можно использовать следующие общие действия SharePoint для документов:
- Изменить
- Удалить
- Вернуть
- Извлечь
- Отменить извлечение
- Редактировать свойства

## <a name="files-tab-faq"></a>Часто задаваемые вопросы о вкладке "Файлы"
*Почему расположение для доступа к документам было перемещено?* 
- Мы перенесли команду, чтобы упростить поиск документов, сократив при этом число щелчков мышью.

*Исчезла ли вкладка "Документы"?*
- Нет, он никуда не делась. Пользователи по-прежнему могут получить доступ к документам, связанным с соответствующей записью, просто щелкнув соответствующее меню, а затем перейдя по ссылке "Документы".

*Будут ли автоматически создаваться подпапки в SharePoint после этого изменения?*
- Да. Поведение аналогично поведению ссылки **Документы** в меню **Связанные**. Когда пользователь выбирает в первый раз вкладку **Файлы**, система создает соответствующую вложенную папку SharePoint. 

*Есть ли способ добавить вкладку "Файлы" в другие сущности или удалить ее?*
- Да. Чтобы добавить или удалить вкладку "Файлы", выполните действия, описанные в этой статье. [Добавление вкладки "Документы SharePoint" в главную форму сущности](../maker/model-driven-apps/add-documents-tab-entity-main-form.md)  

*Где можно отправить отзыв об этом изменении?*
- Вы можете отправить отзыв в группу интеграции Dynamics 365 Sales, Office и Teams по адресу электронной почты: d365_ot_crew@microsoft.com
