# <a name="collaborate-using-sharepoint"></a>Совместная работа с помощью SharePoint 

Управление общими типами документов, такими как Word, Excel и PowerPoint, и создание папок для сохранения и управления документами, которые легко хранятся в SharePoint из приложения, управляемого моделью. 

> [!NOTE]
> Эта функция требует, чтобы системный администратор включил управление документами SharePoint. Дополнительные сведения см. [в документе Управление документами с помощью SharePoint](/power-platform/admin/manage-documents-using-sharepoint) .

Для записей Account и Contact папка расположения документов по умолчанию автоматически создается в SharePoint при первом переходе на вкладку **файлы** . Для других стандартных или пользовательских записей сущностей перейдите на **соответствующую** вкладку > **документы** . Имя расположения документа имеет следующий формат: < record_name > _ < record_id >.

По умолчанию для расположения задано значение документы на сайте по умолчанию 1.

## <a name="add-a-document"></a>Добавление документа
1.  Откройте запись учетной записи или контакта и выберите вкладку **файлы** . Для других стандартных или настраиваемых сущностей, включенных для управления документами, выберите вкладку **связанные** , а затем выберите **документы**.
2.  Выберите один из следующих параметров. 
    - Чтобы создать новый документ, выберите **создать**, выберите нужный тип документа, например Word, Excel или OneNote, а затем введите имя. Щелкните **Сохранить**. На новой вкладке откроется пустой документ. 
    - Чтобы добавить существующий документ, выберите **Отправить**, щелкните **выбрать файл**, перейдите к и выберите нужный файл, а затем нажмите кнопку **Открыть**. Выберите **ОК**. 

Файл документа появится в представлении **сетки, связанной с документом** . 

> [!div class="mx-imgBorder"] 
> ![](media/add-doc-sharepoint.png "Add document to SharePoint")

Документ также появится в расположении папки сайта SharePoint. 

> [!div class="mx-imgBorder"] 
> ![](media/doc-on-sharepoint.png "Document on SharePoint")

## <a name="manage-sharepoint-locations"></a>Управление расположениями SharePoint
Можно создать новые или изменить существующие расположения SharePoint из приложения, управляемого моделью.

1. В списке **файлы** на панели команд выберите **открыть расположение**, а затем выберите расположение.
2. Чтобы изменить расположение, на панели команд выберите **изменить расположение** <location name>.
Откроется диалоговое окно **изменение расположения** .
3. Отображаемое имя, родительский сайт и имя папки заполняются автоматически. Введите сведения о новом расположении и нажмите кнопку **сохранить**.
4. Чтобы добавить расположение, на панели команд выберите **Добавить расположение**.
5. Откроется диалоговое окно **Добавление расположения** .

    > [!div class="mx-imgBorder"] 
    > ![](media/add-location-dialog-box.png "Add location dialog box")
6. Отображаемое имя, родительский сайт и имя папки заполняются автоматически. При необходимости измените сведения, а затем нажмите кнопку **сохранить**.

## <a name="actions-on-documents"></a>Действия с документами
При выборе одного или нескольких документов в списке документы можно использовать следующие общие действия SharePoint для документов:
- Edit
- Оператор delete
- Следите за мнением пользователей
- См. статью
- Отменить извлечение
- Изменить свойства

## <a name="files-tab-faq"></a>Вкладка "файлы" — вопросы и ответы
*Почему расположение для доступа к документам было перемещено?* 
- Мы переехали команду, чтобы упростить поиск документов с помощью меньшего числа щелчков мышью.

*Исчезла ли вкладка "документы"?*
- Нет, он не исчезает. Пользователи по-прежнему могут получить доступ к документам, связанным с соответствующей записью, просто щелкнув соответствующее меню, а затем перейдя по ссылке документы.

*После изменения будет ли автоматически создаваться подпапки в SharePoint?*
- Да. Поведение аналогично ссылке « **документы** » в **соответствующем** меню. Когда пользователь выбирает вкладку « **файлы** » в первый раз, система создает соответствующую вложенную папку SharePoint. 

*Есть ли способ добавить вкладку "файлы" в другие сущности или удалить ее?*
- Да. Чтобы добавить или удалить вкладку "файл", выполните действия, описанные в этой статье. [Добавление вкладки "документы SharePoint" в главную форму сущности](../maker/model-driven-apps/add-documents-tab-entity-main-form.md)  

*Где можно отправить отзыв об этом изменении?*
- Вы можете отправить отзыв в группу интеграции Office Dynamics 365 Sales and Teams по адресу электронной почты: d365_ot_crew@microsoft.com
