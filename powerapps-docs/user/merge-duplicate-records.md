# <a name="merge-duplicate-records"></a>Объединение повторяющихся записей 

Дубликаты записей могут перезаписываться в данные, когда пользователь или другие пользователи вводят данные вручную или импортируют данные в виде пакета. Common Data Service помогает устранить потенциальные дубликаты, предоставляя обнаружение дубликатов для активных записей, таких как, учетные записи и контакты. При объединении записи все связанные или дочерние записи также будут объединены. Администратор может также настроить правила обнаружения повторяющихся данных для других ситуаций.  
  
Например, предположим, что вы вводите контактную запись, Джим Глинн, а также номер мобильного телефона.  Правило обнаружения дубликатов обнаруживает, что у вас уже есть аналогичная запись, и отображает это диалоговое окно.  
  
 > [!div class="mx-imgBorder"] 
 > ![Повторяющаяся запись контакта детектиед](media/duplicates-detected.png "Повторяющаяся запись контакта детектиед")  
  
 Вы не уверены, что это новая запись (которая будет иметь то же имя, что и существующий контакт) или дубликат, поэтому выберите **Пропустить и сохранить**.  
  
 Затем перейдите в список **все контакты** и проверьте, что теперь у вас есть две записи с одинаковым именем. После просмотра записей необходимо определить, что они являются дубликатами, которые необходимо объединить.  
 
 > [!div class="mx-imgBorder"] 
 > ![Повторяющаяся запись контакта детектиед](media/duplicates-detected_1.png "Повторяющаяся запись контакта детектиед")  
 
Common Data Service содержит правила обнаружения повторений для учетных записей и контактов. Эти правила включены автоматически, поэтому вам не нужно ничего делать, чтобы настроить обнаружение дубликатов для этих типов записей.  
  
> [!NOTE]
>  При наличии в системе можно также проверить наличие дубликатов других типов записей в дополнение к контактам и учетным записям. Обратитесь к системному администратору. [Поиск администратора или специалиста службы поддержки](find-admin.md)  
  
## <a name="merge-duplicate-records"></a>Объединение повторяющихся записей  
  
1. Выберите дублирующиеся записи, а затем нажмите кнопку **объединить**.  
  
   > [!div class="mx-imgBorder"] 
   > ![Повторяющаяся запись контакта детектиед](media/duplicates-detected_2.png "Повторяющаяся запись контакта детектиед")  
  
2. В диалоговом окне **слияние записей** выберите главную запись (которую нужно сохранить), а затем выберите поля в новой записи, которые необходимо объединить с главной записью. Данные в этих полях могут переопределять существующие данные в главной записи. Выберите **ОК**.  
  
     
   > [!div class="mx-imgBorder"] 
   > ![Диалоговое окно для слияния записей](media/merge-records-dialog.png "Диалоговое окно для слияния записей")  
  
> [!NOTE]
>  Существует несколько ситуаций, когда можно найти дубликаты:  
> 
> - При создании или обновлении записи.  
>   - При использовании Dynamics 365 для Outlook и переходе из режима «вне сети» в режим «в сети».  
>   - При импорте данных с помощью мастера импорта данных.  
> 
>   Дубликаты не обнаруживаются при слиянии записей, сохранении действия как завершенных или изменении состояния записи, например активации или повторной активации записи.
