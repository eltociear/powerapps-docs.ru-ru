#  <a name="basic-navigation-in-a-model-driven-app"></a>Базовая навигация в приложении на основе модели 

В этом вводе объясняется, как найти и открыть приложение, а также как работать с его общими элементами пользовательского интерфейса, включая списки, формы и бизнес-процессы.

## <a name="navigating-among-apps-areas-and-entities"></a>Навигация между приложениями, областями и сущностями

Приложение, управляемое моделью, основано на приложениях (приложениях), областях и сущностях.

- *Приложения* предоставляют набор функций для выполнения определенного класса действий, таких как управление учетными записями и контактами. Используйте меню выбора приложений для перехода между приложениями, доступными для вашей организации.

- *Рабочая область* — это подразделение приложения, выделенное для определенной функции. Каждая Рабочая область предоставляет целевую коллекцию сущностей для работы в этой области. В некоторых случаях одна и та же сущность отображается в нескольких областях (или даже в нескольких приложениях). Например, сущности контактов и учетных записей отображаются в различных приложениях и рабочих областях. Используйте меню рабочей области для перехода между рабочими областями текущего приложения.

- *Сущности* представляют конкретный тип данных, например контакты и учетные записи. Сущности используют структурированный формат данных, который определяет коллекцию полей, доступных для сущности. Каждая сущность состоит из коллекции отдельных записей. Например, для сущности Contact каждая запись описывает одного человека, а каждая запись включает в себя набор полей, таких как имя, фамилия и адрес электронной почты. Сущности обычно представляют два представления: представление списка, которое обычно представляет собой таблицу с перечнем доступных записей. и представление формы, в котором отображаются все доступные данные и параметры для одной записи. Используйте боковое навигатор для перемещения между сущностями в текущей рабочей области.

### <a name="move-between-apps"></a>Перемещение между приложениями

Используйте меню выбора приложений для переключения между приложениями.

![Меню выбора приложения](media/app-selector.png "Меню выбора приложения")

Приложения, которые отображаются в меню выбора приложения, будут зависеть от того, к каким приложениям у вас есть доступ. 

### <a name="move-between-entities-records-and-work-areas"></a>Перемещение между сущностями, записями и рабочими областями

Это легко сделать и вернуться к избранным или наиболее часто используемым записям. На следующем рисунке показаны основные элементы навигации.

![Элементы управления навигацией, развернутое представление](media/nav-expanded.png "Элементы управления навигацией, развернутое представление")

Условные обозначения:

1. **Селектор приложений**: откройте это меню, чтобы перейти между приложениями.
1. **Кнопка "свернуть/развернуть"** : Выберите этот параметр, чтобы свернуть навигатор, чтобы разрешить больше места для основной части страницы. Если навигатор уже свернут, нажмите эту кнопку, чтобы снова развернуть ее.
1. **Недавние записи**: разверните эту запись, чтобы просмотреть список недавно использованных записей. Выберите запись здесь, чтобы открыть ее. Щелкните значок push-уведомления рядом с записью в списке, чтобы добавить ее в избранное (закрепленные записи).
1. **Избранные записи**: разверните эту запись, чтобы просмотреть и открыть избранные (закрепленные) записи. Используйте список **недавние записи** , чтобы добавить записи. Щелкните значок Удалить ПИН-код рядом с записью в списке, чтобы удалить ее из списка.
1. **Навигатор сущностей**. в этой области перечислены все сущности и панели мониторинга, доступные для текущей рабочей области. Выберите любую запись, чтобы открыть именованную панель мониторинга или представление списка для этой сущности.
1. **Область выделения рабочей области**: откройте это меню, чтобы перейти к другой рабочей области. Текущая рабочая область называется здесь.

## <a name="working-with-list-views"></a>Работа с представлениями списков

Обычно при первом открытии сущности отображается представление списка, в котором отображается список записей, относящихся к этой сущности, в формате таблицы. Например, если открыть сущность **учетные записи** , отобразится список учетных записей.

![Типичное представление списка](media/list-view.png "Типичное представление списка")

Условные обозначения:

1. **Выбор записей**: выберите одну или несколько записей, установив флажок в этом столбце. В зависимости от того, где вы работаете, вы можете применить одну операцию ко всем выбранным записям одновременно с помощью кнопок на панели команд.
2. **Открыть запись**. Выберите любую запись в списке, чтобы открыть ее представление записей, в котором отображаются все сведения о записи. Обычно выбирается из столбца **имя** , чтобы открыть запись из текущей сущности. Некоторые сущности предоставляют ссылки на записи из связанных сущностей в других столбцах (например, связанный контакт).
3. **Сортировать или фильтровать список**: выберите, чтобы отсортировать список по значениям в этом столбце или отфильтровать список по значениям в этом столбце. Стрелка в заголовке столбца указывает, какой столбец сортируется и в каком направлении. 
4. **Панель команд**: используйте команды на панели команд для работы с записями в списке и выполнения связанных действий. Для некоторых команд (например, **Delete**) необходимо сначала выбрать одну или несколько целевых записей, установив флажок в крайнем левом столбце, а другие — во всем списке. Список можно экспортировать в книгу Excel (возможно, на основе шаблона), открыть диаграммы и панели мониторинга и многое другое, в зависимости от типа записей, с которыми вы работаете.
5. **Поиск в представлении**: введите текст в поле поиска над списком, чтобы показать только те записи в текущем представлении, которые содержат текст.
6. **Фильтрация и разбиение на страницы**. Выберите букву, чтобы отобразить только те записи, имена которых начинаются с этой буквы. Если список содержит больше записей, чем может быть отображено на одной странице, воспользуйтесь кнопками со стрелками в нижней части списка, чтобы перейти вперед и назад по страницам.

## <a name="working-with-record-views"></a>Работа с представлениями записей

В представлениях записей отображаются все сведения об отдельной записи, а иногда также предоставляются специальные функции для работы с ней. Обычно вы открываете представление записей, выбрав запись, которая отображается в представлении списка, но можно также открыть представление записей, выполнив ссылку из связанной записи.

![Типичное представление записей](media/form-view.png "Типичное представление записей")

Условные обозначения:


1. **Вкладки**: большинство представлений записей разделены на вкладки. Каждая вкладка содержит коллекцию связанных полей из записи. Если вкладки доступны, они перечислены под именем записи. Выберите любое имя вкладки, чтобы открыть эту вкладку. Текущая вкладка отображается подчеркнутой.
2. **Связанные**: почти все типы записей показывают **связанную** вкладку после того, как вы сохранили их по крайней мере один раз. Эта вкладка на самом деле представляет собой раскрывающийся список, который можно использовать для поиска других типов записей, использующих отображаемую запись или ссылающихся на нее. При выборе имени сущности из **связанного** раскрывающегося списка открывается новая вкладка с именем для этой сущности, в которой отображается список всех связанных записей этого типа. **Связанная** вкладка остается доступной, и ее по-прежнему можно использовать для поиска других типов записей, ссылающихся на текущий.
3. **Панель команд**: используйте команды на панели команд для работы с текущей записью или выполнения задачи, связанной с записью. Доступные команды зависят от типа записи, но обычно для сохранения изменений можно использовать панель команд, удалить запись, обновить страницу, отправить ссылку на запись, переназначить владельца записи или экспортировать запись с помощью шаблона Word.
4. **Строка заголовка**. в некоторых представлениях записей в строке заголовка отображаются несколько особенно важных полей, противоположных имени записи. Обычно это поля, которые являются фундаментальными для работы с записями текущего типа (например, имя записи или владелец записи).
5. **Просмотр и изменение значений всех полей**: в основном тексте представления записей можно найти все поля, связанные с текущей вкладкой, представлением формы и типом записи. Поля, отмеченные красной звездочкой, являются обязательными, и запись не может быть сохранена без допустимых значений. Поля, отмеченные синим знаком «плюс», особенно важны или рекомендуются, но не являются обязательными. Поля, отображающие значок блокировки, доступны только для чтения и не могут быть изменены.

## <a name="notifications"></a>Уведомления 

В форме отображаются три типа уведомлений: информационное, предупреждение и ошибка. Уведомления всегда доступны наверху формы, прямо над заголовком.

При выборе уведомления об ошибке вы перейдете к полю в форме, в которой произошла ошибка.

![Пример уведомлений](media/notifications.png "Примеры уведомлений")

Условные обозначения:

1. **Info**: уведомление является информационным.
2. **Предупреждение**: уведомление является предупреждением. 
3. **Ошибка**: уведомление является ошибкой. 



### <a name="single-notification"></a>Одно уведомление

Если имеется только одно уведомление, вы увидите одну строку.

![Пример одиночных уведомлений](media/single_notification.png "Пример одного уведомления")

### <a name="multiple-notifications"></a>Несколько уведомлений

Если имеется несколько уведомлений, вы увидите количество уведомлений. Щелкните значок шеврона, чтобы просмотреть каждое сообщение.

![Пример нескольких уведомлений](media/multiple_notification.png "Пример нескольких уведомлений")
