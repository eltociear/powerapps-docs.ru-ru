#  <a name="basic-navigation-in-a-model-driven-app"></a>Базовая навигация в приложении на основе модели 

В этом введении объясняется, как найти и открыть приложение, а также как работать с его общими элементами пользовательского интерфейса, включая списки, формы и бизнес-процессы.

## <a name="navigating-among-apps-areas-and-entities"></a>Навигация между приложениями, областями и сущностями

Приложение на основе модели состоит из приложений, областей и сущностей.

- *Приложения* предоставляют набор функциональных возможностей для выполнения определенного класса действий, таких как управление учетными записями и контактами. Используйте меню выбора приложений для перехода между приложениями, доступными для вашей организации.

- *Рабочая область* — это подразделение приложения, выделенное для конкретного компонента. Каждая рабочая область предоставляет целевую коллекцию сущностей для работы в этой области. В некоторых случаях одна и та же сущность отображается в нескольких областях (или даже в нескольких приложениях). Например, сущности "Контакт" и "Учетная запись" отображаются в различных приложениях и рабочих областях. Используйте меню рабочей области для перехода между рабочими областями текущего приложения.

- *Сущности* представляют данные определенного типа, например контакты и учетные записи. Сущности используют структурированный формат данных, который определяет коллекцию полей, доступных для сущности. Каждая сущность состоит из коллекции отдельных записей. Например, для сущности "Контакт" каждая запись описывает одного человека и включает набор полей, таких как имя, фамилия и адрес электронной почты. Сущности обычно представляют два представления: представление списка (таблица с перечнем доступных записей) и представление формы, в котором отображаются все доступные данные и параметры для одной записи. Используйте боковой навигатор для перемещения между сущностями в текущей рабочей области.

### <a name="move-between-apps"></a>Перемещение между приложениями

Используйте меню выбора приложений для переключения между приложениями.

![Меню выбора приложений](media/app-selector.png "Меню выбора приложений")

В меню выбора приложений отображаются приложения, к которым у вас есть доступ. 

### <a name="move-between-entities-records-and-work-areas"></a>Перемещение между сущностями, записями и рабочими областями

Вы можете легко перемещаться между элементами и возвращаться к избранным или наиболее часто используемым записям. На следующей иллюстрации показаны основные элементы навигации.

![Элементы управления навигацией, развернутое представление](media/nav-expanded.png "Элементы управления навигацией, развернутое представление")

Условные обозначения

1. **Выбор приложений**. Откройте это меню для перемещения между приложениями.
2. **Кнопка "Свернуть или развернуть"** . Выберите этот параметр, чтобы свернуть навигатор, освободив место для основной части страницы. Если навигатор уже свернут, нажмите эту кнопку, чтобы снова развернуть его.
3. **Последние записи**. Разверните эту запись, чтобы просмотреть список недавно использованных записей. Выберите здесь запись, чтобы открыть ее. Щелкните значок кнопки рядом с записью, чтобы добавить ее в избранное (закрепленные записи).
4. **Избранные записи**. Разверните эту запись, чтобы просмотреть и открыть избранные (закрепленные) записи. Для добавления записей сюда используйте список **Последние записи**. Щелкните значок удаления рядом с записью в списке, чтобы удалить ее из списка.
5. **Навигатор по сущностям**. В этой области перечислены все сущности и панели мониторинга, доступные для текущей рабочей области. Выберите любую запись, чтобы открыть именованную панель мониторинга или представление списка для этой сущности.
6. **Выбор рабочей области**. Откройте это меню, чтобы перейти к другой рабочей области. Текущая рабочая область указана здесь.

## <a name="working-with-list-views"></a>Работа с представлениями списков

Обычно при первом открытии сущности отображается представление списка, содержащее список записей, относящихся к этой сущности, в формате таблицы. Например, если открыть сущность **Учетные записи**, отобразится список учетных записей.

![Типичное представление списка](media/list-view.png "Типичное представление списка")

Условные обозначения

1. **Выбор записи**. Выберите одну или несколько записей, установив флажок в этом столбце. В зависимости от того, где вы работаете, вы можете применить одну операцию ко всем выбранным записям одновременно с помощью кнопок на панели команд.
2. **Открытие записи**. Выберите любую запись в списке, чтобы открыть ее представление записей, в котором отображаются все сведения о записи. Обычно вы выбираете значение из столбца **Имя**, чтобы открыть запись из текущей сущности. Некоторые сущности предоставляют ссылки на записи из связанных сущностей в других столбцах (например, связанный контакт).
3. **Сортировка и фильтрация списка**. Выберите, чтобы отсортировать список по значениям в этом столбце или отфильтровать список по значениям в этом столбце. Стрелка в заголовке столбца указывает, какой столбец сортируется и в каком направлении. 
4. **Панель команд**. Используйте команды на панели команд для работы с записями в списке и выполнения связанных действий. Для некоторых команд (например, **Удалить**) необходимо сначала выбрать одну или несколько целевых записей, установив флажок в крайнем левом столбце, тогда как другие действуют для всего списка. Вы можете экспортировать в книгу Excel (возможно, на основе шаблона), открывать диаграммы и панели мониторинга и многое другое в зависимости от типа записей, с которыми вы работаете.
5. **Поиск представления**. Введите текст в поле поиска над списком, чтобы показать только те записи в текущем представлении, которые содержат текст.
6. **Фильтрация и переход по страницам**. Выберите букву, чтобы отобразить только те записи, имена которых начинаются с этой буквы. Если список содержит больше записей, чем может отображаться на одной странице, воспользуйтесь стрелками перехода по страницам в нижней части списка, чтобы переходить вперед и назад по страницам.

## <a name="working-with-record-views"></a>Работа с представлениями записи

В представлениях записей отображаются все сведения об отдельной записи, а иногда также предоставляются специальные функции для работы с ней. Обычно вы открываете представление записей, выбрав запись, которая отображается в представлении списка, но можно также открыть представление записей по ссылке из связанной записи.

![Типичное представление записей](media/form-view.png "Типичное представление записей")

Условные обозначения


1. **Вкладки**: Большинство представлений записей делятся на вкладки. Каждая вкладка содержит коллекцию связанных полей из записи. Если вкладки доступны, они перечислены под именем записи. Выберите любое имя вкладки, чтобы открыть ее. Текущая вкладка подчеркнута.
2. **Связанные**. Почти все типы записей имеют вкладку **Связанные** после того, как вы хотя бы раз их сохраните. Эта вкладка представляет собой раскрывающийся список, который можно использовать для поиска других типов записей, использующих отображаемую запись или ссылающихся на нее. При выборе имени сущности из раскрывающегося списка **Связанные** открывается новая вкладка для этой сущности, в которой отображается список всех связанных записей этого типа. Вкладка **Связанные** остается доступной, и ее по-прежнему можно использовать для поиска других типов записей, ссылающихся на текущую.
3. **Панель команд**. Используйте команды на панели команд для работы с текущей записью или выполнения задачи, связанной с записью. Доступные команды зависят от типа записи, но обычно можно использовать панель команд, чтобы сохранить изменения, удалить запись, обновить страницу, отправить ссылку на запись по электронной почте, переназначить владельца записи или экспортировать запись с помощью шаблона Word.
4. **Строка заголовка**. В некоторых представлениях записей рядом с именем записи в строке заголовка отображается несколько особенно важных полей. Обычно это поля, которые являются фундаментальными для работы с записями текущего типа (например, имя записи или владелец записи).
5. **Просмотр и изменение значений всех полей**. В основном тексте представления записей можно найти все поля, связанные с текущей вкладкой, представлением формы и типом записи. Поля, отмеченные красной звездочкой, являются обязательными, и запись нельзя сохранить без допустимых значений. Поля, отмеченные синим знаком "плюс", особенно важны или рекомендуются, но не являются обязательными. Поля со значком блокировки доступны только для чтения и не могут быть изменены.

## <a name="notifications"></a>Уведомления 

Существует три типа уведомлений, отображаемых в форме: информация, предупреждение или ошибка. Уведомления всегда доступны наверху формы, прямо над заголовком.

При выборе уведомления об ошибке вы перейдете к полю в форме, в которой произошла ошибка.

![Пример уведомлений](media/notifications.png "Примеры уведомлений")

Условные обозначения

1. **Сведения**. Уведомление является информационным.
2. **Предупреждение**. Уведомление является предупреждением. 
3. **Ошибка**. Уведомление является ошибкой. 



### <a name="single-notification"></a>Одиночное уведомление

Если у вас только одно уведомление, оно отобразится на одной строке.

![Пример одиночного уведомления](media/single_notification.png "Пример одиночного уведомления")

### <a name="multiple-notifications"></a>Несколько уведомлений

Если имеется несколько уведомлений, вы увидите количество уведомлений. Щелкните шеврон, чтобы просмотреть каждое сообщение.

![Пример нескольких уведомлений](media/multiple_notification.png "Пример нескольких уведомлений")
