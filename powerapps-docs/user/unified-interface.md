# <a name="enhanced-user-experience-with-the-unified-interface-for-model-driven-apps"></a>Улучшенное взаимодействие с пользователем с единым интерфейсом для приложений на основе моделей 

Единый интерфейс для приложений на основе моделей обеспечивает единообразное и доступное взаимодействие с пользователем на разных устройствах — на настольном компьютере, ноутбуке, планшете или телефоне. Масштабирование приложений происходит за счет перетекания компонентов на экране. Высокодоступный дизайн адаптируется к вашей среде в зависимости от размера экрана, что позволяет показать дополнительные сведения, если присутствует дополнительное свободное место.

> [!div class="mx-imgBorder"]
> ![Единый интерфейс адаптируется к экрану](media/Reflow.png "Единый интерфейс адаптируется к экрану")

Общие сведения о едином интерфейсе в приложениях на основе моделей см. в этом видео: [Введение в единый интерфейс](https://www.youtube.com/watch?v=_VPOi_Iq6ko)

> [!NOTE]
> Устаревший веб-клиент станет нерекомендуемым, и клиентам нужно будет перейти на единый интерфейс до 1 октября 2020 г. Для получения дополнительных сведений см. статью [Часто задаваемые вопросы: опубликовано расписание перехода к единому интерфейсу](https://cloudblogs.microsoft.com/dynamics365/it/2019/09/10/announcing-the-timeline-to-move-to-unified-interface/). Дополнительные сведения о переходе см. в разделе [Краткое руководство по переходу](https://docs.microsoft.com/powerapps/maker/model-driven-apps/transition-web-app).

## <a name="navigation"></a>Навигация

Параметры меню позволяют быстро перемещаться между различными приложениями в системе. Они обеспечивают быстрый доступ к недавно просмотренным записям и закрепленному избранному.

![Элементы управления навигацией, развернутое представление](media/nav-expanded.png "Элементы управления навигацией, развернутое представление")

Условные обозначения

1. **Выбор приложений**. Откройте это меню для перемещения между приложениями.
1. **Кнопка "Свернуть или развернуть"** . Выберите этот параметр, чтобы свернуть навигатор, освободив место для основной части страницы. Если навигатор уже свернут, нажмите эту кнопку, чтобы снова развернуть его.
1. **Последние записи**. Разверните эту запись, чтобы просмотреть список недавно использованных записей. Выберите здесь запись, чтобы открыть ее. Щелкните значок кнопки рядом с записью, чтобы добавить ее в избранное (закрепленные записи).
1. **Избранные записи**. Разверните эту запись, чтобы просмотреть и открыть избранные (закрепленные) записи. Для добавления записей сюда используйте список **Последние записи**. Щелкните значок удаления рядом с записью в списке, чтобы удалить ее из списка.
1. **Навигатор по сущностям**. В этой области перечислены все сущности и панели мониторинга, доступные для текущей рабочей области. Выберите любую запись, чтобы открыть именованную панель мониторинга или представление списка для этой сущности.
1. **Выбор рабочей области**. Откройте это меню, чтобы перейти к другой рабочей области. Текущая рабочая область указана здесь.

Дополнительные сведения см. в статье [Базовая навигация в приложении на основе модели](navigation.md).

## <a name="dashboards-and-charts"></a>Панели мониторинга и диаграммы
Вы можете получить доступ ко всем системным и пользовательским панелям мониторинга в приложениях с единым интерфейсом. Интерактивные панели мониторинга с расширенными возможностями взаимодействия теперь доступны для всех типов записей. Дополнительные сведения см. в разделе [Отслеживание хода выполнения с помощью панелей мониторинга и диаграмм](track-your-progress-with-dashboard-and-charts.md).

## <a name="timeline-control"></a>Элемент управления "Временная шкала" 
Представление временной шкалы помогает совместно работать с командой путем отслеживания взаимодействия с клиентами в записи на одной странице в удобном для чтения виде. Здесь можно видеть сразу все, от записей и голосовых вложений до сообщений электронной почты и примечаний. Оно обеспечивает быстрый способ просмотра всего потока сообщений. Дополнительные сведения см. в разделе [Добавление встречи, электронной почты, телефонного звонка, заметок или задачи во временной шкале](add-activities.md).

## <a name="business-process"></a>Бизнес-процесс 
Поток бизнес-процессов был улучшен за счет добавления механизма стыковки. Вы можете пристыковать этап бизнес-процесса на экране, чтобы сосредоточиться на текущей задаче в потоке бизнес-процесса. Это особенно полезно, когда этап процесса содержит сложные для выполнения шаги. Дополнительные сведения см. в статье [Работа с бизнес-процессами](work-with-business-processes.md).

## <a name="accessibility"></a>Возможности доступа
Улучшенный интерфейс позволяет использовать средства чтения с экрана для преобразования информации на экране в звук и печати на устройстве Брайля, чтобы самые разные пользователи могли использовать приложение. Дополнительные сведения см. в статье [Использование средства чтения с экрана](screen-reader.md).

## <a name="create-a-unified-interface-app"></a>Создание приложения с единым интерфейсом
Если у вас есть требования для создания собственного проекта на основе единого интерфейса, можно создать приложение на основе модели с помощью конструктора приложений. См. раздел [Общие сведения о создании приложения на основе модели](https://docs.microsoft.com/powerapps/maker/model-driven-apps/model-driven-app-overview).
