# <a name="run-canvas-apps-and-model-driven-apps-on-a-mobile-device"></a>Запуск приложений на основе холста и приложений на основе моделей на мобильном устройстве

[!INCLUDE [cc-beta-prerelease-disclaimer](../includes/cc-beta-prerelease-disclaimer.md)]

При создании или получении совместного доступа к &mdash;приложению на основе холста или приложению на основе модели&dash; вы можете запускать это приложение на устройствах с iOS и Android с помощью мобильного приложения Power Apps. Если вы используете устройство с Windows, можно запускать только приложения на основе холста. Приложения на основе моделей не поддерживаются в мобильном приложении Power Apps для устройств с Windows. В этом разделе вы узнаете, как начать работу, чтобы запустить приложение на основе холста или приложение на основе модели на мобильном устройстве. 

Сведения об использовании приложений на основе модели, работающих в мобильном приложении Power Apps, см. в разделе [Руководство пользователя по приложениям на основе моделей, выполняющихся в мобильном приложении Power Apps](use-custom-model-driven-app-on-mobile.md).

> Приложения на основе моделей для Dynamics 365 Sales, Dynamics 365 Customer Service, Dynamics 365 Field Service, Dynamics 365 Marketing и Dynamics 365 Project Service Automation<!--Via Dynamics Style Guide. Дополнительные сведения: [Руководство пользователя по Dynamics 365 для телефонов и планшетов](https://docs.microsoft.com/dynamics365/mobile-app/dynamics-365-phones-tablets-users-guide)

1. **Приложения на основе модели**

**Чтобы зарегистрироваться для использования предварительной версии приложений Power Apps для мобильных устройств**

1. Установите [приложения Power Apps для iOS](https://go.microsoft.com/fwlink/?linkid=2125171) или [приложения Power Apps для Android](https://go.microsoft.com/fwlink/?linkid=2125172).
2. Если приложение уже установлено, то оно будет заменено предварительной версией приложения.
3. Не забудьте вручную выполнить обновление до последней доступной версии, чтобы получить преимущества от последних улучшений. 
3. После установки приложения можно начать тестирование. Чтобы дать отзыв, обратитесь к нам по адресу [pamobsup@microsoft.com](mailto:pamobsup@microsoft.com). 

## <a name="open-power-apps-and-sign-in"></a>Открытие и вход в Power Apps

Откройте Power Apps на мобильном устройстве и выполните вход, используя учетные данные Azure Active Directory.

![Войдите в Power Apps](media/powerapps_mobile_app_signin_screen.png "Вход в Power Apps")

Если на мобильном устройстве установлено приложение Microsoft Authenticator, просто введите имя пользователя, когда появится запрос, а затем подтвердите уведомление, отправленное на устройство.


## <a name="find-the-app"></a>Поиск приложения

При входе в приложение по умолчанию установлен фильтр **Мои приложения**. Если искомое приложение не найдено, можно открыть меню **Power Apps**, а затем выбрать другой фильтр. 


![Фильтры приложений](media/filter-menu.png "Фильтры приложений")

Доступны указанные ниже фильтры.

* **Все приложения**: отображаются все приложения на основе холста и приложения на основе моделей, к которым у вас есть доступ, включая созданные вами и предоставленные вам в совместный доступ.

* **Мои приложения**: отображаются приложения на основе холста, которые вы открыли, приложения, владельцем которых вы являетесь, и приложения, которые вы можете изменять. В случае приложений на основе модели отображаются все приложения, к которым у вас есть доступ. 

* **Примеры приложений** (только для приложений на основе холста): отображаются предоставленные корпорацией Майкрософт примеры приложений на основе холста с вымышленными данными, которые помогут вам изучить возможности разработки.

* **Избранное** (только для приложений на основе холста): отображаются приложения на основе холста, которые вы пометили, нажав многоточие (...) на плитке приложения и выбрав пункт **Избранное**. Чтобы удалить приложение из этого списка, нажмите многоточие (...) на плитке приложения и выберите пункт **Удалить из избранного**.

    ![Добавить в избранное](media/add_favorite_app.png "Добавление в избранное")

* **Подборка приложений** (только для приложений на основе холста): отображаются приложения на основе холста, помеченные администратором как "Рекомендуемые приложения".

### <a name="sort-apps"></a>Сортировка приложений

Отфильтровав приложения, можно отсортировать полученный список по дате последнего открытия или изменения приложения либо по имени в алфавитном порядке. Настройки сортировки сохраняются, если вы закроете и снова откроете приложения. Сортировать можно как приложения на основе холста, так и на основе моделей.

![Меню сортировки](media/sort_apps.png "Меню сортировки")

### <a name="search-apps"></a>Поиск приложений

Если вы знаете имя приложения, которое нужно запустить, можно коснуться значка поиска в верхней части окна, а затем ввести часть этого имени в поле поиска. Искать можно как приложения на основе холста, так и на основе моделей.

![Поиск приложений](media/search_apps.png "Поиск приложений")

Если приложения отфильтрованы, поиск будет производиться в отфильтрованном списке.

### <a name="refresh-the-list-of-apps"></a>Обновление списка приложений

Щелкните значок обновления ![Значок обновления](media/refresh_icon.png) для обновления списка приложений. В результате будут обновлены списки приложений на основе холста и приложений на основе моделей. 

## <a name="pin-an-app-to-the-home-screen"></a>Закрепление приложения на начальном экране

Вы можете закреплять как приложения на основе холста, так и приложения на основе модели на начальном экране устройства для быстрого доступа. Коснитесь многоточия (...) на плитке приложения, выберите пункт **Закрепить на начальном экране** и следуйте инструкциям на экране.

![Закрепление приложения](media/pin_to_home.png "Закрепление приложения")

## <a name="see-non-production-apps"></a>Просмотр приложений, отличных от рабочих

По умолчанию в списке приложений отображаются только рабочие приложения на основе модели.

Чтобы просмотреть приложения на основе модели из нерабочих сред, выберите меню **Параметры** ![значок "Параметры"](media/settings_icon.png), а затем включите параметр **Показать приложения, отличные от рабочих**. Следуйте появившимся инструкциям.

![Переключатель "Приложения, отличные от рабочих"](media/non_prod_toggle.png "Переключатель "Приложения, отличные от рабочих"")

## <a name="run-an-app"></a>Запуск приложения

Чтобы запустить приложение на мобильном устройстве, коснитесь плитки приложения. Если другой пользователь создал приложение и предоставил вам доступ к нему по электронной почте, его можно запустить, перейдя по ссылке в сообщении.

### <a name="run-a-canvas-app"></a>Запуск приложения на основе холста

Если приложение на основе холста выполняется в первый раз с помощью мобильного приложения Power Apps, на экране отобразятся жесты прокрутки.

#### <a name="close-a-canvas-app"></a>Закрытие приложения на основе холста

Чтобы закрыть приложение, проведите пальцем от левого края окна приложения вправо. На устройствах с Android можно также нажать кнопку перехода назад и подтвердить намерение закрыть приложение.

![Закрытие приложения](media/swipe.gif "Закрытие приложения")

#### <a name="pinch-and-zoom-in-on-a-canvas-app"></a>Сжатие и увеличение масштаба приложения на основе холста

![Сжатие для увеличения](media/pinchtozoom.jpg "Сжатие для увеличения")

#### <a name="give-consent-to-a-canvas-app"></a>Предоставление согласия для приложения на основе холста

Если приложению требуется подключение к источнику данных или разрешение на использование возможностей устройства (например, камеры или служб определения местоположения), вам необходимо дать согласие, прежде чем использовать приложение. Обычно такой запрос появляется только при первом запуске приложения.

![Предоставление согласия](media/give_consent_canvas.jpg "Предоставление согласия")

Сведения об использовании приложений на основе модели, работающих в мобильном приложении Power Apps, см. в разделе [Руководство пользователя по приложениям на основе моделей, выполняющихся в мобильном приложении Power Apps](use-custom-model-driven-app-on-mobile.md).

Если приложению требуется подключение к источнику данных или разрешение на использование возможностей устройства (например, камеры или служб определения местоположения), вам необходимо дать согласие, прежде чем использовать приложение. Обычно такой запрос появляется только при первом запуске приложения.

![Предоставление согласия](media/give_consent.png "Предоставление согласия")

#### <a name="close-a-model-drive-app"></a>Закрытие приложения на основе модели

Выберите карту сайта ![Значок карты сайта](media/pa_mobile_sitemap_icon.png "Значок карты сайта"), а затем выберите **Приложения**.

![Закрытие приложения на основе модели](media/pa_mobile_close_app.png "Закрытие приложения на основе модели")

## <a name="known-issues"></a>Известные проблемы

Мы по-прежнему работаем над некоторыми известными проблемами и выпускаем исправления для них с течением времени. Не забудьте вручную обновить предварительную версию сборки, как только обновление станет доступно. 

Некоторые известные проблемы:

- При открытии приложения Power Apps в автономном режиме приложения не отображаются в списке приложений.
- Значки внутри приложений на основе модели иногда исчезают.
