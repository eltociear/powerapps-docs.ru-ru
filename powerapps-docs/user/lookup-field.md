#  <a name="use-the-lookup-field-on-a-record"></a>Использование поля подстановки в записи

Подстановка позволяет выбрать записи из связанной сущности. При выборе связанной сущности и вводе условий поиска, таких как имя или адрес электронной почты, подстановка автоматически начинает разрешать частичный текст и отображает все совпадающие записи. Если после ввода полного текста условий поиска никакие записи не отображаются, отображается сообщение об отсутствии записей.

Например, можно выполнить поиск по имени **Эдриан Думитраску**. При вводе **Эд** автоматически отображаются совпадающие записи.

  > [!div class="mx-imgBorder"]
  > ![Автоматически заполняет совпадающие записи](media/automatically-populate-matching-records.png "Автоматически заполняет совпадающие записи")
  
>[!NOTE] 
>Администратор может определить критерии, используемые подстановкой для разрешения частичного поиска текста.

Кроме того, можно создать новую запись, нажав кнопку **Создать**. Необходимо иметь достаточные разрешения, чтобы кнопка **Создать** отображалась и вы могли создать запись. При выборе поля подстановки отображаются пять последних использованных записей вместе с пятью избранными записями. Отображаемые записи зависят от истории просмотра и закрепленного избранного. 

Например, если в журнале имеется только три записи, подстановка отобразит эти три записи, а также семь избранных записей. Если у вас нет закрепленных избранных записей, будут отображаться только самые последние просмотренные записи.

## <a name="types-of-lookups"></a>Типы подстановок

Подстановки классифицируются следующим образом. 

- **Простая подстановка.** Выберите одну запись в поле из одной сущности. 

- **Поля типа PartyList.** Используется для выбора нескольких записей из нескольких сущностей в подстановке. Используйте поля PartyList, чтобы выбрать несколько записей. Это позволяет добавить каждую запись, выполняя новый поиск несколько раз. Каждый раз, когда вы выбираете запись, вы сможете выполнить новый поиск для другой записи.
  
- **Поля по теме.** Используется для выбора одной записи из нескольких сущностей в подстановке. 

## <a name="search-in-a-lookup-field"></a>Поиск в поле подстановки 
Чтобы выполнить поиск по подстановке, выберите текстовое поле и введите условия поиска. Если для подстановки включены последние записи, они будут отображаться при выборе текстового поля.

  > [!div class="mx-imgBorder"]
  > ![Просмотр поля подстановки](media/MRU.png "Просмотр поля подстановки")  
  
>[!NOTE]   
> Результат поиска по умолчанию для поиска подстановки **учитывает начальные слова**. Это означает, что результаты включают записи, которые начинаются с определенного слова. Например, если вы хотите найти **Горнолыжная база**, введите **гор** в поле поиска. Если ввести **база**, запись не будет отображаться в результатах поиска.
>
> Для поиска с подстановочными знаками используйте звездочки. Например, введите \*база или \*база\*.

Чтобы просмотреть подстановку, щелкните значок поиска (лупа). В раскрывающемся списке будет отображаться полный список элементов.

## <a name="create-a-new-record-if-you-dont-find-an-existing-record"></a>Создание новой записи, если не найдена существующая запись

Если запись не найдена, выберите **Создать** в области поиска, чтобы создать новую запись.


### <a name="replace-an-existing-record-from-a-lookup-field"></a>Замена существующей записи из поля подстановки

Можно заменить существующую запись при помощи простой подстановки и подстановки по теме. Поиск записи. Затем выберите запись и замените ее новой записью.

### <a name="change-a-view-in-a-lookup-field"></a>Изменение представления в поле подстановки 

Выбор варианта **Изменить представление** позволяет определить следующее.
 - Как просматривать записи, например **Контакты по подписке**, **Представление подстановки контактов** или **Активные контакты**.
 - Данные, которые нужно просмотреть в записях, например имя, адрес электронной почты или номер телефона. Например, если вы хотите просмотреть только контакты, на которые вы подписаны, выберите **Изменить представление** \> **Контакты по подписке**. Будут показаны только те контакты, на которые вы подписаны, как показано здесь. 

    ![Изменить тип представления контактов](media/change-view.png "Изменить тип представления контактов")

### <a name="filter-by-only-my-records-or-filter-by-related-primary-contact"></a>Фильтрация по собственным записям или по связанному основному контакту

Чтобы применить дополнительные фильтры, в меню **Изменить представление** выберите пункт **Только мои записи** или **Фильтр по связанному основному контакту**.

![Добавить еще фильтры](media/extra_filters.png "Добавить еще фильтры")

### <a name="choose-from-multiple-records"></a>Выбор из нескольких записей

Если в подстановке больше записей, чем может поместиться в доступной области отображения, область отображения сворачивается, то есть записи, которые помещаются в отображаемую область, отображаются рядом с числом записей, которые не поместились. Чтобы просмотреть все записи, нажмите на это число. На следующих изображениях показано различие между свернутыми и не свернутыми полями.
