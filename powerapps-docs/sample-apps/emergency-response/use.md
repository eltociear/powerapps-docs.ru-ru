# <a name="use-the-hospital-emergency-response-mobile-app"></a>Использование мобильного приложения для службы экстренной медицинской помощи

Персонал больницы сталкивается с проблемой увеличения числа пациентов и управления цепочкой поставок при чрезвычайных ситуациях. Используя мобильное приложение для службы экстренной медицинской помощи соответствующие сотрудники могут быстро просматривать и добавлять данные об аппаратах ИВЛ, количестве персонала, выписках пациентов, а также пациентах с COVID-19.

## <a name="prerequisites"></a>Предварительные требования

Чтобы приступить к работе с мобильным приложением Power Apps Mobile, его необходимо скачать в магазине приложений мобильного устройства.

- **Скачайте** [**Power Apps Mobile**](https://powerapps.microsoft.com/downloads).
    - Для устройств **Apple** с iOS, например iPhone и iPad, используйте [**App Store**](https://aka.ms/powerappsios).
    - Для устройств **Android** используйте [**Google Play**](https://aka.ms/powerappsandroid).
- Убедитесь, что ваша организация развернула и настроила приложение для службы экстренной медицинской помощи, как описано в статье [Развертывание и настройка приложения для службы экстренной медицинской помощи](deploy-configure.md).

После установки Power Apps Mobile откройте приложение на устройстве и выполните вход, используя учетную запись Azure Active Directory своей компании. После входа вы можете просмотреть все приложения с общим доступом для вас, предоставленным вашей организацией. Дополнительные сведения см. в разделе о входе в Power Apps с помощью мобильного устройства см. [здесь](https://docs.microsoft.com/powerapps/user/run-app-client#open-power-apps-and-sign-in).

## <a name="hospital-emergency-response-mobile-app"></a>Мобильное приложение для службы экстренной медицинской помощи

![Приложение для службы экстренной медицинской помощи](media/use/app-launcher.png)

Мобильное приложение для службы экстренной медицинской помощи имеет модульную структуру и предусматривает различные приложения в зависимости от вашей роли. Откройте мобильное приложение для службы экстренной медицинской помощи из Power Apps Mobile, выберите значения в полях **Hospital system** (Система больницы), **Region, Facility** (Регион, учреждение), а затем нажмите кнопку **Далее**, чтобы приступить к работе.

> [!NOTE]
> Когда вы запускаете мобильное приложение для службы экстренной медицинской помощи или любой из его компонентов *первый раз*, вам будет предложено разрешить приложению считывать данные профиля *Пользователи Office 365* и данные о вашем *расположении*. Чтобы можно было использовать выбранное приложение, необходимо щелкнуть **Разрешить**. Дополнительные сведения см. в разделе [Предоставление согласия](https://docs.microsoft.com/powerapps/user/run-app-client#give-consent).

## <a name="app-components"></a>Компоненты приложения

![Компоненты мобильного приложения для службы экстренной медицинской помощи](media/use/app-components.png)

Пример решения для службы экстренной медицинской помощи состоит из нескольких приложений для повышения удобства работы пользователей. В зависимости от вашей роли вы можете увидеть один или несколько компонентов в **приложении для службы экстренной медицинской помощи**.

- **Bed capacity**
     (Количество коек)<br> Сбор информации о койках (общее разрешенное количество, для интенсивной терапии и т. д.).

- **COVID-19 stats**
     (Статистика COVID-19)<br> Собирает данные о количестве пациентов на обследовании относительно COVID-19 и пациентов с положительным результатом.

- **Equipment**
     (Оборудование)<br> Отслеживание сведений об аппаратах ИВЛ, НИВПД и PAPR.

- **Staff**
     (Персонал)<br> Получение сведений о количестве пациентов и зарегистрированных медсестер: партнерские ресурсы, назначенные, запрошенные и не назначенные.

- **Supplies**
     (Снабжение)<br> Обрабатывает данные об основных категориях снабжения для более эффективного отслеживания и прогнозирования запасов, а также управления ими. 

- **Staffing needs**
     (Потребности в персонале)<br> Собирает запросы относительно персонала по подразделениям, ролям и срочности.

- **Discharge planning**
     (Планирование выписки)<br> Собирает данные о текущей ситуации с выписками и проецировании выписок.

> [!NOTE]
> По умолчанию информацию можно отслеживать на уровне *расположения* в следующих приложениях: **COVID-19 stats** (Статистика COVID-19), **Equipment** (Оборудование) и **Staff** (Персонал). В остальных приложениях информацию можно по умолчанию отслеживать на уровне *учреждения*. При необходимости администратор может изменить уровень отслеживания по умолчанию. Дополнительные сведения: [Управление уровнем отслеживания для мобильных приложений](configure-data-reporting.md#manage-tracking-level-for-mobile-apps)

## <a name="bed-capacity"></a>Bed capacity (Количество коек)

![Bed capacity (Количество коек)](media/use/bed-capacity.png)

Отправьте сведения о койках, например об их общем разрешенном количестве, предназначенных для интенсивной терапии (с изоляцией от воздушно-капельных болезней и без), неотложной помощи (с изоляцией от воздушно-капельных болезней и без), а также сведения об укомплектованности для обслуживания всего количества разрешенных коек.

Щелкните значок **Назад** вверху слева, если хотите вернуться к **приложению для службы экстренной медицинской помощи** без отправки каких-либо изменений. Кнопка **Отправить** позволяет отправить введенные значения.

После отправки данных можно вернуться в приложение **Bed capacity** (Количество коек), чтобы создать еще одну запись с помощью кнопки **Track another** (Отслеживать другое). Щелкните **Главная**, чтобы вернуться в **приложение для службы экстренной медицинской помощи**.

### <a name="fields-and-description"></a>Поля и описание

| **Имя параметра**                                               | **Описание**                                                                       |
|---------------------------------------------------------------|---------------------------------------------------------------------------------------|
| How many licensed beds are currently in use in this facility? | Общее количество разрешенных коек, используемых в настоящее время в этим учреждении.                            |
| Number of ICU Beds (AIIR Room) currently in use               | Число коек для интенсивной терапии (в палатах с изоляцией от воздушно-капельных болезней), которые сейчас используются.                                      |
| Number of ICU Beds (non-AIIR Room) currently in use           | Число коек для интенсивной терапии (в палатах без изоляции от воздушно-капельных болезней), которые сейчас используются.                                  |
| Number of Acute Care Beds (AIIR Room) currently in use        | Число коек для неотложной помощи (в палатах с изоляцией от воздушно-капельных болезней), которые сейчас используются.                               |
| Number of Acute Care Beds (non-AIIR Room) currently in use    | Число коек для неотложной помощи (в палатах без изоляции от воздушно-капельных болезней), которые сейчас используются.                           |
| Is your facility staffed to its full license bed capacity?    | Укомплектовано ли учреждение всем необходимым для обслуживания всего количества разрешенных коек — Yes (Да) или No (Нет). При ответе "Нет" можно указать недостающие ресурсы: <br> - Staff (Персонал); <br> - Space (Помещения); <br> - PPE (Защитное оборудование); <br> - Equipment (Оборудование); <br> - Low Patient Volume (Мало пациентов). |
| Are you able to surge beyond your licensed beds?              | Укомплектовано ли учреждение всем необходимым для обслуживания всего количества разрешенных коек — Yes (Да) или No (Нет). При ответе "Нет" можно указать недостающие ресурсы: <br> - Staff (Персонал); <br> - Space (Помещения); <br> - PPE (Защитное оборудование); <br> - Equipment (Оборудование); <br> - Low Patient Volume (Мало пациентов).  |
| Number of surge beds currently in use                         | Число дополнительных коек, используемых в настоящее время.                                                |

## <a name="staff"></a>Персонал

![Персонал](media/use/staff.png)

Отправьте сведения на основе местоположения о текущей ситуации относительно пациентов, дипломированных медсестер и оборудования. Вам не нужно вводить значения для каждого поля на экране. Введите число в поле, которое необходимо сохранить в базе данных решения.

Например, если есть запросы на трех дипломированных медсестер и необходимо добавить это количество, введите 3 в поле **Registered nurses on duty - Requested** (Дипломированные медсестры на дежурстве — запрос) и выберите **Submit** (Передать). Если вы также хотите указать для параметра **Registered nurses on duty - Assigned** (Дипломированные медсестры на дежурстве — назначено) значение 6, введите 3 в поле **Registered nurses on duty — Requested** (Дипломированные медсестры на дежурстве — запрос), а затем введите 6 для параметра **Registered nurses on duty - Assigned** (Дипломированные медсестры на дежурстве — назначено) и нажмите кнопку **Submit** (Передать).

Щелкните значок **Назад** вверху слева, если хотите вернуться к **приложению для службы экстренной медицинской помощи** без отправки каких-либо изменений. Кнопка **Отправить** позволяет отправить введенные значения.

После отправки данных можно вернуться в приложение **Staff** (Персонал), чтобы создать еще одну запись с помощью кнопки **Track another** (Отслеживать другое). Щелкните **Главная**, чтобы вернуться в **приложение для службы экстренной медицинской помощи**.

### <a name="fields-and-description"></a>Поля и описание

| **Имя параметра**               | **Описание**                                                                                   |
|-------------------------------|---------------------------------------------------------------------------------------------------|
| Расположение                      | Название и тип палаты, отделения, а также любое другое специализированное расположение в выбранной больнице. |
| Число пациентов            | Текущее общее число пациентов в выбранном расположении.                                        |
| **Дипломированные медсестры на дежурстве** |                                                                                                   |
| *Партнеры*                    | Число медбратьев в выбранном расположении.                             |
| *Requested* (Запрошены)                   | Число дипломированных медсестер, запрошенных для выбранного расположения.                                  |
| *Assigned* (Назначены)                    | Число дипломированных медсестер, назначенных для выбранного расположения.                                    |
| *Unassigned* (Не назначены)                  | Число дипломированных медсестер, не назначенных для выполнения какой-либо задачи для выбранного расположения.                    |
| **Facility level tracking** (Отслеживание на уровне учреждения) |                                                                                                   |
| % of essential care personnel currently absent in the this facility                  | Нехватка штата основного медперсонала в процентах от общего штатного расписания для **всего учреждения**.                    |

### <a name="fields-and-description"></a>Поля и описание

Список элементов приложения Supplies (Снабжение) может отличаться в зависимости от требований организации. Описание названий элементов поставок см. в ресурсах вашей организации.

ИТ-администраторы могут добавлять или обновлять список элементов приложения Supplies (Снабжение) с помощью приложения на основе моделей для Power Apps. Дополнительные сведения см. в статье [Развертывание и настройка приложения Emergency Response](deploy-configure.md).

> Значения позиций запасов вспомогательных материалов должны быть в числовом формате.

Щелкните значок **Назад** вверху слева, если хотите вернуться к **приложению для службы экстренной медицинской помощи** без отправки каких-либо изменений. Кнопка **Отправить** позволяет отправить введенные значения. Выберите **Главная**, чтобы вернуться в **приложение для службы экстренной медицинской помощи** после отправки.

### <a name="fields-and-description"></a>Поля и описание

| **Имя поля**           | **Описание**                                                                            |

| Требуется сейчас или для следующей смены | Выберите смену для запрошенной рабочей силы: текущая или будущая смена.                |
| How many (Количество)                 | Необходимое количество ресурсов в числовом формате.                |
| Сведения                  | Введите дополнительные сведения или комментарии для запроса по резерву рабочей силы.                        |



## <a name="discharge-planning"></a>Discharge planning (Планирование выписки)

![Выписка пациентов из стационара](media/use/discharge.png)

Отправьте сведения о выписке пациентов из стационара, состоянии пациентов и их общем количестве, используя приложение  **Discharge planning** (Планирование выписки). Вы можете обновить сведения о выписке за последние 24 часа, текущих препятствиях выписке и их преодолении.

Щелкните значок **Назад** вверху слева, если хотите вернуться к **приложению для службы экстренной медицинской помощи** без отправки каких-либо изменений. Кнопка **Отправить** позволяет отправить введенные значения. Выберите **Главная**, чтобы вернуться в **приложение для службы экстренной медицинской помощи** после отправки.

### <a name="fields-and-description"></a>Поля и описание

| **Имя поля**            | **Описание**                                                    |
|---------------------------|--------------------------------------------------------------------|
| Авторизация             | Количество пациентов в процессе авторизации.                   |
| Durable medical equipment (Медицинское оборудование длительного пользования) | Количество пациентов, использующих медицинское оборудование длительного пользования.            |
| Guardianship (Режим карантина)              | Количество пациентов в режиме карантина.                             |
| Durable medical equipment (Услуги, предоставляемые по месту жительства) | Количество пациентов, получающих услуги, предоставляемые по месту жительства.               |
| Размещение                 | Необходимое количество мест.                                       |
| Skilled Nursing Facility (Центр сестринского ухода)  | Количество центров сестринского ухода.                              |
| **Discharges** (Выписки)            |                                                                    |
| Past 24 h (За следующие 24 часа)                 | Ожидаемое количество пациентов, которые должны быть выписаны за следующие 24 часа.  |
| Likely next 24 h (За последние 24 ч)          | Количество пациентов, выписанных за последние 24 часа.                    |

## <a name="other-options"></a>Другие варианты

В этом разделе описываются другие действия, которые можно выполнить с помощью компонентов мобильного приложения для службы экстренной медицинской помощи.

### <a name="end-shift---sign-out"></a>End shift — sign out (Закончить смену — выход)

Вы можете выйти из приложения, используя значок профиля в верхней левой части экрана.  

![Функция выхода](media/use/sign-out.png)

Чтобы завершить сеанс и выйти из него, нажмите кнопку **End shift** (Завершить смену).

> [!NOTE]
> Функция *End shift* (Завершить смену) может быть недоступной, если ИТ-администратор отключил общий доступ к устройствам.

### <a name="app-feedback"></a>Отзыв о приложении

Вы можете отправить отзыв с помощью кнопки **App feedback** (Отзыв о приложении) из любого раздела мобильного приложения для службы экстренной медицинской помощи. Чтобы поделиться своими отзывами, выберите профиль в левом верхнем углу, а затем нажмите кнопку **Submit feedback** (Отправить отзыв).

![Отзывы](media/use/give-feedback.png)

При выборе параметра **App feedback** (Обратная связь приложения) вы можете выразить благодарность, поделиться мнением или сообщить о проблеме в работе приложения.

### <a name="switch-facility"></a>Switch facility (Сменить учреждение)

![Switch facility (Сменить учреждение)](media/use/switch-facility.png)

Сменить учреждение можно в любое время, выбрав название в верхней правой части экрана. После выбора имени расположения вы перейдете на экран **приложения для службы экстренной медицинской помощи**, где можно выбрать другую больницу, другой регион или другое медицинское учреждение.
