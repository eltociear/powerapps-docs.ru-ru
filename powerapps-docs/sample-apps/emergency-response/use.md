# <a name="use-the-emergency-response-app"></a>Использование приложения для реагирования на чрезвычайные ситуации

Персонал больницы сталкивается с проблемой увеличения числа пациентов и управления цепочкой поставок при чрезвычайных ситуациях. Используя мобильное приложение для реагирования на чрезвычайные ситуации соответствующие сотрудники могут быстро просматривать и добавлять данные об аппаратах ИВЛ, количестве персонала, выписках пациентов, а также пациентах с COVID-19.

## <a name="prerequisites"></a>Предварительные требования

Чтобы приступить к работе с приложением, необходимо скачать Power Apps Mobile на устройство, используя магазин приложений устройства.

- **Скачайте** [**Power Apps Mobile**](https://powerapps.microsoft.com/downloads).
- Для устройств **Apple** с iOS, например iPhone и iPad, используйте [**App Store**](https://aka.ms/powerappsios).
- Для устройств **Android** используйте [**Google Play**](https://aka.ms/powerappsandroid).

После установки Power Apps Mobile откройте приложение на устройстве и выполните вход, используя учетную запись Azure Active Directory своей компании. После входа вы можете просмотреть все приложения с общим доступом для вас, предоставленным вашей организацией. Дополнительные сведения см. в разделе о входе в Power Apps с помощью мобильного устройства см. [здесь](https://docs.microsoft.com/powerapps/user/run-app-client#open-power-apps-and-sign-in).

## <a name="app-launcher"></a>Средство запуска приложений

![Средство запуска приложений](media/use/app-launcher.png)

Мобильное приложение для реагирования на чрезвычайные ситуации имеет модульную структуру и различные приложения в зависимости от вашей роли. Откройте мобильное приложение для реагирования на чрезвычайные ситуации из Power Apps Mobile, выберите значения в полях **Hospital system** (Система больницы), **Region, Facility** (Регион, учреждение), а затем нажмите кнопку **Далее**, чтобы приступить к работе.

> [!NOTE]
> Когда вы запускаете средство запуска мобильных приложений для реагирования на чрезвычайные ситуации или любой из его компонентов *первый раз*, вам будет предложено разрешить приложению считывать данные профиля *Пользователи Office 365* и данные о вашем *расположении*. Чтобы можно было использовать выбранное приложение, необходимо щелкнуть **Разрешить**. Дополнительные сведения см. в разделе [Предоставление согласия](https://docs.microsoft.com/powerapps/user/run-app-client#give-consent).

## <a name="app-components"></a>Компоненты приложения

![Компоненты мобильного приложения для реагирования на чрезвычайные ситуации](media/use/app-components.png)

Пример решения для реагирования на чрезвычайные ситуации состоит из нескольких приложений для повышения удобства работы пользователей. В зависимости от вашей роли вы можете увидеть одно или несколько приложений в **средстве запуска приложений** для мобильного приложения для реагирования на чрезвычайные ситуации.

- **Staff + equipment**
     (Персонал + оборудование)<br> Собирает данные о текущем количестве медсестер и критически необходимом оборудовании по расположению в конкретной больнице.

- **Supplies**
     (Снабжение)<br> Обрабатывает данные об основных категориях снабжения для более эффективного отслеживания и прогнозирования запасов, а также управления ими. 

- **Staffing needs**
     (Потребности в персонале)<br> Собирает запросы относительно персонала по подразделениям, ролям и срочности.

- **COVID-19 stats**
     (Статистика COVID-19)<br> Собирает данные о количестве пациентов на обследовании относительно COVID-19 и пациентов с положительным результатом.

- **Discharge planning**
     (Планирование выписки)<br> Собирает данные о текущей ситуации с выписками и проецировании выписок.

## <a name="staff--equipment"></a>Staff + equipment (Персонал + оборудование)

![Staff + equipment (Персонал + оборудование)](media/use/staff-equipment.png)

Отправьте сведения на основе местоположения о текущей ситуации относительно пациентов, дипломированных медсестер и оборудования. Список областей состоит из всех расположений, относящихся к конкретной больнице, выбранной в **средстве запуска приложений**. Выберите расположение из доступных вариантов, чтобы обновить другие поля.

После выбора области введите необходимые для полей значения, чтобы сохранить записи в базе данных решения. Вам не нужно вводить значения для каждого поля на экране. Введите число в поле, которое необходимо сохранить в базе данных решения.

Например, если есть запросы на 3 дипломированных медсестер и необходимо добавить это количество, введите 3 в поле **Registered nurses on duty - Requested** (Дипломированные медсестры на дежурстве — Запрос) и выберите **Отправить**. Если вы также хотите обновить число аппаратов ИВЛ в работе на значение 6, введите 3 в поле **Registered nurses on duty — Requested** (Дипломированные медсестры на дежурстве — запрошены), а затем введите 6 для параметра **Vents** (Аппараты ИВЛ) в разделе **Equipment in use** (Используемое оборудование) и нажмите кнопку **Отправить**.

Щелкните значок **Назад** вверху слева, если хотите вернуться к **средству запуска приложений** без отправки каких-либо изменений. Кнопка **Отправить** позволяет отправить введенные значения.

| **Имя параметра**               | **Описание**                                                                                   |
|-------------------------------|---------------------------------------------------------------------------------------------------|
| Расположение                      | Название и тип палаты, отделения, а также любое другое специализированное расположение в выбранной больнице. |
| Число пациентов            | Текущее общее число пациентов в выбранном расположении.                                        |
| **Дипломированные медсестры на дежурстве** |                                                                                                   |
| *Партнеры*                    | Число медбратьев в выбранном расположении.                             |
| *Requested* (Запрошены)                   | Число дипломированных медсестер, запрошенных для выбранного расположения.                                  |
| *Assigned* (Назначены)                    | Число дипломированных медсестер, назначенных для выбранного расположения.                                    |
| *Unassigned* (Не назначены)                  | Число дипломированных медсестер, не назначенных для выполнения какой-либо задачи для выбранного расположения.                    |
| **Equipment in use** (Используемое оборудование)          |                                                                                                   |
| *Vents* (Аппараты ИВЛ)                       | Число аппаратов ИВЛ, используемых в выбранном расположении.                                            |
| *PAPR hoods* (Капюшоны с фильтрующим респиратором с принудительной подачей воздуха)                  | Число капюшонов с фильтрующим респиратором с принудительной подачей воздуха, используемых в выбранном расположении.                 |
| *PAPR belts* (Пояса для фильтрующих респираторов с принудительной подачей воздуха)                  | Число поясов для фильтрующих респираторов с принудительной подачей воздуха, используемых в выбранном расположении.                 |
| *PAPR chargers* (Зарядные устройства для респираторов с принудительной подачей воздуха)               | Число зарядных устройств для фильтрующих респираторов с принудительной подачей воздуха, используемых в выбранном расположении.              |

## <a name="supplies"></a>Supplies (Снабжение)

![Supplies (Снабжение)](media/use/supplies.png)

Просмотрите запас вспомогательных материалов с помощью приложения **Supplies** (Снабжение). Вы можете обновить объем поставок вспомогательных материалов с помощью функции полной инвентаризации больницы и определения ежедневных затрат в этом приложении.

> [!NOTE]
> Введите значения в оба поля — **In stock** (В запасе) и **Used past 24 hr** (Использовались за последние 24 часа) перед нажатием кнопки "Отправить".

Щелкните значок **Назад** вверху слева, если хотите вернуться к **средству запуска приложений** без отправки каких-либо изменений. Кнопка **Отправить** позволяет отправить введенные значения.

### <a name="fields-and-description"></a>Поля и описание

Список элементов приложения Supplies (Снабжение) может отличаться в зависимости от требований организации. Описание названий элементов поставок см. в ресурсах вашей организации.

ИТ-администраторы могут добавлять или обновлять список элементов приложения Supplies (Снабжение) с помощью приложения на основе моделей для Power Apps. Дополнительные сведения см. в статье [Развертывание и настройка приложения Emergency Response](deploy-configure.md).

**Примечание.** Значения позиций запасов вспомогательных материалов должны быть в числовом формате.

## <a name="staffing-needs"></a>Staffing needs (Потребности в персонале)

![Staffing needs (Потребности в персонале)](media/use/staffing-needs.png)

Собирает данные о запросах по резерву рабочей силы для выбранного учреждения. Перед отправкой запроса по резерву рабочей силы для больницы убедитесь, что *обязательные* поля заполнены.

Щелкните значок **Назад** вверху слева, если хотите вернуться к **средству запуска приложений** без отправки каких-либо изменений. Кнопка **Отправить** позволяет отправить введенные значения.

### <a name="fields-and-description"></a>Поля и описание

| **Имя поля**           | **Описание**                                                                            |
|--------------------------|--------------------------------------------------------------------------------------------|
| Отдел               | Название подразделения, выполняющего запрос по резерву рабочей силы. Это *обязательное* поле.             |
| Расположение подразделения                 | Расположение, в котором находится подразделение.                                                                |
| Тип запроса             | Тип запроса по резерву рабочей силы: клинический и неклинический резерв. Это *обязательное* поле. |
| Role needed (Требуемая роль)              | Требуемая роль для запроса по резерву рабочей силы, например, сиделка или дипломированная медсестра.                          |
| Требуется сейчас или для следующей смены | Выберите смену для запрошенной рабочей силы: текущая или будущая смена.                |
| Количество | Требуемое количество в числовом формате.                |
| Сведения                  | Введите дополнительные сведения или комментарии для запроса по резерву рабочей силы.                        |

## <a name="covid-19-stats"></a>COVID-19 stats (Статистика COVID-19)

Можно также добавить другое расположение, используя кнопку **+ Add another location** (+ Добавить другое расположение) для отправки статистики относительно нескольких расположений.

Щелкните значок **Назад** вверху слева, если хотите вернуться к **средству запуска приложений** без отправки каких-либо изменений. Кнопка **Отправить** позволяет отправить введенные значения.

### <a name="fields-and-description"></a>Поля и описание

| **Имя поля**  | **Описание**                                                                                    |
|-----------------|----------------------------------------------------------------------------------------------------|
| Расположение        | Название и тип палаты, отделения, а также любое другое специализированное расположение в выбранной больнице.  |
| PUIs (Пациенты на обследовании)            | Количество пациентов на обследовании.                                                            |
| Положительное        | Число пациентов с положительным результатом анализов на COVID-19.                                                         |

## <a name="discharge-planning"></a>Discharge planning (Планирование выписки)

![Выписка пациентов из стационара](media/use/discharge.png)

Отправьте сведения о выписке пациентов из стационара, состоянии пациентов и их общем количестве, используя приложение  **Discharge planning** (Планирование выписки). Вы можете обновить сведения о выписке за последние 24 часа, текущих препятствиях выписке и их преодолении.

Щелкните значок **Назад** вверху слева, если хотите вернуться к **средству запуска приложений** без отправки каких-либо изменений. Кнопка **Отправить** позволяет отправить введенные значения.

### <a name="fields-and-description"></a>Поля и описание

| **Имя поля**            | **Описание**                                                    |
|---------------------------|--------------------------------------------------------------------|
| Авторизация             | Количество пациентов в процессе авторизации.                   |
| Durable medical equipment (Медицинское оборудование длительного пользования) | Количество пациентов, использующих медицинское оборудование длительного пользования.            |
| Guardianship (Режим карантина)              | Количество пациентов в режиме карантина.                             |
| Durable medical equipment (Услуги, предоставляемые по месту жительства) | Количество пациентов, получающих услуги, предоставляемые по месту жительства.               |
| Размещение                 | Необходимое количество мест.                                       |
| Skilled Nursing Facility (Центр сестринского ухода)  | Количество центров сестринского ухода.                              |
| **Discharges** (Выписки)            |                                                                    |
| Past 24 h (За следующие 24 часа)                 | Ожидаемое количество пациентов, которые должны быть выписаны за следующие 24 часа.  |
| Likely next 24 h (За последние 24 ч)          | Количество пациентов, выписанных за последние 24 часа.                    |
## <a name="other-options"></a>Другие варианты

В этом разделе описываются другие действия, которые можно выполнить с помощью компонентов мобильного приложения для реагирования Emergency Response.

### <a name="end-shift---sign-out"></a>End shift — sign out (Закончить смену — выход)

Вы можете выйти из приложения, используя значок профиля в верхней левой части экрана.  

![Функция выхода](media/use/sign-out.png)

> [!NOTE]
> *Функция выхода* может быть недоступной, если ИТ-администратор отключил общий доступ к устройствам.

### <a name="give-feedback"></a>Отправить отзыв

![Отправить отзыв](media/use/give-feedback.png)

Вы можете отправить отзыв с помощью параметра **App feedback** (Обратная связь приложения) в любом компоненте мобильного приложения для реагирования на чрезвычайные ситуации. При выборе параметра **App feedback** (Обратная связь приложения) вы можете выразить благодарность, поделиться мнением или сообщить о проблеме в работе приложения.

### <a name="switch-facility"></a>Switch facility (Сменить учреждение)

![Switch facility (Сменить учреждение)](media/use/switch-facility.png)

Сменить учреждение можно в любое время, выбрав название в верхней правой части экрана. После выбора имени расположения вы перейдете на экран **Средство запуска приложений**, где можно выбрать другую больницу, другой регион или другое медицинское учреждение.
