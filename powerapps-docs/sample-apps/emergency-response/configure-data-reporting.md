# <a name="configure-master-data-and-view-dashboards"></a>Настройка основных данных и просмотр панелей мониторинга

Эта статья содержит сведения о том, как можно использовать приложение администрирования (приложение, управляемое моделью) для добавления основных данных в решение, управления этими данными и применения панели мониторинга Power BI для просмотра ключевых сведений и метрик.

Эти задачи обычно выполняются бизнес-администраторами организации.

## <a name="configure-and-manage-master-data-for-your-organization"></a>Настройка основных данных об организации и управление ими

Приложение администрирования позволяет настроить основные данные об организации и управлять ими. Эти данные необходимы для работы приложения для службы экстренной медицинской помощи.

> [!IMPORTANT]
> - Убедитесь, что ИТ-администратор развернул решение в организации и предоставил бизнес-администраторам разрешения для использования приложения администрирования. Дополнительные сведения: [Развертывание приложения для службы экстренной медицинской помощи](deploy-configure.md#deploy-the-hospital-emergency-response-app)
> 
> - Вы также можете импортировать данные из файлов данных, доступных в пакете развертывания. Дополнительные сведения: [Шаг 4. Загрузка конфигурации и основных данных организации](deploy-configure.md#step-4-load-configuration-and-master-data-for-your-organization).

Необходимо добавить основные данные в эти сущности в следующей последовательности:

1. [Systems](#systems-data) (Системы)

1. [Регионы](#regions-data)

1. [Facilities](#facilities-data) (Учреждения)

1. [Расположения](#locations-data)

1. [Departments](#departments-data) (Подразделения)

Управление основными данными осуществляется из области **Locations** (Местоположения) в левой панели навигации в приложении администрирования.

> [!div class="mx-imgBorder"]
> ![locations-area](media/locations-area.png)

Сущности в разделе **Иерархия** перечислены в том порядке, в котором следует заполнять данные.

> [!NOTE]
> Данные о неотложности медицинской помощи импортируются во время развертывания решения. Дополнительные сведения: [Шаг 4. Загрузка конфигурации и основных данных организации](deploy-configure.md#step-4-load-configuration-and-master-data-for-your-organization).

### <a name="systems-data"></a>Системные данные

Сущность **Systems** (Системы) позволяет создавать записи для больничных систем и управлять ими. Это позволяет управлять несколькими больничными системами в одной организации.

Чтобы создать запись, выполните следующие действия:

1. Войдите в приложение администрирования (на основе модели), используя предоставленный ИТ-администратором URL-адрес.

1. В левой области выберите сущность **Systems** (Системы) и нажмите кнопку **Создать**:  
    > [!div class="mx-imgBorder"]
    > ![select-systems-new](media/select-systems-new.png)

1. На странице **New System** (Новая система) укажите соответствующие значения.  
   > [!div class="mx-imgBorder"]
   > ![enter-details-new-system](media/enter-details-new-system.png)

   | **Поле**            | **Описание**                                    |
   |----------------------|----------------------------------------------------|
   | Имя системы          | Введите название вашей больницы.                     |
   | Описание          | Введите дополнительное описание.                      |
   | Effective Start Data (Дата начала действия) | Введите дату и время начала действия для этой системы больницы. |
   | Effective End Date (Дата завершения действия)   | Введите дату и время завершения действия для этой системы больницы.   |

1. Выберите **Сохранить и закрыть**. Созданная запись будет доступна в списке **Systems** (Системы).

Чтобы изменить запись, выберите запись, обновите значения по мере необходимости и выберите **Save & Close** (Сохранить и закрыть).

### <a name="regions-data"></a>Данные регионов

Сущность **Regions** (Регионы) позволяет управлять географическими регионами для систем больниц.

Чтобы создать запись, выполните следующие действия:

1. Войдите в приложение администрирования (на основе модели), используя предоставленный ИТ-администратором URL-адрес.

1. В левой области выберите сущность **Regions** (Регионы) и нажмите кнопку **Создать**:

1. На странице **Новый регион** укажите соответствующие значения.  

    > [!div class="mx-imgBorder"]
    > ![enter-details-new-region](media/enter-details-new-region.png)

    | **Поле**            | **Описание**                                                                                          |
    |----------------------|----------------------------------------------------------------------------------------------------------|
    | Система               | Выберите систему больницы. Этот список заполняется на основе данных в разделе **Systems** (Системы), созданных ранее. |
    | Имя региона          | Введите имя региона. Например, Сиэтл.                                                              |
    | Описание          | Введите дополнительное описание.                                                                            |
    | Effective Start Data (Дата начала действия) | Введите дату и время начала для этого региона.                                                       |
    | Effective End Date (Дата завершения действия)   | Задайте дату и время окончания действия для этого региона.                                                         |

1. Выберите **Сохранить и закрыть**. Созданная запись будет доступна в списке **Regions** (Регионы).

Чтобы изменить запись, выберите запись, обновите значения по мере необходимости и выберите **Save & Close** (Сохранить и закрыть).

### <a name="facilities-data"></a>Данные учреждений

Сущность **Facilities** (Учреждения) позволяет управлять местоположениями больницы в каждом регионе. Например, **Редмонд** и **Белвью** в регионе **Сиэтл**.

Чтобы создать запись, выполните следующие действия:

1. Войдите в приложение администрирования (на основе модели), используя предоставленный ИТ-администратором URL-адрес.

1. В левой области выберите **Facilities** (Учреждения) и нажмите кнопку **Создать**:

1. На странице **New Facility** (Новое учреждение) укажите соответствующие значения. 

    > [!div class="mx-imgBorder"] 
    > ![enter-details-new-facility](media/enter-details-new-facility.png)

    | **Поле**            | **Описание**                                                                                 |
    |----------------------|-------------------------------------------------------------------------------------------------|
    | Регион               | Выберите регион, с которым связано это учреждение. Этот список заполняется на основе данных в разделе **Regions** (Регионы), созданных ранее. |
    | Facility Name (Название учреждения)        | Введите название учреждения. Например, Белвью.                                                  |
    | Total Vents (Всего аппаратов ИВЛ)          | Введите общее число аппаратов ИВЛ, доступных в учреждении.                                  |
    | Описание          | Введите дополнительное описание.                                                                   |
    | Effective Start Data (Дата начала действия) | Введите дату и время начала для этого учреждения.                                                     |
    | Effective End Date (Дата завершения действия)   | Задайте дату и время окончания действия для этого учреждения.                                                       |
    |Total Beds (Всего коек)      | Рассчитывается автоматически.|
    |Total Occupied (Всего занятых)      | Рассчитывается автоматически.|
    |Facility Address (Адрес учреждения)      | Введите улицу, город, округ, штат, почтовый индекс, широту и долготу для этого учреждения.|

    При необходимости введите адрес учреждения.

1. Выберите **Сохранить и закрыть**. Созданная запись будет доступна в списке **Total Vents** (Учреждения).

Чтобы изменить запись, выберите запись, обновите значения по мере необходимости и выберите **Save & Close** (Сохранить и закрыть).

### <a name="locations-data"></a>Данные о местоположениях

Сущность **Locations** (Местоположения) позволяет управлять конкретными расположениями в рамках каждой больницы.

> [!NOTE]
> Прежде чем создавать записи сущности **Locations** (Местоположения), убедитесь, что вы импортировали данные о неотложности медицинской помощи с помощью файла **00 - Acuities Import.xlsx**, как описано выше в разделе [Шаг 4. Загрузка конфигурации и основных данных в организации](deploy-configure.md#step-4-load-configuration-and-master-data-for-your-organization). Это обусловлено тем, что для создания записи сущности **Locations** (Местоположения) требуются сведения о неотложности медицинской помощи.

Чтобы создать запись, выполните следующие действия:

1. Войдите в приложение администрирования (на основе модели), используя предоставленный ИТ-администратором URL-адрес.

1. В левой области выберите сущность **Locations** (Местоположения) и нажмите кнопку **Создать**:

1. На странице **New Location** (Новое местоположение) укажите соответствующие значения.  
 
    > [!div class="mx-imgBorder"]
    > ![enter-details-new-location](media/enter-details-new-location.png)

    | **Поле**            | **Описание**                                                                                      |
    |----------------------|------------------------------------------------------------------------------------------------------|
    | Имя расположения        | Введите имя местоположения.                                                                       |
    | Facility             | Выберите учреждение. Этот список заполняется на основе данных в сущности **Facilities** (Учреждения), созданных ранее. |
    | Этаж                | Введите сведения об этаже в учреждении.                                                         |
    | Единица                 | Введите сведения об отделении учреждения.                                                           |
    | Occupancy Percentage (Процент занятости) | Автоматически рассчитывается на основе последних данных переписи и общего количества коек.                                         |
    | Acuity (Неотложность медицинской помощи)      | Выберите степень неотложности медицинской помощи, связанную с этим местоположением.                                                                                                     |
    | COVID Location (Местоположение для лечения COVID)      | Выберите, следует ли использовать это местоположение для лечения пациентов COVID (**Да** или **Нет**).                                                                                                      |
    | Total Beds (Всего коек)           | Введите общее число коек в учреждении.                                                       |
    | Surge Beds (Дополнительные койки)           | Введите общее число дополнительных коек в учреждении. Surge Beds (Дополнительные койки) — это койки, которые могут быть предоставлены сверх разрешенного количества, если пациенты нуждаются в госпитализации.                                                      |
    | Blocked beds (Занятые койки в учреждении)         | Введите число занятых коек в учреждении.                                                     |
    | Last Census (Последняя перепись)          | Заполняется на основе последней создаваемой переписи.                                             |
    | Effective Start Data (Дата начала действия) | Введите дату и время начала для этого местоположения.                                                   |
    | Effective End Date (Дата завершения действия)   | Задайте дату и время окончания действия для этого местоположения.                                                     |
    | Location Order (Порядок местоположений)       | Введите число для сортировки расположений в раскрывающихся списках местоположений.                               |
    | Alternate Site Flag (Альтернативный флаг сайта)  | Для внутреннего использования.                                                                                     |

1. Выберите **Сохранить и закрыть**. Созданная запись будет доступна в списке **Locations** (Местоположения).

Чтобы изменить запись, выберите запись, обновите значения по мере необходимости и выберите **Save & Close** (Сохранить и закрыть).

Вы можете просмотреть связанные данные для расположения, такие как **Census** (Перепись), **COVID Tracking** (Отслеживание COVID) и **Equipment Needs** (Потребности в оборудовании), открыв существующую запись для расположения и выбрав нужные вкладки. Связанные данные вводятся персоналом, работающим с пациентами, с помощью [мобильных приложений](use.md).

## <a name="manage-tracking-level-for-mobile-apps"></a>Управление уровнем отслеживания для мобильных приложений

Работники, взаимодействующие с пациентами, могут отслеживать сведения по *расположениям* или *учреждениям* с помощью мобильных приложений (приложения холста). Ниже приведен уровень отслеживания по умолчанию для каждого мобильного приложения. 

|Приложение  |Уровень отслеживания по умолчанию  |
|--|--|
|COVID tracking (Отслеживание COVID)|Расположение|
|Персонал|Расположение|
|Equipment (Оборудование)|Расположение|
|Bed capacity (Количество коек)|Facility|
|Supplies (Снабжение)|Facility|
|Staffing needs (Потребности в персонале)|Facility|
|Discharge tracking (Отслеживание выписок)|Facility|

Администратор может изменить уровень отслеживания по умолчанию для мобильных приложений.

1. Войдите в приложение администрирования (на основе модели), используя предоставленный ИТ-администратором URL-адрес.

1. В области навигации слева выберите область **Administration** (Администрирование), а затем выберите **Apps** (Приложения). 

    > [!div class="mx-imgBorder"]
    > ![config-admin-app-records](media/admin-apps.png)

1. Выберите одно из приложений холста, чтобы открыть запись.

1. В записи приложения выберите подходящее значение в поле **Tracking Level** (Уровень отслеживания).

    > [!div class="mx-imgBorder"]
    > ![app-tracking-level](media/app-tracking-level.png)

    - Если для приложения выбрано значение **Location** (Расположение), созданные в этом мобильном приложении записи будут содержать сведения о расположении и учреждении наряду с другими данными. Кроме того, в мобильном приложении будет доступен раскрывающийся список **Location** (Расположение), где пользователи смогут выбрать расположение для отслеживания данных.

    - Если для приложения выбрано значение **Facility** (Учреждение), созданные в этом мобильном приложении записи будут содержать сведения только об учреждении наряду с другими данными.

    - Если вы не выберете значение для поля **Tracking Level** (Уровень отслеживания), то для мобильных приложений применяется уровень отслеживания *по умолчанию*, как описано выше.

1. Нажмите кнопку **Сохранить** в правом нижнем углу, чтобы сохранить изменения.

## <a name="view-common-data-service-dashboards"></a>Просмотр панелей мониторинга Common Data Service

Следующие панели мониторинга доступны по умолчанию в приложении для администрирования службы экстренной медицинской помощи (на основе модели):

- **Bed Management** (Управление койками)

   Показывает наличие коек, процент занятости и общее число коек в разных учреждениях.

- **Equipment and Supply** (Оборудование и поставки)

  Показывает, какое оборудование и запасы сейчас доступны в различных учреждениях.

- **Staff Management** (Управление персоналом)

  Показывает число запрошенных, назначенных и доступных сотрудников в разных учреждениях.

- **COVID Patients** (Пациенты, зараженные COVID)

  Показывает количество пациентов в процессе обследования и фактически зараженных COVID-19 в различных учреждениях.

- **Discharges** (Выписки)

  Показывает число пациентов, ожидающих выписку, и уже выписанных.

Кроме того, можно создавать собственные панели мониторинга в дополнение к доступным по умолчанию.

### <a name="manage-dashboards"></a>Управление панелями мониторинга

Для управления панелями мониторинга необходимо сделать следующее:

1. Войдите в [Power Apps](https://make.powerapps.com) и перейдите к своему приложению администрирования.

2. В области навигации слева щелкните **Панели мониторинга** в средстве выбора области:

    > [!div class="mx-imgBorder"]
    > ![select-dashboards](media/select-dashboards.png)

3. Выберите имя панели мониторинга в области навигации слева, чтобы просмотреть диаграммы:

    > [!div class="mx-imgBorder"]
    > ![view-charts](media/view-charts.png)

    > [!NOTE]
    > Вы можете фильтровать данные в нижней части экрана, а диаграммы сверху автоматически будут обновляться отфильтрованными значениями.

4. Щелкните значок **развертывания**, чтобы просмотреть диаграмму в полноэкранном режиме:

    > [!div class="mx-imgBorder"]
    > ![select-expand](media/select-expand.png)

Фильтры в правой части можно использовать для фильтрации данных по связанным с COVID расположениям, учреждениям, регионам и больничным системам.

#### <a name="system-at-a-glance-page"></a>Система на странице с кратким обзором 

**Страница с кратким обзором систем** — это страница по умолчанию или страница верхнего уровня, которая предоставляет общие сведения.  

Здесь отображается сводное представление следующих данных: 

- **COVID Patient Information** (Сведения о пациентах COVID): показывает общее число пациентов с COVID, число пациентов с положительным результатом анализов на наличие COVID-19 и число пациентов, находящихся на обследовании.

- **Bed Management** (Управление койками) — показывает наличие коек, процент заполненности, общее число коек и количество дополнительных коек. Приведенную ниже сетку можно использовать для просмотра количества по отделениям неотложной помощи.

- **Staffing Information** (Сведения о персонале): показывает количество пациентов в отделении интенсивной терапии, назначенных медсестер и соотношение медсестер к пациентам.  

- **Patient Discharge Information** (Сведения о выписке пациентов): показывает число пациентов, пребывающих на длительном лечении, ожидающих выписку, а также уже выписанных.

- **Equipment Information** (Сведения об оборудовании): показывает общее количество аппаратов ИВЛ, количество используемых и доступных аппаратов ИВЛ. 

- **Supplies Information** (Сведения о снабжении): показывает количество запасов в наличии по дням.

> [!NOTE]
> - При выборе заголовка в любой из областей сводки отображается соответствующая страница сведений для этой области. 
> - С отчетами можно также выполнять другие действия, такие как фильтрация и сортировка данных, экспорт отчета в формате PDF и PowerPoint, добавление фокуса и т. д. Подробные сведения о возможностях отчетов в Power BI см. в статье [Отчеты в Power BI](https://docs.microsoft.com/power-bi/consumer/end-user-reports).
> - В столбцах времени последнего и недавнего обновления в некоторых из этих отчетов отображаются дата и время последнего обновления данных. Можно также легко определить актуальность по цвету значений даты и времени в этих столбцах:
>    - черный — данные обновлены менее 20 часов назад;
>    - серый — данные обновлены 20–24 часа назад;
>    - красный — данные обновлены более 24 часов назад.

## <a name="view-and-manage-app-feedback"></a>Просмотр отзывов о приложении и управление ими

Все отзывы, предоставленные персоналом, использующим приложения холста на своих мобильных устройствах, хранятся в сущности **App Feedback** (Отзывы о приложении), администраторы могут просматривать эту сущность, а также управлять ею с помощью области **Администрирование** на панели навигации слева в приложении администрирования.

Чтобы просмотреть отзывы о приложении и управлять ими:

1. Войдите в [Power Apps](https://make.powerapps.com) и перейдите к своему приложению администрирования.

2. В области навигации слева щелкните область **Администрирование** в средстве выбора области:

3. Щелкните сущность **App Feedback** (Отзывы о приложении), чтобы просмотреть список отзывов о приложении, отправленных пользователями. Можно щелкнуть запись, чтобы просмотреть подробные сведения и пометить, просмотрена она или нет.

    > ![select-app-feedback](media/select-app-feedback.png)
