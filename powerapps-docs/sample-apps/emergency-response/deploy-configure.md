# <a name="deploy-and-configure-the-hospital-emergency-response-app"></a>Разверните и настройте приложение Hospital Emergency Response

Приложение Hospital Emergency Response требует небольшой настройки для адаптации к вашим потребностям. Эта статья предоставляет пошаговые инструкции для ИТ-администраторов больниц по развертыванию и настройке приложения для своей организации.

### <a name="step-1-sign-up-for-power-apps-and-create-an-environment"></a>Шаг 1: войдите в Power Apps и создайте среду

Если у вас еще нет Power Apps, зарегистрируйтесь в Power Apps и купите соответствующую лицензию.

Дополнительные сведения:

-   [Цены Power Apps](https://powerapps.microsoft.com/pricing/)
-   [ПриобрестиPower Apps](https://docs.microsoft.com/power-platform/admin/signup-for-powerapps-admin)

После того как вы приобрели Power Apps, создать среду с база данных Common Data Service.

1.  Выполните вход в [центр администрирования Power Platform](https://aka.ms/ppac).

2.  Создайте среду Common Data Service с базой данных. Дополнительные сведения: [Создание сред и управление ими](https://docs.microsoft.com/power-platform/admin/create-environment)

    > [!IMPORTANT]
    > При создании базы данных, если вы выбираете группу безопасности для базы данных, приложения могут быть общими *только* с пользователями, которые являются участниками группы безопасности.

3.  Создайте подходящих пользователей в вашей среде. Дополнительные сведения: [Создание пользователей и назначение ролей безопасности](https://docs.microsoft.com/power-platform/admin/create-users-assign-online-security-roles)

### <a name="step-2-download-the-deployment-package"></a>Шаг 2: загрузите пакет развертывания

Получите последний пакет развертывания (.zip) от <https://aka.ms/emergency-response-solution> который содержит файл решения, изображения и файлы данных для настройки приложений и бизнес-логики для приложения Hospital Emergency Response.

Чтобы начать процесс развертывания, распакуйте файл развертывания (.zip) в папку на вашем компьютере. Извлеченная папка будет содержать следующие папки:

| **Папка/Файл**       | **Описание**  |
|-----------------------|------------------|
| **Иконки приложений**         | Содержит значки приложений по умолчанию для мобильных приложений (приложений на основе холста)|
| **Файлы данных**        | Содержит файлы основных и демонстрационных данных (.xlsx) для решения/приложения для работы. Вы можете импортировать данные из этих файлов, чтобы начать работу над приложением. Дополнительные сведения: см. [Шаг 4: загрузите конфигурацию и основные данные для вашей организации](#step-4-load-configuration-and-master-data-for-your-organization) |
| **Power BIШаблон** | Содержит файл шаблона отчета Power BI (.pbit), который вы будете использовать для настройки отчетов для своей организации. Дополнительные сведения: [Получайте аналитики с помощью панелей мониторинга Power BI](#get-insights-using-power-bi-dashboards)|
| **PowerShell**        | Содержит сценарии, которые вы будете использовать для настройки своих мобильных приложений (приложений на основе холста). |
| **Файл решения**     | Содержит файл решения Common Data Service, который создает приложения и метаданные, необходимые для приложения Hospital Emergency Response.  |

### <a name="step-3-import-the-solution-file-into-your-environment"></a>Шаг 3: импортируйте файл решения в свою среду

1.  Перейдите в расположение, в которое вы распаковали файл развертывания (.zip); вы найдете папку **Файл решения**. Мы импортируем файл управляемое решение (.zip) в папке **Файл решения** в нашу среду.

2.  Выполните вход в [Power Apps](https://make.powerapps.com).

3.  Выберите свое окружение в правом верхнем углу.

4.  В левой области выберите **Решения**, а затем выберите **Импорт**.

    > [!div class="mx-imgBorder"] 
    > ![Импорт решения](media/conf-import-solution.png "Импорт решения")

5.  В диалоговом окне **Импорт решения** выберите файл решения, упомянутый в шаге 1, а затем выполните шаги мастера, чтобы импортировать решение.

6.  После успешного импорта решения выберите **Закрыть** в диалоговом окне импорта.

Теперь вы увидите новые приложения в **Приложения**:

> [!div class="mx-imgBorder"] 
> ![Новые приложения](media/conf-apps-new-apps.png "Новые приложения")

Выберите **Приложение администрирования** чтобы открыть приложение на основе модели, позволяющее настроить остальные параметры развертывания. Дополнительные сведения: [Что такое управляемые моделью приложения?](https://docs.microsoft.com/powerapps/maker/model-driven-apps/model-driven-app-overview)

> [!div class="mx-imgBorder"] 
> ![Открыть приложение администрирования](media/conf-admin-app-open.png "Откройте приложение администрирования")

Приложение администрирования имеет ряд сущностей, в которые вы можете добавлять и управлять данными для вашей больничной системы. Вы можете использовать область выбора в нижней части левой панели навигации, чтобы выбрать другую область.

#### <a name="step-41-load-mandatory-configuration-data"></a>Шаг 4.1: загрузите обязательные данные конфигурации

Импортировать данные конфигурации в следующих сущностях в приложении администрирования является **обязательным** прежде чем перейти к следующему шагу:

| Имя области | имя сущности,| Имя файла
|-|-|-
| Расположения | Acuities | 00 - Acuities Import.xlsx
| Администрирование | Apps Config | 00 - App Config Import.xlsx
| Администрирование | Приложения | 00 - App Import.xlsx
| Обеспечение персоналом | Request Roles | 00 - Request Roles Import.xlsx
| Расположения | Supplies | 00 - Supplies Import.xlsx

##### <a name="how-to-load-data-from-data-files"></a>Как загрузить данные из файлов данных?

Чтобы импортировать данные из одного из файлов данных в сущность:

1.  На левой панели навигации приложения администрирования выберите сущность, для которого вы хотите загрузить данные. Например, выберите **Администрирование** из области выбора, а затем выберите **Acuities**.

2.  Выбрать **Импорт из Excel** и выберите файл **00 - Acuities Import.xlsx** из папки **Файлы данных**.

    > [!div class="mx-imgBorder"]
    > ![Импорт из Excel](media/conf-import-from-excel.png "Импорт из Excel")

3.  Продолжайте с помощью шагов мастера импорта для импорта данные.

4.  После импорта данных вы увидите импортированную запись в сущности:

    > [!div class="mx-imgBorder"] 
    > ![Запись сущности](media/conf-entity-record.png "Запись в сущности после импорта")

Повторите вышеуказанные шаги с другими сущностями данных конфигурации.

В качестве альтернативы, если вы хотите ввести основные данные вручную, см. [Ручная настройка и управление основными данными для вашей организации](#manually-configure-and-manage-master-data-for-your-organization).

#### <a name="step-42-load-master-data"></a>Шаг 4.2: загрузите основные данные

Как объяснено ранее:
- Вы можете использовать файлы демонстрационных данных для сущностей основных данных в папке **Файлы данных/демонстрационные данные** для импорта демонстрационных данных в требуемые сущности. 

- Вы можете использовать «пустые» файлы данных для основных сущностей в папке **Файлы данных/файл шаблона данных для основных данных**, которую вы можете использовать для заполнения данных вашей организации, а затем импортировать данные в требуемые сущности.

Вы также можете вручную добавить основные данные позже. Дополнительные сведения: [Ручная настройка и управление основными данными для вашей организации](#manually-configure-and-manage-master-data-for-your-organization)

### <a name="step-5-update-the-mobile-app-branding"></a>Шаг 5: обновите фирменное оформление мобильного приложения

Вы можете изменить значок приложения, цветовую схему или отображаемое имя мобильных приложений, чтобы они соответствовали фирменному оформлению вашей организации.

Вы делаете это с помощью сущностей **Приложение** и **Конфигурация приложения** в области **Администрирование** путем импорта данных приложения и конфигурации приложения из файлов Excel, доступных в папке **Файлы данных** и файлы значков в папке **Значки приложений** в пакете развертывания, как описано в [Шаг 2: загрузите пакет развертывания](#step-2-download-the-deployment-package).

> [!div class="mx-imgBorder"] 
> ![Сущности "Приложения" и "Конфигурация приложения"](media/conf-app-app-config-entities.png "Сущности "Приложения" и "Конфигурация приложения"")

1.  Убедитесь, что вы импортировали данные конфигурации для сущностей **Приложения** и **Конфигурация приложения**, использующие файлы **App Import.xlsx** и **App Config Import.xlsx**, соответственно.

1.  Теперь мы скопируем идентификаторы приложений на основе холста, чтобы заполнить их в записях **Приложения**, которые мы импортировали. Выполните вход в [Power Apps](https://make.powerapps.com).

1.  Выберите свое окружение в правом верхнем углу.

1.  На левой панели выберите **Приложения**, а затем выберите многоточие (…) для приложения на основе холста, а затем **подробно**.

    > [!div class="mx-imgBorder"] 
    > ![Сведения о приложениях](media/conf-environments-apps-details.png "Сведения о приложении")

1.  Идентификатор приложения отображается внизу панели **подробно** для приложения. Скопируйте идентификатор приложения вместе с его именем в файл Notepad.

    > [!div class="mx-imgBorder"] 
    > ![Сведения об идентификатор приложения](media/conf-details-app-id.png "Сведения об идентификатор приложения")

1.  Повторите шаги 4 и 5 для каждого приложения на основе холста.

1.  Откройте приложение администрирования и в левой области навигации приложения администрирования выберите **администрирование** из области выбора, а затем выберите **Приложения**. Это покажет все записи приложения на основе холста, которые вы импортировали из файла **App Import.xlsx**.

    > [!div class="mx-imgBorder"] 
    > ![Приложения администрирования](media/conf-admin-app-records.png "Приложения администрирования")

1.  Откройте одну из записей приложения, выбрав ее. Обратите внимание, что поле **идентификатор Power App** пустое.

    > [!div class="mx-imgBorder"] 
    > ![Поле идентификатора Power App](media/conf-powerapp-id-field.png "Поле идентификатора Power App")

1.  На странице сведений о приложении:

    1. Скопируйте идентификатор приложения из блокнота (куда вы скопировали ранее) в поле **Идентификатор Power App**.

    2. Дважды щелкните значок приложения и выберите файл значка для приложения из **Значки приложений**. Файлы изображений названы интуитивно, чтобы вы могли легко выбрать правильный значок. Например, выберите файл «Emergency Response App.png» для **Приложение Emergency Response**. Вы также можете выбрать собственное изображение согласно фирменному оформлению вашей организации.

    3. При необходимости обновите **Описание** или же **отображаемое имя** приложения.

    4. При необходимости обновите значение **Скрыть приложение из меню**, чтобы установить, должно ли приложение отображаться в списке приложений. Так как **Приложение Emergency Response** является контейнерным приложением, значение установлено в **нет** по умолчанию.

    5. При необходимости обновите значение **Отображаемый рейтинг приложения**, чтобы установить позицию отображения приложения в списке приложений.

    6. Нажмите кнопку **Сохранить**.

1.  Повторите шаги 8 и 9 для каждой записи приложения на основе холста в разделе **Приложения**.

1.  Выберите **Конфигурация приложения** в левой панели.

1.  Выберите запись **Приложение Emergency Response**, чтобы открыть ее для изменения.

    1.  При необходимости обновите цвета для вашего приложения.

    2.  Выбрать **Да** или же **Нет** в поле **Общий доступ к устройствам включен**, чтобы указать, является ли параметр **Выход** доступным в мобильных приложениях или нет. Выбор **да** сделает параметр **Выход** доступным. Дополнительные сведения: [Окончание смены — выход](use.md#end-shift---sign-out) в руководстве пользователя.

    > [!div class="mx-imgBorder"]

### <a name="step-6-bypass-consent-for-mobile-apps"></a>Шаг 6: обход согласия для мобильных приложений

Для завершения этого шага необходимо быть администратором клиента. Кроме того, перед выполнением этого шага вам потребуется идентификатор приложения каждого мобильного приложения (приложения на основе холста).

Чтобы получить идентификатор приложения для вашего приложения, в левой области навигации приложения администрирования выберите **администрирование** из области выбора, а затем выберите **Приложения**. Это отображает все мобильные приложения (приложения на основе холста). Выберите мобильное приложение, чтобы просмотреть его идентификатор приложения. Скопируйте идентификатор приложения для каждого приложения в файл блокнота.

> [!div class="mx-imgBorder"] 
> ![Получить идентификатор приложения](media/conf-admin-get-app-id.png "Получить идентификатор приложения")

Затем выполните следующие действия:

1.  Откройте Блокнот и скопируйте этот скрипт PowerShell:

    ```powershell
    # MUST BE A TENANT ADMIN TO RUN THIS
    Install-Module -Name Microsoft.PowerApps.Administration.PowerShell
    Install-Module -Name Microsoft.PowerApps.PowerShell -AllowClobber
    Import-Module -Name Microsoft.PowerApps.Administration.PowerShell
    Import-Module -Name Microsoft.PowerApps.PowerShell
    
    # This call opens prompt to collect credentials 
    # (Azure Active Directory account and password) 
    # used by the commands
    Add-PowerAppsAccount
    
    # Change the App ID for each new app (APPGUIDHERE)
    Set-AdminPowerAppApisToBypassConsent -AppName APPGUIDHERE
    ```

2.  Заменить значение `APPGUIDHERE` фактическим идентификатором приложения на основе холста.

3.  Сохраните файл в формате (.ps).

4.  Запустите PowerShell как Администратор и выполните только что созданный файл .ps.

5.  Повторите шаги 2 - 4 для каждого приложения на основе холста.

  Показывает количество пациентов, находящихся под наблюдением и с положительным результатом теста на COVID-19 в разных помещениях.

- **Выписки пациентов**

  Показывает количество пациентов, ожидающих выписки и фактической выписки.

## <a name="get-insights-using-power-bi-dashboards"></a>Получите аналитики, используя панели мониторинга Power BI

Опубликовать панель мониторинга Power BI и делитесь ею с пользователями в вашей организации, чтобы они могли использовать панель мониторинга для анализа и принятия решений.

### <a name="prerequisites"></a>Предварительные условия

- Лицензии Power BI Premium Capacity или Power BI Pro назначаются пользователям, получающим доступ к отчету. 

- Создать рабочее область в Power BI, где вы публикуете отчет. Войти в Power BI и создать рабочее область. Дополнительные сведения: [Создайте новые рабочие области в Power BI](https://docs.microsoft.com/power-bi/service-create-the-new-workspaces)

- Устанавливать Power BI Desktop из магазина приложений Windows: <https://aka.ms/pbidesktop>

   > [!NOTE] 
   > Если вы установили Power BI загружая прямо с Сайта Power BI как исполняемый файл в прошлом, удалите его и используйте файл из магазина приложений. Версия из магазина приложений будет обновляться автоматически по мере появления новых версий.

- После установки Power BI Desktop из магазина приложений запустите его, войдите в систему, используя учетную запись, которая имеет право на публикацию приложений Power BI в вашей организации.

### <a name="publish-the-power-bi-dashboard"></a>Опубликовать панель мониторинга Power BI

1. Перейдите в расположение, в котором вы извлекли пакет развертывания. Вы найдете файл **Emergency Response App.pbit** в папке **шаблон Power BI**.

2. Откройте файл **Emergency Response App.pbit** в Power BI Desktop. Вам будет предложено ввести следующие значения:

    - **Название организации**: введите название своей организации, которое будет заполняться в верхнем левом углу каждой страницы отчета.
    - **CDS_base_solution_URL**: введите URL-адрес вашего экземпляр среды Common Data Service. Например: https://*[myenv]*.crm.dynamics.com

    > [!div class="mx-imgBorder"]
    > ![указать название организации и базовый URL-адрес](media/pbi-pub-rep1.png)

    Выберите **Загрузить**.

3. Вам будет предложено ввести учетные данные для подключения к вашему среде Common Data Service. Выбрать **Учетная запись организации** > **Вход**, чтобы указать свои учетные данные Common Data Service.  

    > [!div class="mx-imgBorder"]
    > ![выбор-учетные данные-организации](media/select-organizational-account.png)

4. После входа выберите **Подключить**, чтобы подключиться к вашим данным в Common Data Service.

5. При успешном подключении ваши данные будут отображаться в отчете Power BI. Вам будет предложено применить ожидающие изменения к вашему запросу; выбрать **Применить изменения**.

6. Выбрать **опубликовать**, чтобы опубликовать данные в вашей рабочей области Power BI. Вам будет предложено сохранить изменения; выбрать **Сохранить**.

    > [!div class="mx-imgBorder"]
    > ![выбор-обновить-опубликовать](media/select-refresh-publish.png)

7. Вам будет предложено сохранить файл в формате .pbix вместе со сведениями о вашей среде Common Data Service. Укажите имя и сохраните его на своем компьютере.

8. После сохранения файла .pbix вам будет предложено опубликовать отчет. На странице **Опубликовать в Power BI** выберите рабочую область, в которой вы хотите опубликовать, а затем нажмите **Выбрать**.

12. Отчет становится доступным в вашей рабочей области. Теперь мы настроим параметры обновления данных для набора данных. Выберите набор данных в вашей рабочей области и выберите значок **запланировать обновление**.  
    
    > [!div class="mx-imgBorder"]
    > ![запланировать-обновление](media/schedule-refresh.png)

13. При первой попытке установить параметры обновления данных вы увидите страница **настройки** с сообщением о том, что ваши учетные данные недействительны. В **учетные данные источника данных** выбрать **Изменить учетные данные**, чтобы указать свои учетные данные.  

    > [!div class="mx-imgBorder"]
    > ![выбор-изменить-учетные данные](media/select-edit-credentials.png)

14. На следующем экране:
    - Выберите **способ проверки подлинности** как **OAuth2**.
    - Выбрать **Настройка уровня конфиденциальности для этого источник данных** как **организация**.
    - Выберите **Вход**.

    Вам будет предложено указать свои учетные данные и войти в систему. После успешного входа вы вернетесь к странице **Параметры**.

15. На странице **Параметры** развернуть **Запланированное обновление** и укажите необходимые данные для обновления данных на основе расписания. Выберите **Применить**.

    > [!div class="mx-imgBorder"]
    > ![Запланированное обновление](media/refresh-schedule.png)

    > [!NOTE] 
    > Существуют ограничения на количество обновлений данных. Power BI ограничивает наборы данных по общей емкости до восьми ежедневных обновлений. Если набор данных находится на емкости Премиум, вы можете запланировать до 48 обновлений в день в параметрах набора данных. Дополнительные сведения см. в разделе [Обновить данные](https://docs.microsoft.com/power-bi/refresh-data#data-refresh)

16. Выберите имя рабочей области на левой панели, а затем выберите **Создать приложение** в правом верхнем углу.  

    > [!div class="mx-imgBorder"]
    > ![выбор-создать-приложение](media/select-create-app.png)

17. На странице публикации приложения:

    1. На вкладке **Настройка** укажите название и описание вашего приложения.

    2. На вкладке **Навигация** укажите расположение панели мониторинга, где вы ее опубликуете.

    3. На вкладке **разрешения** укажите пользователей или группу, которые смогут просматривать это приложение. Убедитесь, что вы установите флажок **Установите это приложение автоматически** для автоматической установки этого приложения для конечных пользователей. Дополнительные сведения: [Автоматически устанавливать приложения для конечных пользователей](https://docs.microsoft.com/power-bi/service-create-distribute-apps#automatically-install-apps-for-end-users)  

        > [!div class="mx-imgBorder"]
        > ![выбор-установить-приложения-автоматически](media/select-install-apps-automatically.png)

18. Выберите **Опубликовать приложение.** Дополнительные сведения о публикации приложений в Power BI, см. [Публикация приложения](https://docs.microsoft.com/power-bi/service-create-distribute-apps#publish-your-app).

#### <a name="system-at-a-glance-page"></a>Система с первого взгляда 

**Системы с первого взгляда** является страницей по умолчанию или страницей верхнего уровня, которая обеспечивает общее представление.  

На странице отображается сводное представление следующего: 

- **пациенты с COVID**: показывает общее количество пациентов с COVID, количество пациентов с положительным результатом теста на COVID-19 и количество пациентов, находящихся под наблюдением. 

- **Управление койками**: показывает наличие коек, процент занятости, количество функциональных кроватей и общее количество коек. Вы также можете использовать сетку ниже, чтобы просмотреть показатели по острым единицам. 

- **Управление штатом медсестер** показывает количество пациентов в отделении интенсивной терапии, назначенных медсестер и соотношение медсестер к пациентам.  

- **Выписки пациентов**: показывает общее количество пациентов с длительным пребыванием, количество пациентов, ожидающих выписки и фактической выписки. 

- **Оборудование**: показывает общее количество вентиляторов, количество используемых вентиляторов и доступные вентиляторы. 

- **Медикаменты**: показывает количество медикаменты в наличии по дням.

> [!NOTE]
> - Выбор значка информации (i) в любой обобщенной области приведет вас к соответствующей странице сведений для этой области. 
> - Вы также можете выполнять другие действия с отчетами, например фильтровать и сортировать данные, экспортировать отчет в PDF и PowerPoint, добавлять полезные сведения и т.д. Для получения подробной информации о функциях отчета в Power BI см. [Отчеты в Power BI](https://docs.microsoft.com/power-bi/consumer/end-user-reports)
> - Самые последние или последние обновленные столбцы в некоторых из этих отчетов показывают дату и время последнего обновления данных. Также легко определить свежесть, посмотрев цвет значений даты и времени в следующих столбцах:
>    - Черный: данные обновлялись менее 20 часов назад
>    - Серый: данные обновлялись 20 - 24 часа назад
>    - Красный: данные обновлялись более 24 часов назад

2. В левой области навигации выберите **администрирование** из области выбора.

3. Выбрать **Отзыв о приложении**, чтобы просмотреть список отзывов о приложении, представленных пользователями. Вы можете нажать на запись, чтобы просмотреть сведения и пометить их как проверенные или нет.
